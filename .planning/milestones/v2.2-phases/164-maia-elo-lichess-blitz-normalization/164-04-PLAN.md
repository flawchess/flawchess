<objective>
Close the single verified gap in Phase 164 (VERIFICATION.md Truth #13 / REVIEW.md WR-01): a
chess.com game bucketed `classical` that is NOT correspondence (a genuine long real-time game such
as `3600+45`, `1500+10`, `3600`) currently makes `normalize_to_lichess_blitz` return `None`, so the
frontend's `*_lichess_blitz ?? raw` fallback seats the Maia ELO slider at the RAW chess.com rating —
the exact scale mismatch Phase 164 exists to eliminate.

Purpose: The module already treats a chess.com `classical`-bucketed rating as rapid-scale
(`_BUCKET_TO_SOURCE_TC = {"classical": "rapid", ...}` in the SQL composed-grid pipeline). The Maia
normalization path diverges by returning None. This plan makes the two paths agree: map chess.com
`classical` -> `rapid` before converting. chess.com Daily / correspondence is unaffected (guarded one
line earlier).

Output: a one-branch fix in `normalize_to_lichess_blitz` plus its docstring, an inverted unit test,
and one new integration test at the `_build_card` boundary.

Scope: FIX (not accept-the-risk). The deferred DB-count check from RESEARCH Pitfall 2 / assumption A1
was run at plan time: chess.com classical-bucketed non-correspondence games are 105 / 149,553 =
0.070% of chess.com games in the dev DB — all genuine long real-time games (top values `3600+45` x73,
`1500+10` x11, `3600` x6). Rare but real and legitimate, so the user decided to convert them, not
override.
</objective>

## Supersedes note (must-have reversal)

This gap-closure plan REVERSES a must-have from 164-01-PLAN.md, which intentionally specified
"chess.com classical returns None" as correct behavior (traceable to the SEED-093 routing table never
enumerating a `chess.com | classical` row). The NEW truth is: **chess.com classical-bucketed,
non-correspondence -> rapid-scale-converted Lichess-blitz equivalent (not None); chess.com Daily /
correspondence still None.** Rationale for the reversal: the DB-count above (0.070%, all legitimate)
plus the locked user decision to fix rather than accept. 164-01's original tests for the correspondence
case (`is_correspondence=True` -> None) stay intact and still pass; only the non-correspondence
classical assertion is inverted.
