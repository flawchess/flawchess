### Phase 87.2: Section 2 ‚Äî eval-based ŒîES Score Gap on Conv/Parity/Recov + Endgame Skill cards (INSERTED 2026-05-15)

**Goal:** Replace the rate-based mirror-bucket peer-diff Gap bullet (`You / Opp / Gap` row + `MiniBulletChart` centred on 0) on each of the four Section 2 cards ‚Äî Conversion, Parity, Recovery, and Endgame Skill ‚Äî with an **eval-based ŒîES Score Gap bullet** that reuses the Phase 87.1 per-span ŒîES infrastructure already shipped for Section 3 per-type cards. The current rate-based peer-diff is mathematically degenerate by construction:

- **Conv-Gap ‚â° Recov-Gap.** Because `opp_conv_win% = user_recov_loss%` (mirror-bucket same-game symmetry) and `opp_recov_save% = user_conv_loss% + user_conv_draw%`, the two Gaps are algebraically identical for every user / filter combination. The "Conv vs Recov" framing is illusory.
- **Parity-Gap is a deterministic affine of the gauge.** `opp_parity = 1 ‚àí user_parity` by chess-score symmetry, so `Parity-Gap = 2¬∑user_parity ‚àí 1`. The Gap bullet plots the same information as the absolute Parity gauge through a confusing mirror frame.
- **Skill-Gap inherits both pathologies.** `Skill = mean(Conv, Parity, Recov)` ‚áí `Skill-Gap = (2¬∑Conv-Gap + Parity-Gap) / 3`, i.e. two degrees of freedom and one of them (`user_parity`) is already on the Parity gauge.

The ŒîES Score Gap sidesteps all three: it measures eval delta *during the span* (entry-eval expected-score minus exit-score), not symmetric outcomes of a shared set of games, so Conv-bucket spans and Recov-bucket spans yield genuinely independent values.

Each Section 2 card gets one ŒîES bullet restricted to the eval-entry bucket that defines the card:

| Card | ŒîES cohort |
|---|---|
| Conversion | spans with `entry_eval ‚â• +100 cp` (user perspective) |
| Parity | spans with `‚àí100 cp ‚â§ entry_eval ‚â§ +100 cp` |

### Phase 87.6: Endgame ELO via Performance Rating ‚Äî drop K, dual PR lines + signed band (INSERTED 2026-05-17)

