### Phase 85.1: Hypothesis tests + CIs for Endgame Score Differences (INSERTED 2026-05-14)

**Goal:** Add two-sided p-values and 95% CIs to the two derived metrics in the Phase 85 "Endgame Score Differences" card. Endgame Score Gap (Endgame Score − Non-Endgame Score) is an **independent two-sample** z-test on the chess-score difference, using the trinomial empirical variance per side. Achievable Score Gap (Endgame Score − Achievable Score) is a **paired one-sample** z-test on per-game differences `d_i = actual_score_i − expected_score_i` over the same `bucket_rows` cursor that already feeds `entry_expected_score`. Surface CIs as bullet whiskers on each `ScoreGapRow`. Optionally tighten the gap font color via the new significance signal (reopens Phase 85 D-04 zone-only coloring) — decision recorded in plan-phase.

**Depends on:** Phase 85 (3-card composite section, `ScoreGapRow`, and `EndgameOverallPerformanceSection` shipped).

  2. `EndgamePerformanceResponse` carries `achievable_score_gap`, `achievable_score_gap_p_value`, `achievable_score_gap_ci_low`, `achievable_score_gap_ci_high` computed from the paired z-test on per-game `(actual_i − expected_i)` over the existing `bucket_rows` cursor, with the gating policy from SEC1-10. ### Phase 87.1: Per-span ΔES metric for endgame types (INSERTED 2026-05-15)

**Goal:** Add a new per-endgame-type performance metric — mean per-span expected-score delta (ΔES) — that complements the existing game-result-derived Conversion/Recovery rates. For each endgame sequence: `ΔES_span = ES_sigmoid(entry_eval, user_perspective) − exit_score`, where `exit_score = ES_sigmoid(next_span_entry_eval)` for transitory spans (followed by another span in the same game) and `exit_score = game_result_score` (1.0 / 0.5 / 0.0 from user perspective) for terminal spans. Aggregate as mean per (user, endgame_class). Display as a percentile-zoned gauge alongside Conv/Recov on the Phase 87 per-type cards. ΔES isolates per-span quality from game-outcome noise, eliminates the ±1.0 threshold cliff in Conv/Recov, and uses every span (no cohort thinning). Sigmoid choice and calibration caveat documented in `.planning/notes/lichess-sigmoid-endgame-calibration.md`.

**Depends on:** Phase 87 (per-type card layout shipped — ΔES gauge slots into the existing card shell).

**Requirements:** TBD (DELTA-01..DELTA-NN to be drafted during spec-phase; not yet appended to `.planning/REQUIREMENTS.md`)

  1. `EndgameCategoryStats` carries `delta_es_mean`, `delta_es_n`, `delta_es_p_value`, `delta_es_ci_low`, `delta_es_ci_high` per endgame_class, computed from per-span ΔES values using `compute_paired_difference_test` (or equivalent) from `app/services/score_confidence.py`.
