## Goal

Replace three table-driven sections on the Endgames page with the established WDL + ScoreBullet card pattern from `EndgameStartVsEndSection.tsx` / `OpeningStatsCard.tsx`. Single-bullet doctrine on Conv/Parity/Recov/Skill: each card carries one peer bullet (`You − Opp` vs 0) using the mirror-bucket opponent baseline. The peer frame is rating-tier-conditioned by construction via the mirror bucket, filter-responsive, and self-calibrating.

### Phase 85: Section 1 — Games with vs without Endgame cards

**Goal:** Replace the `EndgamePerformanceSection` table with two side-by-side cards (No / Yes) on lg+, stacked on mobile. Each card carries a WDL bar + Score row + score bullet vs 0.50 (natural balanced-WDL anchor, not a population statistic). A full-width Score Gap footer bullet (Yes − No vs 0) spans both cards. Significance gating follows `EndgameStartVsEndSection` convention (`n ≥ 10` ∧ `p < 0.05` ∧ value outside neutral band).
**Depends on:** Phase 84 (mirror-rate audit complete — not strictly blocking since Section 1 uses no peer bullet)
**Requirements:** SEC1-01, SEC1-02, SEC1-03, SEC1-04, SEC1-05, SEC1-06, SEC1-07
**Success Criteria** (what must be TRUE):
  1. On lg+ the Endgames page shows two side-by-side cards labelled "Games without Endgame" / "Games with Endgame", stacked on mobile (375px parity).
  2. Each card shows its WDL bar and a Score row with chess score `(W + 0.5·D)/n` + Wilson CI whiskers + score bullet centred at 0.50 with the standard neutral band.
  3. A full-width footer Score Gap bullet (Yes − No, centre 0, neutral band `[SCORE_GAP_NEUTRAL_MIN, SCORE_GAP_NEUTRAL_MAX]`) spans both cards.
  4. Card score-row font color paints only when `n ≥ MIN_GAMES_FOR_RELIABLE_STATS` AND `p < 0.05` AND value lands outside the neutral band; an `InfoPopover` on each Score row explains that 0.50 is the balanced-WDL natural anchor (no rating-tier confound applies — this is conceptually different from the dropped global p50 frame).
  5. The legacy `EndgamePerformanceSection` table component is removed from the codebase (knip clean).
**Plans:** 4 plans
- [ ] 85-01-PLAN.md — Backend: add non_endgame_score_p_value field + service mirror + test (D-01)
- [ ] 85-02-PLAN.md — Frontend type extension + extract EndgameScoreOverTimeChart to its own file (D-09)
- [ ] 85-03-PLAN.md — Build EndgameGamesWithWithoutSection (two cards + footer gap bullet) + tests
- [ ] 85-04-PLAN.md — Mount swap in Endgames.tsx + delete legacy section + knip/build verification (SEC1-07)
**UI hint:** yes

### Phase 86: Section 2 — Endgame Metrics 4-card layout (Conv / Parity / Recov / Skill)

**Goal:** Replace `EndgameScoreGapSection` table + 4-gauge strip with 4 side-by-side cards on lg+, stacked on mobile, in order Conversion → Parity → Recovery → Endgame Skill. Conv/Parity/Recov share an identical layout (gauge → percent + games → WDL bar → single peer bullet `You − Opp` vs 0). Skill card has no WDL bar; its peer bullet uses derived `Opp Skill = composite(opp_conv, opp_recov)` computed frontend-side using the same Skill composite formula as `Your Skill`. Mirror-bucket opponent baseline logic (`opponentRate` / `MIRROR_BUCKET` / `MIN_OPPONENT_BASELINE_GAMES`) is preserved and feeds the peer bullets. Open question on Skill peer-bullet sig-test handling resolved during plan-phase per SEC2-08.
**Depends on:** Phase 84 (DATA-02 mirror-rate audit) and Phase 85 (Section 1 score-bullet pattern locked in for reuse).
**Requirements:** SEC2-01, SEC2-02, SEC2-03, SEC2-04, SEC2-06, SEC2-07, SEC2-08, SEC2-09, SEC2-10
**Success Criteria** (what must be TRUE):
  1. User sees 4 side-by-side cards (Conv / Parity / Recov / Skill) on lg+, stacked on mobile; Conv/Parity/Recov each show gauge + percent + WDL bar + single peer bullet `You − Opp` vs 0.
  2. Skill card shows gauge + percent + peer bullet `Your Skill − Opp Skill` vs 0 (no WDL bar); `Opp Skill` is derived frontend-side from `opp_conv` + `opp_recov` using the existing Skill composite formula; sig-test handling is documented in the plan-phase decision log.
  3. Peer bullets use the mirror-bucket opponent rate per-card with Wald-z sig test on the signed difference vs 0, gated on `MIN_OPPONENT_BASELINE_GAMES`; gauge bands continue to use `FIXED_GAUGE_ZONES.{conversion,parity,recovery}` (p25/p75) unchanged.
  4. An `InfoPopover` on each peer bullet explains the filter-responsive mirror-bucket interpretation (baseline shifts with rating × TC × color × opponent-type filters).
  5. The legacy `EndgameScoreGapSection` table and its 4-gauge strip are removed (knip clean).
**Plans:** TBD
**UI hint:** yes
**Context:** Smaller than the pre-pivot two-bullet version of this phase, but Skill peer-bullet sig-test methodology (SEC2-08) remains the main scope risk — options are Wald-z directly on the derived difference / propagation from component CIs / computation on raw underlying outcomes. The Skill composite formula is already implemented elsewhere; the plan-phase should reference it rather than redefining.

### Phase 87: Section 3 — Per-type Endgame Type Breakdown cards

**Goal:** Replace the grouped `EndgameWDLChart` and extend `EndgameConvRecovChart` into 5 full per-type cards (rook / minor_piece / pawn / queen / mixed; pawnless hidden per `HIDDEN_ENDGAME_CLASSES`). Each card carries side-by-side Conv + Recov gauges, a WDL bar, Conv peer bullet (`You − Opp` vs 0), Recov peer bullet (`You − Opp` vs 0), and a Games deep-link to `/endgames/games?type=…`. No cohort bullets (single-bullet doctrine). Mobile density check on real device before locking; fewer elements per card than the pre-pivot design, so the fallback path is to drop the WDL bar (not the peer bullets) if scroll bloat is still unacceptable.
**Depends on:** Phase 86 (single peer-bullet doctrine and mirror-bucket peer-bullet plumbing established).
**Requirements:** SEC3-01, SEC3-02, SEC3-04, SEC3-05, SEC3-06, SEC3-07
**Success Criteria** (what must be TRUE):
  1. User sees 5 per-type cards in a 3-col grid on lg+, 2-col on sm, 1-col on mobile, each labelled by class with side-by-side Conv + Recov gauges, a WDL bar, two peer bullets (Conv + Recov), and a Games deep-link `/endgames/games?type=…`. No cohort bullets.
  2. Per-type peer bullets use the per-type mirror-class rate, gated on `MIN_OPPONENT_BASELINE_GAMES` with a sparse-n indicator. Gauge bands continue to use `PER_CLASS_GAUGE_ZONES[<class>].{conversion,recovery}` (p25/p75) — no per-class p50 needed.
  3. Real-device mobile density check is performed and documented; if WDL bar is dropped under the fallback path, the decision log records the loss and updates the `InfoPopover` copy accordingly.
  4. The legacy `EndgameWDLChart` component is removed; `EndgameConvRecovChart` no longer exists as a standalone gauge-only component (extended into the per-type card shell). Knip clean.
