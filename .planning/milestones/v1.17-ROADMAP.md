The peer frame is rating-tier-conditioned by construction via the mirror bucket, filter-responsive, and self-calibrating.

### Phase 85.1: Hypothesis tests + CIs for Endgame Score Differences (INSERTED 2026-05-14)

**Goal:** Add two-sided p-values and 95% CIs to the two derived metrics in the Phase 85 "Endgame Score Differences" card. Endgame Score Gap (Endgame Score − Non-Endgame Score) is an **independent two-sample** z-test on the chess-score difference, using the trinomial empirical variance per side. Achievable Score Gap (Endgame Score − Achievable Score) is a **paired one-sample** z-test on per-game differences `d_i = actual_score_i − expected_score_i` over the same `bucket_rows` cursor that already feeds `entry_expected_score`. Surface CIs as bullet whiskers on each `ScoreGapRow`. Optionally tighten the gap font color via the new significance signal (reopens Phase 85 D-04 zone-only coloring) — decision recorded in plan-phase.

**Depends on:** Phase 85 (3-card composite section, `ScoreGapRow`, and `EndgameOverallPerformanceSection` shipped).

**Requirements:** SEC1-08, SEC1-09, SEC1-10, SEC1-11

**Success Criteria** (what must be TRUE):
  1. `ScoreGapMaterialResponse` carries `score_difference_p_value`, `score_difference_ci_low`, `score_difference_ci_high` computed from the independent two-sample z-test on `endgame_score − non_endgame_score`, with the gating policy from SEC1-08 / SEC1-09.
  2. `EndgamePerformanceResponse` carries `achievable_score_gap`, `achievable_score_gap_p_value`, `achievable_score_gap_ci_low`, `achievable_score_gap_ci_high` computed from the paired z-test on per-game `(actual_i − expected_i)` over the existing `bucket_rows` cursor, with the gating policy from SEC1-10. The frontend derivation in `EndgameOverallPerformanceSection.tsx:80-81` is removed in favor of the server value.
  3. Test math lives in `app/services/score_confidence.py` (or a sibling) as two new helpers — `compute_score_difference_test(eg_w, eg_d, eg_l, eg_n, ne_w, ne_d, ne_l, ne_n)` returning `(p_value, ci_low, ci_high)`, and `compute_paired_difference_test(diffs: Sequence[float])` returning `(mean_d, p_value, ci_low, ci_high)`. Single source of truth, unit-tested at boundaries (n=0, all-wins both sides, all-equal pairs, n=1 paired).
  4. Both `ScoreGapRow` rows in `EndgameOverallPerformanceSection` render 95% CI whiskers on the bullet, mirroring the Score-row whiskers from Phase 85 (SEC1-03).
  5. The plan-phase decision log records whether Phase 85 D-04 (gap font color is zone-only) is reopened. If kept, sig + CI are informational only via the existing `InfoPopover`. If reopened, the same triple gating used elsewhere in v1.17 applies (`n ≥ MIN_GAMES_FOR_RELIABLE_STATS` AND `p < 0.05` AND value outside neutral band).
  6. Tests cover the new test helpers, the service-layer wiring, the response-schema gates (None vs value at the n thresholds), and the frontend CI whisker rendering.

**Plans:** 4 plans
- [ ] 85.1-01-PLAN.md — Backend math helpers: `compute_score_difference_test` + `compute_paired_difference_test` in `score_confidence.py` with boundary unit tests
- [ ] 85.1-02-PLAN.md — Service wiring + schema fields + `PVALUE_RELIABILITY_MIN_N` extraction (IN-01); paired-diff loop merged into existing `ex_sum`/`ex_n` accumulator
- [ ] 85.1-03-PLAN.md — Frontend TS types + CI whisker prop threading + server-side `achievable_score_gap` migration (delete `EndgameOverallPerformanceSection.tsx:80-81` derivation); folds in WR-04 + IN-02
- [ ] 85.1-04-PLAN.md — D-04 reopen decision (recommend keep zone-only) + InfoPopover sigmoid-bias caveat + verification sweep

**UI hint:** yes

**Context:** Math is straightforward — both tests are textbook and the codebase already has the trinomial empirical variance (`compute_confidence_bucket`) and Wilson CI helper (`wilson_bounds`). Two scope risks: (a) Phase 85 D-04 explicitly chose zone-only coloring for gaps; this phase reopens that, which is a normal in-milestone iteration but worth flagging. (b) The Achievable test treats Stockfish-derived expected scores (Lichess sigmoid) as ground truth; sigmoid bias per rating bucket would surface as user-skill signal. Document the caveat in the InfoPopover but don't block on it. Future iteration FUT-05 could quantify sigmoid bias on the benchmark cohort and adjust the test, but that's a benchmarks-skill rework.

Skill card has no WDL bar; its peer bullet uses derived `Opp Skill = composite(opp_conv, opp_recov)` computed frontend-side using the same Skill composite formula as `Your Skill`. Mirror-bucket opponent baseline logic (`opponentRate` / `MIRROR_BUCKET` / `MIN_OPPONENT_BASELINE_GAMES`) is preserved and feeds the peer bullets. Open question on Skill peer-bullet sig-test handling resolved during plan-phase per SEC2-08.

### Phase 88: Time Pressure stats rework with hypothesis tests + CIs (INSERTED 2026-05-14)

**Goal:** Replace the Time Pressure at Endgame Entry table and the Time Pressure vs Performance line chart with a unified per-TC card design that mirrors the v1.17 single-bullet doctrine. One card per time control (bullet / blitz / rapid / classical) in a 4-col grid on `xl`, 2-col on `lg`, 1-col below. Each card stacks 6 horizontal bullets: 1 Clock Gap bullet at the top (mean `(my_clock − opp_clock) / base_clock` at endgame entry, centred on 0%, Wilson-style CI as whiskers, color zone from benchmark global distribution) + 5 Score-Delta bullets below (one per 20% pressure quintile measured as base-clock remaining at endgame entry, metric `user_score − cohort_score` centred on 0, Wilson CI from user bin score with cohort N treated as fixed reference, color zone from per-bin cohort inter-user spread). Closes v1.17 endgame rework arc by applying CI + hypothesis-testing rigor to the last table-driven Endgames-page section.

**Depends on:** Phase 87 (per-TC card layout pattern + bullet primitives + mirror-bucket plumbing established). Compatible with Phase 89 Polish running after to extend popover / gating / 375px sweep to the new card surfaces.

**Requirements:** TBD (TIME-01..TIME-NN to be drafted during spec-phase; not yet appended to `.planning/REQUIREMENTS.md`)

**Success Criteria** (what must be TRUE):
  1. Endgames page Time Pressure section renders one card per TC; legacy `TimePressureAtEndgameEntrySection` table and `TimePressureVsPerformanceChart` line chart are removed (knip clean).
  2. Each rendered TC card carries 6 bullets: 1 Clock Gap bullet (centre 0%, Wilson-style CI whiskers, color zone from benchmark global distribution per /benchmarks collapse verdict — expected single global zone) + 5 stacked Score-Delta bullets, one per 20% pressure quintile (centre 0, Wilson CI from user bin score, color zone from per-bin cohort inter-user spread).
  3. Sparse-TC gating: the entire TC card hides when total user games for this TC < `MIN_GAMES_PER_TC_CARD` (proposed 20). Individual Score-Delta bullets within a rendered card show a dash + sample count when their bin n < `MIN_GAMES_PER_PRESSURE_BIN` (proposed 5). Clock Gap bullet always renders when the card renders.
  4. /benchmarks skill ships two new metrics: clock-gap-% (per TC × ELO; expected to collapse to a single global zone) and chess-score-per-pressure-bin (per TC × ELO × pressure bin; per-bin Cohen's d collapse verdict). Generated zone constants are surfaced in `frontend/src/generated/endgameZones.ts` (or sibling) and consumed by the bullets.
  5. Backend math reuses Phase 85.1 helpers where possible (`compute_paired_difference_test` for Clock Gap). A new helper `compute_score_delta_vs_reference(user_w, user_d, user_l, user_n, cohort_score)` returns `(delta, p_value, ci_low, ci_high)` treating the cohort score as a fixed reference — unit-tested at boundaries.
  6. InfoPopovers on every bullet explain the reference (0% / cohort baseline), the filter-responsive interpretation, and sparse-n handling. Significance gating follows the established v1.17 triple gate (`n ≥ threshold` ∧ `p < 0.05` ∧ outside neutral band).
