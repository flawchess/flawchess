<objective>
Build the per-run / per-xdist-worker test-DB mechanism inside `tests/conftest.py`, replacing
the shared `flawchess_test` database + session-start `TRUNCATE ... RESTART IDENTITY CASCADE`
with a private database cloned from a migrated `flawchess_test_template` via
`CREATE DATABASE ... TEMPLATE`. Add `pytest-xdist` as a dev dependency so `-n auto` becomes
available (measured in Plan 02). This is the core of Phase 100 (SC-2, SC-4) and the enabler
for SC-1 and SC-3.

Purpose: Concurrent `pytest` runs (multiple agents + IDE coverage) currently deadlock on the
shared DB's whole-schema `ACCESS EXCLUSIVE` truncate lock, and committing fixed-ID fixtures
collide across runs. Giving each run its own freshly-cloned DB removes both root causes
structurally (SEED-031): a fresh clone is already clean, so the hostile truncate is retired,
and fixed IDs (e.g. `test_eval_drain.py`'s 99100/99101) can no longer collide across runs.

Output: Modified `tests/conftest.py` (new helpers + rewritten `test_engine` fixture),
`pyproject.toml` + `uv.lock` with `pytest-xdist`. The existing ~2200-test suite passing
serially under the new infrastructure is the safety net.

Scope fence: the benchmark suite (`tests/scripts/benchmarks`, port 5433) is explicitly OUT of
scope and left untouched (D-03) â€” it uses the read-only `flawchess_benchmark_ro` role (no
`CREATE DATABASE` privilege), is already `--ignore`'d by `addopts`, and has no TRUNCATE
deadlock. This plan only touches the main suite against dev Postgres (port 5432).
</objective>
