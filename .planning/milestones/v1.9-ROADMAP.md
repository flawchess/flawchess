## Overview

Restructure page layouts across desktop and mobile based on UI/UX expert recommendations — improve information hierarchy, reduce scrolling, and unify desktop/mobile interaction patterns. Delivers a collapsible left-edge sidebar for the Openings desktop layout, a unified mobile control row that stays visible when the board is collapsed, visual alignment across Openings/Endgames mobile surfaces, a 2-column Stats subtab, a 2-column homepage hero, and a full "Stats" → "Global Stats" rename with opponent filters enabled end-to-end.

## Milestone Summary

**Key Decisions:**
- Unified row outside board collapse region (D-03) — Openings mobile controls stay visible when the board is collapsed
- Left-to-right order Tabs | Color | Bookmark | Filter (D-02) — lock mobile control order across pages
- Overlay/push breakpoint at 1280px — sidebar overlays on smaller desktop, pushes on larger
- Bot games excluded by default on Global Stats (D-21) — `opponentType='human'` from `DEFAULT_FILTERS` propagates through opponent filter enablement
- URL route `/global-stats` unchanged (D-17) — only the display label + h1 were renamed, no redirect needed
- Explicit `leftRows`/`rightRows` array split for the 2-col Bookmarked Results (not CSS `columns-2`) — deterministic odd-count behavior, no `break-inside` edge cases
- Static 2-col hero on homepage (not a carousel) — simpler, preserves scroll-free visibility without JS state

**Issues Resolved:**
- MMOB-01 (subtab placement TBD) resolved with unified row holding Tabs | Color | Bookmark | Filter inside the sticky wrapper but outside the board collapse region
- Collapse handle on mobile Openings grew from a ~4px sliver to a 44px tappable strip
- Global Stats page previously exposed no opponent filter — now defaults to human-only with full opponent type + strength controls
