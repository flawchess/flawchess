<objective>
Wire the two style hooks into `selectBotMove`'s existing two regimes (STYLE-03/04) behind an optional `BotSettings.style?: BotStyleParams` field (STYLE-05): prior reweighting in the `blend<=0` raw-policy branch, score shaping in the search branch. The field stays structurally separate from `budget` so the D-02/WR-04 policyTemperature exclusion is preserved, and `undefined` style keeps the code path byte-identical to today (D-03).

Purpose: The two regimes operate on structurally different data (`Record<string,number>` raw policy vs `RankedLine[]`) at different control-flow points, so they need two distinct hook calls, never one unified entry point (Pitfall 5). This plan touches only the orchestrator; the pure transforms come from Plan 04, so `selectBotMove` gains orchestration calls only (the pure/impure split from D-09).
Output: `BotSettings.style?` field + two guarded hook calls in `selectBotMove.ts` + a `selectBotMove.test.ts` regression proving undefined-style is unchanged and styled paths call the transforms.
</objective>
