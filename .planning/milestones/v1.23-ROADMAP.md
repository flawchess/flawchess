# v1.23 LLM Endgame-Insights Statistical-Reasoning Rework — Roadmap (ARCHIVED)

**Shipped:** 2026-06-03
**Phases:** 102, 103
**Milestone goal:** Rework the endgame-insights LLM payload + prompt so the model reasons explicitly over the v1.17–v1.21 statistical-rigor metric set (Endgame Score Gap, Achievable Score Gap, Section 2 ΔES Score Gap family, Time Pressure hypothesis tests) and the v1.19 peer-relative percentile annotations, with prompt guardrails that prevent narrating small-but-significant findings. Preserve `feedback_llm_significance_signal`: the cohort `zone` field stays the sole gate on *whether* a metric is narrated; percentile informs only *how*. No new frontend cards — payload + prompt only.

## Summary

- **Phase 102 — Endgame LLM Statistical-Reasoning Rework** wired cohort-framed **percentile annotations** (the page-level, game-count-weighted value the chip shows) into the endgame-insights payload alongside the existing `zone` + `sample_quality` fields, and taught the prompt to weave them into narration naturally ("vs other ~{anchor}-rated players"). Added **LLM narration of time pressure** — Score Gap by Remaining Time (the per-bucket decomposition Phase 88.1 had stripped from the payload), Clock Gap, and Net Flag Rate. Relaxed the `overview` ("Data Analysis" card) word cap so the model may narrate more when ≥3 distinct signals genuinely exist, keeping the no-fabrication / within-noise guards. Audited the prompt's UI-vocabulary mapping against both the Endgame Statistics Concepts accordion **and** the tooltip info-icon popovers so report and hover-help never contradict. **p-values and CI bounds stayed OUT** (redundant with the zone band; conflicts with `feedback_llm_significance_signal`). Prompt version walked `endgame_v35` → through `endgame_v43` across the GSD auto-chain (per-TC metrics_elo, per-TC time-pressure chart block, per-TC type_breakdown), each cache-invalidating via the `_PROMPT_VERSION` key. HUMAN-UAT (LLM-07) was the primary verification: multiple passes against short-history / sparse-section / full-history production users, signed off 2026-06-02.
- **Phase 103 — Endgame report LLM prompt refinements** (unplanned follow-on, direct commits) applied three recommendation-quality fixes from chess-GM feedback on generated reports and then condensed the prompt ~35%. Time-trouble advice was reframed from opening repertoire to decision speed ("make quicker decisions earlier", "play okay-looking moves faster"); a new register item bans the "do X → to achieve effect Y" fabricated-mechanism construction (recommendations state the WHAT, leave the HOW to the player); and naming specific theoretical positions (Philidor, Lucena, opposition, …) as study targets is now forbidden at every Elo, since the data is type-level aggregate and never proves the player mishandled those exact positions. A fixed GM Noël Studer Lichess endgame study link was added to the Recommendations card (frontend-only; the prompt deliberately does not emit the URL). Prompt version `endgame_v44`.

Phase 102 landed on `main` via local squash-merge (PR #173 closed in favour of the squash) after full HUMAN-UAT; Phase 103 landed as direct gated commits on `main`. Both shipped to production together with this milestone close.

## Milestone Summary

**Key decisions:**

- **Percentile-only enrichment under the zone gate** — no parallel p-value/CI significance fields in the payload; percentile annotations inform only *how* a zone-opened narration is phrased, never *whether* (LLM-03, preserving `feedback_llm_significance_signal`).
- **p-values and CI bounds OUT** — redundant with the cohort zone band; the existing `sample_quality` / within-noise / `[near edge]` markers already cover precision and hedging.
- **No new frontend cards** — the metric set was already aligned with the page; the rework is payload + prompt only.
- **Phase 103 recommendation register** — decision-speed time-trouble advice, no fabricated mechanism, no named theoretical positions at any Elo. The causal tail of "do X → effect Y" is where the model hallucinates / shows half-knowledge, so directives state the WHAT only.
