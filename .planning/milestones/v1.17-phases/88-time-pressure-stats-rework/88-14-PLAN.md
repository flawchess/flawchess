Inline helpers (place them above `ThreeStatRow` in the same file):
- `function formatPctSecs(pct: number | null, secs: number | null): string` — returns "—" if either is null; otherwise `${Math.round(pct * 100)}% (${Math.round(secs).toLocaleString()}s)`.
- `function formatNetTimeoutRate(rate: number): string` — returns `${rate > 0 ? '+' : ''}${(rate * 100).toFixed(1)}%` for nonzero, `0.0%` for zero. (Negative gets a `-` sign naturally from JS string conversion.)
- `function tintForNetTimeoutRate(rate: number): string | undefined` — uses `NEUTRAL_TIMEOUT_THRESHOLD` (imported from `@/generated/endgameZones`) to return `ZONE_SUCCESS` (positive past threshold), `ZONE_DANGER` (negative past threshold), or `undefined` (neutral, no tint). **Unit convention LOCKED:** `rate` is a fraction (`0.005` = 0.5%); `NEUTRAL_TIMEOUT_THRESHOLD = 5.0` is percent. Implement as: `const pct = rate * 100; if (pct > NEUTRAL_TIMEOUT_THRESHOLD) return ZONE_SUCCESS; if (pct < -NEUTRAL_TIMEOUT_THRESHOLD) return ZONE_DANGER; return undefined;`.
