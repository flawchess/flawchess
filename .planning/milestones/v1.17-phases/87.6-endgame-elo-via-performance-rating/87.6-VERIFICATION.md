### Code Review Findings Surfaced

The following findings from `87.6-REVIEW.md` are reproduced here for traceability. They are **advisory** (not blocking success criteria) unless explicitly noted:

**CR-01 (Advisory — regression risk, not a goal blocker):** `query_endgame_performance_rows` NULL-rating WHERE guards now filter rows that feed `get_endgame_performance` WDL aggregation (the "Endgame vs Non-Endgame" Performance card), not just the ELO timeline path. Any user with NULL-rated games will see a quiet game-count drop in the WDL comparison card. No test seeds a NULL-rating game against this query. The Success Criteria do not require the WDL caller to be unaffected, but the ROADMAP says "Existing callers of this function (Time Pressure sections per the docstring) ignore the new columns; planner verifies no breakage" — that breakage check was documented as "no breakage" only for column access, not for row-population change. This is a meaningful behavior change to `get_endgame_performance` that is invisible to the test suite.

**CR-02 (Advisory — copy accuracy, not a goal blocker):** Popover line "Actual ELO sits inside the band by construction" and the corresponding LLM prompt sentences drop the qualifying conditions from the schema docstring (`|gap| ≤ 0.20` and `equal opp pools`). Provably false for users with asymmetric opp pools. Does not block the phase goal but may cause incorrect LLM narration.
