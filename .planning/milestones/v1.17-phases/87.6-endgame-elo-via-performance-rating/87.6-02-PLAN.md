The verbatim algorithm body (to lift from `EndgameScoreOverTimeChart.tsx:118-163`, parameterising `SCORE_TIMELINE_FILL_ABOVE` → `colors.positive` and `SCORE_TIMELINE_FILL_BELOW` → `colors.negative`):

1. If rows.length = 0, return [].
2. Find the first non-zero `sign` in rows; if none found, use `colors.positive` as default. Set `currentColor` to the matching color.
3. Push `{offset: 0, color: currentColor}`.
4. For each i from 0 to N-2:
   - Let `dA = rows[i].sign`, `dB = rows[i+1].sign`.
   - Let `colorA = dA >= 0 ? colors.positive : colors.negative` (note: `dA === 0` maps to positive by the `>= 0` branch; the chart's algorithm uses `diff >= 0 ? ABOVE : BELOW`).
   - Same for `colorB`.
   - If `colorA !== colorB`: compute `t = dA / (dA - dB)` (in [0, 1]); push `{offset: (i+t)/(N-1)*100, color: currentColor}` AND `{offset: (i+t)/(N-1)*100, color: colorB}`; update `currentColor = colorB`.
5. Push `{offset: 100, color: currentColor}`.

The "first non-zero" rule from step 2 handles the Phase 68 UAT bug 260424: if `rows[0].sign === 0`, naïvely starting from `colorFor(0) = positive` would silently miss subsequent negative segments because `0 * dB = 0` is not strictly negative. Finding the first non-zero sign makes the initial color match the first visible band direction.
