### 2. Midpoint property edge case (CR-02 validation) — issue: design pivot required
expected: Actual ELO sits inside the band most of the time, with at most rare edge cases outside it.
result: **issue — fundamental design problem, not an edge case**

Investigation 2026-05-17 (script: `/tmp/investigate_actual_above_pr.py`, against prod, 10 highest-volume combos, 2 248 weekly points):

| | weeks | share |
|---|---:|---:|
| Actual ELO inside the band       |   270 | **12.0 %** |
| Actual ELO **above** both PRs    | 1 111 | **49.4 %** |
| Actual ELO **below** both PRs    |   867 | **38.6 %** |

The midpoint property holds in only 1 weekly point out of 8. Root cause: PR is a trailing-100-game-window estimator, Actual ELO is a live Glicko-1/Glicko-2 snapshot — the two diverge whenever the user's rating is moving. Median 4-week ΔActual: +27 ELO (above-both), 0 ELO (in-band), −20 ELO (below-both). The popover and LLM prompt's "by construction" claim is empirically wrong.

**Decision:** Amend Phase 87.6 in-flight (do not archive, do not spin up a new phase) to replace the PR-direct mapping with a **logistic stretch anchored on Actual ELO** — see [`.planning/notes/endgame-elo-logistic-anchored.md`](../../../notes/endgame-elo-logistic-anchored.md) for the full design.

Summary of the amendment:
- `endgame_elo = actual_elo + spread / 2`, `non_endgame_elo = actual_elo − spread / 2`, where `spread = 400 · log10((s_E / (1 − s_E)) / (s_N / (1 − s_N)))`.
- Midpoint = Actual ELO **exactly**, restoring the visual invariant Phase 87.5 had.
- Zero free parameters (the `400` is the same Elo logistic scale FIDE PR uses) — no eyeballed K.
- Drops `Game.white_rating` / `Game.black_rating` SELECT, the `opp_rating_avg` accumulators, and the `endgame_opp_rating_avg` / `non_endgame_opp_rating_avg` schema fields added by Phase 87.6.
- Touched files enumerated in the note. Implementation delta is ~150 LOC net deletion + ~10 LOC change in `_compute_endgame_elo_weekly_series` + popover and LLM prompt copy rewrite + `_PROMPT_VERSION` bump to `endgame_v35`.
