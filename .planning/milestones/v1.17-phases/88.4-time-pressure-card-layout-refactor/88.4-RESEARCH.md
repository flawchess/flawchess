## Summary

Phase 88.4 is a pure frontend reshape of `EndgameTimePressureCard.tsx` and its section orchestrator. No backend changes, no new stats, no benchmark refresh. The three Success Criteria each touch a distinct concern: (SC-1) the CSS grid layout in `EndgameTimePressureSection.tsx`; (SC-2) restructuring the per-card "Remaining Time at Endgame Entry" header to a 3-column row above the Clock Gap bullet; and (SC-3) replacing four stacked `QuintileRow` bullet charts with one `ComposedChart` using Recharts `ErrorBar` on a `Line` with 4 ordinal x-axis datapoints and three `ReferenceArea` zone bands.

The biggest gray area is the bucket count: the ROADMAP SC-3 says "4 datapoints" and the screenshot confirms 4 visible rows (Q0ŌĆōQ3), but the backend always emits 5 quintiles (Q0ŌĆōQ4). The existing Phase 88 frontend already filters `quintile_index <= 3` (`MAX_VISIBLE_QUINTILE_INDEX = 3`), so "4 buckets" is the correct display count. The new line chart consumes the same filtered set. This is not a discrepancy ŌĆö it is a confirmed design decision (Phase 88 CONTEXT ┬¦2 A-4 / Plan 88-13 A-4).

The TC-collapse question for the zone band is settled by data: under opp-quintile semantics (the live delta formula), all 20 (TC, quintile) cells cap at ┬▒0.06 (┬¦3.3.3.b, benchmarks-latest.md lines 1169ŌĆō1192). The live `PRESSURE_BIN_SCORE_NEUTRAL_ZONES` already implements these bands. The user explicitly confirmed the collapse is acceptable (ROADMAP Phase 88.4 Context entry). The concrete action for the planner is to derive a per-quintile TC-pooled band by picking the most conservative (widest) bound across the 4 TCs ŌĆö which in practice collapses to `[ŌłÆ0.06, +0.06]` for all four displayed quintiles.

**Primary recommendation:** Build `ScoreGapByTimePressureChart` as a new self-contained `ComposedChart` component alongside `EndgameClockDiffOverTimeChart` (not a fork of it), reusing the same Recharts primitives and theme constants. The zone-band pattern from `EndgameClockDiffOverTimeChart` is directly portable; replace date XAxis with a `type="category"` XAxis using the 4 ordinal bucket labels, add `ErrorBar` as a child of `Line`, and strip the volume bar layer.

## Q2: Reuse Target ŌĆö `EndgameClockDiffOverTimeChart.tsx`

### Recharts construction

The chart is a `ComposedChart` inside `ChartContainer` (which wraps `ResponsiveContainer`). Key elements:

- **Zone bands:** Three `ReferenceArea` elements with `yAxisId="value"`:
  - Danger: `y1=yDomain[0]` to `y2=-NEUTRAL_PCT_THRESHOLD` with `fill={ZONE_DANGER}` at `fillOpacity={ZONE_OPACITY=0.15}`

### Recharts ErrorBar for CI whiskers

`ErrorBar` is available in recharts 2.15.x (confirmed in installed version). It is a child of `Line`. When `errorVal` is a 2-element array `[lowOffset, highOffset]`, it renders an asymmetric I-beam whisker. The offsets are distances from the data value (not absolute bounds).

For a datapoint with `delta=d`, `ci_low=lo`, `ci_high=hi`:
```tsx
// errorVal format: [distance_below_value, distance_above_value]
// ci_low and ci_high are absolute bounds (fractions), so:
errorVal = [d - lo, hi - d]  // both non-negative when CI contains the value
```

The `ErrorBar` needs a `dataKey` that points to this pair. Approach: compute `ci_error: [delta - ci_low, ci_high - delta]` in the data transform step (or use a `dataPointFormatter`). The `direction` prop defaults to `'y'` for a horizontal line chart (values on y-axis), which is correct.

**Null handling:** `ci_low` and `ci_high` can be null when `bin.n === 0`. The datapoint for a zero-n bin should either be omitted from the chart data or have `errorVal` set to `[0, 0]` to suppress the whisker (Recharts skips rendering if errorVal is falsy).

**Alternative whisker approach:** The existing `MiniBulletChart` implements a custom HTML/CSS whisker (not Recharts). For the line chart, Recharts `ErrorBar` is the standard approach and integrates directly with the `Line`'s coordinate system. It renders standard SVG `<line>` elements.

## Q4: Zone-Band Collapse Across TC

### ROADMAP claim and data support

ROADMAP ┬¦88.4 Context: "TC d_max within collapse threshold; bounds close enough to pool, user-confirmed."

The relevant benchmark section is **┬¦3.3.3.b** (benchmarks-latest.md lines 1134ŌĆō1223), which covers the **opp-quintile semantics** actually used by the live frontend (delta = user_score_in_user_Q ŌłÆ opp_score_in_opp_Q).

**Key finding from ┬¦3.3.3.b (lines 1169ŌĆō1192):** Under opp-quintile semantics, all 20 (TC, quintile) cells cap at ┬▒0.06 after the delta-IQR transform. The editorial cap `PRESSURE_BIN_NEUTRAL_CAP = 0.06` fully dominates; the raw IQR widths (0.166ŌĆō0.370) are ~2├Ś wider than the cap. **This means TC differences in the neutral band cannot be observed** ŌĆö every TC gives the same ┬▒0.06 band for every quintile.

**Concrete implication for the new chart:** The TC-collapsed neutral zone is `[ŌłÆ0.06, +0.06]` for all 4 displayed quintiles (Q0ŌĆōQ3). This is a symmetric band centered at 0.

### How to derive the single TC-collapsed band

The existing `PRESSURE_BIN_SCORE_NEUTRAL_ZONES` in `frontend/src/generated/endgameZones.ts` (lines 93ŌĆō101) already holds per-(TC, quintile) bands. For the zone-band line chart, the planner needs a per-quintile collapsed band. The derivation:

```
For each quintile q in {0, 1, 2, 3}:
  collapsed_min[q] = min(PRESSURE_BIN_SCORE_NEUTRAL_ZONES[tc][q].min for tc in {bullet, blitz, rapid, classical})
  collapsed_max[q] = max(PRESSURE_BIN_SCORE_NEUTRAL_ZONES[tc][q].max for tc in {bullet, blitz, rapid, classical})
```

Because ┬¦3.3.3.b confirms all cells cap at ┬▒0.06, this derivation yields `[-0.06, +0.06]` for every quintile. The planner can hardcode a single constant:

```typescript
// Derived from PRESSURE_BIN_SCORE_NEUTRAL_ZONES TC-collapse: all cells cap at ┬▒0.06
// Source: benchmarks-latest.md ┬¦3.3.3.b (opp-quintile rerun, 2026-05-17)
const PRESSURE_SCORE_GAP_NEUTRAL_MIN = -0.06;
const PRESSURE_SCORE_GAP_NEUTRAL_MAX = 0.06;
```

**HOWEVER ŌĆö important nuance for the line chart zone bands:** The line chart must show the same collapsed band regardless of which TC card the user is looking at. Since the chart is displayed inside a per-TC card, the band is per-quintile-collapsed across TC (a single symmetric band), not per-(TC, quintile). The `ReferenceArea` bands in the chart are therefore:

- Danger: below `ŌłÆ0.06`
- Neutral: `[ŌłÆ0.06, +0.06]`
- Success: above `+0.06`

These are y-axis threshold values, constant across all 4 x-axis positions. There are no per-x-position band jumps (unlike a grouped bar approach). This matches `EndgameClockDiffOverTimeChart`'s ReferenceArea pattern exactly (flat horizontal bands).

### Why the user confirmed collapse is acceptable

From ┬¦3.3.3.b line 1221: "Under opp-quintile semantics the per-user delta is intrinsically more variable... The ┬▒0.06 editorial cap fully dominates in all 20 cells." The TC differences in the raw IQRs are real (e.g., bullet has narrower raw IQR than classical in some quintiles) but they are all swamped by the cap. The user-visible behavior difference from showing a per-TC band vs a collapsed band is zero ŌĆö both would show ┬▒0.06 regardless.

### What ┬¦3.3.3 (cohort version) says about TC collapse

For completeness: ┬¦3.3.3 (lines 1032ŌĆō1044) shows that under the older cohort semantics, TC d_max per quintile ranged from 0.22 to 0.63 ŌĆö Q0 and Q2 were "keep separate". **This version is NOT what the live frontend uses** (Phase 88.1 retired cohort semantics). The ┬¦3.3.3.b rerun (opp-quintile semantics) is the relevant one, and it shows full ┬▒0.06 cap saturation with no meaningful TC differentiation.

## Q6: Patterns, Pitfalls, and Tests

### Recharts `ComposedChart` with ordinal x-axis

For a category x-axis, use `XAxis type="category"` (the default) with `dataKey="label"`. The 4 labels are ordinal strings; Recharts will render them at equal intervals. No tick formatter needed (the labels are already human-readable).

**The Y axis** should use `type="number"` with a fixed domain of `[-PRESSURE_DELTA_DOMAIN, PRESSURE_DELTA_DOMAIN]` (i.e., `[-0.30, 0.30]`) to match the existing bullet chart domain from `PRESSURE_DELTA_DOMAIN = 0.30` in `pressureBulletConfig.ts`. The tick formatter should sign-prefix positives: `(v) => v > 0 ? \`+${(v*100).toFixed(0)}%\` : \`${(v*100).toFixed(0)}%\``.

### Recharts `ResponsiveContainer` inside a TC card grid

Each TC card will contain a `ChartContainer` (which wraps `ResponsiveContainer`). When 4 cards are in a `xl:grid-cols-4` layout, each card is narrow. The chart height should be fixed (e.g., `h-48` or `h-40` ŌĆö smaller than the timeline chart's `h-72`) to fit without excessive vertical space per card. The existing `EndgameClockDiffOverTimeChart` uses `h-72` which would be too tall for a card that also has the header + bullet sections. The planner should choose a height that leaves the chart readable at ~200ŌĆō250px width on small breakpoints.

### `ErrorBar` with nullable CI

Recharts `ErrorBar` will not render a bar when `errorVal` is null or undefined. The data transform should use:
```typescript
const ciError: [number, number] | undefined =
  bin.ci_low != null && bin.ci_high != null
    ? [bin.delta - clampDeltaCi(bin.ci_low), clampDeltaCi(bin.ci_high) - bin.delta]
    : undefined;
```

For zero-n bins that are excluded from the chart (omitted from the data array), no special handling is needed for ErrorBar. If a zero-n quintile is shown as a gap in the line (`connectNulls={false}`), the line will break at that x-position.

### `noUncheckedIndexedAccess` compliance

When converting `PressureQuintileBullet[]` to chart data, use:
```typescript
const label = PRESSURE_LABELS[bin.quintile_index as 0|1|2|3];
```
or use a helper with a defensive check (as `pressureLabel()` already does in `EndgameTimePressureCard.tsx`).

### knip-clean requirement

SC-3 requires knip to stay clean after removing the 4 per-bucket bullet rows. The items at risk:

- `QuintileRow` ŌĆö currently unexported (module-private); removing it is fine for knip since it is only consumed within `EndgameTimePressureCard.tsx`
- `EmptyBinRow` ŌĆö same, module-private
- `pressureLabel()` function ŌĆö module-private
- `PRESSURE_LABELS` constant ŌĆö module-private
- `MetricStatPopover` ŌĆö still used in the new header row (moved from `QuintileRow` to the tooltip), but the info-icon instances in `QuintileRow` are removed. Check if `MetricStatPopover` is used elsewhere (it is ŌĆö `EndgameOverallEntryCard.tsx`, `EndgameTypeCard.tsx`)
- `MiniBulletChart` ŌĆö still used for the Clock Gap bullet; not orphaned

## Common Pitfalls

### Pitfall 1: ErrorBar Offset Direction

**What goes wrong:** `ErrorBar` `dataKey` expects offsets (distances from the value), not absolute bounds. Passing `[ci_low, ci_high]` directly (as absolute values) produces inverted or misaligned whiskers.

**Root cause:** The recharts `ErrorBar` renders `value ┬▒ offset`, not an absolute `[min, max]` interval.

**How to avoid:** Compute `[delta - ci_low_clamped, ci_high_clamped - delta]`. Both values should be non-negative when the CI properly brackets the point.

**Warning signs:** Whisker appears on the wrong side of the data point; whisker extends far beyond the chart boundary despite clamping.

### Pitfall 2: `connectNulls={false}` breaks at zero-n bins

**What goes wrong:** If a user has no games in quintile Q1 but has games in Q0 and Q2, the line chart shows a broken line (gap between Q0 and Q2). The chart needs to handle this gracefully ŌĆö either omit the zero-n datapoint from the data array (line breaks), or include it with `delta: null` (Recharts skips the segment).

**Root cause:** `connectNulls={false}` (the correct setting to avoid drawing through gaps) means zero-n bins produce a visual discontinuity.

**How to avoid:** Filter out zero-n bins from the data array. The x-axis will still show all 4 category labels because `type="category"` can include all labels independent of the data array ŌĆö OR accept the gap as informative (mirrors the "dash + no games" slot in the current bullet stack). The tooltip should not appear for missing bins.

**Warning signs:** Line jumps from Q0 directly to Q2, skipping Q1 tick ŌĆö the label appears but there is no data dot at that position.

### Pitfall 3: `noUncheckedIndexedAccess` on `PRESSURE_LABELS`

**What goes wrong:** `PRESSURE_LABELS[bin.quintile_index]` returns `string | undefined` when `quintile_index` is typed as `number`. TypeScript will error.

**Root cause:** `noUncheckedIndexedAccess` is enabled in the project.

**How to avoid:** Cast to the union type `(0|1|2|3)` only after the filter step that confirms `quintile_index <= 3`. Use `PRESSURE_LABELS[bin.quintile_index as 0|1|2|3] ?? ''` or reuse `pressureLabel()`.

### Pitfall 4: Grid layout with null-returning card children

**What goes wrong:** The grid div has a fixed number of columns (e.g., `grid-cols-4`), but `EndgameTimePressureCard` returns null for sub-threshold TCs. The grid renders visible cards but its `grid-cols-N` is based on payload count, not visible count. The backend already filters `data.cards` so this is not usually a problem ŌĆö but if the threshold changes or a TC card unexpectedly returns null, the grid can have empty cells.

**How to avoid:** SC-1 should use `data.cards.length` (the payload length) to set grid columns, not a hardcoded 4. Since the backend pre-filters by `MIN_GAMES_PER_TC_CARD`, `data.cards.length` is the visible count.

### Pitfall 5: Recharts ResponsiveContainer zero-size in tests

**What goes wrong:** `ChartContainer` (which wraps `ResponsiveContainer`) measures its parent via `ResizeObserver`. In jsdom, the parent has zero dimensions and the chart refuses to render, causing test queries to fail.

**Root cause:** jsdom has no layout engine.

**How to avoid:** Apply the same `vi.mock('recharts')` stub used in `EndgameClockDiffOverTimeChart.test.tsx` ŌĆö stub `ResponsiveContainer` with a fixed 800├Ś400 div and inject width/height into the child chart element via `cloneElement`.
