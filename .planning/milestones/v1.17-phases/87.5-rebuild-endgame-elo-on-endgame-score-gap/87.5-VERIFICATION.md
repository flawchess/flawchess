### 1. Visual chart relocation

**Test:** Visit `/endgames` after running `bin/run_local.sh` and importing a user with sufficient endgame history.
**Expected:** "Endgame ELO Timeline" chart appears in the "Endgame Overall Performance" section, **visually below** the "Endgame Score Gap over Time" chart, with no other chart interleaved between them. The "Endgame Metrics" section below contains only the three Conv/Parity/Recov gauges (no chart).
**Why human:** Compares rendered pixel layout, not just DOM order. The Vitest assertion locks DOM order via `compareDocumentPosition`, but a CSS rule (`order:`, `grid-template-rows: dense`, `flex-direction: column-reverse`) could rearrange visual order while DOM order stays unchanged. Eyes confirm the rendered result matches the documented intent.

### 2. Mobile 375px parity

**Test:** Open Chrome DevTools, switch to iPhone SE (375px) emulation, navigate to `/endgames`, scroll through the "Endgame Overall Performance" section.
**Expected:** The Endgame ELO Timeline chart appears below the Score Gap chart with the same vertical-stack order as desktop. No horizontal scroll, no chart clipped at the viewport edge, popover triggers reachable.
**Why human:** Mobile layout differences are rendered, not encoded. CLAUDE.md's "always apply changes to mobile too" rule requires both layouts to be exercised. Plan 02 noted that `Endgames.tsx` uses a unified responsive layout (no separate mobile branch), but a CSS rule for `max-width: 768px` or `@media` could still misbehave on the relocated chart.

### 3. LLM Insights narration uses restored metaphor

**Test:** Trigger an Endgame Insights regeneration (the `_PROMPT_VERSION` bump to `endgame_v33` invalidates the v32 cache, so the next render hits the LLM). Read the narration for the "Endgame Overall Performance" section, specifically the per-combo Endgame ELO Timeline bullets.
**Expected:** Bullets use phrasing like "your endgame is lifting your rating by X Elo" or "your endgame is holding back your rating by Y Elo" (or analogous wording). Not "what your ELO would be if everyone played the way you do" (the v32 framing). No `ΔES` / `delta_es` / `Conv ΔES` tokens appear in the user-facing narration.
**Why human:** LLM output is non-deterministic and the metaphor restoration is a prompt-quality assertion, not a deterministic invariant. Prompt-snapshot tests confirm the prompt text contains "lifts" / "holds back" tokens, but the LLM choosing to use them in narration must be eyeballed on real data.
