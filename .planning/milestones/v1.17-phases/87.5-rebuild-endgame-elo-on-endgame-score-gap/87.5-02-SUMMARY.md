## Decisions Made

1. **Section h2 fold-in: "Endgame Metrics and ELO" -> "Endgame Metrics".** RESEARCH.md Q3 + CONTEXT.md left this discretionary. Folded in because the "and ELO" clause is misleading after the chart moves out — deferring would have shipped a confusing header for the duration of the Phase 87.5 release window. The `sectionId="metrics_elo"` value stays unchanged (it maps to an internal LLM-payload Literal; Plan 03 owns the section grouping move).

2. **Used the actual `data-testid="endgame-score-timeline-chart"` for the positioning assertion.** The plan referred to it as `endgame-score-over-time-chart`; the wrapper has carried `endgame-score-timeline-chart` since Phase 68 (visible in `EndgameScoreOverTimeChart.tsx:179`). Cross-referenced before writing the assertion to avoid a phantom-selector failure.

3. **`frontend/src/lib/endgameMetrics.ts` not modified.** The plan's interfaces block listed a "Conversion ELO" comment at line ~70. The actual line 70 references the planning note `endgame-skill-dropped-conversion-elo.md` (a path-string referencing an existing file), not the rendered surface phrase. No `git diff` change is appropriate.

4. **Positioning-test fixture populated with one combo.** Plan 01's backend rename ships the field; the frontend page mounts the chart only when `eloTimelineData.combos.length > 0`. The Phase 85 Plan 05 fixture had carried an empty `combos: []`, which was fine for the legacy mock pattern but silently skipped the chart for the new positioning assertion. Promoting the fixture to one populated combo fixed the assertion without changing the chart wiring contract.

5. **Popover prose follows CLAUDE.md popover minimalism.** Phase 87.4's prose described Conv ΔES sigmoid framing with affine-recenter caveats; Phase 87.5's prose is WHAT + sign convention only ("Endgame ELO is your actual rating plus a small offset proportional to your Endgame Score Gap; positive lifts your rating, negative holds it back"). No Wilson / sigmoid / affine jargon.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Positioning-test fixture had empty combos; the chart wrapper never rendered.**

- **Found during:** Task 3 quality gates (the new positioning assertion failed: `expected 0 to be greater than or equal to 1`).
- **Issue:** The Phase 85 fixture set `conversion_elo_timeline: { combos: [], timeline_window: 100 }`. After Task 3 wired the relocated chart with the `eloTimelineData.combos.length > 0` guard, an empty combos array silently skipped the chart, so the positioning selector saw zero matches.
- **Fix:** Promoted the fixture's `endgame_elo_timeline` to one combo + one point. Both pre-existing tests (which only assert on score-gap + WDL surfaces) and the new positioning test now pass.
- **Files modified:** `frontend/src/pages/__tests__/Endgames.overallPerformance.test.tsx`.
- **Commit:** 706e9a1c (Task 3).

After reading, the file has no rendered "Conversion ELO" surface and no functional constant tied to the rename — its only "conversion-elo" string is a path to an existing planning note.
