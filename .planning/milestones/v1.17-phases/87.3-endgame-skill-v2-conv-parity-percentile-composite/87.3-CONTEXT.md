<decisions>
## Implementation Decisions

### Frozen percentile reference artifact

- **D-01: Empirical quantile table.** Store interpolated quantile knots of the per-user composite over the benchmark population. Map a user composite → percentile by interpolation. No normality assumption. (Rejected: full empirical CDF — finer than the signal warrants; μ/σ + Φ — assumes normality the Conv+Parity composite likely violates.)
- **D-02: Backend Python constant + manual gen script + CI drift-gate; NOT mirrored to TS.** The quantile table lives as a constant in a dedicated backend module (suggest `app/services/endgame_skill_reference.py` — planner finalizes). A gen script (parallels `scripts/gen_endgame_zones_ts.py`) regenerates it from the benchmark DB; CI runs `git diff --exit-code` on the **Python artifact itself** to block silent drift. The percentile is computed backend-side; the FE only consumes the resulting value + zone via the API/LLM payload.
- **D-03: Manual-refresh-only versioning.** A `REFERENCE_VERSION` constant is bumped in the same commit as any regeneration. The script regenerates only when run explicitly (intentional, reviewed). Historical Endgame ELO timeline points are non-drifting *by construction* — the artifact only changes on a deliberate commit (satisfies SC#3). (Rejected: auto-regenerate on benchmark rebuild — would drift historical points.)
- **D-04: FE gets only the Skill neutral-band zone constant** via the existing `endgameZones.ts` registry/codegen — no new TS mirror for the quantile table (no FE consumer; avoids dead-code/drift-gate surface).

### Recovery standalone tile — placement & anatomy

- **D-05: Keep the 3-col grid; reorder reading order to Conversion → Parity → Skill → Recovery.** Skill stays positioned under the two cards that feed it (Conv/Parity); Recovery drops *below* Skill as a clearly downstream descriptive card. No row collapse, no grouping boxes/connectors (per memory `feedback_mobile_layout_scope` — inside-card changes, keep the outer grid). Exact grid-cell placement (which row/col Recovery and Skill land in after the reorder) is a planner implementation detail; the locked decision is the *reading order* and *no structural grid change*.
- **D-06: Recovery tile keeps full card anatomy.** Retain both the Recov Save % rate gauge AND the ΔES Score Gap bullet (the bullet is SC#4's "standalone descriptive tile"). Only framing/heading/popover change to descriptive. Card anatomy stays consistent with Conv/Parity cards.

### Endgame Skill headline framing

- **D-07: Headline reads "Better than X% of players"** on a 0–100 gauge, 50 = typical (population midpoint). Most honest/self-explanatory. (Rejected: opaque "Skill Index" framing that hides the percentile basis; "Endgame Skill label + percentile subtext" — less direct.)
- **D-08: Editorial neutral band centered at 50, weak/typical/strong labels** consistent with the other Section 2 gauges (per memory `feedback_zone_band_judgement` — tighten when small percentile gaps are meaningful). **Exact band bounds are finalized in plan-phase** against the benchmark composite spread (the gap discriminates ELO at d≈1.39 per the LOCKED SPEC — band must respect that, not a naive symmetric 40–60).

### Recovery exclusion copy & LLM payload

- **D-09: Explicit one-liner, no hedging.** Recovery popover states it is opponent-dependent (reflects how often opponents failed to convert against you, not your defensive skill) and is deliberately excluded from Endgame Skill. One sentence, no methodology jargon (per memory `feedback_popover_copy_minimalism`). No "vs peer" framing implying it's a skill measure. (Rejected: soft "comeback" framing — too weak for SC#4; visual de-emphasis badge — not required, the one-liner suffices.)
- **D-10: Endgame Skill popover cites "the FlawChess Benchmark" + GitHub link.** Explicit user override of the generic "no internal naming" default: the popover names the FlawChess Benchmark as the percentile reference and links to `https://github.com/flawchess/flawchess/blob/main/reports/benchmarks-latest.md`. This is a deliberate transparency choice (labeled source link, not inline jargon). Planner wires the attribution + URL verbatim. The standard sigmoid caveat one-liner reused from Phase 87.1/87.2 still applies.
- **D-11: Prompt version bump + changelog.** Bump `_PROMPT_VERSION` to the next `endgame_vNN` (planner reads the current value at `app/services/insights_llm.py` — Phase 87.2 landed `endgame_v31`, so this is likely `v32`). Changelog entry narrates: Skill is now the Conv+Parity percentile composite; Recovery dropped (opponent-confounded) and reframed as a descriptive tile, with the confound rationale. Skill payload carries the percentile value + zone label; Recovery framing updated to descriptive language. **No parallel `verdict`/`p_value` field** (memory `feedback_llm_significance_signal` — the zone band IS the significance signal).
