<objective>
SC-1: Add inactivity-gap annotations to the two ordinal-axis over-time charts on `/endgames/stats` ("Endgame Score Gap over Time" and "Endgame ELO Timeline"). The x-axis stays ordinal-by-activity; a visible axis-break glyph plus a compact "≈Ny inactive" label is drawn at each gap between consecutive data points that exceeds the inactivity threshold.

Purpose: A long inactive stretch silently compresses into a single ordinal step today, misleading the reader. The annotation makes the compression honest without sacrificing chart resolution (decisions note Thread 2).
Output: New pure helper `frontend/src/lib/inactivityGaps.ts` + companion unit test (Nyquist Wave 0), and gap-annotation rendering wired into both chart components with added test cases.
</objective>

export const INACTIVITY_GAP_THRESHOLD_DAYS = 56;  // 8 weeks = ~2 months

export interface InactivityGap {
  afterIndex: number;   // gap occurs AFTER sortedDates[afterIndex]
  gapDays: number;
  label: string;        // "≈3.1y inactive" (>=365 days) or "≈4mo inactive" (< 365 days)
}

export function computeInactivityGaps(
  sortedDates: string[],
  thresholdDays?: number,   // defaults to INACTIVITY_GAP_THRESHOLD_DAYS
): InactivityGap[];

    - Single date `['2024-01-01']` → returns `[]`
    - All consecutive dates 7 days apart → returns `[]` (every gap below 56-day threshold)
    - Exactly one pair 57 days apart → returns one gap with `afterIndex: 0`, `gapDays: 57`
    - Gap >= 365 days → `label` matches `/^≈\d+(\.\d)?y inactive$/` (years, 1 decimal)
    - Gap < 365 days and > threshold → `label` matches `/^≈\d+mo inactive$/` (months, rounded)
    - `afterIndex` always points to the date BEFORE the gap (the last active point)
    - Custom `thresholdDays` argument overrides the default constant
  </behavior>

Iterate `i` from `0` to `sortedDates.length - 2`; compute `days = Math.round((new Date(sortedDates[i + 1]!).getTime() - new Date(sortedDates[i]!).getTime()) / 86400000)`; push `{ afterIndex: i, gapDays: days, label }` when `days > thresholdDays`. Label: `` `≈${(days / 365).toFixed(1)}y inactive` `` when `days >= 365`, else `` `≈${Math.round(days / 30)}mo inactive` ``. No imports, no React, no side effects (mirror signedBandGradient.ts).
