# Phase 87.1 Plan 02: Backend math + repo + service + schema — Summary

Per-class per-span Score Gap metric wired end-to-end on the backend. `query_endgame_entry_rows` now exposes next-span entry eval via a LEAD() window; `_aggregate_endgame_stats` accumulates per-class `gap = exit_score - ES_entry` vectors and pipes them through `compute_paired_difference_test`; 5 new fields on `EndgameCategoryStats` carry the result downstream to Plan 03 (frontend card) and Plan 04 (LLM payload).

## What Shipped

### Task 1 — Repository LEAD extension (commit 884589d8)

`query_endgame_entry_rows` row shape grew from 6 to 8 columns:

```
(game_id, endgame_class, result, user_color,
 eval_cp, eval_mate, next_entry_eval_cp, next_entry_eval_mate)
```

Implementation:
- Added `func.min(GamePosition.ply).label("span_min_ply")` to the existing `span_subq` SELECT (free against `ix_gp_user_endgame_game` since ply is the index sort key).
- Wrapped `span_subq` with a `span_with_next` subquery applying `func.lead(...)` over `partition_by=game_id, order_by=span_min_ply ASC` for both `entry_eval_cp` and `entry_eval_mate`.
- Main query joins `Game -> span_with_next` (was `span_subq`); SELECT projects two new LEAD columns. `apply_game_filters` stays on the main query (D-06).
- Docstring updated for the 8-column shape and the NULL semantics on terminal spans.

Existing callers (`_aggregate_endgame_stats`, `_compute_score_gap_material` at `endgame_service.py:792`) use attribute access on Row objects, so the extra columns are silently ignored — no regression for Phase 87 / Conv/Recov wiring.

Tests added in `tests/test_endgame_repository.py::TestQueryEndgameEntryRowsNextSpanEval`:
1. New columns accessible by attribute name.
2. LEAD ordering matches `span_min_ply ASC` across a 2-span game (rook span min_ply=20 lifts pawn span eval=−75 onto the rook row).
3. Terminal span returns NULL on the new columns.
4. Legacy 6-column attribute access preserved (regression guard).

Two pre-existing tests updated from positional unpacking to attribute access since the row shape grew.

### Task 2 — Service math + schema (commit be0150b0)

**`app/schemas/endgames.py`** — extended `EndgameCategoryStats` with 5 new fields:
```python
type_achievable_score_gap_mean:   float | None = None
type_achievable_score_gap_n:      int | None = None
type_achievable_score_gap_p_value: float | None = None
type_achievable_score_gap_ci_low:  float | None = None
type_achievable_score_gap_ci_high: float | None = None
```

All five default to `None` so existing `EndgameCategoryStats(...)` constructor call sites keep working. Comment cluster cites Phase 87.1 + SEED-016 D-05 and documents the dual-label rule (internal `achievable_score_gap` for math-family grep with Phase 85.1; user-facing "Score Gap" / "Endgame Type Score Gap").

**`app/services/endgame_service.py`** — three additions:

1. Module-level `_GAME_RESULT_TO_SCORE: dict[Literal["win","draw","loss"], float]` for terminal-span exit_score lookup (CLAUDE.md "no magic numbers"). Typed with `Literal` keys so a future caller indexing with an arbitrary string is caught by ty at edit time.

2. `_compute_span_gap(entry_eval_cp, entry_eval_mate, next_entry_eval_cp, next_entry_eval_mate, result, user_color) -> float | None` helper. Sign convention `exit_score − ES_entry` (positive = user outperformed). Returns `None` when both entry eval fields are NULL (D-07: skip span). Reuses `eval_cp_to_expected_score` / `eval_mate_to_expected_score` (mate saturation preserved).

3. `_aggregate_endgame_stats` loop body restructured: iterates `for row in rows:` and branches on `isinstance(row, tuple)` to absorb both 8-column SA Rows (prod) and 6-tuple legacy test fixtures (next-eval padded NULL). Per-class gap vectors accumulate into `gaps_by_class: dict[EndgameClass, list[float]]`. The per-class builder loop calls `compute_paired_difference_test(gaps_by_class[endgame_class])` after `score_p_value` and threads the 4-tuple result + `n` into the `EndgameCategoryStats(...)` kwargs. Mean is surfaced as `None` (not `0.0`) when `n == 0` so the frontend's hide-row gate has a clean sparse signal.

Tests added in `tests/test_endgame_service.py`:
- `TestComputeSpanGap` (9 tests): NULL entry eval → None; pure transitory (white perspective, hand-computed reference); pure terminal win/draw/loss; mate at entry; mate at next-entry; mate precedence over cp at entry; sign-convention anchor (ES_entry < 0.5 < exit → gap > 0).
- `TestAggregateEndgameStatsTypeScoreGap` (9 tests): 6-tuple regression guard; all 5 fields present; n=0 when all NULL eval; NULL-eval exclusion preserves WDL count; p_value gated below n=10; p_value populated at n=10; hand-computed mean matches reference to 1e-6; per-class accumulation is independent; mate at endpoint handled.

## Math Contract Honored (LOCKED — CONTEXT.md)

```
gap_span = exit_score_span − ES_sigmoid(entry_eval_span, user_perspective)
```

- `exit_score_span` (transitory) = `ES_sigmoid(entry_eval_of_next_span, user_color)` — driven by the LEAD column.
- `exit_score_span` (terminal) = `_GAME_RESULT_TO_SCORE[derive_user_result(result, user_color)]` → 1.0 / 0.5 / 0.0.
- NULL entry eval span: excluded from cohort (returns `None` from `_compute_span_gap`).
- Aggregate: `mean(gap_span)` per `(user, endgame_class)`; paired one-sample two-sided z-test via `compute_paired_difference_test`.
- Sign convention: `actual − expected`. Positive = outperformed Stockfish. Matches `higher_is_better` on `achievable_score_gap` ZoneSpec.
