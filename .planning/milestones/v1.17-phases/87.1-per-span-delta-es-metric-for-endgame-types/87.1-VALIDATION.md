## Nyquist Sampling — Discrete Output Signals

Per RESEARCH.md §Validation Architecture, the executor / plan-checker sample these discrete signals:

**Math helper (per-span gap unit tests)** — synthetic span vectors covering CONTEXT.md D-11 boundary cases:
- Pure transitory span (next_entry_eval populated, both endpoints non-NULL)
- Pure terminal span (next_entry_eval NULL, game_result drives exit_score)
- Mixed transitory + terminal in same cohort
- Mate at entry (mate_in sign maps via `_signed_pawns_from_mate` → sigmoid clamps)
- Mate at next-entry (mate at exit)
- Mate at game-result terminal (no eval needed)
- All-NULL-eval span (excluded from cohort — n decrements)
- Single-span cohort (n=1: returns mean, p_value=None, ci=None per CONFIDENCE_MIN_N gate)
- Empty cohort (n=0: returns mean=0.0, p/CI=None)
- Variance-0 cohort (all identical diffs — already trapped by `compute_paired_difference_test`)

**Service-layer integration** — fixture with 2-3 games per class:
- All 5 new fields populated on `EndgameCategoryStats` for each rendered class
- Hand-computed reference matches `type_achievable_score_gap_mean` to 1e-6
- p_value None at n < CONFIDENCE_MIN_N (=10); ci None at n < 2
- Mirror flip NOT involved (no peer comparison) — simpler than Phase 87 D-02

**Repo query** — `query_endgame_entry_rows` extended with `next_entry_eval_cp/mate`:
- Window-function LEAD ordering matches `span_min_ply ASC` per game
- Terminal spans return NULL `next_entry_eval_*`
- Existing callers (Phase 87 categories builder + conv/recov path) unaffected — extra columns ignored by attribute access
- Filter predicates (`apply_game_filters`) still apply

**Codegen drift gate (CI)** — `frontend/src/generated/endgameZones.ts`:
- Contains `ENDGAME_TYPE_SCORE_GAP_NEUTRAL_MIN/MAX` (or per-class equivalent map per RESEARCH §D-04 decision)
- `MetricId` Literal in `endgame_zones.py` includes `"endgame_type_achievable_score_gap"`
- `PER_CLASS_GAUGE_ZONES` (or sibling) carries the new band per rendered class
- `scripts/gen_endgame_zones_ts.py --check` exits 0

**Frontend Vitest (`EndgameTypeCard.test.tsx`)** — extend with:
- Row renders at `n >= CONFIDENCE_MIN_N` with value + CI whiskers + sig-gated color
- Row renders at `n < CONFIDENCE_MIN_N` with value but neutral font color
- Row hidden at `n == 0` (or wrapped in opacity-50 per Phase 87 D-13)
- `MetricStatPopover` opens via help trigger; explanation copy contains "Score Gap" + sigmoid-bias caveat
- Testid sub-elements present: `${tileTestId}-asg-bullet`, `${tileTestId}-asg-value`, `${tileTestId}-asg-info`

**LLM payload (`test_insights_llm.py`)** — extend with:
- `type_breakdown.<class>.type_achievable_score_gap` block present under sparse cohort (n < gate → `mean: None, zone: null`)
- Same block present under full cohort with `{ mean, n, zone, neutral_band }` shape
- Glossary block defines "Endgame Type Score Gap" terminology (per D-10)
- Prompt version bumped from `endgame_v28` → `endgame_v29` (per RESEARCH Open Question #2)
