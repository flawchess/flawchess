## Pattern Assignments

### `app/services/score_confidence.py` — new helpers `compute_score_difference_test` and `compute_paired_difference_test`

**Role:** service utility (pure math), transform data flow

**Analog for `compute_score_difference_test`:** `app/services/eval_confidence.py` (full file, lines 55-141)

The `compute_eval_confidence_bucket` function is the closest structural template: it computes a Wald-z test, handles the SE=0 degenerate case, gates p-value and CI independently by n, and returns a tuple. The new helper returns `(p_value, ci_low, ci_high)` — a subset of that tuple shape (no confidence bucket, no mean).

**Imports pattern** (lines 54-63 of `score_confidence.py`):
```python
import math
from typing import Literal

from app.services.opening_insights_constants import (
    OPENING_INSIGHTS_CI_Z_95 as CI_Z_95,
    OPENING_INSIGHTS_CONFIDENCE_HIGH_MAX_P as CONFIDENCE_HIGH_MAX_P,
    OPENING_INSIGHTS_CONFIDENCE_MEDIUM_MAX_P as CONFIDENCE_MEDIUM_MAX_P,
    OPENING_INSIGHTS_CONFIDENCE_MIN_N as CONFIDENCE_MIN_N,
    OPENING_INSIGHTS_SCORE_PIVOT as SCORE_PIVOT,
)
```
The new helpers also need `CI_Z_95` and `CONFIDENCE_MIN_N` — import via the same alias chain already in scope.

**Trinomial variance formula** (lines 219-220 of `score_confidence.py`):
```python
empirical_variance = max(0.0, (w + 0.25 * d) / n - score * score)
se = math.sqrt(empirical_variance / n)
```
`compute_score_difference_test` reuses this formula per side (once for `eg`, once for `ne`). The variance formula is already annotated inline with its derivation.

**Variance-0 / SE=0 trap pattern** (lines 116-119 of `eval_confidence.py`):
```python
if se == 0.0:
    # Degenerate: all evals identical. mean != baseline -> perfectly determined
    # result (z = +-inf, two-sided p = 0.0); mean == baseline -> p = 1.0.
    p_value = 0.0 if mean != baseline_cp else 1.0
else:
    z = (mean - baseline_cp) / se
    p_value = math.erfc(abs(z) / math.sqrt(2.0))
```
For `compute_score_difference_test`, SE_diff = 0 → check `score_eg == score_ne`; return `p_value = 1.0` if equal, `0.0` otherwise. Exact same pattern, different variable names.

**Bessel-corrected variance + Wald CI + SE=0 trap** (lines 110-128 of `eval_confidence.py`):
```python
# Bessel-corrected sample variance: (sumsq - n * mean^2) / (n - 1).
# Clamped at 0.0 to handle floating-point rounding where the raw value is
# slightly negative despite mathematically being non-negative.
variance = max(0.0, (eval_sumsq - n * mean * mean) / (n - 1))
se = math.sqrt(variance / n)

if se == 0.0:
    p_value = 0.0 if mean != baseline_cp else 1.0
else:
    z = (mean - baseline_cp) / se
    p_value = math.erfc(abs(z) / math.sqrt(2.0))

ci_half_width = CI_Z_95 * se
```
`compute_paired_difference_test` reuses this pattern verbatim (Bessel-corrected sample variance on diffs, same SE=0 trap, same Wald CI). The only change: return `(mean_d, p_value, ci_low, ci_high)` where `ci_low = mean_d - CI_Z_95 * se` and `ci_high = mean_d + CI_Z_95 * se` (not symmetric half-width, but explicit bounds like the `entry_eval_ci_low/high` convention at lines 1748-1754 of `endgame_service.py`).

**n=0 and n=1 guard pattern** (lines 98-108 of `eval_confidence.py`):
```python
if n <= 0:
    return "low", 1.0, 0.0, 0.0

mean = eval_sum / n

if n == 1:
    return "low", 1.0, mean, 0.0
```
For `compute_paired_difference_test(diffs: Sequence[float])`: guard `len(diffs) == 0` → return `(0.0, None, None, None)`; `len(diffs) == 1` → return `(diffs[0], None, None, None)` (Bessel undefined at n=1).

**n-gate on wire-format p_value** (lines 1744-1753 of `endgame_service.py`):
```python
entry_eval_p_value: float | None = p_eval_raw if eval_n >= 10 else None
# ...
if eval_n >= 2:
    entry_eval_ci_low_pawns = (mean_cp - ci_half_cp) / 100.0
    entry_eval_ci_high_pawns = (mean_cp + ci_half_cp) / 100.0
else:
    entry_eval_ci_low_pawns = None
    entry_eval_ci_high_pawns = None
```
Both new helpers encapsulate these gates internally, returning `None` directly rather than leaving gating to the caller (cleaner than `eval_confidence.py` which returns a raw value and requires the service to gate). The `CONFIDENCE_MIN_N = 10` constant already imported in `score_confidence.py` provides the p-value gate; CI gate is `n < 2`.

**Helper signature templates** — copy structure from `compute_eval_confidence_bucket` (lines 55-57):
```python
def compute_eval_confidence_bucket(
    eval_sum: float, eval_sumsq: float, n: int, baseline_cp: float = 0.0
) -> tuple[Literal["low", "medium", "high"], float, float, float]:
```
New helpers:
```python
def compute_score_difference_test(
    eg_w: int, eg_d: int, eg_l: int, eg_n: int,
    ne_w: int, ne_d: int, ne_l: int, ne_n: int,
) -> tuple[float | None, float | None, float | None]:
    """Return (p_value, ci_low, ci_high). All None when gate fails."""

def compute_paired_difference_test(
    diffs: Sequence[float],
) -> tuple[float, float | None, float | None, float | None]:
    """Return (mean_d, p_value, ci_low, ci_high)."""
```
Note: `Sequence` (not `list`) per CLAUDE.md — list is invariant, Sequence is covariant.
