## Decisions Made

- **Null pValue UX.** Chose silent omission of the `(p = …)` parenthetical over an explicit `(p unavailable)` phrase. The headline (`Inconclusive.` / `Possibly a real strength.`) already conveys the reliability state, and the parenthetical was always a supporting detail. Adding a "unavailable" phrase would read more clinically than user-facing copy elsewhere in the popover. Documented with an inline comment at the seam.
- **Per-row mock disambiguation via data-aria-label.** Both rows render through the same `MiniBulletChart` mock and the same `mock-mini-bullet` testid. The section test needed to attribute CI props to a specific row, so I extended the mock to surface `ariaLabel` as a `data-aria-label` data-attr. This is consistent with the existing `data-domain` / `data-center` pattern already in the mock and avoids fragile reliance on DOM order. The mock is now richer but still entirely test-scoped.
- **`?? undefined` coercion at the call site** (matches the plan literal). The schema returns `number | null` at the gated fields; `ScoreGapRow.ciLow` / `ciHigh` are `number | undefined`. The cleanest seam is `?? undefined` at the boundary so both `null` and missing-key cases collapse to the same downstream behavior (no whisker). This mirrors how `EndgameOverallEntryCard` already coerces `entry_expected_score_ci_low/high`.
