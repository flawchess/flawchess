## Skill card cohort

User flagged a sharp objection to the ROADMAP's pooled-all-spans recommendation: "Would the Pooled all-spans mean based Endgame skill be basically the same as the Mean Score of Games with an Endgame? That feels anti-climactic."

Claude verified the concern: by telescoping over per-game spans, `Σ_spans (exit_score − ES_entry) = game_result − ES_first_entry`. The page-level Achievable Score Gap (Phase 85.1) is exactly `actual − expected` per game = `game_result − ES_at_first_endgame_entry` — averaged per game. So pooled all-spans mean ΔES = span-count-reweighted version of Achievable Score Gap. Near-duplicate signal to a card that already sits in Section 1.

Reformulated the options to give the user a non-overlapping signal.

| Option | Description | Selected |
|--------|-------------|----------|
| Equal-weighted mean of 3 bucket means | `(ΔES_conv + ΔES_parity + ΔES_recov) / 3`. Mirrors existing rate-based Skill = mean(Conv, Parity, Recov). Independent from Achievable Score Gap (each bucket has its own anchor). | ✓ |
| Games-weighted mean of 3 bucket means | Same 3 buckets, weighted by n_spans per bucket. Still independent; loses equal-importance framing. | |
| Drop the Skill ΔES bullet | Skill card keeps composite gauge only; siblings carry the ΔES signal. Avoids overlap entirely; loses one-number composite. | |
| Pooled all-spans mean (original ROADMAP) | Accept near-duplication with Achievable Score Gap. Simplest math. | |

**User's choice:** Equal-weighted mean of 3 bucket means
**Notes:** Mirrors the existing rate-based Skill formula 1-to-1 (just swaps Conv/Parity/Recov rates for their per-bucket ΔES means). Per-bucket means CI propagation pattern is planner's call — see CONTEXT D-01.
