### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed ty type inference for metric_id in loop**
- **Found during:** Task 1 GREEN (ty check)
- **Issue:** `for bucket_id, metric_id in [("conv", "section2_score_gap_conv"), ...]` infers `metric_id` as `str`, not `MetricId`, causing `invalid-argument-type` on `SubsectionFinding.metric` and `assign_zone()`.
- **Fix:** Changed the loop variable annotation to `list[tuple[str, MetricId]] = [...]` so ty infers the correct Literal type.
- **Files modified:** app/services/insights_service.py
- **Commit:** 68700409

**2. [Rule 3 - Style] Removed `# ty: ignore[attr-defined/arg-type]` comments**
- **Found during:** Task 1 GREEN (ty check reported `ignore-comment-unknown-rule` for `attr-defined` and `arg-type`)
- **Fix:** ty uses rule names like `invalid-argument-type`, not mypy's `attr-defined`/`arg-type`. The typed loop approach eliminated the need for any ty ignores.
- **Commit:** 68700409

**3. [Style] Replaced em-dash in REQUIREMENTS.md heading with colon**
- **Found during:** Task 3 (acceptance criterion `grep -c '^+.*—' ... returns 0` would fail with em-dash heading)
