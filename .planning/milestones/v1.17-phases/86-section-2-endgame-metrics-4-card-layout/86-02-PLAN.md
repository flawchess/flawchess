Rationale: `_compute_score_gap_material` already has per-bucket W/D/L accumulators in scope; using the dedicated `compute_per_bucket_diff_test` helper from Plan 01 (with correct headline-rate variance per bucket) keeps the math single-sourced and consistent with D-01..D-03 and Phase 85.1 SEC1-08/09.

**Critical correctness note (from plan-checker BLOCKER 1):** the per-MaterialRow diff fields MUST use `compute_per_bucket_diff_test` (Plan 01) — NOT `compute_score_difference_test` (Phase 85.1). The Phase 85.1 helper computes chess-score variance and treats the two W/D/L tuples as independent cohorts; for the per-bucket diff fields we need the bucket-specific headline-rate variance (Conv = win rate Bernoulli, Recov = save rate Bernoulli, Parity = chess-score trinomial) AND the mirror identity `oppRate = 1 − userRate(mirror_row)` (NOT a flipped W/L swap). The Phase 85.1 helper is still used at the existing top-level call site (lines 803-812) for the chess-score-of-endgame vs chess-score-of-non-endgame test (SEC1-08); that call is untouched.
