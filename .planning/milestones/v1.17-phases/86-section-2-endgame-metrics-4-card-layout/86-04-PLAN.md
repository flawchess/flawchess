<objective>
Build the two sibling card components for Phase 86's 4-card layout, mirroring the Phase 85 sibling-component pattern (`EndgameOverallCard.tsx` is the direct template):

1. **`EndgameMetricCard`** — shared shell for Conversion, Parity, Recovery cards. Layout: gauge (using `FIXED_GAUGE_ZONES[bucket]` per SEC2-04) → games-count row with `Swords` icon (`Games: {sharePct}% ({total})` per D-15) → `MiniWDLBar` → peer-bullet row (`You: X% · Opp: Y% · Diff: ±Zpp` text + `MiniBulletChart` for the signed diff vs 0). Per-card `MetricStatPopover` with content locked at D-16. Sig-gating triple applied to the diff-percent font color only (D-14).

2. **`EndgameSkillCard`** — Skill variant. Layout: gauge (using `ENDGAME_SKILL_ZONES`) → games-count row (just `Games: {totalGames}` per D-15) → peer-bullet row (`Your Skill: X% · Opp Skill: Y% · Diff: ±Zpp` + `MiniBulletChart`). **No `MiniWDLBar`** (single-ply composite has no W/D/L definable per SEC2-03). Per-card `MetricStatPopover` for the Skill composite explanation.

Cards are presentation-only: all data + sig fields come from props. The orchestrator in Plan 05 derives them from `ScoreGapMaterialResponse` (Conv/Parity/Recov via `row.diff_p_value` / `row.diff_ci_low` / `row.diff_ci_high` from Plan 02 wire; Skill via `data.skill_diff_p_value` etc.).

Purpose: Plan 05 mounts these into the new orchestrator. Splitting cards from orchestrator lets the orchestrator focus on grid composition + connector-arrow plumbing.

Output: 2 new components (~100 LOC + ~70 LOC), 2 new test files (~80 LOC each).
</objective>
