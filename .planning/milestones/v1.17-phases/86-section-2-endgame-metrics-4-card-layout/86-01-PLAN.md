<objective>
Add TWO backend math helpers in `app/services/score_confidence.py` that power Phase 86's peer-bullet sig tests:

1. **`compute_skill_diff_test`** (D-01 / SEC2-08) — Wald-z on the aggregate Skill composite. Used for the Skill card's peer bullet.
2. **`compute_per_bucket_diff_test`** (D-05 / D-06 / SEC2-06) — Wald-z on a single bucket's `userRate − opponentRate` headline-rate diff. Used for the Conversion / Parity / Recovery cards' peer bullets.

**Critical math fix from plan-checker BLOCKER 1 + 2:** both helpers compute variance using the bucket's HEADLINE-RATE distribution, not the chess-score distribution. Conversion is Bernoulli on wins (`p = W/N`, `var = p(1-p)`); Recovery is Bernoulli on saves (`p = (W+D)/N`, `var = p(1-p)`); Parity uses the trinomial chess-score variance `var = max(0, (W + 0.25·D)/N − p²)`. The chess-score variance is only correct for Parity.

**Critical mirror-identity fix:** for both helpers, the "opp_row" argument is the USER'S row in the mirror bucket (per `EndgameScoreGapSection.tsx:141-146` legacy convention — same-game symmetry means we never observe the opponent's W/D/L directly; we observe the user's W/D/L in the mirror bucket and invert via `oppRate = 1 − userRate(mirror_row)`). The helpers compute `1 − userRate(opp_row)` internally; do NOT flip W↔L at the call site.

Purpose: D-01 + D-05 + D-06 locked these math choices. This plan ships the pure-math layer with boundary tests — no schema, no service wiring, no frontend yet. Plan 02 wires both helpers from `_compute_score_gap_material`.

Output: Two new functions in `score_confidence.py` (~70 LOC + ~50 LOC including docstrings), two new test classes in `tests/services/test_score_confidence.py` (~6 + ~5 test methods).
