# Phase 94.1 Plan 10: _compute_and_count classifier IN-03 fix Summary

**One-liner:** `_compute_and_count` now reaches all three summary counters (upserted / skipped_below_floor / skipped_no_canonical_games) by returning False when the post-compute row has `percentile IS NULL`, and the dead `row_before` pre-probe is removed.

## What Was Built

### Task 1: Classifier fix + 5 unit tests (TDD)

**File:** `scripts/backfill_user_percentiles.py` (`_compute_and_count`, lines 401-453 → 401-451)

Three changes to the classifier:

1. **Removed the dead `row_before = await _row_exists(...)` pre-probe** at line 419. Its result was never used in the return path (`return True` was returned regardless of whether the row pre-existed). Pure dead code costing one DB round-trip per (user, metric) pair. Insert-vs-update is not observable from outside the UPSERT anyway — the only meaningful states are "row exists with percentile" / "row exists with NULL percentile" / "no row".

2. **Added the `return False` branch** when the post-compute row exists but `pctl_row.percentile is None`. This is the canonical "below inclusion floor" outcome from `app/services/user_benchmark_percentiles_service.py` (a row is still written, but `percentile` is NULL so the chip won't render). Before this fix, `_backfill_user`'s `else` branch at line 378 was unreachable and below-floor users were silently mis-counted as `upserted`.

3. **Updated docstring + inline comments** to spell out the three-branch contract:
   - `True` → row upserted with non-null percentile
   - `False` → row exists but percentile IS NULL (below floor)
   - `None` → no row written (zero canonical-slice games)

   IN-03 comment cites the rule and pins the fix to the consumer at `_backfill_user:378/:398`.

**File:** `tests/scripts/test_backfill_user_percentiles_classifier.py` (new — 5 tests)

- `test_returns_true_when_row_written_with_non_null_percentile` — mocks `_row_exists=True` + percentile=42.0 → asserts True.
- `test_returns_false_when_row_written_with_null_percentile` — mocks `_row_exists=True` + percentile=None → asserts False (the previously unreachable branch).
- `test_returns_none_when_no_row_written` — mocks `_row_exists=False` → asserts None.
- `test_stage_b_routes_to_compute_stage_b` — smoke check that the stage-routing arm still calls `compute_stage_b` and NOT `compute_stage_a` when `stage="B"`.
- `test_backfill_user_increments_skipped_below_floor_when_classifier_returns_false` — consumer-level: drives `_backfill_user` with `_compute_and_count` mocked to return False; asserts `summary[STAGE_A_METRIC].skipped_below_floor == 1` (and `upserted == 0`, `skipped_no_canonical_games == 0`).

All tests are pure unit tests — no real DB. `_row_exists`, `compute_stage_a`, `compute_stage_b`, and the inline percentile SELECT session are mocked via `unittest.mock.AsyncMock` + `MagicMock` with `__aenter__`/`__aexit__`. Real-DB coverage of the classifier already lands via the Plan 09 integration tests + the backfill HUMAN-UAT, so the unit-test ceiling here is the right altitude.
