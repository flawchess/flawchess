# Phase 94.1 Plan 13: Drop n_cells_floor + apply_floor fix + Stage B gate Summary

Closes two correctness bugs surfaced during Plan 11 HUMAN-UAT: (1) `_compute_metric_for_user` used `apply_floor=False` causing below-floor cells to distort reported values; (2) Stage B re-fired during active imports producing transient partial percentiles.

## What Was Built

### Bug 1 Fix: Single-query apply_floor=True

`_compute_metric_for_user` previously ran two CTEs: one with `apply_floor=False` for the stored `value`, and one with `apply_floor=True` to count `n_cells_floor`. This caused the stored value to include tiny below-floor cells (n=1, n=3) that were excluded from the benchmark CDF. User 28's `achievable_score_gap` showed Top 6% (+0.1204) vs the floor-respecting Top 71% (-0.0322).

The fix collapses to a single `apply_floor=True` query. The returned value is now the same weighted average the CDF uses. If the CTE returns no rows (user has zero floor-passing cells for this metric), the function returns `None` and no row is written. Row absence is the new below-floor signal.

### Bug 2 Fix: Stage B active-import gate

`users_with_zero_pending` now checks two conditions: (1) no pending evals AND (2) no active import job (`status IN ('pending', 'in_progress')`). This prevents Stage B from firing during import, where the pending-eval count flickers between zero and non-zero as batches drain. User 28's `achievable_score_gap` was observed to flip between -0.0511 and +0.1204 in a 20-second window due to this re-fire pattern.

### Schema Change: Drop n_cells_floor

The `user_benchmark_percentiles.n_cells_floor` column is dropped via reversible Alembic migration. It was write-only from production code's POV (only compared `== 0` as a binary gate). With the floor-respecting value query, the gate becomes implicit via row-absence.

### Backfill Script Simplification

`backfill_user_percentiles.py` removes the `skipped_below_floor` counter and the `False` return branch from `_compute_and_count`. Both zero-game and below-floor users now return `None` and increment `skipped_no_canonical_games`.
