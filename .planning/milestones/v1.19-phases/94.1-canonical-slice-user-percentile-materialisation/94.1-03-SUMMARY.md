## Decisions Made
- `sparse_exclusion_sql(elo_col, tc_col)` takes positional (not keyword-only) args — Plan 01's test called it as `fn("elo_bucket", "tc_bucket")` with positional args, matching the original script's signature
- `apply_floor=False` drops only the per-user inclusion-floor HAVING (>= 20 or >= 30 games), not the structural `HAVING count(*) >= 6` inside `endgame_game_ids`/`spans` CTEs — the Test 7 assertion was updated to check specific floor HAVING fragments rather than any HAVING presence
- The script's `INCLUSION_FLOOR_*` constants are kept as aliases pointing to the imported shared-module constants to avoid breaking the `_registry_entry_comment` non-SQL code path
- Only `selected_users_cte` and `per_user_cte_for` are imported into the script (ruff F401 removed unused per-metric builders that are called internally via dispatch)
