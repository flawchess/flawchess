<objective>
Close the CRITICAL runtime bug shipped in Phase 94.1 (VERIFICATION.md gap #1 + #2): `selected_users_cte(source="single_user")` emits `SELECT :user_id::int AS user_id`, which SQLAlchemy's `text()` tokeniser silently fails to recognise as a bindparam because of the immediately-following `::int` Postgres cast operator. Every `.bindparams(user_id=user_id)` call raises `ArgumentError` at runtime; the exception is swallowed by the Stage A/B Sentry capture, so no row is ever written and the failure is invisible. Tests do not catch it because the entire happy-path test module is `pytest.skip("implementation pending Plans 05/06")` and the SC-7 parity test is `pytest.skip("implementation pending Plan 03")`.

This plan: (a) replaces `:user_id::int` with `CAST(:user_id AS int)`, (b) lifts every skip marker in the two test modules, (c) adds a new real-data integration test that seeds a user with canonical-slice-qualifying games and asserts `compute_stage_a` / `compute_stage_b` actually persist rows, and (d) gates final acceptance on a HUMAN-UAT re-run of the dev backfill confirming high-game-count users (user_id=8 has 18,226 equal-footing rated games) flip from `no_canonical_games` to `upserted=1`.

Purpose: Unblock Stage A/B at runtime so the chip lights up; install regression tests that would have caught the bug pre-ship.
Output: 1-token SQL fix + 2 unblocked test modules + 1 new integration test module + dev backfill HUMAN-UAT.
</objective>
