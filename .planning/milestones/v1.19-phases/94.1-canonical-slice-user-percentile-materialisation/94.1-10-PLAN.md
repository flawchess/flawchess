<objective>
Close VERIFICATION.md gap #3 / REVIEW.md IN-03: the backfill script's per-user outcome classifier `_compute_and_count` (`scripts/backfill_user_percentiles.py:401-453`) returns only `True` or `None` — never `False`. The downstream `_backfill_user` logic at lines 373-378 maps `False` to the `skipped_below_floor` counter, but no code path can produce it, so below-floor users are mis-classified as `upserted`. Additionally, `row_before` is queried but unused — pure dead code that costs one round-trip per (user, metric) pair.

This plan: (a) makes `_compute_and_count` return `False` when the post-compute row exists but `percentile IS NULL` (the canonical below-floor outcome), (b) removes the unused `row_before` probe OR uses it to distinguish insert-from-update (planner picks "remove" — simpler, fewer queries, matches IN-03 recommendation), and (c) adds a unit test that drives all three branches.

Purpose: Restore meaningful per-skip-reason reporting in the backfill summary so operators can distinguish "user has no canonical games" from "user has games but doesn't meet the floor" at rollout time.
Output: 1 function rewrite + 1 new unit test module.
</objective>
