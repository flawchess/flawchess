<objective>
Reshape the `user_rating_anchors` table per the D-12 Reversal Amendment (CONTEXT.md, 2026-05-27). The amendment locks a game-weighted blended anchor that pools converted-chess.com ratings with native-lichess ratings into a single median. The schema must shed the columns that modelled the old per-platform-source rule (`source_platform`, `chesscom_raw_rating`) and surface the new tooltip-disclosure columns (`n_chesscom_games`, `n_lichess_games`, `chesscom_median_native`, `lichess_median_native`).

Purpose: This is the foundation plan for the D-12 reversal. The Alembic migration TRUNCATEs both derived tables (`user_rating_anchors` and `user_benchmark_percentiles`) and reshapes the columns. The ORM model and repository are rewritten to match. The CONSUMER files (`app/services/user_benchmark_percentiles_service.py`, `app/schemas/endgames.py`, `frontend/src/components/charts/PercentileChip.tsx`) are LEFT BROKEN — they reference `source_platform` / `chesscom_raw_rating` and will fail their imports until Plans 10 + 11 land. This is an intentional Wave 1 broken-state cluster (Plans 09 + 10 ship together as a single PR; the executor MUST NOT push 09 alone).

Output: 1 Alembic migration (NEW) + 1 ORM model file rewrite + 1 repository file rewrite + 1 test file rewrite. Plan 10 (anchor compute rewrite) lands in the same Wave 1, with both Plans rolling up into the cross-stack atomic cutover PR.
</objective>
