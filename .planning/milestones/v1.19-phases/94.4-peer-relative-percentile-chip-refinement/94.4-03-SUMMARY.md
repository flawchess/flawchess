# Phase 94.4 Plan 03: Per-TC ΔES Builders + Pitfall 1 Widening Summary

Extends `app/services/canonical_slice_sql.py` with 4 new per-TC ΔES builders (3 new functions plus a Literal widening on the existing section2 builder), applies the Pitfall 1 user_id widening to all 7 per-TC builders (3 existing 94.3 + 4 new 94.4), and locks the source-mode parity invariant across 32 new parity cases. Plan 03 is the first commit in the Wave 2 atomic-cutover quartet (Plans 03 → 04 → 05 → 06) — Plan 04 generates the cohort CDF against these builders, Plan 05 consumes them per-user, Plan 06 backfills.

## What Shipped

### 4 new per-TC ΔES builders (Task 2)

| Builder | Signature | Mirrors |
|---------|-----------|---------|
| `per_user_cte_score_gap_tc` | `(tc, *, source, snapshot_date)` | non-per-TC `per_user_cte_score_gap` (eg_score − non_eg_score; dual ≥30 floor) |
| `per_user_cte_achievable_tc` | `(tc, *, source, snapshot_date)` | non-per-TC `per_user_cte_achievable` (avg d_i; ≥30 d_i-non-null floor) |
| `per_user_cte_section2_tc` | `(tc, *, source, snapshot_date, bucket_label)` | non-per-TC `per_user_cte_section2` (per-bucket span aggregate; ≥30 per-bucket floor) |
| `per_user_cte_section2` (modified) | bucket_label widened: `Literal['conversion', 'parity'] → Literal['conversion', 'parity', 'recovery']` | (no body changes — see B4 lock) |

All 3 new builder functions:
- Use `_recent_capped_per_tc_cte(snapshot_date, tc)` for the per-TC restriction (no consumer-layer per-TC predicate — D-10 invariant).
- Accept `tc: TimeControlBucket` (Literal-typed; ty-enforced 4-value set; safe f-string interpolation).
- Discard `source` with `_ = source` — the cohort difference lives entirely in `selected_users_cte`, not in the pooled body (D-10 byte-identical guarantee).
- Project `user_id` in `per_user_values` (Pitfall 1 — cohort-CDF JOIN requirement).
- Carry the standard provenance comment: `-- user_id widened per Phase 94.4 Pitfall 1 (cohort-CDF JOIN against per_user_anchor).`

### Section2 `bucket_label` recovery widening (Task 2.3)

The existing `per_user_cte_section2` signature widened from `Literal["conversion", "parity"]` to `Literal["conversion", "parity", "recovery"]`. **No SQL body changes** — verified per the B4 lock in the plan:

- The existing `gap_rows` CASE classification (lines 502-512 of `canonical_slice_sql.py`) already emits `'recovery'` rows when `entry_eval_mate * user_color_sign < 0` OR `entry_eval_cp * user_color_sign <= -100` (user enters down material at endgame entry).
- The downstream `per_user_bucket` `GROUP BY user_id, bucket` (line 553) and `per_user_values WHERE bucket = '{bucket_label}'` (line 561) dispatch parametrically. The widening is purely at the Literal type level — Python f-string interpolates `'recovery'` into the WHERE clause; the SQL builder already does the right thing.
- The `per_user_cte_section2_tc` builder accepts the same 3-value bucket_label Literal.

### Pitfall 1 user_id widening (Task 1)

The 3 existing 94.3 per-TC builders (`per_user_cte_time_pressure_score_gap`, `per_user_cte_clock_gap`, `per_user_cte_net_flag_rate`) and the 3 new Task 2 per-TC builders (`per_user_cte_score_gap_tc`, `per_user_cte_achievable_tc`, `per_user_cte_section2_tc`) all project `user_id` in `per_user_values`:

```sql
-- Before (94.3):
per_user_values AS (
  SELECT (user_pressured_score - opp_pressured_score) AS metric_value, ... FROM per_user
)

-- After (94.4 Plan 03):
per_user_values AS (
  -- user_id widened per Phase 94.4 Pitfall 1 (cohort-CDF JOIN against per_user_anchor).
  SELECT user_id, (user_pressured_score - opp_pressured_score) AS metric_value, ... FROM per_user
)
```

Per RESEARCH Pitfall 1 lines 1168-1177: the cohort-CDF JOIN (Plan 04) joins `per_user_values` against `per_user_anchor` on `user_id`. Without `user_id` exposed in `per_user_values`, the JOIN loses user identity and the K=200 ranking by anchor distance is undefined.

The upstream `per_user` CTE already `GROUP BY`s `user_id` on all 6 builders (per-user aggregate by construction), so no upstream change was needed.

**Total Pitfall 1 widened `per_user_values` blocks**: 6 (3 existing 94.3 per-TC + 3 new Task 2 per-TC).
**Non-per-TC builders untouched**: `per_user_cte_score_gap`, `per_user_cte_achievable`, `per_user_cte_section2` pooled. Plan 04's cohort-CDF JOIN consumes per-TC builders only.
