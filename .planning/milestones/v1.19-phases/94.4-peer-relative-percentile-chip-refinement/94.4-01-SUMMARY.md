# Phase 94.4 Plan 01: chesscom_to_lichess Module Summary

One-liner: New pure-Python module shipping the canonical 2026-05-26 ChessGoals snapshot (3 tables, 5 accessors) — chess.com↔Lichess conversion ready for Phase 94.4 Plan 05 Stage A anchor compute, with D-14 USCF + FIDE future-use accessors.

## What Was Built

`app/services/chesscom_to_lichess.py` (415 LOC, mostly lookup-table literals) exposes:

**Lookup tables (Final[Mapping[...]], verbatim from RESEARCH Pattern 8b re-fetched canonical snapshot):**

- `CHESSCOM_INTRA_TC` — 33 anchor rows, chess.com Blitz 500..3000 → chess.com Bullet, Rapid, USCF, FIDE. The 1100/1150 irregularity in the source (both Bullet=1150) is preserved as written; the inversion helper handles the tied case deterministically.
- `CHESSCOM_BLITZ_TO_LICHESS` — 35 anchor rows, chess.com Blitz 500..3000 → Lichess Bullet, Blitz, Rapid, Classical. `classical=None` at chess.com Blitz 2900 and 3000 per source (insufficient classical data above that rating).
- `LICHESS_BLITZ_INTRA_TC` — 35 anchor rows, Lichess Blitz 1030..2850 → Lichess Bullet, Rapid, Classical, USCF, FIDE. Multiple None entries per source: 6 lowest-rated rows (1030..1420) have FIDE=None; top row 2850 has Classical=USCF=FIDE=None; top 3 rows (2695, 2780, 2850) have Classical=None.

**Module constants (D-14 provenance):**

- `CHESSCOM_TO_LICHESS_TABLE_SNAPSHOT: Final[str] = "2026-05-26"`
- `CHESSCOM_TO_LICHESS_SOURCE: Final[str] = "https://chessgoals.com/rating-comparison/"`

**Converter (Phase 94.4 Plan 05 caller):**

- `convert_chesscom_to_lichess(rating, source_tc, target_tc) -> int | None`
  - `source_tc == "daily"` → returns None (Pitfall 2 — no published mapping, caller suppresses chip).
  - `source_tc == "blitz"` → linear-interp `CHESSCOM_BLITZ_TO_LICHESS[*][target_tc]`.
  - `source_tc in ("bullet", "rapid")` → invert `CHESSCOM_INTRA_TC[*][source_tc]` to recover chess.com Blitz equivalent, then chain Table 2 at the recovered anchor.
  - Clamps at [500, 3000] chess.com Blitz; returns None outside the published range (no extrapolation).

**USCF + FIDE accessors (D-14 future-use, no Phase 94.4 caller):**

- `lookup_uscf_from_chesscom_blitz(rating) -> int | None`
- `lookup_fide_from_chesscom_blitz(rating) -> int | None`
- `lookup_uscf_from_lichess_blitz(rating) -> int | None`
- `lookup_fide_from_lichess_blitz(rating) -> int | None`

All four: linear-interp against the relevant table's USCF or FIDE column. Each returns None outside the published range OR when the bracket of two anchors flanking `rating` contains a None sentinel (refuses to cross None gaps rather than guess).

### Editorial Choices Worth Recording

- **Inversion at tied Bullet column.** `CHESSCOM_INTRA_TC[1100]["bullet"] == CHESSCOM_INTRA_TC[1150]["bullet"] == 1150` per source. When inverting `convert_chesscom_to_lichess(1150, "bullet", "...")`, the helper returns the lower Blitz anchor (1100) per SEED-026 conservative-estimate principle. This is the only tied column in any of the three tables.
- **None-gap handling.** `_interp_int_column` refuses to interpolate across a None gap (returns None instead). This means e.g. `lookup_fide_from_lichess_blitz` returns None for Lichess Blitz in [1030, 1474] because FIDE=None for the 6 lowest rows AND for any rating where one of the bracketing anchors is None. This is the safe default — guessing across a published-as-undefined column would silently fabricate values.
