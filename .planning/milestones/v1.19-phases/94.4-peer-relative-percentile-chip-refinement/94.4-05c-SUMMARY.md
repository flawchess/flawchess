### 1. Aggregation helper (`app/services/endgame_service.py`)

**`_aggregate_per_tc_percentile(per_tc_rows: Mapping[TimeControlBucket, PercentileRow] | None) -> float | None`** — new module-level helper per PATTERNS lines 510-532 verbatim. Game-count-weighted mean across per-TC PercentileRow entries:

```
weighted_sum = Σ (percentile_i * n_games_i)   for each TC where percentile_i is not None
weight_sum   = Σ n_games_i                    for each TC where percentile_i is not None
return None if (per_tc_rows is None) or (empty) or (weight_sum == 0); else weighted_sum / weight_sum
```

- D-08a: percentile-None TCs (CDF out-of-range for the user's value on that TC) drop from both numerator AND denominator; their n_games is ignored.
- Single-TC dict passes the percentile through verbatim (the weighted mean of one row is itself).
- Empty dict / None / all-percentile-None returns None — chip suppresses naturally.

### 4. Schema extension (`app/schemas/endgames.py`)

**`RatingAnchorOut`** — new Pydantic model per PATTERNS lines 546-552:

```python
class RatingAnchorOut(BaseModel):
    anchor_rating: int
    source_platform: Literal["lichess", "chesscom"]
    chesscom_raw_rating: int | None = None
    n_games: int
```

Docstring cites D-07 bullet 4. `chesscom_raw_rating` is the pre-conversion chess.com median (when `source_platform == 'chesscom'`); `anchor_rating` is always the Lichess-equivalent (post-conversion for chess.com sources). The cohort CDF is Lichess-keyed.

**`rating_anchors: dict[TimeControlBucket, RatingAnchorOut] = Field(default_factory=dict)`** — added to `EndgameOverviewResponse` as a **top-level** block (not embedded per-chip). Plan 07's tooltip body builder picks the dominant TC's anchor from this dict for the 4th bullet.

**`recovery_score_gap_percentile: float | None = None`** — added to `ScoreGapMaterialResponse` per **D-05a rescue**. Replaces the prior Phase 94 D-12 "NO recovery percentile" suppression comment with a positive declaration + rescue docstring. Field name mirrors the `CdfMetricId` Literal `recovery_score_gap` (D-13 ENUM), NOT the `section2_*` prefix used by conv/parity — the CdfMetricId-mirror name aligns with the new PK column shape.
