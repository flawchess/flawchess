## Surgical-correction approach

The plan anticipated "1–5 paragraphs" of false-correspondence claims to rewrite individually. Inspection found those claims concentrated in **Chapter 4 alone** (the Global Percentile CDF chapter):

- L3022: "The committed artifact lives at `app/services/global_percentile_cdf.py`"
- L3024: "The downstream consumer is Phase 94 backend only: it interpolates the user's per-metric value against `GLOBAL_PERCENTILE_CDF`"
- L3047: "the authoritative SQL lives in `scripts/gen_global_percentile_cdf.py`"
- L3049: "Bypassing the canonical CTE produces a wrong global distribution and therefore wrong percentiles for every user"
- L3148: "The 99-element `breakpoints` array is the row that lands in `GLOBAL_PERCENTILE_CDF[...]`. Phase 94's interpolation helper consumes this array"

Rather than edit each paragraph individually (which would have grown the diff substantially and littered the chapter with parenthetical disclaimers), I added **one chapter-level note immediately after the `## 4. Global Percentile CDF` heading** that re-frames every `chip` / `GLOBAL_PERCENTILE_CDF` / "backend interpolation" reference in the chapter as historical to Phase 93 v1 and points readers to `canonical_slice_sql.py`. This:

1. Keeps the diff minimal (+2 lines for the chapter-level note vs ~10+ for paragraph-by-paragraph edits).
2. Preserves the chapter's per-cohort sanity-check methodology verbatim — it remains valid as an analytical breakdown (same role as `_build_per_bucket_sanity_query` post-94.2 Plan 02).
3. Avoids "Superseded by Phase 94.2" callouts (explicitly rejected by acceptance criteria).
4. Avoids touching the per-metric inclusion-floor table (the ≥20 / ≥30 game floors at L528–529 and L3042–3045 are SKILL-analytical floors and remain correct for the SKILL's own purpose).

Outside Chapter 4, no SKILL.md paragraphs make claims of correspondence between SKILL-computed numbers and the production chip — the rest of the file is per-cohort analysis that never claimed to be the production CDF, so no further surgical edits were needed.
