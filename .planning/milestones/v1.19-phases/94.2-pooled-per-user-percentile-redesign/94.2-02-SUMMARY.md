## Decisions Made

1. **Preserve `_build_per_bucket_sanity_query` as a zero-row stub.** Plan called for keeping it as a diagnostic; CONTEXT.md Claude's Discretion explicitly permits dropping it if it adds friction. Reconstructing the per-cell diagnostic post-pooled means widening the canonical builders to project `user_id` (which they deliberately strip per D-5) — that's a future-phase decision, not in scope here. The function stays present (importable, callable, grep-able) and its docstring + the report's per-table callout flag the limitation honestly.
2. **Fail-soft runtime around `_query_per_bucket_sanity_check`.** A try/except logs a WARNING and continues with `[]` rows if the diagnostic SQL ever raises. The production CDF regen MUST NOT be blocked by a deprecated diagnostic.
3. **Section-heading anchor for drift-table parser.** First-pass parser used `text_blob.find("(`{metric_id}`)")` which matched the cohort-listing line (`- **Endgame Score Gap** (`score_gap`): n_users = 2003`) before the section heading (`## Endgame Score Gap (`score_gap`)`), clipping the wrong region and parsing zero rows. Fixed with a multiline `^## [^\n]*(\`{metric_id}\`)` regex. Caught by the WARNING in the first regen-run stdout; fixed inline.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 — Bug] Drift-table parser anchored on wrong line**
- **Found during:** Task 2 (first regen run produced `WARNING: could not parse breakpoints out of reports/archive/global-percentile-cdf-94.1-2026-05-24.md; drift section skipped.`)

- **Fix:** Replaced the literal `.find` with a `re.compile(r"^## [^\n]*\(\`{metric_id}\`\)", re.MULTILINE).search(...)` so the anchor requires a section-heading prefix.
- **Files modified:** `scripts/gen_global_percentile_cdf.py`
- **Verification:** Second regen run produced no WARNING and emitted populated `Δ` rows in the drift table (visible at `reports/global-percentile-cdf-latest.md` — e.g. `| p99 | +29.2pp | +27.9pp | -1.4pp |` for score_gap).
- **Committed in:** `a5ee1f32` (folded into Task 2's commit since the fix was discovered during the regen run and the corrected output is what got committed).
