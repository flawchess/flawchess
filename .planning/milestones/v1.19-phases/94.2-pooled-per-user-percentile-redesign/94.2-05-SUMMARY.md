# Phase 94.2 Plan 05: PercentileChip D-4 popover disclosure + 4-flavor enum

## One-liner

PercentileChipFlavor widened from 2 generic ('skill-isolating' / 'improvement-focus') to 4 metric-named variants ('score-gap' / 'achievable' / 'parity' / 'conversion'); popover body now discloses 4 D-4 bullets per metric with rating-correlation framing calibrated per Cohen's d.

## What changed

### `PercentileChip.tsx`

- `PercentileChipFlavor` type widened to `'score-gap' | 'achievable' | 'parity' | 'conversion'` (matches backend metric IDs).
- Popover body rewritten to render four `<p>` blocks for every flavor:
  1. Benchmark composition — *"Compared against benchmarked Lichess players of all ratings and time controls."*
  2. Recent-games basis — *"Uses your most recent 1000 games per time control played, last 36 months."*
  3. Filter independence — *"UI filters do not affect this percentile."*
  4. Per-metric rating-correlation framing — differs per flavor (per Cohen's d in `reports/benchmarks-gap-metrics-percentile-candidacy.md`):
     - `score-gap` (d=0.19): *"Endgame Score Gap is mostly independent of rating, so this reflects endgame ability separate from overall strength."*
     - `achievable` (d=0.32): *"Achievable Score Gap mildly correlates with rating."*
     - `parity` (d=0.30): *"Parity ΔES mildly correlates with rating."*
     - `conversion` (d=1.37): *"Conversion ΔES tracks rating strongly: stronger players tend to score higher here because they blunder less when up material."*
- Copy hoisted to seven module-level constants (`COPY_BENCHMARK_COMPOSITION`, `COPY_RECENT_GAMES_BASIS`, `COPY_FILTER_INDEPENDENCE`, four `COPY_RATING_NOTE_*`).
- Lookup table `RATING_NOTE_BY_FLAVOR` typed `as const satisfies Record<PercentileChipFlavor, string>` for compile-time exhaustiveness — adding a fifth flavor without a copy entry will fail tsc.
- D-14 preserved: `deriveBandColor`, `deriveFlameCount`, `formatTopXPercent`, the Popover hover-mechanics, the chip trigger element, animation classes — all unchanged.
- `text-xs` on popover Content retained per CLAUDE.md hover-tooltip exception (PercentileChip is the explicit reference example).

### Call sites

- `EndgameOverallPerformanceSection.tsx` — Achievable Score Gap chip: `flavor="skill-isolating"` → `"achievable"`; Endgame Score Gap chip: `flavor="skill-isolating"` → `"score-gap"`.
- `EndgameMetricCard.tsx` — bucket-to-flavor dispatch rewritten: `bucket === 'conversion' ? 'improvement-focus' : 'skill-isolating'` → `bucket === 'conversion' ? 'conversion' : 'parity'`. The pre-existing guard `bucket !== 'recovery'` already excludes `recovery` from chip rendering, so the binary ternary is exhaustive against `MaterialBucket = 'conversion' | 'parity' | 'recovery'`.

### `PercentileChip.test.tsx`

- `renderChip` helper default flavor: `'skill-isolating'` → `'score-gap'`; signature now imports `PercentileChipFlavor` from the component.
- New `describe` block: *"PercentileChip — D-4 popover disclosure (Phase 94.2)"* with:
  - `it.each(ALL_FLAVORS)` parametrized assertion that every flavor's popover discloses benchmark composition, recent-games basis, and filter independence.
  - Per-flavor framing assertions: `score-gap` MUST contain "mostly independent of rating" and MUST NOT contain "mildly correlates" or "tracks rating strongly"; `achievable` + `parity` MUST contain "mildly correlates with rating"; `conversion` MUST contain "tracks rating strongly".
- Removed the prior `POPOVER_MAX_CHARS = 280` minimalism-budget guard and the three `Phase 94.1 D-13 canonical-slice tooltip framing` tests — the new copy intentionally exceeds 280 chars per D-4.
- All Phase 94 label-formatter, band-color, flame-tier, aria-label, data-testid tests preserved verbatim.
