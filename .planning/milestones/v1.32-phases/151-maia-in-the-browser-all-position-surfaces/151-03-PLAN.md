today: `useUserProfile()`'s `UserProfile` type has no rating field, and no backend computes one
(RESEARCH Pitfall 2). Extend the existing read-only `/users/me/profile` endpoint with a
`current_rating` derived from the user's most-recent game rating (per platform/color), using the
already-present `ix_games_user_played_at (user_id, played_at DESC)` index — no migration, no new
endpoint, no write. Independent of the ONNX contract; runs in Wave 1.

Purpose: Give engaged users a meaningful free-play "you are here" default instead of the flat 1500
fallback. Output: one repository query, one response field, one frontend type field.
</objective>
