  <behavior>
    classifyMoveQuality(gradeMap: Map<string, MoveGrade>, mover: MoverColor):
      - For each SAN, compute expectedScore = evalToExpectedScore(grade.evalCp, grade.evalMate, mover). Track the max (bestEs) and its SAN (bestSan).
      - The SAN with the max expected score → quality 'best'.
      - Every other SAN → classifyLiveSeverity(bestEs, thisEs): 'blunder'|'mistake'|'inaccuracy' maps straight through; null → 'good'.
      - Returns Map<string, MoveQualityInfo> where MoveQualityInfo = { quality: MoveQuality; expectedScore: number }.
      - Test boundaries against flawThresholds: drop 0 → best; 0 < drop < 0.05 → good; drop === 0.05 → inaccuracy; drop === 0.10 → mistake; drop === 0.15 → blunder (classifyLiveSeverity uses >= so exact-threshold lands in the higher band).
      - Test: empty gradeMap → empty result; single entry → that entry is 'best'.
    selectCandidatesByMass(perElo, selectedElo, playedSan, bestSan): string[]:
      - Pick the perElo rung nearest selectedElo (mirror useMaiaEngine's nearestByElo).
      - Sort that rung's moveProbabilities descending; accumulate SANs until cumulative probability first reaches CUMULATIVE_MASS_THRESHOLD (0.95) — include the move that crosses the threshold, then stop.
      - Take the top CANDIDATE_HARD_CAP (5) of that mass set by probability.
      - Union {bestSan} (if non-null) and {playedSan} (if non-null) — D-07 forces the played move in even when below the cut.
      - Return in probability-descending order for stable stacking.
      - Test: mass accumulation stops correctly; a low-probability played move outside the top-5 is still included; a null played/best is skipped; empty perElo → [].
    Constant assertions: CUMULATIVE_MASS_THRESHOLD === 0.95, CANDIDATE_HARD_CAP === 5.
  </behavior>
