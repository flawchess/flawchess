# Phase 151.1 Plan 04: Live graded Moves-by-Rating chart Summary

**The Moves-by-Rating chart now draws the 0.95-mass Maia candidate set colored by streamed Stockfish quality (dark-green best → red blunder) with decoupled played/best stroke emphasis and a quality+eval hover tooltip, driven by a second independent grading worker that leaves the primary eval bar / engine card untouched — signed off live on desktop Chrome + mobile Safari.**

## Performance

- **Duration:** ~40 min (execution + live UAT round-trip)
- **Started:** 2026-07-05T11:36:00+02:00
- **Completed:** 2026-07-05T12:22:00+02:00
- **Tasks:** 3 (2 auto + 1 human-verify checkpoint)
- **Files modified:** 6

## Accomplishments
- **Quality recoloring (SC1):** `MovesByRatingChart` now colors each shown line + its end-of-line SAN label by its `MoveQuality` bucket via `colorForQuality()` → `MOVE_QUALITY_*` tokens; ungraded shown moves render `MOVE_QUALITY_PENDING` until the first grade arrives (D-05).
- **Decoupled encodings (D-01/D-07):** stroke width keys off played/best emphasis only; color keys off quality only — a played blunder keeps `EMPHASIZED_STROKE_WIDTH` while rendering red.
- **D-08 tooltip:** rows now carry `{san}{role}: {QualityWord} · {eval} · {prob}%` with white-POV eval (`+1.2` cp / `#N` mate / `—` ungraded), extracted into an exported `MovesByRatingTooltipContent` for direct unit testing.
- **Candidate selection hoisted (SC2):** the chart's internal top-6-by-peak cap (`capMovesByPeak`/`collectCandidateSans`/`peakProbability`) is deleted; `Analysis.tsx` computes `shownSans = selectCandidatesByMass(...)` once and passes it to the grading hook (as `candidateSans`) and the chart (as `shownSans`).
- **Grading wired end-to-end (SC3):** `Analysis.tsx` instantiates `useStockfishGradingEngine` gated on the same `engineEnabled` toggle, derives `qualityBySan` via `classifyMoveQuality(gradeMap, sideToMoveFromFen(position))` merged with each grade's raw white-POV eval, and passes `shownSans` + `qualityBySan` to BOTH `MaiaHumanPanel` call sites (shared mobile `humanTab` + desktop left column).
- **Dead-token cleanup:** `MOVES_BY_RATING_PLAYED_LINE/BEST_LINE/OTHER_LINES` removed from `theme.ts` (knip clean); `MOVES_BY_RATING_REFERENCE_LINE` kept for the you-are-here line.
- **SC4 latency:** judged acceptable by the user at the default grading budget; no degradation knobs changed.
