## What Was Built

### Task 1: Generator refactor (`scripts/gen_global_percentile_cdf.py`)

Retired the entire `.py` source-emission stack:
- Removed: `_emit_breakpoints_literal`, `_emit_registry_block`, `_rewrite_module_source`, `_write_module`
- Removed constants: `OUTPUT_MODULE_PATH`, `REGISTRY_BEGIN_MARKER`, `REGISTRY_END_MARKER`
- Added: `OUTPUT_SEED_PATH = Path("app/data/cohort_cdf.tsv")`
- Added: `_write_seed_tsv(cells, path)` using `csv.writer(delimiter="\t")` with header `metric/anchor_elo/tc/percentile/value/n_users/snapshot_month`, one row per `(metric, anchor_elo, tc, percentile)` breakpoint
- Iteration order preserves the retired `_emit_registry_block` sort contract: outer `IN_SCOPE_METRICS`, inner `sorted((anchor, tc_order[tc]))`, innermost p1..p99
- `FLOAT_PRECISION = 4` formatting preserved for bit-stable values
- `regenerate()` now calls `_write_seed_tsv(cells, OUTPUT_SEED_PATH)` instead of `_write_module(cells)`
- Added `csv` import; kept `re` import (still used in `_archive_prior_report`)
- Added `BREAKPOINT_PERCENTILES` to the import from `global_percentile_cdf`

### Task 2: Committed artifact + drift test

**`app/data/cohort_cdf.tsv`** (122,662 lines = 1 header + 122,661 data rows):
- Minted by calling `_write_seed_tsv` over the current in-source `COHORT_PERCENTILE_CDF` dict
- First rows: `score_gap, 850, blitz, 1..99` -- matches registry values exactly
- Header: `metric\tanchor_elo\ttc\tpercentile\tvalue\tn_users\tsnapshot_month`

**`tests/scripts/test_cohort_cdf_seed_artifact.py`** (3 tests):

- `test_committed_tsv_row_count`: asserts > 100,000 data rows

- Does NOT import `COHORT_PERCENTILE_CDF` (survives Plan 04 registry removal)
