<objective>
Stop the generator from rewriting `global_percentile_cdf.py` between the BEGIN/END
sentinels; make it emit a compact long-row seed file `app/data/cohort_cdf.tsv` (CONTEXT
D-02). Produce the committed `cohort_cdf.tsv` so its content is byte-for-byte derivable
from the CURRENT in-source `COHORT_PERCENTILE_CDF` registry (this guarantees the
post-refactor chip output is byte-for-byte equivalent — SC#6). Add a drift test comparing
regenerated TSV CONTENT against the committed file.

Purpose: replace 130k-line `.py` diffs with a clean TSV artifact; preserve deterministic
ordering so regen stays byte-identical.
Output: `app/data/cohort_cdf.tsv` (committed), TSV-emitting generator, drift test repointed.

DESIGN RESOLUTION (Claude's Discretion — byte-for-byte parity): The committed
`cohort_cdf.tsv` MUST be produced by serializing the EXISTING in-source
`COHORT_PERCENTILE_CDF` dict (the authoritative current data), NOT by re-running the
generator against the benchmark DB. Re-running against the benchmark DB could yield
different values if that DB has drifted, breaking SC#6. The `_write_seed_tsv` emitter walks
the same `cells` dict either way; we feed it the in-source registry once to mint the
committed artifact while the registry still exists in source (it is removed in Plan 04, so
the TSV MUST be committed in THIS plan). The drift test then proves `_write_seed_tsv` over
the same data reproduces the committed file deterministically.
</objective>
