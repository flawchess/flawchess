  <action>
    1. Install knip as a dev dependency:
       ```bash
       cd frontend && npm install -D knip
       ```

    2. Add a `"knip"` script to `frontend/package.json` scripts section:
       ```json
       "knip": "knip"
       ```

    3. Create `frontend/knip.json` with this exact content (per D-03, research recommends minimal config since Vite and Vitest plugins auto-activate from devDependencies):
       ```json
       {
         "$schema": "https://unpkg.com/knip@6/schema.json",
         "entry": ["src/main.tsx", "src/prerender.tsx"],
         "project": ["src/**/*.{ts,tsx}"]
       }
       ```

    4. Run `cd frontend && npm run knip` to verify it executes. Review output — it may report issues that Plan 03 will address. The goal here is just to confirm knip runs without crashing.

    NOTE: Knip may report false positives for exports used only in test files (Pitfall 3 from research). If the Vitest plugin doesn't auto-detect test patterns, Plan 03 will add `"ignoreExportsUsedInFile": true` to `knip.json`. Do NOT fix any reported dead code in this task — just confirm the tool runs.

<task type="auto">
  <name>Task 2: Add frontend build, test, and knip steps to CI</name>
  <files>.github/workflows/ci.yml</files>
  <read_first>
    - .github/workflows/ci.yml (current pipeline structure — must understand step ordering and working-directory patterns)
    - frontend/package.json (verify script names: "build", "test", "knip")
  </read_first>
  <action>
    Per D-09, add three new steps to the CI workflow AFTER the existing "Lint (eslint)" step (which already has `working-directory: frontend` and runs after `npm ci`).

    Add these three steps in this exact order after the eslint step:

    ```yaml
      - name: Type check and build (tsc + vite)
        run: npm run build
        working-directory: frontend

      - name: Run tests (vitest)
        run: npm test
        working-directory: frontend

      - name: Dead code check (knip)
        run: npm run knip
        working-directory: frontend
    ```

    Rationale for ordering:
    - `npm run build` runs `tsc -b && vite build` — catches TypeScript compilation errors first (including noUncheckedIndexedAccess once Plan 04 enables it)
    - `npm test` runs `vitest run` — catches test failures
    - `npm run knip` runs dead export detection — catches regressions in unused code

    IMPORTANT: These steps go inside the existing `test` job, AFTER the "Lint (eslint)" step and BEFORE the `deploy` job. They use the same Node.js setup and `npm ci` install that already exists.

    Do NOT modify the deploy job or any backend steps.
  </action>
  <verify>
    <automated>grep -A2 "Dead code check" .github/workflows/ci.yml && grep -A2 "Run tests (vitest)" .github/workflows/ci.yml && grep -A2 "Type check and build" .github/workflows/ci.yml</automated>
  </verify>
  <acceptance_criteria>

    - `.github/workflows/ci.yml` contains step "Run tests (vitest)" with `run: npm test` and `working-directory: frontend`
    - `.github/workflows/ci.yml` contains step "Dead code check (knip)" with `run: npm run knip` and `working-directory: frontend`

  </acceptance_criteria>
