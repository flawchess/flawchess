<objective>
Build the pure-logic, engine-free bot-cell two-pass scheduler module `scripts/lib/calibration-bot-cell-schedule.mjs` and its `.check.mjs` assertion suite. This module owns the internal-scale anchor accessor (`internalRatingFor`), the locate-pass anchor pick, and the measure-pass bracket selection — the load-bearing logic that replaces the Phase-173 anchor-graph scheduler for hub-and-spoke bot cells (D-07). It is consumed by the harness in Plan 03.

Purpose: Isolate the strength-relative anchor-selection logic into a testable, engine-free module so the D-02(a) logic-test layer can prove it with fabricated providers before any real engine runs.
Output: Two new files under `scripts/lib/`.

**Planner decision (deviation from RESEARCH Pattern 1, with rationale):** `internalRatingFor` is DEFINED AND EXPORTED HERE (in the new pure-logic module), not in `calibration-harness.mjs` as RESEARCH Pattern 1 suggested. Reason: the harness must import the two-pass selection functions from this module, and those functions need `internalRatingFor`; defining it here keeps the dependency one-directional (harness -> schedule) and avoids a circular import (harness <-> schedule). The harness imports `internalRatingFor` from this module in Plan 03.
</objective>
