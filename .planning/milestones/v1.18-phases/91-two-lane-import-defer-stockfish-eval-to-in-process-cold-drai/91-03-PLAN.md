<objective>
Strip the Stockfish evaluation pass from the hot-lane `_flush_batch` in `app/services/import_service.py` and add Stage 5c — the "lichess-already-covered" gate that sets `evals_completed_at = NOW()` for games whose entry plies need no further engine work.

Purpose: The OOM-kill failure mode in the 2026-05-20 stress test was driven by 20-40 s eval-stage transactions in the hot lane (`asyncio.gather` over `engine.evaluate` inside an open `AsyncSession`). Removing this from the hot lane is the structural fix. Plan 91-02 already moved the work to `eval_drain.py`; this plan removes the duplicate from `import_service.py` and adds the covered-game gate so games that need no eval don't sit in the pending queue forever.

Output: Hot-lane `_flush_batch` with Stages 3a/4/eval-UPDATE gone, Stage 5c added, and a regression test that proves `engine.evaluate` is never called during import.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| _flush_batch → cold drain | Hot lane writes `evals_completed_at = NULL` for pending games; cold drain picks them up asynchronously |
| _flush_batch → DB | Single per-batch session; per Phase 90 WR-01 lifecycle preserved |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-91-10 | Tampering | Stage 5c marks wrong games as covered (false positive) | mitigate | `_collect_covered_game_ids` calls the SAME helpers (`_collect_midgame_eval_targets`, `_collect_endgame_span_eval_targets`) that the cold drain uses; a game marked covered by Stage 5c is by definition one that the drain would also have found "empty targets" for. Integration test `test_stage5c_marks_covered_games` verifies. |
| T-91-11 | Repudiation | Stage 5c MISSES games that should be covered (false negative) | accept | Worst case: cold drain picks the game, finds no targets, calls `_mark_evals_completed` with empty `eval_targets` list, marks game done. Identical end state, just one drain tick later. No correctness impact. |
| T-91-12 | Denial of Service (regression) | Future edit re-introduces eval call in hot lane | mitigate | `test_flush_batch_no_engine_calls` is a hard regression gate. CI runs it on every push. |
| T-91-13 | Information Disclosure | Cross-module use of `_`-prefixed helpers | accept | The `_`-prefix is a convention, not an enforcement mechanism. Inline comment + SEED-023 docs the intentional cross-module use. Phase 91 scope only. |
</threat_model>
