## Milestone Goal

Replace the material-imbalance + 4-ply persistence proxy for endgame conversion / parity / recovery classification with Stockfish eval (depth 15) populated into the existing `eval_cp` / `eval_mate` columns on `game_positions`. Backfill historical span-entry positions across benchmark + prod, eval new span-entry positions during import, refactor endgame queries to threshold on eval, and remove the proxy entirely (hard cutover). Phase 79 folded in a per-position `phase` SmallInteger column (0=opening, 1=middlegame, 2=endgame) computed via a Python port of lichess `Divider.scala` and extended the eval pass + backfill script to also evaluate the middlegame-entry position — substrate for v1.16 opening-stats analyses.

## Phase Details

### Phase 78: Stockfish-Eval Cutover for Endgame Classification

**Shipped behavior:** Backend Docker image ships pinned Stockfish (sf_17 AVX2 with SHA-256 supply-chain verification, bumped to sf_18 in a follow-up). `app/services/engine.py` exposes an async-friendly Stockfish wrapper with FastAPI lifespan integration (`start_engine` / `stop_engine`, idempotent, depth-15 `evaluate()`). `scripts/backfill_eval.py` is an idempotent + resumable CLI driver (`--db dev/benchmark/prod`, `--user-id`, `--limit`, `--dry-run`, `--workers N` for parallel evaluation, COMMIT-every-100). Import path evaluates per-class span-entry rows in `_flush_batch` between `bulk_insert_positions` and the `move_count` UPDATE, in the same transaction. `_classify_endgame_bucket(eval_cp, eval_mate, user_color)` is the single helper for endgame conv/parity/recov classification — SQL projects raw white-perspective eval; the service layer applies the user-color sign flip. ±100 cp threshold for cp; mate scores at any value count as max conv/recov. Alembic `c92af8282d1a` reshapes `ix_gp_user_endgame_game INCLUDE` from `material_imbalance` to `eval_cp` / `eval_mate` so rewritten queries stay index-only. `_MATERIAL_ADVANTAGE_THRESHOLD`, `PERSISTENCE_PLIES`, and the `array_agg(... ORDER BY ply)[PERSISTENCE_PLIES + 1]` contiguity case-expression are deleted from the codebase. `material_imbalance` column retained for other consumers.

**Requirements:** ENG-01..03, FILL-01..04, IMP-01..02, REFAC-01..05, VAL-01, VAL-02 (VAL-01 rescinded post-cutover — proxy removed → agreement metric undefined; the `/conv-recov-validation` skill was deleted on 2026-05-03).

### Phase 79: Position-phase classifier and middlegame eval

**Shipped behavior:** `game_positions` gains a `phase` SmallInteger column (0=opening, 1=middlegame, 2=endgame) populated on every new import and backfilled across benchmark + prod. The classifier is a Python port of lichess [`Divider.scala`](https://github.com/lichess-org/scalachess/blob/master/core/src/main/scala/Divider.scala) using existing `piece_count`, `backrank_sparse`, `mixedness` inputs — no second board scan. 11 Divider-sourced parity assertions in `tests/test_position_classifier.py` lock the output to the lichess reference on a curated FEN fixture set. The middlegame entry position (`MIN(ply)` of `phase = 1` per game) is also Stockfish-evaluated at depth 15 alongside endgame span-entry positions, populated into the same `eval_cp` / `eval_mate` columns. The combined Phase 78+79 backfill ran on benchmark first, then prod, then PR #78 was merged and deployed.

**Requirements:** CLASS-01..02, SCHEMA-01..02, PHASE-IMP-01..02, PHASE-FILL-01..03, PHASE-VAL-01..03, PHASE-INV-01 (PHASE-VAL-01 rescinded with VAL-01 — same root cause, agreement metric undefined post-cutover).

## Milestone Summary

**Key Decisions:**
- Hard cutover, no fallback: removed proxy code path entirely (REFAC-03) rather than running both side-by-side. The validation report had already established the proxy's structural ceiling on queen + pawnless classes; a fallback would have re-introduced the failure mode for any game without lichess `%eval` annotation.
- VAL-01 / PHASE-VAL-01 rescinded as moot: the `/conv-recov-validation` skill compared proxy vs eval on the populated subset; once the proxy was deleted, the agreement metric became undefined. Skill deleted, requirements marked rescinded.
- Combined Phase 78 + Phase 79 cutover (D-79-10): single backfill pass on benchmark + prod, single PR #78, single deploy. Saved an operational round-trip and consolidated the deployment risk window.
- Frontend-only changes via per-class endgame gauge cards (quick task 260501-s0u) instead of a separate phase — the v1.15 milestone scope was strictly backend; UI calibration shipped as inline quick tasks.

**Issues Resolved:**
- Endgame conv/parity/recov misclassification on queen + pawnless classes (~24% of substantive material-edge sequences missed by the proxy).
- Index-only scan preserved for `query_endgame_*` queries via Alembic `c92af8282d1a` reshape of `ix_gp_user_endgame_game INCLUDE`.
- Short and repeated-class endgame spans now eval-covered via 5937602 (`fix(eval): cover short and repeated-class endgame spans`).

**Issues Deferred:**
- VAL-01 / PHASE-VAL-01 — rescinded, not deferred (agreement metric undefined post-cutover).
- v1.16 (Phase 80+) Stockfish Eval Analyses — opens immediately. First phase: opening-stats columns for middlegame-entry eval and clock diff.

**Technical Debt Incurred:**
- `STOCKFISH_POOL_SIZE` defaults to 1 outside prod; prod ships 2 via `docker-compose.yml`. No autotune. Worth re-visiting if import latency p99 regresses or if dev environments hit eval throughput ceilings during local benchmarking.
- `STOCKFISH_PATH` env-var setup is ad-hoc for standalone runs (documented in CLAUDE.md). A wrapper in `bin/` could harden the local-dev experience.
