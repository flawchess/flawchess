# Milestone v1.26: Full-Game Eval Pipeline

**Status:** ✅ SHIPPED 2026-06-14
**Phases:** 116, 117, 117.1 (INSERTED), 117.2 (INSERTED), 118, 119, 120 — 7 phases
**Total Plans:** 18 (116: 3 · 117: 3 · 117.1: 2 · 117.2: data migration, no plan · 118: 3 · 119: 3 · 120: 4)

## Overview

v1.26 turned FlawChess's eval coverage from "endgame-entry plies only" into a **full-game background analysis pipeline**: every move of a user's imported games is now evaluated by Stockfish at Lichess-parity strength (1M nodes/move), drained by a tiered priority queue that puts explicit user requests first, recent windows second, and an idle backlog third, with the results flowing automatically into the Library's flaw surfaces.

The milestone was the server-first v1 of SEED-012 (client-side analysis deferred), with all queue/eval decisions locked on 2026-06-12. It grew two inserted correctness phases (117.1, 117.2) when a pre-move/post-move eval off-by-one was discovered, a coverage-hardening phase (119) from SEED-045/046, and an off-box compute phase (120) from SEED-048.

Dependency-ordered:

- **Phase 116 — All-Ply Engine Core.** The eval drain analyzes every ply at the 1M-node / NNUE / multiPV=1 Lichess-parity budget, stores results in `game_positions.eval_cp/eval_mate` with a ply≤20 `full_hash` dedup (shared opening region stays cheap, no opening gap in eval charts), and a distinct full-analysis completion marker. Worker-pool memory was explicitly bounded against the backend container's 4g limit before raising `STOCKFISH_POOL_SIZE`.
- **Phase 117 — Priority Queue + Flaw Integration.** A PostgreSQL SKIP-LOCKED tiered priority queue (tier-1 explicit > tier-2 recent windows > tier-3 idle backlog) with round-robin per-user fairness, a lease/report contract (so a future browser/external worker is additive), tier-1 ~10s fan-out across the whole pool, `best_move` (PV[0]) for every position + full PV adjacent to flaws, automatic `classify_game_flaws` flow-through, and guest exclusion on every path.
- **Phase 117.1 (INSERTED, SEED-044) — Flaw-Eval Convention Fix.** A single post-move eval-storage convention across all sources, fixing an off-by-one where the engine drain stored the pre-push position's eval while the classifier assumed post-move — which mis-scored flaws for every chess.com (engine-evaluated) game. Terminal-position eval + one-ply-shift dedup transplant rework + clean-slate re-eval migration.
- **Phase 117.2 (INSERTED, SEED-044 follow-up) — Wipe Eval-Only Engine Residue.** A data-only migration NULLing dense evals on 3,497 engine games (3 users) carried over from a pre-Phase-117 eval-only pass with no `best_move`/flaw classification/marker, so they stopped showing as analyzed-with-no-flaws until the tier-3 drain re-materializes them properly. Gated on `lichess_evals_at IS NULL` — lichess data untouched.
- **Phase 118 — Demand UX + Auto-Enqueue.** Explicit on-demand single-game analysis, an "analyze more" affordance, "N of M analyzed" coverage indicators + in-flight status across the Library flaw surfaces, and guest promotion as the unlock for full-game analysis.
- **Phase 119 (SEED-045, SEED-046) — Eval-drain coverage.** Bounded-retry hole-filling (no false "fully analyzed" stamp while genuine mid-game holes remain; cap at MAX_EVAL_ATTEMPTS), a recency-weighted Efraimidis–Spirakis tier-3 lottery (returning users' coverage ticks briskly), the lichess-%eval leak fix (the drain stops burning passes on already-analyzed lichess games), and an honest pulsing coverage badge with `in_flight_count` removed end-to-end.
- **Phase 120 (SEED-048) — Headless remote trusted-operator eval worker.** Off-box CPU for the tier-3 drain via an operator-token-authed lease/submit HTTP contract + a headless Python CLI worker running the existing `EnginePool` natively, with server-side SF-version-mismatch rejection and weighted-random within-user game pick to cut multi-worker collisions.

**Scope notes:** the milestone roadmap bullet originally read "Phases 116–119"; at close it was widened to 116–120 (incl. 117.1/117.2) to cover the full body of shipped full-game-eval work. SEED-049 (game-ending ply false-hole fix) shipped as quick task `260614-tgs` under this milestone. The original REQUIREMENTS.md covers Phases 116–118 + 117.1 (EVAL / QUEUE / EVUX / EVALFIX); Phases 119 and 120 were seed-driven additions tracked via SEED-045/046 and SEED-048 respectively (decisions D-1..D-7 locked in the SEED-048 file).

### Key Decisions

- **Server-first v1, queue shape kept additive** (SEED-012 D-1/D-8) — no client-side WASM analysis in v1; the lease/report contract means a browser or external worker is a later additive change, realized early in Phase 120.
- **Post-move eval convention, one storage rule** (SEED-044, EVALFIX-01) — `eval_cp`/`eval_mate` store the eval *after* the move for every row and every source; `classify_game_flaws` has no per-source branch. Fixed an off-by-one that mis-scored every chess.com game's flaws.
- **No book-skip; ply≤20 `full_hash` dedup instead** (EVAL-01/03) — eval charts get no opening gap and the collector needs no openings-table dependency; the heavily-shared opening region stays cheap via transplant.
- **`best_move` for every position, full PV only adjacent to flaws** (EVAL-04, D-117-01/02) — engine-best-move step-through for free; the heavy full-PV-for-all (~5 GB) rejected.
- **Explicit on-demand over tier-2 auto-enqueue** (Phase 119) — tier-2 auto-enqueue from Phase 118 was dropped; coverage advances via tier-1 explicit requests + the recency-weighted tier-3 idle drain. `EVAL_AUTO_DRAIN_ENABLED` defaults off in dev/CI.
- **Recency-weighted Efraimidis–Spirakis tier-3 lottery** (SEED-046) — replaced winner-take-all `last_activity DESC`; returning users' coverage badge moves briskly. τ/floor kept as tunable constants for post-deploy prod tuning.
- **Hole-aware completion gate with bounded retry** (SEED-045) — stop stamping `full_evals_completed_at` while genuine non-terminal/non-mate holes remain; cap at MAX_EVAL_ATTEMPTS then stamp + one aggregated Sentry event.
- **Guest exclusion everywhere** (QUEUE-08) — guests never enqueue/drain; UX presents account promotion as the unlock.
