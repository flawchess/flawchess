# Milestone v1.24: Library Page

**Status:** ✅ SHIPPED 2026-06-09

## Overview

v1.24 delivered the **Library** — a new top-level destination that grew from a simple shell-and-migration into the full eval-driven mistake/flaw archive of SEED-036's analysis half. It started (Phase 104) by folding the existing Import and Overview pages into deep-linkable Library subtabs, then built the on-the-fly mistake-detection kernel (105), the Games-surface backend (106), the Games subtab UI (107), the Flaws subtab with a materialized `game_flaws` table and a shared cross-tab Flaw filter (108), per-card expected-score eval charts (109), a full flaw-tag taxonomy overhaul (110), a filter-UX polish pass (111), and a Flaws-card rework with a single-game modal (112).

The original 2026-06-05 requirements covered only LIB-01..09 + the analysis-half LIBG requirements; the milestone expanded through roadmap evolution (SEED-036 / SEED-038 + locked `/gsd-explore` decisions). The deferred SEED-036 surfaces — the Analysis detail viewer (LIBG-04) and the on-demand best-move endpoint (LIBG-05) — were intentionally left for a later phase.

**Pure-frontend → full-stack arc:** Phase 104 had no backend changes; by Phase 108 the milestone added a derived `game_flaws` table (composite PK `(user_id, game_id, ply)`, typed tag-family columns + display payload), two Alembic migrations (create + impact-family alter), a `scripts/backfill_flaws.py` driver, and several new `/api/library/*` endpoints.

## Milestone Summary

**Decimal phases:** none (104–112 are all whole numbers).

**Key decisions:**

- **On-the-fly first, materialize later** — Phases 105/106 deliberately shipped the mistake classifier with no schema change; the `game_flaws` materialization (108) was a drop-in optimization once the Flaws subtab needed paginable per-flaw rows. LIBG-07's typed contract was designed for exactly this.
- **M+B-only materialization** (108 D-02/D-03) — inaccuracies stay aggregate / recomputed on the fly (eval chart 109); `game_flaws` stores only mistakes + blunders.
- **Single classify path** (108 D-10) — `classify_game_flaws` is invoked identically from the import hook, `reclassify_positions.py`, and `backfill_flaws.py`.
- **Outcome-independent impact tags** (110) — `reversed`/`squandered` reflect how far the position swung, not the eventual game result, replacing the outcome-dependent `result-changing`/`while-ahead`.
- **Inline payload over new endpoints** (109) — the per-ply ES series + flaw markers ride the existing `GET /api/library/games` response; no N+1, no migration.
- **Apply-only filters** (111) — every filter panel applies on demand, replacing the prior live-apply-on-desktop / apply-on-close-on-mobile split.

**Issues resolved during milestone:**

- Flaws miniboard showed the position one ply too early (108).
- Severity filter leaked "or worse" tiers; tag filters had no effect on the Games subtab (108).
- Eval chart broke off before checkmate; flaw dots clipped at chart edges (109).
- Mobile flaw-filter selections didn't reliably apply on close (108/111).

**Technical debt incurred / carried:**

- **Phase 111 has no GSD plan/summary artifacts** — shipped via direct commits; the planning record for it lives only in this archive + git history.
- The deferred SEED-036 surfaces (Analysis detail viewer LIBG-04, best-move endpoint LIBG-05) remain unbuilt — still specified in `.planning/seeds/SEED-036-library-page-milestone.md`.
- Dev-only `game_flaws` backfill (users 28 & 44) — other dev users may hold stale flaw rows; no prod data exists yet (`game_flaws` ships empty to prod on the v1.24 release).
- SEED-030 Track A (split oversized multi-concern modules) still open.
