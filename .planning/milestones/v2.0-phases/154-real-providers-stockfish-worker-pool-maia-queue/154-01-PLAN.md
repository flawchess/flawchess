<objective>
Implement `frontend/src/lib/engine/workerPool.ts`: a pool of 2–4 single-threaded
Stockfish.wasm Web Workers that grade candidate moves via `searchmoves`-restricted MultiPV
searches, fronted by a plain-array priority queue that dispatches work toward the
currently-highest-scoring root line first. This is the real implementation of the frozen
`EngineProviders.grade()` method (Phase 153), generalizing the already-shipped single-worker
`useStockfishGradingEngine.ts` state machine into N independent instances.

Purpose: Give the Phase 153 MCTS search core real parallel Stockfish grading (POOL-01),
priority scheduling (POOL-02), device-adaptive sizing within mobile Safari's memory ceiling
(POOL-04), and a clean abort surface Phase 155 will gate against the standalone eval bar.
Output: `workerPool.ts` (new) + `workerPool.test.ts` (new). No React, no UI, no wiring into
any hook — that is Phase 155. `types.ts`/`mctsSearch.ts` and every other Phase 153 core file
stay UNTOUCHED.
</objective>
