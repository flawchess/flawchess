<objective>
Implement `frontend/src/lib/engine/maiaQueue.ts`: a dedicated single-instance Maia policy
Web Worker (separate from the existing `useMaiaEngine` hook) that supplies per-node,
UCI-keyed move-probability distributions for an explicit per-side ELO, requesting only the
distinct ELOs the search needs (D-04) rather than the full ladder. This is the real
implementation of the frozen `EngineProviders.policy()` method (Phase 153), forking the
already-shipped `useMaiaEngine.ts` `{type:'analyze', fen, eloInputs}` protocol into a
non-React async queue.

Purpose: Give the Phase 153 MCTS search core real Maia priors per node (POOL-03) from an
isolated worker + cache (POOL-04 isolation), converting Maia's SAN-keyed softmax to the
UCI-keyed contract at the boundary.
Output: `maiaQueue.ts` (new) + `maiaQueue.test.ts` (new). No React, no UI, no wiring into any
hook — Phase 155. `useMaiaEngine.ts` and `maia-worker.js` stay UNTOUCHED (the worker already
supports `eloInputs.length===1`).
</objective>
