<objective>
Thread B, the committed real fix for SEED-085: fold the root player's own move-probability `P_you` back into the FlawChess Engine's root ranking so a move the user will almost never find (Nb5 @600, ~5%) can no longer be recommended as "Best", while leaving the per-move practical score `V(X)`, the Phase 153 search core, the backup rule, and leaf grades untouched.

The mechanism is D-01's locked saturating linear factor `rankScore = min(1, P_you/P_ref(ELO)) · V(X)` (β fixed at 1). Saturation is the load-bearing property: any move at or above `P_ref` gets factor 1, so the modal move can never be boosted above its own V — the rejected greedy `P·V` modal-move engine is impossible by construction, not calibrated away.

Purpose: change the ONE ranking function (`buildRankedLines`) that currently discards `P_you`, at the single seam both search runners snapshot through, so the fix covers both engines and propagates to arrows/badges/verdict without further wiring.
Output: a new pure `findability.ts` module, a `rootElo`-parameterized `buildRankedLines`/`buildSnapshot`, and regression tests including the three D-03 cases.
</objective>
