<objective>
Convert the `/analysis` **desktop** layout from scrolling fixed-pixel columns to a `100dvh`-locked,
fluid, height-aware layout (chess.com/lichess model) so content stops cutting off at the bottom on
small-laptop screens. Delivers SEED-088 decisions D-01…D-09 in one cohesive layout refactor across
4 existing files (+ one small extracted sizing helper and its test).

Purpose: Fix the reported small-laptop bottom-cutoff (eval chart falling off the fold) by locking the
frame to the viewport, making the board size to the available height/width budget, converting the
fixed-pixel flex row to a fluid CSS grid, and relocating the Tags panel out of the overflowing middle
column.

Output:
- `frontend/src/components/board/boardSize.ts` (new) + its unit test — the one piece of genuinely new
  logic (the `clamp(420, min(w,h), 600)` board-size computation).
- Height-aware `ChessBoard.tsx` (extended, not replaced, ResizeObserver).
- New Tailwind v4 `--breakpoint-desk3col` token + `short` custom variant in `index.css`.
- Inverted shell lock polarity in `App.tsx`; grid + tags-relocation + `min-h-0` chain in `Analysis.tsx`.

**This is a pure client-side CSS/JSX/layout refactor. No backend, no schema, no migrations, no new
npm packages, no new endpoints.**
</objective>
