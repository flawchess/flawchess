## Decisions Made

- **Collapsed three parallel bucket-emit blocks into one `_format_bucket_zones(bucket)` helper.** The plan action sketched three explicit `_format_gauge_zones_for_bucket("conversion")` / `"parity"` / `"recovery"` calls with per-branch registry lookups inlined in a single function via nested ternaries. Refactored that to a single helper keyed on bucket name with an if/elif/else branch — the call sites are symmetric (just the string argument changes) and the registry-lookup branching lives in one place. Output bytes are identical; the diff guard confirms.
- **Did not introduce `# ty: ignore` anywhere.** The codegen script passes ty on first try. Imports are guarded by `# noqa: E402` (the `sys.path.insert` must happen before the import) — no type-checker suppression needed.
- **Python float repr stability documented.** Phase 66 FE consumers will see `-0.1` not `-0.10` because Python's repr trims trailing zeros (`f"{-0.10}"` → `"-0.1"`). This is stable across Python 3.13 and deterministic across machines, so the CI diff guard compares exact bytes safely. Documented here for Phase 66 consumer switchover — TS numeric comparison handles `-0.1` and `-0.10` identically, so no behaviour difference, just a visual note.
