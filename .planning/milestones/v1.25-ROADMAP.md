# Milestone v1.25: Flaw-Stats Opponent Comparison

**Status:** ✅ SHIPPED 2026-06-12
**Phases:** 113–115 (incl. inserted 114.1) — 4 phases
**Total Plans:** 8 (113: 3 · 114: 1 · 114.1: 2 · 115: 2)

## Overview

v1.25 reworked the Library flaw-stats surface from a self-only descriptive panel into a **you-vs-opponent comparison**. Flaw rates only reveal a *specific* recurring weakness when contrasted against ELO-matched peers, not in absolute terms — so this milestone made the Library statistics actionable by comparing the user against their actual opponents.

Three dependency-ordered phases plus one inserted optimization:

- **Phase 113** — the data foundation: opponent flaws materialized into `game_flaws` via a nearly-free extension of the existing classifier (both colors were already evaluated), with the player/opponent split derived at query time (no column, no migration, no index).
- **Phase 114** — the benchmark backfill: per-cohort you−opponent deltas, Q1/Q3 quartile zones per ELO×TC cell, Cohen's-d collapse verdicts (the lightweight delta-IQR zone, deliberately NOT the heavy 99-breakpoint endgame CDF).
- **Phase 114.1 (INSERTED)** — SEED-041 item 9: replace `games.move_count` (±1 half-move accurate) with an exact `games.ply_count`, giving an exact per-game user-move denominator with zero `game_positions` access.
- **Phase 115** — the comparison surface: a unified per-100-moves paired-delta endpoint feeding a uniform 15-bullet grid (measure + CI + benchmark "typical" zone) replacing the old tag-distribution zone.

**Scope amendments during the milestone:** the original SEED-040 count-rate-vs-proportion family split (FLAWCMP-01/02) was superseded by a **unified per-100-moves paired-delta estimator for all 15 metrics** (114-CONTEXT D-01/D-04); FLAWCMP-02 (Wilson difference-of-proportions) was voided. FLAWX-03 (an `is_opponent` column + index) was voided in favour of a query-time `is_opponent_expr(ply, user_color)` helper (113-CONTEXT D-01/D-02/D-03) — no column, no migration, no index.

## Milestone Summary

**Inserted Phases:**

- Phase 114.1: Replace `move_count` with exact `ply_count` (inserted 2026-06-10 from SEED-041 §9 for an exact per-game denominator).

**Key Decisions:**

- Derive the player/opponent split at query time via `is_opponent_expr(ply, user_color)` rather than an `is_opponent` column + index (113-CONTEXT D-01/D-02/D-03). `is_opponent` is ~50% selective (no useful index) and a `GROUP BY` dimension, not a selective filter; the existing PK `(user_id, game_id, ply)` already covers two-sided per-game reads. The helper keeps the fragile ply-parity convention in one tested place.
- Unify on a single per-100-moves paired per-game delta estimator for all 15 flaw metrics (114-CONTEXT D-01/D-04), superseding SEED-040's count-rate-vs-proportion family split. FLAWCMP-02 (Wilson difference-of-proportions) voided. One estimator + one CI method across the whole grid is simpler and keeps the benchmark zone portable across ELO cells.
- Lightweight delta-IQR benchmark zone (Q1/Q3 per ELO×TC) instead of a 99-breakpoint percentile CDF — the CDF cost trajectory does not survive the future SEED-039 tactic-motif roadmap.
- Replace `move_count` with exact `ply_count` (114.1, SEED-041 §9) to get an exact per-game user-move denominator without scanning the 190M-row positions table.

**Issues Resolved:**

- Library flaw stats were self-only and not actionable in absolute terms — now contrasted against ELO-matched opponents.
- Mate transitions are now graded with the lichess mate ladder (shipped in the same release window), bringing blunder/mistake counts on analyzed lichess games to ~100% agreement with lichess (vs ~92% before).

**Issues Deferred / Tech Debt:**

- **FLAWTAC (SEED-039)** — tactic-motif comparison families (missed forks/pins/skewers) and the literal `missed-X` upgrade of the `miss` proxy: deferred to a future milestone; the comparison backbone already accommodates zoneless bullets.
- **FLAWCOV (SEED-012)** — analyzed-game coverage raising (client-side `stockfish.wasm` / server idle-time analysis): the feature ships for the ~37–51 heavy-analysis users; broader reach is owned by SEED-012.
- Phase 115 UAT (2 manual visual/data-basis scenarios) and verification (human_needed) were not formally signed off before close; the feature is shipped and live in production.
- Dev/benchmark-only `game_flaws` backfill; prod `game_flaws` continues to ship empty.
