## Summary

Add distinct playstyles to the bot, surfaced as a roster of 24 named bot personas
(4 styles × 6 ELO rungs, 800–1800) on the Bots page. Each persona is a complete
pinned opponent — preset, calibrated ELO, style params, opening book,
resign/draw-offer policy, avatar, bio — with no persona × strength picker (the
persona frame carries the personality, cf. chess.com bots); Custom mode keeps the
raw (ELO, preset) knobs for power users. Built in perceptibility-per-effort order:
style levers first, then the persona registry + Bots page UI (independently
playable), then per-persona harness calibration finalizes the labels, then a
roster re-layout + win stars.

### Phase 182: Style Levers

**Goal**: The bot's play is steerable by style parameters across the full ELO range — distinct opening choices, resign/draw behavior, and move preferences per style — without touching Custom mode or existing invariants (`policyTemperature`, player-derived inputs).
**Depends on**: Nothing (builds on existing v2.0/v2.3 engine primitives — `botSampling.ts`, `selectBotMove`, `trollOpenings.ts`)
**Requirements**: STYLE-01, STYLE-02, STYLE-03, STYLE-04, STYLE-05
**Success Criteria**:
  1. A bot configured with a given style plays its style-specific opening book for the first several moves (Trickster → a `trollOpenings.ts` trap line; others → their own curated book) instead of generic book/search moves.
  2. A Grinder-style draw/resign policy keeps playing where a neutral-style bot would resign or accept a draw; contempt is configurable per style.
  3. At Human-rung strength (800-1400), the move distribution visibly shifts toward the style's favored move features via a cheap chess.js move classifier reweighting Maia's policy probabilities.
  4. At Light/Deep-rung strength (1600-1800), the chosen move reflects the style's score bonus/malus and variance preference added to `practicalScore` before the existing softmax in `selectBotMove`.
  5. `botSampling.ts` helpers stay pure (no new search machinery); no style param is derived from the player's rating/play — entirely new bot-only fields, never `policyTemperature`.

**Plans**: 7/7 — completed 2026-07-22
- 182-01 — `RankedLine.childScoreSpread` variance signal (types.ts + treeCommon.ts)
- 182-02 — Draw contempt + pure `wouldBotResign` predicate (botDrawGate.ts)
- 182-03 — Curated per-style×color opening-line prefix sets (styleOpeningLines.ts)
- 182-04 — botStyle engine helpers: classifier + prior reweighting + score shaping + book weighting
- 182-05 — 4 named style bundles + D-11 measurement script + D-12 hand-tuning
- 182-06 — selectBotMove wiring: BotSettings.style + two regime hooks
- 182-07 — useBotGame integration: style book, contempt, resign hysteresis

### Phase 184: Persona Calibration & Strength Honesty

**Goal**: Each persona's displayed ELO label reflects its actual measured strength (style-induced delta included) rather than the raw underlying preset target, and labels honor the measured floor/ceiling honesty constraints.
**Depends on**: Phase 183
**Requirements**: CAL-04, CAL-05
**Success Criteria**:
  1. Every persona's ELO label is the calibrated ELO measured via harness runs on the Phase-173 internal anchor scale, with a per-persona offset absorbing the style-induced strength delta.
  2. The bottom rung (800) is labeled honestly as below the measured floor (~900 approx blitz, `beyond_ladder`-flagged) rather than a precise value.
  3. The top rung (1800) is capped at Deep's measured ceiling.
  4. The ~2 overnight harness runs (24 cells × ~4 anchors × ~24 games) produce per-persona offset data replacing the provisional labels.
