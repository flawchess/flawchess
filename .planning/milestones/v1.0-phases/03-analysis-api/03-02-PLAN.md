  <name>Task 1: Write repository integration tests</name>
  <files>tests/test_analysis_repository.py</files>
  <action>
Create `tests/test_analysis_repository.py` with tests using the `db_session` fixture (real PostgreSQL, rolled-back transactions).

**Test data helper:** Create a helper function `_seed_game(session, **overrides)` that inserts a Game + GamePosition row with sensible defaults (user_id=1, platform="chess.com", result="1-0", user_color="white", rated=True, time_control_bucket="blitz", played_at=recent date). Accepts overrides for any field. Also accepts `hash_overrides` dict for `full_hash`, `white_hash`, `black_hash` on the position. Use `session.add()` + `await session.flush()` to get IDs. Generate unique `platform_game_id` per call (use a counter or uuid).

**TestMatchSide** (ANL-02):
- `test_match_side_white`: Seed a game with known white_hash=1234. Query with hash_column=GamePosition.white_hash, target_hash=1234. Assert 1 result.
- `test_match_side_black`: Same pattern with black_hash.
- `test_match_side_full`: Same pattern with full_hash.
- `test_match_side_no_match`: Query with wrong hash returns 0 results.

**TestFilters**:
- `test_time_control_filter` (FLT-01): Seed 3 games: bullet, blitz, rapid. Filter time_control=["blitz"]. Assert only 1 returned.
- `test_rated_filter` (FLT-02): Seed 2 games: rated=True, rated=False. Filter rated=True. Assert 1 returned.
- `test_recency_filter` (FLT-03): Seed 2 games: one played today, one played 2 years ago. Filter with cutoff = 30 days ago. Assert only recent game returned.
- `test_color_filter` (FLT-04): Seed 2 games: user_color="white", user_color="black". Filter color="white". Assert 1 returned.
- `test_combined_filters`: Apply time_control + rated + color together. Verify intersection behavior.

**TestDeduplication**:
- `test_transposition_counts_once`: Seed 1 game with 2 GamePosition rows at different plies but same full_hash. Query returns count=1 and 1 game (not 2).

**TestPagination**:
- `test_pagination_offset_limit`: Seed 5 games matching same hash. Query with offset=2, limit=2. Assert 2 games returned, total=5.

All tests use `query_matching_games` and/or `query_all_results` directly — no HTTP layer.
