<objective>
Wire the import service orchestrator and API router that ties platform clients, Zobrist hashing, and the repository layer together.

Purpose: This is the capstone plan that delivers the user-facing import functionality. POST /imports launches a background job, GET /imports/{job_id} returns live progress. The import service orchestrates fetching games from platform clients, computing Zobrist hashes, and bulk inserting to the database. It supports incremental sync via last_synced_at and prevents duplicate concurrent imports.

Output: Working import pipeline accessible via REST API. User can trigger chess.com or lichess import and poll progress.
</objective>

  <name>Task 1: Import service with job registry and background orchestrator</name>
  <files>
    app/services/import_service.py,
    tests/test_import_service.py
  </files>
  <action>
    Create `app/services/import_service.py` with:

    1. **In-memory job registry** (module-level dict):
       ```
       _jobs: dict[str, JobState] = {}
       ```
       Where `JobState` is a dataclass with: job_id, user_id, platform, username, status (JobStatus enum: PENDING/IN_PROGRESS/COMPLETED/FAILED), games_fetched (int), games_imported (int), error (str | None).

    2. **Job management functions:**
       - `create_job(user_id, platform, username) -> str` — Creates UUID, stores JobState in _jobs, returns job_id.
       - `get_job(job_id) -> JobState | None` — Returns from _jobs dict.
       - `find_active_job(user_id, platform) -> JobState | None` — Checks _jobs for any PENDING/IN_PROGRESS job for this user+platform. Returns it if found (prevents duplicate imports).

    3. **run_import(job_id: str) -> None** — The main async function launched via asyncio.create_task:
       a. Get JobState from _jobs. Set status = IN_PROGRESS.
       b. Open an async DB session via async_session_maker.
       c. Check for last_synced_at by querying import_job_repository.get_latest_for_user_platform. If found, compute since parameters.
       d. Create DB import_job row via import_job_repository.create_import_job.
       e. Create httpx.AsyncClient.
       f. Based on platform, call fetch_chesscom_games or fetch_lichess_games as an async iterator. Pass on_game_fetched callback that increments job.games_fetched.
       g. Batch games: collect up to 50 game dicts at a time, then:
          - Call bulk_insert_games(session, batch) to get new_game_ids.
          - For each new game ID + its PGN: call hashes_for_game(pgn) then build position row dicts (game_id, user_id, ply, white_hash, black_hash, full_hash).
          - Call bulk_insert_positions(session, position_rows).
          - await session.commit().
          - Increment job.games_imported by len(new_game_ids).
       h. After all games processed: set status = COMPLETED, update DB import_job with final counts, last_synced_at = datetime.now(timezone.utc), completed_at.
       i. On any exception: set status = FAILED, error = str(exception), update DB import_job. Log the error. Do NOT re-raise (this is a background task).
       j. Wrap everything in try/except to ensure the job always reaches a terminal state.

    Create `tests/test_import_service.py`:
    - Test create_job returns a valid UUID and stores in _jobs.
    - Test find_active_job returns existing in-progress job for same user+platform.
    - Test find_active_job returns None when no active job exists.
    - Test run_import with mocked platform client and repository: verify it transitions through PENDING -> IN_PROGRESS -> COMPLETED.
    - Test run_import calls hashes_for_game for each newly inserted game's PGN.
    - Test run_import with client raising ValueError (invalid username): verify status = FAILED with error message.
    - Test incremental sync: mock get_latest_for_user_platform returning a job with last_synced_at, verify since parameter passed to client.
    - Mock asyncio.sleep, httpx.AsyncClient, session, and repositories. Focus on orchestration logic.
    - Important: clear _jobs dict in a fixture before each test to avoid cross-test pollution.
