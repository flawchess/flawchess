<objective>
Close UAT gap: chess.com client crashes with JSONDecodeError on non-200 archives responses instead of raising a clean ValueError.

Purpose: The import pipeline's error handling contract promises that invalid usernames produce a failed job with an error message. Currently any non-200 status on the archives endpoint (410 for email-format names, 403, 500, etc.) falls through to `.json()` on a non-JSON body and crashes the background task with an unhandled exception instead of transitioning the job to "failed" status.

Output: Hardened `chesscom_client.py` with status-code guards before every `.json()` call, plus regression tests covering the new error paths.
</objective>

    ```python
        resp = await client.get(archive_url, headers=_HEADERS)

    # Skip non-200 archive responses rather than crashing on .json()
    if resp.status_code != 200:
        continue

    games = resp.json().get("games", [])
    ```
