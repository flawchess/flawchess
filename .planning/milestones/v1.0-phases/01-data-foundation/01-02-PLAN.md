<feature>
  <name>Zobrist Hash Computation Module</name>
  <files>app/services/zobrist.py, tests/test_zobrist.py, tests/__init__.py, tests/conftest.py</files>
  <behavior>
    - compute_hashes(board) returns (white_hash, black_hash, full_hash) as signed 64-bit integers
    - Starting position hashes are deterministic: compute_hashes(Board()) == compute_hashes(Board()) across calls
    - Different positions produce different full_hash values
    - white_hash only reflects white piece positions: after e4 e5 vs e4 d5, white_hash is identical (same white pawn on e4)
    - black_hash only reflects black piece positions: symmetrically, black_hash unchanged when only white moves
    - All three hash values are within PostgreSQL BIGINT range: -2^63 <= h <= 2^63-1 (ctypes.c_int64 conversion applied)
    - hashes_for_game(pgn_text) returns list of (ply, wh, bh, fh) tuples
    - ply 0 is the initial position (before any moves)
    - For PGN "1. e4 e5 2. Nf3 *", returns 5 entries (ply 0 through 4)
    - Invalid/empty PGN returns empty list
    - Same position reached via transposition produces identical hashes (hash set overlap)
  </behavior>
  <implementation>
    RED phase: Write tests/test_zobrist.py with all the behavior cases above. Tests import from app.services.zobrist which does not exist yet. Run tests — they must all FAIL (ImportError or AssertionError).

    GREEN phase: Create app/services/zobrist.py implementing:
    1. `_color_hash(board, color)` — iterate board.occupied_co[color] squares, XOR POLYGLOT_RANDOM_ARRAY entries using indexing scheme: `64 * ((piece_type - 1) * 2 + color_pivot) + square` where color_pivot is 0 for WHITE, 1 for BLACK. Convert result with ctypes.c_int64(h).value.
    2. `compute_hashes(board)` — calls _color_hash for WHITE and BLACK, uses chess.polyglot.zobrist_hash(board) for full_hash. Converts full_hash with ctypes.c_int64 as well. Returns tuple of three signed int64 values.
    3. `hashes_for_game(pgn_text)` — parses PGN with chess.pgn.read_game, iterates mainline_moves, calls compute_hashes at each position starting from ply 0. Returns list of (ply, wh, bh, fh). Wraps in try/except returning [] on parse failure.

    Run tests — they must all PASS.

    REFACTOR phase: Clean up if needed, ensure docstrings are clear, run ruff for formatting.
