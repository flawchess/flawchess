<objective>
Fix the flaw-pipeline off-by-one (SEED-044): the engine drain stored `eval_cp`/`eval_mate` for the PRE-PUSH position (eval BEFORE the move) while `classify_game_flaws` assumes POST-MOVE (eval AFTER the move). This is correct for lichess `%eval` games and wrong for every chess.com engine-evaluated game — the majority of the dataset.

Standardize on the locked canonical convention: POST-MOVE storage everywhere. Change only the drain WRITE side plus the dedup transplant read side; the classifier needs NO logic change (it already reads post-move). Add the terminal-position eval so game-ending blunders are assessable. Encapsulate the one-ply convention shift in exactly ONE helper.

Purpose: Makes flaw stats correct for all chess.com users (Flaw-Stats panel, opponent comparison, every flaw-derived metric). Closes EVALFIX-01/02/03 and seeds the EVALFIX-05 regression guard.
Output: Corrected `eval_drain.py` write path, comment-only update to `flaws_service.py`, drain-level + dedup-shift unit tests, and the three regression fixtures.
</objective>
