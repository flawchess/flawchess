### `_classify_and_fill_oracle` (new function in `eval_drain.py`)

```python
async def _classify_and_insert_flaws(
    session: AsyncSession,
    game_ids: Sequence[int],
) -> None:
    games_result = await session.execute(select(Game).where(Game.id.in_(game_ids)))
    games = games_result.scalars().all()

    for game in games:
        try:
            positions_result = await session.execute(
                select(GamePosition)
                .where(
                    GamePosition.game_id == game.id,
                    GamePosition.user_id == game.user_id,
                )
                .order_by(GamePosition.ply)
            )
            positions = list(positions_result.scalars().all())

            result = classify_game_flaws(game, positions)
            if "reason" in result:
                continue  # GameNotAnalyzed

            rows = [
                flaw_record_to_row(user_id=game.user_id, game_id=game.id, flaw=flaw)
                for flaw in result
            ]
            await bulk_insert_game_flaws(session, rows)

        except Exception as exc:
            sentry_sdk.set_context("game_flaws", {"game_id": game.id, "user_id": game.user_id})
            sentry_sdk.capture_exception(exc)
            continue
```

## Shared Patterns

### Session Discipline (CLAUDE.md hard rule — all new async functions)

**Source:** `app/services/eval_drain.py` `_full_drain_tick` step structure (lines 909-1027)

**Apply to:** All new functions in `eval_queue_service.py` and the modified `_full_drain_tick`

```python
# CORRECT: short session, close, then gather
async with async_session_maker() as session:
    row = await session.execute(...)
    await session.commit()
# session closed here

results = await asyncio.gather(...)  # NO session open

async with async_session_maker() as write_session:
    await write_session.execute(...)
    await write_session.commit()
```
Never pass an `AsyncSession` across the `asyncio.gather` boundary.

### Sentry Error Reporting

**Source:** `app/services/eval_drain.py` lines 995-1001 (per-game aggregated event)

**Apply to:** `_classify_and_fill_oracle`, `_full_drain_tick` circuit breaker, any new exception handlers

Never embed `game_id` or `ply` in the message string. Per-ply Sentry events are forbidden (WR-05).
