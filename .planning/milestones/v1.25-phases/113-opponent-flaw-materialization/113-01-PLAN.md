<objective>
Generalize the single flaw-classification kernel to emit BOTH movers' mistakes/blunders (FLAWX-02), and introduce the single tested `is_opponent_expr` parity helper that recovers the player/opponent split at read time (FLAWX-01). This is the foundation wave: Plan 02 (reader gating) and Plan 03 (backfill) both depend on these two artifacts existing.

Per CONTEXT D-01/D-02/D-03 (authoritative, overriding stale ROADMAP/SEED-040): there is NO `is_opponent` column, NO Alembic migration, NO new index. The split is derived at query time from ply parity + `games.user_color`. FLAWX-03 is VOIDED — do not plan a column/migration/index.

Purpose: Persist both sides' flaws at zero added engine cost (the all-moves pass already evaluates both colors), so phase 115 can contrast the user against their actual opponents. The parity convention is fragile (documented prior off-by-one bug); centralizing it in one unit-tested helper is the core risk mitigation.

Output: Generalized `classify_game_flaws`, the `is_opponent_expr` helper (+ optional `player_only_gate` wrapper), and the Wave-0 test stubs that subsequent plans rely on.
</objective>
