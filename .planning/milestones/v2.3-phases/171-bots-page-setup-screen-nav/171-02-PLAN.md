<objective>
Expose the user's lichess-blitz-equivalent rating on `/users/me/profile` (D-07) and repoint the analysis
board's free-play ELO default at it (D-08).

The setup screen's ELO default (Plan 05) needs a *converted* rating; the frontend cannot get one today.
`profile.current_rating` is the raw platform rating from the user's most recent game — inflated for
chess.com users. `user_rating_anchors.anchor_rating` is already a blended lichess-equivalent median, and
is already the number Phase 167 trusts to stamp a stored bot game's player rating. Surface it. D-08 rides
along: the analysis board's free-play branch has the same half-conversion bug and is fixed by the same field.

Purpose: one normalized rating source, so the two ELO surfaces agree.
Output: a new read-only response field, its tests, and a one-line hook repoint.
</objective>
