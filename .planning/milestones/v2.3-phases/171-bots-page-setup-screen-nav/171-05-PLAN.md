<objective>
Build the pre-game setup screen (PLAY-02, D-09): ELO, play-style, color, and a lichess-preset time control,
then Start.

This replaces the D-14 hardcoded `BOT_GAME_SETTINGS` stub that Phase 170 deliberately left behind. It is a
pre-game SCREEN, not a modal and not a new route — a modal over the board would force `useBotGame` to mount
with placeholder settings and remount on Start, throwing the warm engine pool away.

The screen is a pure, self-contained form: it takes a profile-derived ELO default and an `onStart` callback,
and it emits a fully-resolved `BotGameSettings`. Plan 06 wires it into `Bots.tsx`.

Purpose: PLAY-02, and the single entry point for every new bot game.
Output: one component, one test file.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → setup screen | Prefill values are untrusted; `readSetupSettings`'s validator (Plan 04) is the gate. |
| setup screen → `useBotGame` → `POST /bots/games` | The settings chosen here are the payload the game and its eventual store request are built from. All four inputs are closed sets: the ladder-snapped ELO, a 0.05-step clamped blend, one of 3 colors, one of 9 TC presets. |
