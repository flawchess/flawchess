<objective>
Fix SEED-095: make harness Stockfish grading + adjudication deterministic under load (depth-only `go` + `Clear Hash` per call, D-10), and harden the watchdog so a single late reply no longer kills a multi-hour sweep (independent generous timeout + retry-in-place, D-11). Measure the Clear-Hash throughput cost on a bounded run so downstream plans size the sweep from real data.

Purpose: Early stopping (Plan 03) thresholds on grade margins, and SEED-096 is explicit that early stopping AMPLIFIES grade noise — so grades must be load-independent BEFORE any stop rule is mirrored into the harness. This is SEED-096 sequencing step 2, and a prerequisite for SC3/SC4/SC5.
Output: Deterministic depth-only grading/adjudication with a decoupled watchdog and retry-in-place; a recorded throughput number.
</objective>
