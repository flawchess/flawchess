## Early-stop rule + budget

**Initial pacing-target question answered with free text:** "Can we make the bot spend more time for complicated positions (e.g. when there are several plausible moves according to maia) and less in opening routine and simple positions (one move clearly best)?" — Confirmed as the mechanism's core intent; surfaced the opening-routine subtlety (a margin-only rule runs LONG in near-tie openings), leading to the two-sided rule below.

| Option | Description | Selected |
|--------|-------------|----------|
| Yes, two-sided rule | Stop when settled either way: clear winner (margin ≥ X, stable K) OR near-tie flatness (top candidates within ε, ranking stable); min-nodes floor | ✓ |
| Margin-only rule | Simpler, but routine openings run to full budget | |

**User's choice:** Yes, two-sided rule (recommended)

| Option | Description | Selected |
|--------|-------------|----------|
| ~5s median, ~15s worst | MAX_NODES sized so a full-budget contested move stays ~15s (~50 nodes ballpark) | ✓ |
| ~10s median, ~30s worst | Stronger top end, long waits | |
| ~2-3s median, ~8s worst | Snappiest, weakest top end | |

**User's choice:** ~5s median, ~15s worst (recommended)

| Option | Description | Selected |
|--------|-------------|----------|
| Same rule all blends, re-measure | SC4 sweep re-measures mid-slider strength; no special-casing | ✓ |
| Full budget for mid-blends | Keeps sampling distribution but mid-slider stays slow | |
| Blend-scaled thresholds | Over-engineering for v1 | |

**User's choice:** Same rule all blends, re-measure (recommended)

| Option | Description | Selected |
|--------|-------------|----------|
| Pin bot-play concurrency | Fixed constant for bot games (analysis board stays adaptive); harness uses same value; full determinism | ✓ |
| Keep adaptive, validate at 2-3 levels | Fastest UX per device, accepts small strength variance | |

**User's choice:** Pin bot-play concurrency (recommended)

Super strong stockfish anchors vs weak FlawChess bots doesn't make any sense.
