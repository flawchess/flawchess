<objective>
Ship the confidence-based early-stop mechanism inside `mctsSearch` and a dedicated, ADDITIVE bot-play budget profile in `useFlawChessEngine.ts`, without changing the analysis board's search at all. This is the frontend half of SC3 (D-05..D-09).

Purpose: The current engine pays the full 400-node budget on every `blend > 0` move (~110s). The two-sided stop rule (clear-winner OR near-tie flatness) is what makes routine positions finish fast while contested positions run toward the (smaller) cap; a smaller bot-only `MAX_NODES` caps the worst case. Both must ship on a bot-only profile so the shipped-in-production analysis board is untouched.
Output: An optional `SearchBudget.stopRule`, its two-sided evaluation in the apply loop, an `EngineSnapshot.stopReason`, and an exported FLAWCHESS_BOT_* constant block with provisional values.
