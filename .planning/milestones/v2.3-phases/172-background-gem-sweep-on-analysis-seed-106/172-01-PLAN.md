  <behavior>
    - Full-book game (every ply matches a trie node with a result): returns `len(moves)`-ish — the index of the LAST node carrying a result, 1-based.
    - Game that leaves book mid-line (e.g. Italian first 6 plies, then a novelty): returns the depth of the deepest *result-carrying* node reached before the divergence, NOT the divergence index.
    - No book match at all (first move unknown to the trie): returns 0.
    - Empty `moves` list: returns 0.
    - A prefix that walks trie children but reaches no `result` node returns 0 (walking ≠ matching).
  </behavior>
  <action>

It walks the same `_TRIE` singleton but tracks the matching index alongside the node: initialise `last_depth = 0`, `enumerate(moves)`, `break` on a missing child, and set `last_depth = i + 1` whenever the reached node has a non-None `result`. Return `last_depth`.

    Do NOT call `_normalize_pgn_to_san_sequence` — the caller supplies already-tokenized SAN (`GameFlawCard.moves`), which is the same SAN dialect `openings.tsv` uses (including `O-O` castling). Do NOT change `find_opening`'s signature or behavior: `app/services/normalization.py` (lines 301, 446, 662) and `app/routers/position_bookmarks.py` (line 115) call it and must remain untouched.

    Docstring must state: 1-based ply depth of the deepest known-opening match, 0 = no match; cite Phase 172 / SEED-106 D-06 and the "no column, no migration, no backfill" rationale.
