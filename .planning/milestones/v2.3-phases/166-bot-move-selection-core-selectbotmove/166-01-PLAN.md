<objective>
Deliver a single pure, provider-agnostic `selectBotMove` function plus its pure
sub-helpers that, given a position and (ELO, play-style) settings, returns a
legal UCI move at the bot's configured strength. The play loop (Phase 169) and
calibration harness (Phase 168) both import and run this unchanged via `@/`.

Purpose: This is the keystone of the v2.3 Bot Play milestone — the one piece of
move-selection logic reused verbatim by the app and the harness, so the harness
measures exactly what users play against.

Output: Two new source modules (`botSampling.ts` pure helpers, `selectBotMove.ts`
async orchestrator) and their two co-located unit-test files, all under
`frontend/src/lib/engine/`. Zero new dependencies, no UI, no backend.
</objective>

    `weightedPick(entries: [string, number][], rng: () => number): string | null` — sort entries UCI-ascending FIRST (same comparator as `select.ts`), sum weights, return null when total <= 0 (D-13 degenerate signal), else cumulative-distribution walk with a draw of `rng() * total`; clamp the exhausted-loop edge case to the last sorted UCI so a `rng()` value of exactly 1 never yields undefined.

    `sampleRankedLines(lines: readonly RankedLine[], tau: number, rng): string | null` — return null when lines is empty; compute `maxScore = max(practicalScore)` across lines and build weights as `Math.exp((line.practicalScore - maxScore) / tau)` (max-subtraction stability trick — mandatory, eliminates the overflow class in RESEARCH Pitfall 2), keyed by `line.rootMove`; delegate to `weightedPick`.
