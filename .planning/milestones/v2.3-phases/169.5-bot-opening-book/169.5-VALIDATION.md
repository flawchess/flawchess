## Non-inferable checks (need a held-out / property-based / revert-proof test)

Per project memory (`feedback_mutation_test_gap_closures`): **a gap fix must be provable by REVERTING it and confirming the test fails.** Symbol presence, grep, and "the constant exists" are NOT acceptable proof. These are the ones most likely to pass vacuously — or, in #5's case, to be silently broken by this phase without any test noticing:

1. **SC1 "no search while in book"** — a test asserting "the bot moved" passes whether or not a search ran. The assertion must be a **provider spy that fails the test if `search` (or the Stockfish `grade` path) is invoked during a book ply**. Revert-check: re-route the book ply through `selectBotMove`'s search regime and this test must go red.
2. **SC3 "openings vary"** — an unseeded randomness assertion is flaky, and a single-run assertion passes vacuously. Use a **seeded PRNG over N runs and assert a distribution property** (>1 distinct first move; the modal move is a plausible main move, not a junk move). Revert-check: replace policy-weighted sampling with argmax and the distinct-move count collapses to 1, going red.
3. **SC4 "one-way exit"** — easy to write a test that never actually re-enters. Assert explicitly that **after a searched move, a subsequent still-in-book position does NOT return to the book path**. Revert-check: remove the latch and the test must go red.
4. **The corpus-parity guard** (all 3,641 TSV lines replay byte-identically through chess.js) is a **property test over the real shipped asset**, not a hand-picked fixture. It is the one thing standing between "the book works" and "the book silently matches nothing and the bot quietly searches every move" — the failure mode is invisible without it, because a book that matches nothing still produces perfectly legal moves.
5. **SC1's grade-suppression must not regress the 169 D-01 draw gate.** SC1 is enforced literally — zero Stockfish leaf evals while in book — which means the post-commit `pool.grade` refresh of `lastRootPracticalScoreRef` never fires during the book window, so the ref holds whatever it was initialized to. That is only safe if the initial value cannot satisfy `wouldBotAcceptDraw`. It could: the gate's endgame condition is `queensAreOff(chess) || moveNumber >= DRAW_ACCEPT_MIN_FULLMOVE` — **queens-off ALONE opens it** — and the shipped corpus contains lines that trade queens inside `BOOK_PLY_CAP` (`openings.tsv:1065`, `B10 Caro-Kann Defense: Endgame Variation — 1. e4 c6 2. Nf3 d5 3. d3 dxe4 4. dxe4 Qxd1+ 5. Kxd1`, queens off by ply 9). With a `0.5` default the near-equal check is trivially true, and the bot would accept a draw in a position it never evaluated. The fix is a not-yet-evaluated sentinel (`null`) that the gate refuses. **Test:** drive that exact line as a book game (bot = black) with `mockGrade` provably never called, offer a draw, assert the bot DECLINES. **Revert-check:** restore the `0.5` default (or drop the sentinel guard) and the test goes red with a draw-by-agreement outcome. A hook test is required — the pure-gate unit test alone cannot see the suppressed-grade wiring that leaves the ref unwritten.
