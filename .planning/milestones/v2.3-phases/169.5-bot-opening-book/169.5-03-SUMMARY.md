# Phase 169.5 Plan 03: Engine Prewarm Summary

**`WorkerPool.warm()` and `MaiaQueue.warm()` — spawn the engine workers without working them, so the opening-book window pays the spawn cost instead of the first move the bot actually has to search.**

## Performance

- **Duration:** ~12 min
- **Completed:** 2026-07-13
- **Tasks:** 2
- **Files modified:** 4 (2 providers, 2 test files)

## Accomplishments
- `WorkerPool.warm(): void` — added to the interface, implemented as a bare `ensureSpawned()` forward, added to the returned object literal. Spawns `computePoolSize()` slots with no `go` and no movetime spend
- `MaiaQueue.warm(): void` — same shape; the worker is constructed and `{type:'init'}` posted (starting the ONNX weight load) with nothing enqueued to `analyze`
- Both interface doc comments name the anti-pattern explicitly, in place, because its failure is silent: `grade(fen, [])` returns on the WR-05 empty-candidates guard *before* `ensureSpawned()` is reached, so it spawns nothing at all
- 7 new tests (5 WorkerPool, 2 MaiaQueue) reusing each file's existing mock-Worker harness — no second harness was built
- All 53 tests across both files pass, including every pre-existing one (no regressions)
- The frozen `EngineProviders` contract in `types.ts` is untouched; both concrete surfaces remain structurally assignable to `EngineProviders.grade` / `.policy` (`tsc -b` clean)

## Decisions Made
- Nothing outside the two forwarding methods was touched: `grade()`, `policy()`, `ensureSpawned()`, `stopAll()`, `terminate()`, `computePoolSize()`, and the queue/cache machinery are all byte-unchanged, as the plan required
- The WorkerPool "no search" assertion scans every constructed worker's `messages` for a `go` prefix rather than asserting an exact message list — spawn-time UCI handshake traffic (`uci`, `setoption`, `isready`) is expected and must stay allowed
- The `warm() spawns computePoolSize() workers` assertions call `computePoolSize()` rather than hard-coding 4, so they stay correct if the sizing constants are ever retuned (the surrounding describe already stubs `hardwareConcurrency` to make it deterministic)
