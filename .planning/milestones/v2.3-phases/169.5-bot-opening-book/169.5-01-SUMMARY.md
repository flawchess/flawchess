**Extended `openings.ts`'s single TSV parse to also build a prefix `Set<string>` (`loadOpeningPrefixSet()`) alongside the existing full-line name `Map`, and committed a permanent whole-corpus SAN-parity property test over the real 3,641-line shipped asset.**

## Accomplishments
- `openings.ts`'s `buildLookup()` now returns `OpeningLookup { fullLineMap, prefixSet }` from a single cached fetch+parse — `prefixSet` holds every non-empty prefix of every ECO line, keyed by space-joined SAN tokens
- New export `loadOpeningPrefixSet(): Promise<ReadonlySet<string>>` — the candidate-membership seam plan 04's bot opening book will consume; does not swallow fetch errors (unlike `preloadOpenings()`)
- `findOpening()` / `preloadOpenings()` kept their exact exported signatures and behavior; `useChessGame.ts` (their only consumer) compiles with zero edits
- Committed `frontend/src/lib/__tests__/openings.test.ts` — a standing whole-corpus SAN-parity property test (reads the real `openings.tsv` off disk, replays every line through `chess.js` independently of `openings.ts`'s own parsing, asserts a failure-array equality plus a `rowCount > 3000` floor) plus synthetic and real-TSV prefix-set assertions
- Revert-checked the corpus-parity test live: mutated one SAN token in `openings.tsv`, confirmed the test goes red and names the exact offending ECO line (`A00`, line 2), then restored the file with zero diff

## Decisions Made
- Kept `OpeningLookup` as a two-field interface (`fullLineMap`, `prefixSet`) rather than two separate cached promises, so the fetch/parse stays a single shared operation as the plan required
- `loadOpeningPrefixSet()` deliberately does not `.catch()` its rejection (contrast `preloadOpenings()`), per the plan's explicit instruction that the future book caller (plan 04) decides how to react to a fetch failure
- Test file re-imports `openings.ts` fresh per test via `vi.resetModules()` + dynamic `import()` after installing that test's own `fetch` stub (mirrors the existing `sounds.test.ts` precedent), so a synthetic-TSV test can't be poisoned by a real-TSV test's cached `lookupPromise`
