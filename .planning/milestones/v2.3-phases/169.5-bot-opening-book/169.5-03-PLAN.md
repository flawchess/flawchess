<objective>
Give the engine providers a way to spawn without working.

`WorkerPool` and `MaiaQueue` both spawn lazily on first real use (D-02): the Stockfish WASM workers appear on the first `grade()`, the Maia ONNX session on the first `policy()`. Under the book, the first `grade()` of a game does not happen until the bot LEAVES book — which is exactly the move we least want to be cold. SC5 wants the spawn cost paid during the book window instead.

**The trap this plan exists to avoid** (RESEARCH.md Pitfall 2): the obvious prewarm ping, `pool.grade(fen, [])`, spawns NOTHING. `grade()`'s WR-05 guard returns `Promise.resolve(new Map())` for an empty `candidateUcis` array BEFORE `ensureSpawned()` is ever reached — a deliberate guard (an empty `searchmoves` list would make Stockfish search ALL moves and burn its full movetime), and a silent trap for anyone reusing `grade()` as a warm signal. It does not throw, it does not error, it simply does not work. So `warm()` is a real, dedicated export that calls `ensureSpawned()` directly.

Purpose: SC5.
Output: `WorkerPool.warm()`, `MaiaQueue.warm()`, and the tests that pin them (including one that pins the trap itself).
</objective>
