## Per-game TSV column contract (shared with Plan 03 — MUST match exactly)

Header row (tab-separated), written once at run start:
`pass  anchor_white  anchor_black  result  reason  plies  game_index  opening  seed  git_sha`
`result` ∈ {`white_win`, `black_win`, `draw`}; `pass` ∈ {`probe`, `measure`}.
One row is streamed the instant each game finishes (WR-01 durability). The raw
ledger's first line is a clean header (no leading comment). The D-13 "internal
scale — NOT human ELO" caveat lives in the per-pair aggregate file's metadata
footer and every rating-bearing artifact, not in this ledger's header.
