## Per-game TSV column contract (shared with Plan 02 â€” MUST match exactly)

`load_games` parses a tab-separated file whose header row is exactly:
`pass  anchor_white  anchor_black  result  reason  plies  game_index  opening  seed  git_sha`
where `result` âˆˆ {`white_win`, `black_win`, `draw`}. `build_win_counts` folds
each game into `win_counts[(i, j)]` (i's score vs j) regardless of color:
`white_win` â†’ +1 to anchor_white vs anchor_black; `black_win` â†’ +1 to
anchor_black; `draw` â†’ +0.5 to each. The raw ledger's first line is a clean
column header (no leading comment) so this parser stays trivial; the D-13
"internal scale â€” NOT human ELO" caveat lives in the output artifacts, not the
input ledger header.
