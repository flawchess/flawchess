<objective>
Build the authoritative two-tier readiness backend and the frontend poll hook that
every gating surface in this phase consumes. This is the contract layer: a new
`GET /imports/readiness` endpoint exposing `{tier1, tier2, pending_count, total_count}`,
a `has_any_rows` existence helper, the matching Pydantic schema + TS type, and the
`useReadiness` hook. No UI gating yet â€” those land in Wave 2 against this contract.

Purpose: Replace the eval-coverage 100%-transition heuristic (which races Stage B) with
a single authoritative signal. Tier 2 derives from percentile-row existence â€” a post-commit
signal that cannot race the `asyncio.create_task(compute_stage_b())` schedule (Constraint 1).
Output: Working, tested endpoint + hook with safe defaults (tier1/tier2 false before first fetch).
</objective>
