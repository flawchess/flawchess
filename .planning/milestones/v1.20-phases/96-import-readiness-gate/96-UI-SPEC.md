## Typography

Minimum font size is `text-sm` (14px) — CLAUDE.md hard rule.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (text-base) | 400 (regular) | 1.5 |
| Label / caption | 14px (text-sm) | 400 (regular) | 1.5 |
| Heading (section) | 20px (text-xl) | 600 (semibold) | 1.2 |
| Display | 28px (text-2xl) | 700 (bold) | 1.2 |

New phase components (`EndgamesProcessingState`, `EvalCpuPlaceholder`) use:
- `text-sm` (14px, weight 400) for inline progress text inside bars — matches `EvalCoverageHeader` exactly.
- `text-base` (16px, weight 400) for body copy on the `EndgamesProcessingState` whole-page locked state.
- `text-xl` (20px, weight 600) for the `EndgamesProcessingState` heading ("Analyzing endgames").

Source: `EvalCoverageHeader.tsx` uses `text-sm`; import page headings already at `text-xl`/`text-2xl`. No new type ramp introduced.

### `useReadiness` hook (new)
**Path:** `frontend/src/hooks/useReadiness.ts`
**Purpose:** Two-tier readiness poll at 3s cadence. Returns `{ tier1, tier2, pendingCount, totalCount, isLoading }`. Stops polling when `tier2=true`. Safe defaults: `tier1=false`, `tier2=false` before first fetch.

**TanStack Query key:** `['imports', 'readiness']` — deduplicates across all consumers.

## Import Page State Machine

The import page (`Import.tsx`) is redriven as a state machine off `useReadiness`. Exact states and copy:

| State | Condition | Heading / Status Copy | CTA |
|-------|-----------|----------------------|-----|
| No games yet | `totalCount == 0`, no active job | (existing import form) | "Sync chess.com" / "Sync lichess" (existing) |
| Importing | active job exists | `ImportProgressBar` — "Importing {username} ({platform})..." (existing) | dismiss |
| Tier 1 reached | `tier1=true`, `tier2=false` | "Games imported — openings ready." | "Explore Openings" button (in-page, `variant="default"`, `data-testid="btn-explore-openings"`) |
| Analyzing endgames | `tier1=true`, `tier2=false`, `pendingCount > 0` | "Analyzing endgames ({analysedCount} / {totalCount})" — below the Tier-1 CTA | (no additional CTA) |
| Fully ready | `tier2=true` | "Ready — all analysis complete." | (nav is fully unlocked; no forced navigation) |

**Constraint:** The "Imported N games" green completion message (current `progressText` when `isDone`) must NOT say "Import complete" at hot-import done, as Constraint 3 forbids over-claiming. The new "Games imported — openings ready." copy replaces the `isDone` branch.

## Interaction Contracts

### Route gating
- **First import (no games):** `ImportRequiredRoute` redirects `/openings` and `/overview` to `/import` when `readiness.tier1=false`. Loading state shows a spinner, not content flash.
- **Endgames lock:** `EndgamesPage` renders `EndgamesProcessingState` in-place when `!readiness.tier2`. Nav link for Endgames stays ENABLED during incremental import — the in-page state carries the message. Nav link disabled only for first-import users (Tier 1 gate via `ImportRequiredRoute`, same as Openings/Overview).
- **Openings/Overview:** Stay accessible during incremental import. No route-level gate change.

### Nav link disabled state
- For users who have never imported: all non-import nav links disabled (existing behavior, now keyed off `readiness.tier1=false` instead of `profileHasCompletedImport()`).
- For users mid-incremental-import: Openings, Overview, Endgames nav links all ENABLED. Endgames shows the in-place processing state on tap.

### `EvalCoverageHeader` global bar
- Stays mounted in `ProtectedLayout` on ALL pages — Constraint 5.
- `useEvalCoverage` hook retained for the header; auto-reload effect removed. No other changes to the header.
- The bar disappears naturally when `isPending=false` (existing `if (!isPending) return null`).

### `EvalConfidenceTooltip` cleanup
- Remove `isPending` / `pendingCount` props from `EvalConfidenceTooltip` and `BulletConfidencePopover`.
- Remove the pending-caveat JSX block (`data-testid="eval-pending-caveat"`).
- Progress signal is now carried by `EvalCoverageHeader` (global) and `EvalCpuPlaceholder` (per-card) — not by tooltips.

### `data-testid` requirements (new elements)
| Element | `data-testid` |
|---------|---------------|
| `EndgamesProcessingState` wrapper | `endgames-processing-state` |
| `EvalCpuPlaceholder` wrapper | `eval-cpu-placeholder` |
| Import page "Explore Openings" CTA | `btn-explore-openings` |
| Tier-2 toast (sonner; no testid on toast itself — test via mock) | n/a |
| `useReadiness` hook result (component gate) | tested via page-level testids |
