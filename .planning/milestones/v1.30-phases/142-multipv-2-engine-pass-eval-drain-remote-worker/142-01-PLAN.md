<objective>
Add a dedicated MultiPV=2 analysis method to `EnginePool` (MPV-01). `_analyse_with_pv`
returns `InfoDict | None` and cannot be reused for the `list[InfoDict]` multi-line return,
so a new `_analyse_multipv2()` method + an `evaluate_nodes_multipv2()` wrapper are required
(D-02). The wrapper extracts best (index 0) and second-best (index 1) eval cp/mate + the
second-best UCI, guarding single-legal-move positions where only one line is returned.

Purpose: This is the engine primitive every other Phase 142 plan consumes — the local
drain (Plan 02), the remote worker (Plan 03), and the validation tool (Plan 04) all depend
on it. It must land first and be unit-tested in isolation.

Output: `_analyse_multipv2`, `EnginePool.evaluate_nodes_multipv2`, the module-level
`evaluate_nodes_multipv2` wrapper, and a `TestEvaluateNodesMultipv2` test class.
</objective>
