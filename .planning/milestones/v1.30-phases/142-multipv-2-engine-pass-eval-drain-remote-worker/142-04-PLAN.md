<objective>
Build the committed, re-runnable margin-histogram validation tool that gates the merge
(MPV-03, D-07). It reads the dev `game_flaws` JSONB blobs populated by Plan 02/03, computes
the only-move win-prob margin at each solver node, plots the distribution, and reports the
fraction within +/-0.05 of `ONLY_MOVE_WIN_PROB_MARGIN`. A `--check-goals` mode gives an
exit-code gate (>10% within the band → fail → raise the node budget to 1.5-2M before merge,
D-06). It also includes a PV1-drift spot-check (the RESEARCH FLAG guard).

Purpose: This is the SC4 automated artifact. The committed tool must be repeatable because
Phases 144 (A/B) and 145 (rollout) re-tune the margin offline against the same stored evals
(NOT a one-off pasted into VERIFICATION.md). The >=200-position histogram read + budget
decision is a HUMAN merge gate.

Output: `scripts/validate_multipv_budget.py` + the `reports/multipv-validation/` directory.
</objective>

    Create `scripts/validate_multipv_budget.py` modeled on `scripts/tactic_tagger_report.py`.
    Bootstrap `sys.path.insert(0, parent.parent)`; argparse with `--db {dev,benchmark,prod}`
    (required), `--limit N` (default e.g. 1000), `--check-goals` (store_true). Set
    `_REPORT_DIR = Path(__file__).resolve().parents[1] / "reports" / "multipv-validation"` and
    a `_MARGIN_BAND = 0.05` named constant, `_MAX_FRACTION_IN_BAND = 0.10` (SC4: >10% fails),
    `_MIN_POSITIONS = 200`.

    Query dev flaw rows via EXPLICIT column projection
    `select(GameFlaw.id, GameFlaw.allowed_pv_lines, GameFlaw.missed_pv_lines).where(
    GameFlaw.allowed_pv_lines.isnot(None))` (never `select(GameFlaw)` — deferred columns,
    Pitfall 4) within a short read session, materialize into a list, then close the session before
    computing. For each blob, for each SOLVER node (even index in the line), compute
    `margin = eval_cp_to_expected_score(b, solver_color) - eval_cp_to_expected_score(s, solver_color)`
    using the white-perspective `b`/`s` (skip mate-scored nodes or handle via
    `eval_mate_to_expected_score`; document the choice). Note: solver_color is recoverable from
    flaw ply parity + line orientation — derive it the same way `_detect_tactic_for_flaw` does, or
    record it alongside the blob query (document the derivation in a comment).

    Produce: (a) an ASCII (or counts-table) histogram of margins binned across 0..1; (b) the count
    and fraction of solver nodes within `+/- _MARGIN_BAND` of `ONLY_MOVE_WIN_PROB_MARGIN`;
    (c) total positions/nodes analyzed. Write a timestamped markdown report to
    `_REPORT_DIR / f"validate-multipv-budget-{date}.md"` (mkdir parents). In `--check-goals` mode:
    if positions < `_MIN_POSITIONS` OR fraction-in-band > `_MAX_FRACTION_IN_BAND`, print a clear
    FAIL line and `sys.exit(1)`; else print PASS and `sys.exit(0)`. No magic numbers (all
    thresholds named). No DB session held during report writing.
  </action>
