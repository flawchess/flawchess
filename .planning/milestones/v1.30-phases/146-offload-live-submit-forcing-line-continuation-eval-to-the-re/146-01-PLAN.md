<objective>
Remove all server-side Stockfish from the live `POST /eval/remote/submit` path and prioritize fresh games for the tier-4 flaw-blob drain.

Implements:
- **D-03** — `_apply_submit` unconditionally takes the empty-`blob_map` path: applies evals, classifies raw flaws, fills flaw PVs, stamps both completion markers, leaves blobs NULL. The live `/submit` payload drops the now-dead per-ply second-best fields (`second_cp`/`second_mate`/`second_uci`) from `SubmitEval`.
- **D-01** — `_claim_tier4_blob` swaps `ORDER BY random()` across the whole backlog for a recency-windowed CTE (random pick among the top-N most-recently-analyzed games) so a just-analyzed live game is gated promptly instead of competing uniformly with the old corpus.

Purpose: Eliminate the `asyncio.gather`-over-continuation-nodes server walk that surfaces as worker `ReadTimeout`s (Sentry FLAWCHESS-7Y) and contends with tier-3 drain. The deferred game self-heals through the existing tier-4 lease/submit + D-07 gated retag path (drained by the worker upgraded in Plan 02).

Output: Live submit runs zero Stockfish; both completion markers stamped with blobs NULL; tier-4 recency-ordered.

## Artifacts this phase produces (Plan 01)
- Modified `_apply_submit` (`app/routers/eval_remote.py`) — `blob_map: dict[int, tuple[list[PvNode], list[PvNode]]] = {}` unconditional; second-best build, `_build_flaw_multipv2_blobs` call, and `_run_multipv2_pass` call removed; their imports removed.
- Narrowed `SubmitEval` (`app/schemas/eval_remote.py`) — three `second_*` fields removed.
- New constant `TIER4_RECENCY_WINDOW: int = 50` and rewritten `_claim_tier4_blob` recency CTE (`app/services/eval_queue_service.py`).
- No new files, no DB schema, no migration.
</objective>
