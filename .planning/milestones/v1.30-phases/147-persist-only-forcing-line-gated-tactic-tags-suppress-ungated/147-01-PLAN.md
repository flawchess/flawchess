<objective>
Part A go-forward (D-01, D-03): thread an explicit `blobs_pending: bool` signal from the remote-submit
call site through the pure classify pipeline so that, when the forcing-line gate cannot yet run
because the continuation blob is deferred, the tactic motif is written as NULL instead of raw.
Mate-adjacent flaws (`pre_flaw_eval_cp IS NULL`) and D-06 `[]`-sentinel flaws are FINAL cases and
KEEP their raw tags. The value self-heals when the tier-4 D-07 gated retag lands.

Purpose: closes the ungated-tag window at the go-forward write site (`_apply_submit`), and provides
the graceful-degradation net Part B relies on under version skew.
Output: the threaded `blobs_pending` parameter + suppression branch + unit and router tests.

</objective>
