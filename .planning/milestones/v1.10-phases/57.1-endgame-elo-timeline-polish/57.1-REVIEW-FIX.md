### WR-01: Asof cutoff bleeds into next ISO week by up to 24 hours

**Files modified:** `app/services/endgame_service.py`, `tests/test_endgame_service.py`
**Commit:** a3521a2
**Applied fix:**
- Added `time` to the `datetime` imports.
- Replaced the `monday_dt = played_at - timedelta(days=iso_weekday - 1)` computation with a pinned-to-midnight version: derive the week's Monday date first, then build `monday_dt = datetime.combine(monday, time.min, tzinfo=played_at.tzinfo)`. `next_monday_dt = monday_dt + timedelta(days=7)` now lands on midnight of the next ISO Monday, so the `bisect_right` cutoff no longer drifts into the following ISO week by up to 24 hours.
- Preserved `played_at.tzinfo` so tz-aware and tz-naive datetimes continue to compare on the same axis (production stores `DateTime(timezone=True)`; tests use naive datetimes).
- Added a block comment marking the fix with the WR-01 reference for future readers.
- Added a focused regression test `test_asof_cutoff_pinned_to_midnight_no_week_bleed` in `TestEndgameEloTimeline`: seeds 10 Sunday endgame games at 14:00 (rating 1500) plus a Monday-morning 08:00 all-game at rating 1600, and asserts the week-2 emission's `actual_elo == 1500` (not 1600). The test is scoped strictly to WR-01's invariant so it does not double-assert WR-02 behavior.

### WR-02: Per-week count can be reported as 0 when an "all" event triggers emission for a future ISO week

**Files modified:** `app/services/endgame_service.py`, `tests/test_endgame_service.py`
**Commit:** 8ed9217
**Applied fix:**
- Added an early `if side != "endgame": continue` at the top of the merged-event loop. Emission now runs only for endgame events, matching the docstring ("only emits on endgame events") and preventing "all" events from producing ghost emissions in weeks with no endgame games.
- Preserved the pre-existing code structure (kept the merged event stream and asof arrays untouched) so the fix is minimal and surgical.
- Added a block comment marking the fix with the WR-02 reference.
- Added a focused regression test `test_all_events_do_not_trigger_emission_in_fresh_week` in `TestEndgameEloTimeline`: seeds 10 endgame games in week 1 plus 3 all-games in week 2 (no endgame games week 2), and asserts `"2026-01-12"` is NOT in the emitted dates while `"2026-01-05"` is (with `per_week_endgame_games == 10`).
