## Anti-Patterns to Avoid (carried over from 57-PATTERNS.md + 57.1-specific)

### All backend files
- **Never use `asyncio.gather` on the same `AsyncSession`** (CLAUDE.md). The orchestrator already executes sequentially — don't parallelise the asof lookup either (it's pure-Python anyway).
- **Never inline filter logic** — route everything through `apply_game_filters`. The repo query is already clean; don't touch it.
- **Never embed variables in error messages** — Sentry grouping. Phase 57.1 is pure math + bisect so exceptions are unlikely, but if an `except` is added use `sentry_sdk.set_context(...)` + `sentry_sdk.capture_exception(...)`.

### `_endgame_elo_from_skill` signature change
- **Don't leave `avg_opp_rating` as a second param "for symmetry"** — the anchor IS the single source of truth. Rename the param to `actual_elo_at_date: float` and drop the alias. Every caller is in-file, so the change is local.
- **Don't widen clamp bounds** (D-04 explicit: still `[0.05, 0.95]`).
- **Preserve the pure-function contract** — no I/O, no session access. The asof lookup happens in the orchestrator BEFORE `_compute_endgame_elo_weekly_series` is invoked.

### Asof-join (net-new territory)
- **Don't do per-event binary search inside the main event loop** — it would be O(W × log N) for each week emission. Instead: pre-compute `per_week_rating: dict[str, int]` once per combo (W binary searches total over the pre-sorted `all_rows`), pass it as a param, O(1) lookup in the hot loop.
- **Don't forward-fill with `None`** — D-02 says the ≥10-endgame-game floor prevents unreachable first-week case. But add a defensive `if played_ats: idx = bisect_right(...) - 1` and skip emission if `idx < 0`. Asserting unreachable is safer than crashing on the edge case.
- **Day-granularity tiebreak for same-day multiple games** (D-03) — the repo already `ORDER BY played_at ASC`, so after `bisect_right(played_ats, week_end_datetime)` we pick the *last* row with played_at ≤ cutoff, which is "the last game played on or before" by construction. Use full `datetime` not `date` for the cutoff key.

### `per_week_endgame_games` (net-new, trivial)
- **Don't double-count** — increment only when `side == "endgame"`, not on "all" events. Use a `collections.Counter` keyed on `(iso_year, iso_week)`; the same key used for `data_by_week`.
- **Don't confuse with `endgame_games_in_window`** — the former is that week's bucket, the latter is the trailing 100-game count. Both appear on every emitted point; D-07 keeps the trailing one for the tooltip "past N games" copy.

### ComposedChart (net-new, first in codebase)
- **Don't forget `yAxisId` on existing `<Line>` elements** — Recharts quietly picks a "default" axis when only one axis exists, but once `yAxisId="bars"` is added the `<Line>` elements lose their default binding and render on the wrong axis. Explicit `yAxisId="lines"` on every `<Line>` is mandatory.
- **Don't let the hidden right axis reserve visual space** — use `hide` prop (not CSS `visibility: hidden`) so Recharts drops the axis from the layout calculation. The `domain={[0, barMax * 5]}` trick is what actually caps bar height, not the axis width.
- **Don't hard-code bar color in JSX** — use `theme.ts` constant (CLAUDE.md §Frontend rule).
- **Recharts 5.x breaking changes**: ComposedChart is stable; no migration concerns. Direct import from `'recharts'`.

## Watch-Out Notes (net-new territory)

### 1. ComposedChart bar height envelope is counterintuitive
The "bars occupy 20% of chart height" trick works by giving the **bar** y-axis a domain 5× the max bar value, so a 40-game bar on a `[0, 200]` axis paints 20% tall. The **line** y-axis is independent (whatever `niceEloAxis` returns). If the user toggles legend combos and `barMax` changes, the bar axis domain recomputes via `useMemo` on `hiddenKeys` — same dep array the line y-axis already uses, so no new hook.

### 2. Asof-join correctness is per-combo, per-week
The asof-join must be scoped to `(platform, time_control_bucket)` — a bullet game's rating does NOT forward-fill the classical line. The orchestrator already partitions by combo before invoking the weekly helper (lines 1776-1786), so the pre-computed `per_week_rating` map is naturally per-combo. Test D-18d (cross-platform isolation) catches regressions here.

### 3. Locked-copy revision precedent — set it right
This is the first time a Phase has superseded a LOCKED UI-SPEC string in the FlawChess GSD history. The trailer-comment pattern proposed above (`<!-- Revised in Phase X.Y ... -->`) sets the precedent for future 57.2 / 58.x / etc. revisions. Keep it boring and grep-friendly: `grep -r "Revised in Phase" .planning/phases/` should yield a clean audit trail.
