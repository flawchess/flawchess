## Accomplishments

- **Anchor change (D-01/D-04):** `_endgame_elo_from_skill` now takes `actual_elo_at_date: float` instead of `avg_opp_rating`; formula simplifies to `round(actual_elo_at_date + 400 * log10(s / (1 - s)))` where `s` is skill clamped to `[0.05, 0.95]`. When skill = 0.5 both lines coincide (log10(1) = 0, zero-delta).
- **Asof-join orchestration (D-02/D-03):** `get_endgame_elo_timeline` pre-computes per-combo `(dates, ratings)` parallel arrays from `all_by_combo` before the main combo loop. The `_compute_endgame_elo_weekly_series` helper uses `bisect.bisect_right(actual_elo_dates, next_monday_dt)` to find the latest game on or before each emitted week's ISO Sunday end (Pitfall 2 cutoff pattern: cutoff = start of next Monday = `monday_dt + timedelta(days=7)`). Forward-fill is implicit — if a week has no games, the bisect returns the prior week's index.
- **Per-week count (D-06):** `per_week_count: dict[tuple[int, int], int]` keyed on `(iso_year, iso_week)` increments on every endgame event. Each emitted point gets `per_week_endgame_games = per_week_count.get((iso_year, iso_week), 0)`, distinct from the trailing-window `endgame_games_in_window`.
- **Schema add:** `EndgameEloTimelinePoint` gains `per_week_endgame_games: int`. Docstring rewritten to drop "performance rating" framing; uses "skill-adjusted rating" instead.
- **Tests:** 6 new unit tests in `TestEndgameEloTimeline` covering asof forward-fill, same-day routing (Sunday + 3 hourly games → latest wins), neutral-skill coincidence (skill=0.5 ⇒ endgame_elo == actual_elo), per-combo isolation, per-week count ≠ trailing-window count, no-prior-game defensive skip. 1 renamed/rewritten test asserts the new asof-not-rolling-mean semantics. 1 new integration test asserts `per_week_endgame_games` presence + shape correctness on `/api/endgames/overview`. 216 tests pass in the focused modules; 835 pass in the full backend suite.
- **Doc supersedes:** `57-UI-SPEC.md` Chart sub-description row + the InfoPopover JSX block (paragraphs 1-3) updated inline per D-12/D-13/D-14/D-16. `57-HUMAN-UAT.md` items 1, 3, 4 superseded; new item 9 added for volume-bars verification (pending Plan 02 frontend); total bumped to 9. Both files get a `<!-- Revised in Phase 57.1 (2026-04-18) ... -->` trailer comment — sets the precedent for future X.Y locked-copy revisions.

- `per_week_endgame_games: int` — count of that specific ISO week's qualifying endgame games (NOT trailing-window). Drives the volume-bar series (D-06/D-09).

**Semantic shift (same wire type):**
- `actual_elo: int` — was the mean user-rating over the trailing all-games window; is now the per-combo asof rating at the point's date (forward-filled from the latest prior game). Frontend label copy changes accordingly; no TypeScript type change needed.
- `endgame_elo: int` — was a classical performance rating anchored on avg_opp; is now a skill-adjusted rating anchored on `actual_elo`. Both lines share the anchor; the gap IS the skill signal. Frontend label copy changes accordingly; no TypeScript type change needed.

- `date`, `endgame_games_in_window` shapes and semantics unchanged.
- `EndgameEloTimelineCombo`, `EndgameEloTimelineResponse` wrappers unchanged.

- Empty-state / error-state / filter responsiveness / ISO-week Monday dates all unchanged.
