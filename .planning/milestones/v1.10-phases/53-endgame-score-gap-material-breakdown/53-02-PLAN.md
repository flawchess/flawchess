Component structure:

1. **Section header**: `<h3>` with text "Endgame Score Gap & Material Breakdown" and an `<InfoPopover>` (ariaLabel="Score Gap info", testId="score-gap-section-info") explaining: "Compares your endgame score (wins + half draws) with your non-endgame score. The material table shows how your performance varies based on whether you entered endgames with a material advantage, equal material, or a deficit."

2. **Score Difference display**: A `<div>` showing:
   - Left: label "Endgame Score Difference"
   - Right: the signed score difference value with 3 decimal places
   - Color: if `data.score_difference >= 0` use `text-green-500`, else `text-red-500`
   - Format: show sign explicitly: `+0.045` or `-0.150`
   - Below: smaller text showing "Endgame: {data.endgame_score.toFixed(3)} | Non-endgame: {data.non_endgame_score.toFixed(3)}"
   - data-testid="score-gap-difference"

3. **Verdict badge mapping** (local constant, NOT exported):
```typescript
const VERDICT_CONFIG: Record<Verdict, { label: string; color: string }> = {
  good: { label: 'Good', color: WDL_WIN },
  ok: { label: 'OK', color: GAUGE_WARNING },
  bad: { label: 'Bad', color: WDL_LOSS },
};
```

4. **Material Table**: Responsive HTML `<table>` with Tailwind classes:
   - Header row: Material at entry | Games | Win | Draw | Loss | Score | Verdict
   - 3 data rows from `data.material_rows` (always 3: ahead, equal, behind)
   - Each row: `data-testid="material-row-{bucket}"` (e.g., `material-row-ahead`)
   - Games column: integer with locale formatting (e.g., `row.games.toLocaleString()`)
   - Win/Draw/Loss columns: formatted as `{pct.toFixed(1)}%`
   - Score column: `row.score.toFixed(3)`
   - Verdict column: colored badge `<span>` with rounded corners, background color from `VERDICT_CONFIG[row.verdict].color` and white text. Use inline `style={{ backgroundColor: config.color }}` since oklch values cannot be Tailwind classes.
   - Dim rows (reduce opacity) where `row.games === 0` using `opacity-50` class on the `<tr>`
   - data-testid="material-table" on the `<table>`

5. **Mobile responsive**: Use `overflow-x-auto` wrapper around the table. The table itself uses `text-sm` on mobile, `text-base` on `sm:` and up. Use `min-w-[480px]` on the table to prevent column squishing — horizontal scroll on very small screens.

6. **All interactive elements get data-testid**: container div (`score-gap-section`), difference display (`score-gap-difference`), table (`material-table`), each row (`material-row-ahead`, `material-row-equal`, `material-row-behind`).

7. Export the component as a named export: `export function EndgameScoreGapSection`.
