## Milestone Summary

**Key Decisions:**
- DB wipe for v1.1: No migration needed — reimport after schema change
- move_san ply semantics: move_san on ply N = move played FROM position at ply N
- DISTINCT + GROUP BY for next-moves: COUNT(DISTINCT g.id) for transposition-safe counting
- Filter state lifted to OpeningsPage: survives tab switches without reset
- result_fen stored at import time: board.board_fen() captured after final PGN push
- QueryClient extracted to lib/queryClient.ts: shared singleton for 401 interceptor + login cache clearing
- Google SSO last_login via direct sa_update: on_after_login not called for OAuth flow in FastAPI-Users

**Issues Resolved:**
- React hooks order error from early return before hooks
- Data leakage between users on same browser (cache not cleared on logout)
- Google SSO double-click-to-login
- 180s time control misclassified as bullet (now blitz)
- Username-scoped sync boundary preventing second-username imports

**Issues Deferred:**
- Move sorting by frequency/win rate/alphabetical (MEXP-08)
- Position thumbnail on move hover (MEXP-09)

**Technical Debt Incurred:**
- clearArrowsOnPositionChange: false is a workaround for react-chessboard v5 clearing arrows on position change
- BoardArrow type defined locally instead of importing from react-chessboard (fragile import path)

**Quick Tasks (19 total):**
UI polish, arrow coloring, tooltip improvements, chart enhancements, and various fixes completed as quick tasks alongside phase work.
