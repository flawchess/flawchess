<objective>
Surface Phase 178's canonical per-color accuracy onto the one shared card payload that
both the Library list and the analysis board consume, so the frontend Move Stats
component (Plans 02-03) has an accuracy value to render.

Purpose: The two-sided Move Stats accuracy strip needs `white_accuracy`/`black_accuracy`
on the wire. RESEARCH confirmed `/analysis?game_id=X` and the Library list are served by
the identical `GameFlawCard` Pydantic model built by the identical `_build_card()` — one
backend edit serves both surfaces. The 14 per-(category × side) counts are already
derivable from existing `flaw_markers` + `eval_series` fields (D-05), so this is the
entire backend delta: two nullable floats.

Output: `GameFlawCard` (backend + frontend type) gains `white_accuracy`/`black_accuracy`.
