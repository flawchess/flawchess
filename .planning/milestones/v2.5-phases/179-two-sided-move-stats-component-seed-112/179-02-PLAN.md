<objective>
Build the shared, presentational two-sided Move Stats component — accuracy strip +
7-row (category × 2 side) count table — plus the pure client-side count-derivation and
mover-color helper it needs. This plan produces a self-contained, unit-tested component;
wiring it into the Library card and analysis panel (cycling, filter ring, mobile collapse)
is Plan 03.

Purpose: One shared component replaces two near-identical badge-row implementations
(RESEARCH: `LibraryGameCard.tsx:915-956` and `AnalysisTagsPanel.tsx:264-297` independently
derive the same data). Both surfaces receive an identical `GameFlawCard`, so a single
component consumes it. The 14 counts are already on the wire (D-05) — this plan derives
them client-side and renders them in the stable chess.com-style 7-row layout.

Output: `MoveStats.tsx` (presentation + cell click/hover dispatch via props), the
`moveStatsCounts.ts` derivation module, `moverColorAtPly` helper, and the two new
Best/Good circular icons — all unit-tested.
