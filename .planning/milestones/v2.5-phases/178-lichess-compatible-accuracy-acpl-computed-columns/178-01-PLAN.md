<objective>
Repurpose the canonical game accuracy/ACPL columns and preserve the platform-provided values.

This plan ships the Alembic migration (D-03): add four new `*_imported` columns, copy
the current platform-provided `white_accuracy` / `black_accuracy` / `white_acpl` /
`black_acpl` values into them, then NULL the canonical columns so the phase's compute
path (Plans 03/04) can refill them with our uniform lichess-formula values. It also adds
the four `*_imported` `mapped_column`s to the SQLAlchemy model and corrects the provenance
comments. The oracle flaw-count columns (the `is_analyzed` sentinel) are left completely
untouched (D-04).

Purpose: Make the canonical accuracy/acpl column names the single uniform cross-platform
metric from now on, with no future "swap the app to `_computed`" migration ever needed,
while keeping the original platform numbers as a `_imported` comparison/validation signal.

Output: One migration revision, four new model columns, and a migration upgrade/downgrade test.
</objective>
