## Overview

LLM-first insights MVP for the Endgame tab. Transforms the existing `/api/endgames/overview` composite into a deterministic findings pipeline (zones, trends, sample quality, cross-section flags), feeds it to a `pydantic-ai` Agent backed by `POST /api/insights/endgame`, renders the overview + 4 Section insights inline on the frontend, and ships a generic `llm_logs` Postgres table designed to host every future LLM feature. Insights were ultimately enabled for all users rather than gated behind a beta cohort (Phase 67 descoped — see Milestone Summary).

Phase 68 was added mid-milestone after UAT feedback: the single-line "Score Gap over Time" chart was replaced with a dual-line "Endgame vs Non-Endgame Score over Time" chart with a colored shaded gap, and the prompt's `score_gap` framing rule was simplified since the chart now makes gap composition self-evident.

**Success Criteria (all met):**
1. Endpoint with valid filter context returns schema-validated `EndgameInsightsReport` whose `overview` is always a non-null paragraph and `sections` contains up to 4 Section insights
2. Equivalent filter states produce cache hits on the second call (no duplicate `llm_logs` row, no new LLM call) — cache invalidates when `prompt_version` or `PYDANTIC_AI_MODEL_INSIGHTS` changes

## Milestone Summary

**Decimal Phases:** None in this milestone.

**Key Decisions:**
- Phase 0 prompt-fluency spike DROPPED — risk absorbed by regression test + admin-impersonation validation loop (later also dropped).
- Zone source of truth is the in-code gauge constants (NOT the 2026-04-18 benchmark report) — no gauge-update phase needed.
- Beta flag is a boolean column on `users` flipped via direct DB operation — no settings UI, no admin page.
- Overview paragraph ALWAYS populated (no null overview in MVP).
- LLM output is NOT style-constrained — no em-dash/noun-label/prescriptive-advice policing; correctness guardrails come from three precomputed cross-section flags.
- Log table is generic `llm_logs` (not `insights_llm_logs`), designed to host future LLM features.
- Parent-lifted mutation state pattern (not Context) — Endgames.tsx holds one `useEndgameInsights` mutation; EndgameInsightsBlock + 4 SectionInsightSlot instances observe the same state.
- Prompt version is the cache-invalidation handle — bumping `_PROMPT_VERSION` forces fresh generation without explicit flush. Iterated v6 → v15 over the milestone.

**Issues Resolved:**
- Pre-merge v1.11 milestone review (Phases 63-68 cohesion pass) surfaced: a critical failing frontend test (fixed), a dead codegen pipeline (resolved by completing the Phase 66 switchover — 3 FE chart components now import from `frontend/src/generated/endgameZones.ts`), a stale `Filters:` header reference in the system prompt (bumped to `endgame_v15`), and a stale CHANGELOG description (updated to v14 three-metric shape).
- Phase 68 UAT surfaced a broken shaded-band render (root-caused to Recharts' `<g>`-boundary-insensitive `findAllByType`) and a y-axis too compressed to read (clamped 20-80%).

**Issues Deferred / Technical Debt:**
- **Phase 67 (Validation & Beta Rollout) descoped.** VAL-01 (ground-truth regression test against SEED-001 canonical user fixture) and VAL-02 (admin-impersonation eyeball validation across 5 real user profiles) were not executed. Insights were enabled for all users via commit `c91478e` rather than gated behind a beta cohort. Consequence: no automated regression guard against prompt changes that break correctness behaviors ("no 'average' Score Gap", "mentions strong Endgame Skill", etc.). Recommended follow-up in v1.12: retrofit the snapshot test against one real production user fixture.
- LOG-02 and LOG-04 (cache-miss write coverage + Sentry `set_context` pattern on LLM failures) remain formally unchecked in the traceability table but are behaviorally implemented in Phase 65 — the requirement markers just weren't updated when the tests landed.
- Open artifact audit flagged 137 legacy residue items (missing quick-task SUMMARY files, 8 debug sessions, 3 dormant seeds) — inherited from pre-v1.10 and not blockers.

**Technical Debt Incurred:**
- Phase 68 `_compute_score_gap_timeline` private helper kept its old name after subsection rename (grep-noise, not a bug).
- `_finding_time_pressure_vs_performance` emits a finding that `_assemble_user_prompt` filters out — kept only for its effect on `findings_hash` (cache key). Mild code smell; either drop entirely or promote the chart payload to own cache-invalidation.
- Pre-existing REAL→Float ORM/DB column drift on `game_positions.clock_seconds`, `games.white_accuracy`, `games.black_accuracy` surfaces on every Alembic autogenerate and was scoped out of this milestone's migrations. Deserves a dedicated cleanup migration.
