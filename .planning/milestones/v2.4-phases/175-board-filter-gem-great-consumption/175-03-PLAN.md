Introduce the "great" tier to the frontend for the first time (the board has only
ever rendered gem) and mirror the backend `EvalPoint` in TypeScript. Build the great
primitives (theme constant, glyph record, icon, live-fallback classifier, MoveQuality

stored `best_move_tier: 'great'` (delivered by Plan 02) can render everywhere gem does.

this plan supplies the reusable great components without yet wiring the Analysis
mainline (that is Plan 04).

MoveQuality bucket, the TS EvalPoint mirror, and great branches on board/badge/
popover/variation-tree.
