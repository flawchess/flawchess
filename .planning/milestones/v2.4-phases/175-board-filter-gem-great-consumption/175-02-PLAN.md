<objective>
Deliver BOARD-01's backend read path: extend `EvalPoint` with a pre-classified
`best_move_tier` and `maia_prob`, batch-load the stored `game_best_moves` rows per
page, and classify them with the authoritative `classify_best_move` during eval-series
assembly. The board then renders the tier string directly — no client classification
and no live engine call for an analyzed game's mainline.

Purpose: make gem/great markers appear instantly from stored data (BOARD-01), and
make row-absence a deterministic "not gem/great" (the structural basis for BOARD-02's
no-sweep-delay guarantee).
Output: `fetch_page_best_moves`, extended `EvalPoint`, and the classified assembly in
`_build_eval_series`.
</objective>
