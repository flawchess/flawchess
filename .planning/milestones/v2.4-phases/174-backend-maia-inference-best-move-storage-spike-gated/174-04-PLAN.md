<objective>
Build the backend Maia inference lifecycle and all the pure, unit-testable candidate logic: the eager-loaded
ONNX session (mirroring Stockfish's start/stop, no-op when onnxruntime absent), `score_move` inference, the
mover's pinned+clamped lichess-blitz ELO derivation (GEMS-05), the write-time inaccuracy candidate gate
(GEMS-02), and the query-time Gem/Great classification constants (GEMS-07). Plan 05 wires these into
eval-apply; this plan delivers and unit-tests them in isolation.

Purpose: Separating the pure logic + lifecycle from the heavy eval_apply integration keeps each testable at
~40-50% context. Every helper here reuses an existing single-source implementation (sigmoid, ELO
normalization, clamp) so backend `maia_prob`/classification can never drift from the live board.

Output: `app/services/maia_engine.py` (start_maia/stop_maia/score_move), `app/services/best_move_candidates.py`
(pinned_elo_for_mover, passes_inaccuracy_gate, classify_best_move + constants), the wired `app/main.py`
lifespan, and two unit-test files.
</objective>
