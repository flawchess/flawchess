<objective>
Wire the isolated `maia-inference` dependency group (created in Plan 01) into the Docker images so the
BACKEND image installs onnxruntime + numpy while the WORKER image stays lean, and prove the isolation
with an automated test. This is GEMS-06: installing the worker image's dependency set must not pull the
inference deps.

Purpose: The remote worker fleet runs `scripts/remote_eval_worker.py` (pure Stockfish) and is updated
manually by operators; bloating it with a ~110-140 MB inference stack it never uses is wasteful and
couples the fleet to an unused dependency. The group is opt-in, so isolation holds unless the backend
Dockerfile explicitly requests it.

Output: a one-line change to the backend `Dockerfile` (`--group maia-inference`) and
`tests/test_dependency_isolation.py` asserting the worker dep-set excludes onnxruntime/numpy.
</objective>
