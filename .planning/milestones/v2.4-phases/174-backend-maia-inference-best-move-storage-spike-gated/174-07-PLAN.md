<objective>
Backfill the ~43k existing lichess-eval games (and any newly-imported complete-%eval lichess games
that never reach the hole-fill drain) so their `game_best_moves` coverage becomes complete —
opportunistically, via the tier-4 lottery pattern, with no deterministic sweep and no completion
deadline (SEED-109 item 2; the lichess-scoped precursor to Phase 176's general backfill).

This plan depends on 174-06: once the unified pass gives a lichess-eval game full-ply best-move
coverage and stamps `full_pv_completed_at`, that stamp is exactly the signal this lottery keys OFF of.
So the predicate `lichess_evals_at IS NOT NULL AND full_pv_completed_at IS NULL AND is_guest=false`
selects games still lacking our best-move pass; draining them through 174-06's pass stamps the column
and removes them from the predicate — a self-terminating, contention-safe backfill.

Purpose: re-establish 174-VERIFICATION Truth 2 for the EXISTING corpus, not just go-forward imports.
Output: an ES-weighted backfill selection + supporting partial index + concurrency-safe routing into
the existing drain.
</objective>
