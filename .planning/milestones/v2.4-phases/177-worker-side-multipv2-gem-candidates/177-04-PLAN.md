<objective>
Upgrade the remote worker to protocol v2: bump the schema version and send it at lease time,
run the targeted MultiPV-2 re-search for fresh-lane gem candidates, and add the tier-4b rung-5
handler that drives the new /bestmove-lease + /bestmove-submit pair.

Purpose: This is the worker half of the shift. The fresh lane must compute runner-up evals only for
plies it already knows are candidates (its own MultiPV-1 best == the played move), keeping the full
pass MultiPV-1 (Phase 146 D-03 invariant). The tier-4b rung is a pure "evaluate these FENs at
MultiPV-2, return second_cp/second_mate" loop — no full pass, structurally simpler than the fresh
lane. Version must be sent on the LEASE call (Pitfall 4), not just the submit body, or the server's
lease gate cannot see it.

Output: WORKER_SCHEMA_VERSION=2, the targeted re-search, the rung-5 handler, and lease-time version.
</objective>
