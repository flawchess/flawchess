<objective>
Give the already-analyzed engine corpus (chess.com-dominant, ~94%) best-move rows
opportunistically, so historical games gradually become eligible for the Phase 175
gem/great markers and the FILT-01 has-gem/has-great filter. Adds a backend-only
tier-4b ES-weighted lottery rung that self-terminates per game via a new
best_moves_completed_at completion marker, gated by a dedicated kill-switch, with a
correctness guardrail that prevents a Maia-absent backend from permanently locking
games out of the lottery.

Purpose: Satisfy BACK-01 / SC1-SC3 — global+random opportunistic backfill, no
deterministic sweep, no operator script, no ETA promise. Coverage grows over time.

Output: One Alembic migration, a new completion-marker column + partial index, a
config gate, a Maia-availability guardrail at the single write choke-point, a new
tier-4b lottery rung, and the tests proving all of the above.
</objective>
