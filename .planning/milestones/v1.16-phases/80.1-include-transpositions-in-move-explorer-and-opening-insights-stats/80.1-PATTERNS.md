**Core pattern — existing `query_transposition_counts` to copy and extend** (lines 457-508):
```python
async def query_transposition_counts(
    session: AsyncSession,
    user_id: int,
    result_hash_list: list[int],
    time_control: Sequence[str] | None,
    platform: Sequence[str] | None,
    rated: bool | None,
    opponent_type: str,
    recency_cutoff: datetime.datetime | None,
    color: str | None,
    opponent_gap_min: int | None = None,
    opponent_gap_max: int | None = None,
) -> dict[int, int]:
    if not result_hash_list:
        return {}

    stmt = (
        select(
            GamePosition.full_hash.label("result_hash"),
            func.count(GamePosition.game_id.distinct()).label("transposition_count"),
        )
        .join(Game, Game.id == GamePosition.game_id)
        .where(
            GamePosition.user_id == user_id,
            GamePosition.full_hash.in_(result_hash_list),
        )
        .group_by(GamePosition.full_hash)
    )

    stmt = apply_game_filters(
        stmt,
        time_control, platform, rated, opponent_type, recency_cutoff, color,
        opponent_gap_min=opponent_gap_min,
        opponent_gap_max=opponent_gap_max,
    )

    rows = await session.execute(stmt)
    return {row.result_hash: row.transposition_count for row in rows}
```

**`query_opening_transitions` — add resulting-position WDL via separate batch query (Option B from RESEARCH.md)**

The function body (lines 511-715) is complex (205 lines, LAG window, phase-71 hotfix). Option B (separate query after the function returns) is the safe approach — it does not modify the complex CTE. The new sibling function follows the same shape as `query_transposition_wdl` above but keys on `resulting_full_hash` values from the transitions result set.

The new sibling (`query_resulting_position_wdl`) receives a list of `resulting_full_hash` values (distinct from the transitions output) and the same filter parameters as `query_opening_transitions`, and returns `dict[int, tuple[int, int, int]]` (same shape as `query_transposition_wdl`). It is called inside `compute_insights` after `query_opening_transitions` returns.

**Source:** `app/repositories/query_utils.py` (lines 12-77)
