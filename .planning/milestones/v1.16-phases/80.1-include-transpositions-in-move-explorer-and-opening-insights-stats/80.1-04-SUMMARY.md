## What Changed

### `scripts/prodcheck_80_1.py` (new)

Read-only standalone async SQLAlchemy script. Given a list of user_ids on the CLI, opens an `AsyncSession` against `PROD_TUNNEL_DATABASE_URL` (which must point at the `bin/prod_db_tunnel.sh` forward on `localhost:15432`) and calls `compute_insights` for each user with default filter parameters — no time-control / platform / rated narrowing — so the numbers match a default dashboard load. Prints, per user and per color, the count of `OpeningInsightFinding` strengths and weaknesses surfaced.

Pre-flight checks:
1. TCP probe on `localhost:15432` (1 s timeout). If closed, prints "tunnel not open" and exits 2.
2. Confirms `PROD_TUNNEL_DATABASE_URL` is set. If missing, prints a clear instruction and exits 2.

Read-only by construction: `compute_insights` performs no INSERT/UPDATE/DELETE; the engine is created with `echo=False`. Safe to run against prod.

`sys.path` shim added at script entry to allow `from app.services.opening_insights_service import compute_insights` to resolve when running from the repo root (matches the pattern in `scripts/import_benchmark_users.py`).

#### D-09 re-run after WR-01 fix (2026-05-07)

After the post-phase code review surfaced WR-01 (cross-color WDL blending in `compute_insights` — `query_resulting_position_wdl` was being called with `color=None`, mixing white-perspective and black-perspective games into the same denominator while `query_opening_transitions` IS color-filtered via an explicit `Game.user_color == color` predicate), the fix was applied as one commit (`bab66b40` — `fix(80.1-03): per-color resulting-position WDL in compute_insights (WR-01)`) plus a regression test commit (`77ed4427` — `test(80.1-03): update tests for per-color pos_wdl_by_color`). The fix changes denominator semantics on insights findings, so D-09 was re-run on the same three users against the same prod tunnel.

#### POST-FIX (WR-01 applied, branch tip):

```

user=    18  white:  1 strengths /  0 weaknesses    black:  0 strengths /  1 weaknesses    total:  2

```

#### Comparison (PRE-SWITCH baseline → POST-SWITCH (color=None bug) → POST-FIX (WR-01)):

| user_id | pre total | post (bug) total | post-fix total | pre→post-fix ratio | band check |
|---------|-----------|------------------|----------------|--------------------|------------|
| 4 | 40 | 40 | 40 | 100% | within 50–200% (caps saturate at every step) |
| 18 | 2 | 3 | 2 | 100% | within 50–200% |
| 35 | 16 | 14 | 16 | 100% | within 50–200% |

#### D-09 re-run verdict: PASS

- All three users land inside the 50–200% PASS band the planner set in 80.1-04-PLAN.md Task 3b — actually tighter than the pre-fix run (every total recovered exactly to its PRE-SWITCH value).
- No user with previously-non-zero findings shows zero post-fix total (no HARD STOP).
- Heavy-user saturation at 40/40 is the expected ceiling regardless of which side the WDL is computed on; both pre and post-fix saturate.
- Light-user user 18 reverts to 2 total findings (was 3 under the bug). The +1 ghost white strength under the bug came from the cross-color blend pulling games into the white-section denominator; with color filtering restored, it falls below the score gate. The black side flips from 1 strength → 1 weakness on the same total — a tiny user (n=261) where single-finding shifts are noise-level.
- Moderate-user user 35 recovers from 14 → 16 total findings: the bug had blended some white-section findings into the neutral band by pulling in black-perspective games (and vice versa); per-color filtering restores 2 findings. Section composition shifts more visibly than user 18 because user 35 has roughly equal play on both colors, so the cross-color blend was meaningfully diluting both sections under the bug. Total of 16 matches PRE-SWITCH exactly.

The recovery to PRE-SWITCH totals is consistent with what we'd expect from WR-01 in isolation: at the click-through Move Explorer's `color=request.color` semantics, transposition-inclusive WDL is in fact close to color-filtered move-played WDL on most positions because most transpositions occur within a color's repertoire (you transpose into your own French Defense from a different move order; you don't typically transpose into the opponent's openings). The bug had introduced a much noisier blended denominator that the WR-01 fix has now eliminated.

Tunnel kept open by orchestrator across the re-run; no `bin/prod_db_tunnel.sh stop` invoked from this re-run.
