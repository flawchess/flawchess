# Milestone v1.29: Live-Engine Analysis Page

**Status:** ✅ SHIPPED 2026-06-29
**Phases:** 136–140
**Total Plans:** 14

## Overview

Ship a standalone `/analysis` board where the user makes any legal move from any position and an in-browser single-thread Stockfish (WASM) evaluates it live (eval bar + top 1–2 lines + best-move arrow), subsuming the static Tactic Line Explorer (Phase 135) as a tactic mode of that one shared board. No backend schema or new endpoints (D-4 locked) — the Phase 135 `tactic-lines` endpoint already surfaces stored PVs; board/variation state lives in the URL. Phase 140 then loaded the *whole* game onto the board behind a single unified `Analyze` entry point, relocated the eval chart below the board, and turned inline missed/allowed tactic tags into navigable PV sidelines. Paste-a-FEN and a multi-thread desktop engine were deferred to v2.

Released to production via PR #227 (commit `e3f652ab`).

## Milestone Summary

**Key Decisions:**

- D-3: Multi-thread WASM (SharedArrayBuffer + site-wide COOP/COEP) explicitly deferred — it breaks the Google OAuth popup and is unreliable on iOS Safari, and an SPA can't toggle cross-origin isolation per-route. v1 ships single-thread lite-single (~7 MB NNUE).
- D-4: No backend schema or new endpoints — analysis state is ephemeral and lives in the URL; the Phase 135 `tactic-lines` endpoint already surfaces stored PVs.
- D-5: Tactic mode opens at the flaw position with the stored PV as the initial mainline; the live engine takes over the moment the user deviates from or walks past the stored line.
- `useAnalysisBoard` was built as a new branching-tree hook rather than a variant of `useChessGame` (which stayed unmodified).
- Phase 140 unified the game-card `Explore`/`Analyze position` pair and flaw-card `Explore`/`Game` modal pair into a single `Analyze` entry; the inline `Game` modal code path was deleted.

**Issues Resolved:**

- Retired the standalone Tactic Line Explorer (`TacticLineExplorer.tsx` + `useTacticLine.ts` + their 26-test suites) without regression — gated on 4 Phase 135 parity behaviors (depth-0 highlight, missed/allowed +1 offset, real-game-ply numbering, tactic-rail state) plus a green knip/tsc/lint/full-suite run.
- First `React.lazy` + Suspense route boundary in the app, so the ~7 MB engine bundle is fetched only on `/analysis`.
- Post-ship production hardening (quick tasks 260628 / 260629): both-color tactic tooltips, engine-line expansion + full-PV click-through, sideline blunder/mistake icon persistence, mobile board-app redesign, and two Sentry-driven fixes — a Stockfish WASM `unreachable` crash from sending `position`+`go` during the `stopping` state (FLAWCHESS-7V), and a duplicate-React-key crash from board arrows sharing a from→to during live streaming.

**Issues Deferred (v2):**

- BOARD-V2: paste-a-FEN / paste-PGN entry, full nested variation-tree display, promote/demote/delete variation.
- ENGINE-V2: multi-thread WASM as a desktop-only progressive enhancement on a hard-loaded cross-origin-isolated document (reopens the COOP/COEP + OAuth + iOS analysis).

**Verification at close:** Phases 136/138/140 VERIFICATION and Phase 138 UAT (4 scenarios) were human-signed-off at milestone close — the feature shipped to production and was exercised live (including two Sentry crash fixes against the running page).
