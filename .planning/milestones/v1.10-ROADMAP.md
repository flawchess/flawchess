## Overview

Advanced analytics milestone focused on the Endgames tab — new metrics, performance rewrites, self-calibrating baselines, a skill-adjusted Endgame ELO per (platform, time-control) combination, and test-suite hardening. Added admin user impersonation for superusers to debug any user's view from their own session. The bulk of the work is analytics surface that answers two questions for each user: "how much worse do I score in endgames compared to the rest of the game?" and "do I crack under time pressure more than my opponents?" — layered on top of a single consolidated `/api/endgames/overview` endpoint that pulls every endgame chart in one round trip.

## Milestone Summary

**Decimal Phases:**

- Phase 57.1: Endgame ELO Timeline — Anchor Change + Volume Bars (inserted after Phase 57 to fix rolling-mean anchor lag and add weekly volume bars per UAT feedback)

**Cancelled / Deferred Phases:**

- Phase 58: Opening Risk & Drawishness — moved to backlog as Phase 999.6 to keep v1.10 endgame-focused

**Key Decisions:**

- **Consolidated `/api/endgames/overview` endpoint** (Phase 52) — all endgame charts served from a single round trip; sequential queries on one AsyncSession; no frontend fan-out
- **2-query timeline rewrite** (Phase 52) — GROUP BY `(game_id, endgame_class)` with HAVING `count(ply) >= 6` replaces 8 sequential per-class queries
- **Deferred filter apply on desktop** (Phase 52) — matches existing mobile pattern; avoids query storm on multi-filter changes
- **Material buckets named Conversion/Parity/Recovery** (post-Phase 53 rename via quick tasks `260413-pwv` and `260415-q75`) — clearer semantic labels than ahead/equal/behind
- **4-ply persistence + 100cp threshold** (Phase 48) — reduces noise from transient captures; validated against Stockfish eval analysis
- **Opponent-based self-calibrating baseline** (Phase 60) — replaces global-average baseline with opponent's rate against the user, muted when sample < 10 games
- **Skill-adjusted Endgame ELO formula** (Phases 57 + 57.1) — `actual_elo + 400·log10(skill/(1-skill))`, anchored on user's actual rating via asof-join, not performance rating
- **Weekly volume bars on timeline charts** (Phase 57.1, and clock/score-diff timeline quick tasks) — visual weight indicator for each weekly point
- **Skill composite = avg(Conv Win %, Parity Score %, Recov Save %)** (Phase 57 + quick task `260418-nlh`) — simple average with 45-55 blue neutral band
- **Single auth_backend + ClaimAwareJWTStrategy wrapper** (Phase 62) — keeps every `Depends(current_active_user)` call site unchanged
- **Nested impersonation rejected via current_superuser dep** (Phase 62) — impersonation token resolves to non-superuser target
- **last_login freeze by construction** (Phase 62) — manual `strategy.write_impersonation_token` bypasses `UserManager.on_after_login`
- **Truncate flawchess_test at pytest session start** (Phase 61) — deterministic integer assertions via shared `seeded_user` fixture
- **Split time_control into base_time_seconds + increment_seconds** (quick task `260414-smt`) — time pressure % denominator is per-game base time, clamped at 2x

**Issues Resolved:**

- Endgame tab load times 150-500s on production (pg_stat_statements) — collapsed 8 queries into 2, deferred filter apply
- Conv+Even+Recov game counts not summing to Games-with-Endgame total — bucket accounting fix in Phase 59
- Global-average baseline fighting the data for weak players — replaced with self-calibrating opponent baseline
- Rolling-mean "Actual ELO" lag confusing users — switched to asof-join anchor on real per-combo rating
- Users couldn't tell if a weekly ELO point was well-supported or marginal — added muted volume bars
- pytest accumulating test data across runs — TRUNCATE on session start
- Flaky aggregation tests (WDL perspective when user plays black, material tally direction) — added sanity test module
