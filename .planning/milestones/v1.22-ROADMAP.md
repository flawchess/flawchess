# v1.22 Maintenance — Test Isolation & Frontend Major Upgrades — Roadmap (ARCHIVED)

**Shipped:** 2026-05-31
**Phases:** 100, 101
**Milestone goal:** Clear two accrued maintenance debts before the next feature milestone: (1) make the test suite safe under concurrent runs and `pytest -n auto`, and (2) bring the frontend onto its latest major dependency versions. Backend had no outstanding major bumps. Sequential, bisectable, each phase gated.

## Summary

Two sequential, requirement-less maintenance phases:

- **Phase 100** gave each `pytest` run (and each `pytest-xdist` worker) its own database, cloned from a migrated template via `CREATE DATABASE … TEMPLATE`, and retired the hostile session-start `TRUNCATE … RESTART IDENTITY CASCADE` whole-schema `ACCESS EXCLUSIVE` lock (a fresh clone is already clean). The template auto-refreshes on Alembic head drift under a `pg_advisory_lock`, killed runs self-heal (drop-if-exists on next create), and per-run DBs drop at teardown. Result: two full suites run simultaneously with zero deadlocks / zero cross-run corruption, and `pytest -n auto` is green at **18.56s vs 40.29s serial (2.2x)**. CI stays serial (D-02).
- **Phase 101** brought 11 majors-behind frontend deps to their latest major across **six atomically-committed clusters** in low→high risk order (lucide-react 1 → Vite 8 + plugin-react 6 → jsdom 29 → eslint 10 stack → TypeScript 6 → recharts 3), each gated by the full local backend+frontend gate so a failure bisects to exactly one cluster. recharts 2 → 3 earned a desktop + mobile visual UAT, which caught one zone-band regression (fixed and locked with a regression test). The typescript-eslint ↔ TS6/eslint-10 peer-compat question resolved cleanly (no overrides; the documented escape hatch was not needed). shadcn straggler advanced 4.8.3 → 4.9.0; `@types/node` held at 24.x (CI tracks Node 24).

Phase clusters landed directly on `main` (gated), per the D-06 design, rather than via a single phase branch. Alongside the two phases, the milestone window also folded in small direct-to-`main` maintenance: backend dependency refresh (`uv lock --upgrade`, pinned patched transitives clearing Dependabot alerts), pydantic-ai-slim 1.85 → 1.104 with deprecation cleanup, and Gemini 3 thinking-level support (default low) in the insights path.
