## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (`text-base`) | 400 (regular) | 1.5 |
| Label / chip / secondary | 14px (`text-sm`) | 500 (medium) | 1.4 |
| Metric delta number | 14px (`text-sm`) | 700 (bold), tabular-nums | 1.4 |
| Card heading / family name | 16px (`text-base`) | 600 (semibold) | 1.2 |

Minimum font size: `text-sm` (14px). The motif chip itself may use `text-xs` — this is the existing TagChip pattern per CLAUDE.md explicit exception (hover/tap-activated info tooltips and family-colored chips that read as visual asides). No other surface goes below `text-sm`.

## Component Inventory

### TACUI-01: Motif Chip (`TacticMotifChip`)

Model directly on `TagChip`. One chip per flaw card showing the `tactic_motif` when:
- `beta_enabled === true` (frontend gate, D-01)
- `tactic_confidence >= MIN_TACTIC_CHIP_CONFIDENCE` (named constant, Claude's discretion — align with Phase 124 threshold, suggest `0.7`)

Chip anatomy: `[FamilyIcon 3×3] [motif-name text]` — `text-xs`, family-colored border + bg, `rounded-full`, `px-2 py-0.5`.

Popover (hover desktop / tap mobile): `"<bold motif-name>: <one-sentence definition>"` — opens `side="top"` desktop, `side="bottom"` mobile (same pattern as TagChip). Popover text `text-xs`, `bg-foreground text-background`.

Chip renders on:
- Library flaw list rows (below the severity badges row)
- Single-game flaw card (same location as existing TagChip chips)

`data-testid`: `chip-tactic-{motif}-{flawId}` (consistent with `chip-{tag}-{gameId}` pattern).

`aria-label`: `"Tactic: {motif} — {definition}"`.

### TACUI-02: Tactic Motif Comparison Grid (`TacticComparisonGrid`)

Model on `FlawComparisonGrid`. Gated behind `beta_enabled`.

**Layout:** Single column on mobile, 2–3 columns on desktop (`grid-cols-1 lg:grid-cols-3`). Up to 6 family cards, ranked by largest significant gap (Wilson-gated), volume fallback.

**Per-row anatomy** (mirrors `FlawBulletRow`):
```
[FamilyIcon 3.5×3.5]  [Family label text-sm font-medium]  …  [±delta text-sm font-bold tabular-nums zone-colored?]  [InfoPopover trigger]
[MiniBulletChart — value=delta, center=0, invertColors=true, barColor="neutral"]
```

**States (in priority order):**
1. `isLoading` → pulse skeleton, `data-testid="tactic-comparison-loading"`, `aria-label="Loading tactic comparison"`
2. `isError` → `<LoadError resource="tactic comparison" />` (CLAUDE.md mandate)
3. `!data` → `null`
4. `data.below_gate` → gate CTA (see copywriting below), `data-testid="tactic-comparison-gate-cta"`
5. normal → grid, `data-testid="tactic-comparison-grid"`

**Zero-data rows:** delta === null → muted italic "No events in current filter" (same as FlawComparisonGrid zero-event placeholder).

**Benchmark zone:** render `neutralMin`/`neutralMax` on `MiniBulletChart` only when the API returns non-null zone bounds. When null, pass `neutralMin=0 neutralMax=0` (collapses the zone band; the bullet still shows the delta bar and CI bars). No "no benchmark" label needed — graceful degradation.

`data-testid` per family card: `tactic-family-card-{familyKey}` (e.g. `tactic-family-card-fork`).

### Motif Filter in FilterPanel (`TacticMotifFilter`)

Beta-gated (`beta_enabled`). Renders inside the existing FilterPanel / drawer, after all current filter sections. Multi-select by family — same toggle-group pattern used for Time Control.

Label: "Tactic motif" (sentence case, consistent with other filter section labels).

Each family rendered as a `ToggleGroupItem`: family-colored chip style matching the motif chip palette, family name as label.

`data-testid`: `filter-tactic-motif-{familyKey}` (e.g. `filter-tactic-motif-fork`).

Applies to both the desktop FilterPanel drawer and the mobile FilterPanel drawer (CLAUDE.md mobile-parity rule).

Popover copy minimalism rules (MEMORY):
- Cover WHAT the motif is and how to read positive vs negative delta (sign convention: negative = you allow fewer than opponents = good).
- No jargon, no p-values, no "Wilson", no "CI" in user-facing copy.
- Keep definitions to one sentence.

Tooltip (InfoPopover on each grid row) copy format:
```
<bold>{Family Label} — {metric label}</bold>
You: {you_rate:.2f} per game  |  Opponents: {opp_rate:.2f} per game
{sign-convention sentence}
{confidence sentence: "Result is [statistically notable / within normal variation]"}
```

## Interaction Contracts

### Beta Gate
- Check `user.beta_enabled` (from `useAuth` / `/users/me`).
- When `false`: render `null` for all three tactic surfaces (chips, grid, filter section). No loading state, no placeholder, no "coming soon" copy — simply absent.
- When `true`: render normally.

### Motif Chip
- Desktop: hover opens definition popover after 100ms delay; moving into popover cancels close (grace period 80ms). Same as TagChip.
- Mobile (≤639px): tap toggles popover. Popover opens below the chip (`side="bottom"`).
- Chip is not a navigation element — no deep-link, no filter activation. Purely informational.
- Keyboard: `tabIndex=0`, `role="button"`, `Enter`/`Space` toggles popover.

### Comparison Grid
- Self-fetching via a `useTacticComparison(filters, flawFilter)` hook (mirrors `useLibraryFlawComparison`).
- Grid re-fetches when any filter in `FilterState` or the motif filter changes.
- No pagination — up to 6 family rows, fully visible.
- Family cards maintain fixed vertical order (rank by significant gap is a server-side sort; the grid renders them in the order returned by the API).

### Motif Filter
- Multi-select: selecting/deselecting families narrows the Library flaw list.
- Flows through `apply_game_filters()` on the backend (D-06).
- "All families selected" is equivalent to "no filter" (show all motifs).
- Filter state persists across tab navigation within the Library page (same as other filters).
- Beta gate: the filter section is not rendered when `beta_enabled === false`. No hidden state — purely absent from DOM.

### Mobile Parity (TACUI-03, 375px)
- All three surfaces (chip, grid, filter section) render correctly at 375px viewport.
- Comparison grid: single-column (`grid-cols-1`). Family cards stack vertically.
- MiniBulletChart: full-width within card padding.
- Motif chips on flaw cards: wrap naturally with other chips, no horizontal scroll.
- Filter section: same toggle-group layout as desktop, wraps to multi-line if needed.
