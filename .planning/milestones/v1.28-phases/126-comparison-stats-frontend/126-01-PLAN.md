<objective>
Backend for Phase 126: a new `GET /api/library/tactic-comparison` endpoint that returns
per-family (collapsed) you-vs-opponent tactic-motif rates with a Wilson-based significance
verdict and a section-level sample gate, honoring all existing game filters + severity. Plus
two supporting backend changes: a new `tactic_families` multi-select filter threaded through
`apply_game_filters()` (D-06), and surfacing `tactic_motif` / `tactic_confidence` (gated by a
named confidence threshold) on the flaw-row schemas so the frontend chip can render (TACUI-01
backend half).

Purpose: turn the populated `game_flaws.tactic_motif` column (Phases 124/125) into queryable,
filter-aware comparison + chip data, reusing the v1.25 `/flaw-comparison` machinery verbatim.
Output: one endpoint, two new schemas, one service fn, one repo fn, one extended filter arg,
two schema field additions, and tests.
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-126-03 | Information Disclosure (beta gate) | tactic data reachable by non-beta user via direct API | accept | D-01a (locked): beta gating is frontend-only, a feature-rollout flag not a security boundary. Data is the user's OWN chess data. Documented residual risk, not a blocker. |
| T-126-04 | Denial of Service | unbounded comparison query | accept | Same gate + analyzed-set bounding as /flaw-comparison (early below_gate short-circuit avoids the expensive query); no new unbounded path introduced. |
| T-126-SC | Tampering | npm/pip/cargo installs | mitigate | No new package installs in this plan (reuses existing deps). No legitimacy checkpoint required. |
