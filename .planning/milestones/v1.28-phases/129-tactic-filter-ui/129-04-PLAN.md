<objective>
Close UAT gap G-01: the "More Tactics" accordion (D-14) is unreachable because today's tactic
taxonomy has exactly 6 families, so the server's top-6-by-Missed selection always equals all families
and overflow is always empty. Replace the 6-family `FAMILY_TO_MOTIF_INTS` with the authoritative
10-family taxonomy from 129-UAT.md "RESOLVED DECISIONS" so >6 families exist, leaving 4 overflow
families and making the accordion exercisable. No accordion-render code changes — the taxonomy change
alone makes it trigger.

This is the backend half (Wave 1). It rewrites the single backend source of truth
(`FAMILY_TO_MOTIF_INTS`) and updates every backend consumer + test that hard-codes the old 6 family
keys (`pin_skewer`, `discovery`, `combinations`). The 10 family-key strings it locks are the
cross-stack contract that plan 129-05 (frontend) must mirror string-for-string.

Purpose: deliver the tactic-family taxonomy redesign the user actually wants (each tier-2 motif its
own family) AND resolve the surface-level accordion-unreachable UAT issue as a side effect.
Output: the 10-family mapping, updated backend consumers, and backend tests proving the new family
set + that dropped `combinations` ints map to no family.
</objective>
