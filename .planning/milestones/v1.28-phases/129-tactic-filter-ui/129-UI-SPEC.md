## Spacing Scale

The app uses a Tailwind 4px base grid. Phase 129 clones spacing from existing filter controls (OpponentStrengthFilter, FilterPanel) — no new spacing tokens.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`) | Icon-to-label gap inside chips |
| sm | 8px (`gap-2`, `mb-2`) | Between chip rows, label-to-control gaps |
| md | 12px (`gap-3`, `mb-3`) | Between filter sections within FlawFilterControl; between preset chips and slider. **Approved exception:** 12px is outside the canonical 4/8/16/24/32 set but is a multiple of 4. Cloned directly from `OpponentStrengthFilter.tsx` / `FilterPanel.tsx` — reusing their exact `gap-3`/`mb-3` values for visual consistency with established filter controls. |
| lg | 16px (`p-4`) | Accordion content padding (matches Endgame Statistics Concepts pattern) |
| xl | 24px | Not used in new controls; inherited from card padding |

Touch targets: `h-11 sm:h-7` on all preset chips and ToggleGroup items (matches existing filter controls exactly — 44px mobile, 28px desktop).

Exceptions:
- Depth slider preset chips: `grid grid-cols-3 gap-1 mb-3` (3-column grid for Beginner / Intermediate / Advanced, vs 4-column for opponent strength presets — same gap/mb values)
- Orientation ToggleGroup: `w-full`, items `flex-1 min-h-11 sm:min-h-0 text-sm` (exact match to "Played as" pattern)

## Typography

The app is forced dark mode. All type uses the system font stack via Tailwind's default sans. Exactly two weights are used throughout filter controls and the comparison grid.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body / filter label / grid cell label | 14px (`text-sm`) | Regular (400) | 1.5 | Filter section labels ("Tactic Difficulty", "Orientation"), summary text, slider tick labels, info-popover body, ToggleGroup items, preset chip labels, family name inside each TacticComparisonGrid card |
| Card / section heading | 16px (`text-base`) | Semibold (600) | 1.25 | "Tactic Motifs" section heading in TacticComparisonGrid; "More Tactics" AccordionTrigger label |

**Weight declaration:** Regular (400) and Semibold (600) only. The existing `TacticComparisonGrid.tsx` uses `font-medium` on the family name label (line 211) — Phase 129 new code must NOT extend this pattern. New grid cell label rows ("Missed {Family}", "Allowed {Family}") use `text-sm` with no weight modifier (Regular 400). The existing `font-bold` on the numeric delta (line 214) is a pre-existing detail in the comparison component, not a declared weight for new Phase 129 surfaces.

**Chip exception (documented):** `TacticMotifChip` uses `text-xs font-bold` — this is an established exception documented in CLAUDE.md and the existing chip code. Do not change. The `missed:`/`allowed:` prefix inherits the same `text-xs font-bold` styling as the motif name.

**Minimum size rule:** All new text uses `text-sm` (14px) or larger. The `text-xs` chip exception applies only to `TacticMotifChip` (hover/tap-activated popovers may also use `text-xs` per CLAUDE.md).

## Component Inventory

### New: `TacticDepthFilter.tsx`

Clone of `OpponentStrengthFilter.tsx` with single-handle slider.

**Structure:**
```
<div>
  <div class="mb-1 flex items-center justify-between gap-2">
    <p class="text-sm text-muted-foreground">
      Tactic Difficulty
      <InfoPopover ariaLabel="Tactic difficulty filter info" testId="filter-tactic-depth-info">
        "Limits tactics by how many moves deep the winning sequence runs.
         Beginner shows simple 1-move threats; Intermediate adds 2-3 move
         combinations; Advanced includes deep sequences. Forced mates always
         show, regardless of difficulty."
      </InfoPopover>
    </p>
    <span data-testid="filter-tactic-depth-summary">{summaryLabel}</span>
  </div>

  {/* Preset chips — 3-column grid */}
  <div class="mb-3 grid grid-cols-3 gap-1" data-testid="filter-tactic-depth-presets">
    <button aria-pressed={isActive} data-testid="filter-tactic-depth-preset-beginner"
            class="rounded border h-11 sm:h-7 text-sm transition-colors {activeClass}">
      Beginner
    </button>
    <button aria-pressed={isActive} data-testid="filter-tactic-depth-preset-intermediate"
            class="rounded border h-11 sm:h-7 text-sm transition-colors {activeClass}">
      Intermediate
    </button>
    <button aria-pressed={isActive} data-testid="filter-tactic-depth-preset-advanced"
            class="rounded border h-11 sm:h-7 text-sm transition-colors {activeClass}">
      Advanced
    </button>
  </div>

  {/* Single-handle slider — domain is FULL MOVES (D-03) */}
  <div class="px-1.5">
    <Slider min={DEPTH_SLIDER_MIN_MOVES} max={DEPTH_SLIDER_MAX_MOVES} step={DEPTH_SLIDER_STEP}
            value={[maxToSlider(value.maxMoves)]}
            thumbLabels={['Maximum tactic depth in moves']}
            data-testid="filter-tactic-depth-slider" />
    <div class="mt-1 flex justify-between text-sm tabular-nums text-muted-foreground">
      <span>{DEPTH_SLIDER_MIN_MOVES}</span>
      <span>{DEPTH_SLIDER_MAX_MOVES}</span>
    </div>
  </div>
</div>
```

**Companion lib: `tacticDepth.ts`**

Named constants (no magic numbers):

| Constant | Value | Rationale |
|----------|-------|-----------|
| `DEPTH_PRESET_BEGINNER_MAX` | `2` | 1-move tactics (depth ≤ 2 half-plies = 1 full move) — Phase 127 depth-vs-Rating correlation: low-depth tactics are most common at all rating bands |
| `DEPTH_PRESET_INTERMEDIATE_MAX` | `6` | Up to 3-move combinations (depth ≤ 6 half-plies) — the median bracket in Phase 127's depth distribution |
| `DEPTH_PRESET_ADVANCED_MAX` | `null` | No cap — shows all depths including deep sequences |
| `DEPTH_SLIDER_MIN_MOVES` | `1` | 1 full move deep minimum (slider domain = full moves, D-03) |
| `DEPTH_SLIDER_MAX_MOVES` | `5` | 5 full moves = 10 half-plies — covers all practical tactics (Phase 127 depth data shows near-zero counts beyond ~8 half-plies); slider at this position = Advanced/no-cap |
| `DEPTH_SLIDER_STEP` | `1` | 1 full move per step |
| `HALF_PLIES_PER_MOVE` | `2` | full-move (slider) ↔ half-ply (API/column) conversion factor |
| `DEPTH_DEFAULT_PRESET` | `'intermediate'` | D-02: always-on, default = Intermediate |

> **WARNING-2 reconciliation (D-03, revision 2026-06-20):** the slider operates and displays in **full moves** (domain 1..5), while `DEPTH_PRESET_*_MAX` and the `max_tactic_depth` API value are **half-plies** (1:1 with the column: Beginner=2, Intermediate=6, Advanced=null). `sliderToMax`/`maxToSlider` bridge the two via `HALF_PLIES_PER_MOVE`. The slider ticks, thumb aria-label, and summary string ALL read in full moves and agree. The earlier mixed-unit `DEPTH_SLIDER_MIN=1`/`DEPTH_SLIDER_MAX=10` framing was the WARNING-2 defect; **plan `129-02` task 1 is the executor's contract** and supersedes any conflicting slider numbers here.

**Note:** "Advanced" maps to `null` (no max) rather than a high number. The slider at `DEPTH_SLIDER_MAX` position means "Advanced / no cap." `derivePreset` returns `'advanced'` when `maxDepth === null`.

**Summary text** (authoritative source = Copywriting Contract table below; D-03 full-move units):
- Beginner preset: "Beginner (1 move)"
- Intermediate preset: "Intermediate (≤ 3 moves deep)"
- Advanced preset: "Advanced (all)"
- Custom (slider at N): "Custom (≤ N moves)"

**Active-state rule:** The depth filter is always on (D-02), so the filter-dot indicator must read as active at the default Intermediate setting. `isFlawFilterNonDefault` must account for this: the depth filter contributes to "non-default" only when the preset is not Intermediate (not when it is simply set).

## Interaction Contracts

### Depth slider behavior

- On mount: preset = Intermediate, `maxMoves = DEPTH_PRESET_INTERMEDIATE_MAX` (6 half-plies = 3 full moves); slider sits at full-move position 3.
- Preset chip click: updates slider position + `maxMoves` atomically; summary updates.
- Slider drag: reads a full-move position, maps to half-ply `maxMoves` via `sliderToMax`; clears active preset when no preset matches (custom summary in full moves).
- At slider max (`DEPTH_SLIDER_MAX_MOVES` = 5 full moves): equivalent to "no cap" = Advanced preset (`maxMoves = null`).
- Filter dot: always shows as active (D-02); `isFlawFilterNonDefault` returns true only when preset is not Intermediate.
- Mates: not filtered regardless of slider position; info popover discloses this.

### Orientation toggle behavior

- Default: "Either" selected.
- Single-select only (`type="single"`). User cannot deselect all — if `onValueChange` receives an empty string, ignore and preserve current value (same guard as "Played as").
- "Either" = OR query: a flaw card appears if its `missed_tactic_motif` OR `allowed_tactic_motif` is in the selected families (within depth bound). Both chips render on each card when both exist.
- "Missed" = only `missed_tactic_motif` contributes; only the missed chip renders.
- "Allowed" = only `allowed_tactic_motif` contributes; only the allowed chip renders.
- The toggle is hidden on the comparison grid stats page (D-09).

### FlawFilterStore additions

Add to `useFlawFilterStore.ts`:
- `tacticOrientation: TacticOrientation` — default `'either'`
- `tacticDepthPreset: TacticDepthPreset` — default `'intermediate'`
- `tacticDepthMax: number | null` — default `DEPTH_PRESET_INTERMEDIATE_MAX` (6 half-plies)

`isFlawFilterNonDefault` must treat `tacticOrientation === 'either'` as default and `tacticDepthPreset === 'intermediate'` as default.
