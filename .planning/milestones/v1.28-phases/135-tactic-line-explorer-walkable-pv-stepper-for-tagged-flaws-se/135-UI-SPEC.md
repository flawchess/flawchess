## Spacing Scale

Standard 4-point grid used throughout the project. All spacing must be multiples of 4px.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, arrow target label padding |
| sm | 8px | Compact element spacing, chip inner padding, SAN ladder row gaps |
| md | 16px | Card body padding, board-to-controls gap, toggle-to-ladder gap |
| lg | 24px | Section padding, dialog header padding |
| xl | 32px | Layout column gap (board column vs SAN ladder column on desktop) |
| 2xl | 48px | Major section breaks (not expected in this phase) |
| 3xl | 64px | Page-level spacing (not expected in this phase) |

Exceptions:
- Touch targets for `BoardControls` icon buttons: 44px minimum on mobile (matches existing `lg` size `h-11 w-11` = 44px).
- Missed/Allowed toggle: minimum touch target height 44px on mobile.

## Layout Specification

### Desktop: Full-Screen Dialog

```
┌─────────────────────────────────────────────────────────────┐
│  Header: [TacticMotifChip] "Explore Tactic"    [Close ×]   │
│  ─────────────────────────────────────────────────────────  │
│  [Missed] [Allowed]   ← toggle (hidden if only one line)   │
│  ─────────────────────────────────────────────────────────  │
│  ┌──────────────────────┐   ┌───────────────────────────┐  │
│  │                      │   │  SAN Ladder               │  │
│  │     Chess Board      │   │  23. Nf5  23…Qd7  24.Nxe7│  │
│  │      (large)         │   │  (scrollable, click-jump) │  │
│  │                      │   │                           │  │
│  └──────────────────────┘   └───────────────────────────┘  │
│  [BoardControls: ↩ ← → ⟳ | Depth: 3]                      │
└─────────────────────────────────────────────────────────────┘
```

- Two-column layout: board (left, ~55% width) + SAN ladder (right, ~45% width).
- `DialogContent` classNames: `no-scrollbar max-w-[calc(100%-1rem)] sm:max-w-4xl overflow-y-auto max-h-[90vh] sm:p-6`.
- BoardControls bar spans the full width below the board column.
- Gap between board and SAN ladder: `gap-8` (32px).

### Mobile: Full-Width Bottom Drawer (D-05)

```
┌─────────────────────┐
│  ─── (drag handle)  │
│  Header + close     │
│  [Missed] [Allowed] │
│  ┌─────────────────┐│
│  │   Chess Board   ││
│  └─────────────────┘│
│  [BoardControls row]│
│  SAN Ladder         │
│  (scrollable)       │
└─────────────────────┘
```

- `DrawerContent` direction: `bottom`. Override width to `w-full` (remove the `sm:max-w-sm` cap from the base drawer). `max-h-[95vh]`.
- Stacked layout: board → BoardControls → SAN ladder (all full-width).
- Board size: responsive, fills available width. Use `w-full aspect-square` on the board container so the board is always square.
- SAN ladder is scrollable inside the drawer body.
- The drawer closes and returns the user to the Game view (D-01/D-05).

### Breakpoint Rule

Use `md` as the breakpoint: `md:` classes switch from mobile-drawer-stacked to desktop-dialog-two-column. Specifically:
- On `< md` (mobile): render `Drawer` bottom sheet.
- On `>= md` (desktop): render `Dialog` full-screen.

### Responsive Switching Pattern

```tsx
// Pseudocode — executor chooses the hook
const isDesktop = useMediaQuery('(min-width: 768px)');
return isDesktop ? <ExplorerDialog /> : <ExplorerDrawer />;
```

## Interaction Specification

### Missed / Allowed Toggle

- Location: below the dialog/drawer header, above the board.
- Visible only when BOTH lines exist (D-01, SEED-065). Hidden (not disabled) when only one line.
- Default selection: **Missed** (SEED-065 locked).
- Switching orientation resets the stepper to the root position (ply 0 of the PV).
- Toggle uses `variant="ghost"` buttons styled with `--toggle-active` background when selected (`oklch(0.55 0 0)`). Active foreground: `--toggle-active-foreground` (`oklch(0.98 0 0)`).
- `data-testid="tactic-toggle-missed"` and `data-testid="tactic-toggle-allowed"`.
- `aria-pressed` attribute reflects the current selection.

### Board Arrows

**At root position (ply 0):**

- **Missed orientation:** Blue best-move arrow (`BEST_MOVE_ARROW = rgba(59, 130, 246, 0.8)`) with depth label (display depth via `toDisplayDepthForOrientation(missed_depth, 'missed')`). Red flaw-move arrow (`TAC_ALLOWED = oklch(0.70 0.15 25)`) with allowed display depth label. Both arrows rendered simultaneously, matching the miniboard.
- **Allowed orientation:** Red flaw-move arrow with depth label (`toDisplayDepthForOrientation(allowed_depth, 'allowed')`). Blue best-move arrow as reference only (no label). Matching miniboard convention.

**After root (ply 1+, pure engine PV):**

- Blue engine arrow only (`BEST_MOVE_ARROW`).
- Depth number on the target square, decrementing with each forward step, flooring at 0.
- At depth 0 (tactic punchline): the target square renders with a brand-brown highlight ring (`--brand-brown` at 0.20 alpha). Depth badge uses `TAC_MISSED` (blue) for missed orientation, `TAC_ALLOWED` (red/label variant) for allowed orientation.

**Payoff plies (past depth-0):**

- Blue arrows, no depth label.
- Rendered in a visually lighter style (use `BEST_MOVE_ARROW` at reduced opacity, e.g. `rgba(59, 130, 246, 0.5)`).
- Truncation: executor picks 2–4 payoff plies within CONTEXT.md's guidance band.

### Depth Counter Readout

- Displayed in `BoardControls` `infoSlot`.
- Format: `"Depth: {N}"` where N is the display depth (1-based via `toDisplayDepth`), except at payoff plies where it shows `"Payoff"` instead of a negative number.
- At root: shows the display depth of the active orientation's tactic.
- `data-testid="tactic-depth-readout"`.

### SAN Ladder

- Linear list — no branch columns (tactic lines have no candidate moves).
- Move numbers anchored to real game ply (e.g. `23. Nf5  23...Qd7  24. Nxe7+`), not restarted at 1.
- Current ply highlighted with brand-brown left border (`border-l-2 border-brand-brown`) and a subtle background tint (`bg-brand-brown/10`).
- Each move row is clickable (jump to that ply): `onClick → goToMove(plyIndex)`.
- `data-testid="tactic-san-ladder"` on the container.
- `data-testid="tactic-san-move-{ply}"` on each move row.
- Keyboard: ArrowLeft/ArrowRight via the `useTacticLine` hook (cloned from `useChessGame`).
- Depth-0 move rendered with `TAC_MISSED` (blue) or `TAC_ALLOWED` (red) text color to match the orientation. Payoff moves rendered in `--muted-foreground` (dimmed).

### Entry Points

**Flaw card button row (D-04):**

- All flaw cards (tagged and untagged) get a dedicated button row below the card body.
- Untagged flaws: button row contains **Game** only (`variant="brand-outline"`).
- Tagged flaws: button row contains **Explore** + **Game** side by side (both `variant="brand-outline"`).
- Button row uses `flex gap-2 mt-2` layout.
- `data-testid="flaw-btn-explore"` on Explore, `data-testid="flaw-btn-game"` on Game.
- Icon for Explore: `Telescope` from Lucide (if available) or `Play` as fallback. Game retains existing `Swords` icon.

**Game card Explore button (D-03):**

- Placed below the eval chart, adjacent to flaw-marker navigation.
- Always visible; disabled (`disabled` attribute + `opacity-50`) when current position is not a tagged flaw.
- Disabled tooltip: `"Select a tagged flaw to explore"` (via `Tooltip` component).
- `data-testid="game-card-btn-explore"`.
- `aria-label="Explore tactic"` when enabled; `aria-label="Select a tagged flaw to explore"` when disabled.

### Keyboard Navigation

- ArrowLeft: step back one ply.
- ArrowRight: step forward one ply.
- Escape: close the explorer (standard Dialog/Drawer behavior).
- All keyboard handlers scoped to the explorer container to avoid conflicting with page-level keyboard shortcuts.

### Close / Return Behavior (D-01, D-05)

- Desktop: `Dialog.onOpenChange(false)` closes the explorer Dialog only; the underlying Game Dialog remains open and focused.
- Mobile: `Drawer.onOpenChange(false)` closes the drawer; the underlying Game view is restored.
- No "close everything" behavior on Escape from the explorer.

## ARIA & Accessibility

- `DialogTitle` or `DrawerTitle` must be visible or have `aria-label` on the container (use `"Explore Tactic"` as the visible title).

- Board container: `aria-label="Tactic board"`.
- `BoardControls` buttons retain their existing `aria-label` attributes.
