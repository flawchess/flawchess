<objective>
Build the storage foundation for the opening-eval dedup cache: the `opening_position_eval` table (Alembic migration + ORM model) and a standalone idempotent backfill script that populates it from existing our-engine opening evals. Per D-123.1-01 the table is keyed by `full_hash` (PK) with the eval resolved onto the row, which is the structure that collapses the common-opening fan-out the self-join cannot. Per D-123.1-06 the one-time data load is a script, NOT the migration (it must not run on the test template or every env, and prod is a ~1M-row op behind the SSH tunnel).

This plan delivers no behavior change to the drain (read/write swap is plan 02). It only creates the table, the model, and the populated cache.
</objective>

    ```sql
    INSERT INTO opening_position_eval (full_hash, eval_cp, eval_mate, best_move)
    SELECT DISTINCT ON (nxt.full_hash) nxt.full_hash, cur.eval_cp, cur.eval_mate, nxt.best_move
    FROM game_positions cur
    JOIN game_positions nxt ON nxt.game_id = cur.game_id AND nxt.ply = cur.ply + 1
    JOIN games g ON g.id = cur.game_id
    ```
