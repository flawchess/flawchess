# Milestone v1.31: Pipeline Consolidation

**Status:** ✅ COMPLETE 2026-07-04 (merged to `main`; prod deploy pending)
**Phases:** 148–150
**Total Plans:** 14 (4 + 5 + 5)

## Overview

Retire the dead Gen-1 eval protocol and unify the copy-pasted eval write path so new
pipeline work threads through one code path instead of 3+, removing the seams that generated
FLAWCHESS-8D and the Phase 146/147 ungated-tag bugs. **Server-side only** — no worker protocol
change, no fleet redeploy (fleet confirmed fully on atomic-lease/submit; a 2026-07-04 prod log
grep showed zero legacy `/lease`+`/submit` hits over 11.3h). Sourced from SEED-080 (the Tier-A/B
recommendations of `reports/pipeline-review-2026-07-04.md`; R-numbers below reference that review).

The milestone's user-visible contract is **no behavior change** — the pipeline produces
identical eval/flaw/tag results through a smaller, single-path implementation. Phases run in
dependency order: 148 (correctness prerequisite) → 149 (Retire & Prune, shrinking the surface)
→ 150 (Consolidate Write Path, so it collapses 2 copies rather than 3).

## Milestone Summary

**Decimal Phases:** none (148, 149, 150 all integer).

**Key Decisions:**

- Retire & prune (149) before consolidating (150) — so Phase 150 collapses 2 copies of the write path, not 3, shrinking the risky R3 authoritative-write rewrite before touching it.
- Delete the Gen-1 `/lease`+`/submit` protocol only after prod-traffic-zero (11.3h grep) + porting the only end-to-end job_id/lichess coverage to the atomic lane.
- Per-ply diff/upsert replaces delete-then-insert + snapshot/restore in `_classify_and_fill_oracle` (R3) — makes blob/tag preservation native to the write, deletes the FLAWCHESS-8D / Phase 146/147 compensation seam.
- Golden-snapshot equivalence harness (8 fixtures from pristine HEAD + regenerate-and-diff) gates the R3 rewrite — the only safe way to rewrite the load-bearing authoritative write.
- Durable `import_jobs` guard row + partial unique index over the in-memory duplicate check (R8) — DB is the single arbiter with `IntegrityError` idempotency.
- Explicit "unknown" chess.com result + Sentry over the silent-draw fallback (R13) — surfaces the gap instead of polluting WDL stats with fabricated draws.

**Issues Resolved:**

- FLAWCHESS-8D root cause (delete-then-insert wiping blobs → fragile snapshot/restore) retired at the source (R3).
- The concurrent-duplicate-import TOCTOU race closed at the DB level (R8).
- Silent-draw pollution from unrecognized chess.com results eliminated (R13).

**Issues Deferred:**

- R14 tier-3 lease (double-claim hardening) — owned by SEED-072, deferred to a later milestone.
- The local in-process drain re-mints ~9/3.36M ungated cp tags (self-heals via tier-4, not rollback-class) — carryover from v1.30, `project_local_drain_ungated_tactic_tags`.

**Technical Debt Incurred:**

- One narrow deliberate residual router import (`_load_pgns_for_games`) remains after the R7 module split, flagged in 150-05.

**Explicitly out of scope:** merging the entry/full lanes (§7 "not recommended"), changing the
post-move eval convention (reopens SEED-044 off-by-one risk), queue/broker rewrite, any worker
protocol change / fleet redeploy, client-facing behavior change (outputs byte-identical), R9
(delivered in Phase 148 as CORR-03), full SEED-078 streaming.
