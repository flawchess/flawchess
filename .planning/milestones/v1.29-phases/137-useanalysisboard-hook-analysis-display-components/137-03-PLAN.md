Build `VariationTree.tsx` — the move list that renders the analysis tree's flat main line plus the
single active variation (BOARD-05; full nested-tree display is deferred to v2). Responsive split
(D-02): mobile extends the existing `HorizontalMoveList` (lichess-style horizontal chips with the
variation inline in parentheses); desktop is a NEW vertical paired `N. white black` list with the

`hidden sm:block`) — there is no `useMediaQuery` hook in the codebase and none should be added.

it depends on Plan 01's exported `MoveNode` / `NodeId` types — hence Wave 2.

any move navigates / jump to any node).
