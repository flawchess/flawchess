# Milestone v1.30: Forcing-Line Tactic Gate

**Status:** ✅ SHIPPED 2026-06-30
**Phases:** 141–147
**Total Plans:** 25 (23 completed; 144-02 HUMAN-UAT + 145-06 prod-drain completed-in-effect through Phase 146/147)

## Overview

Harden the v1.28 tactic "cause-of-error" axis so its tags reflect *real forced tactics*, not
incidental geometry in the non-forced tail of an engine PV. The puzzle-trained tagger fired
noisy tags (clearance / sacrifice / capturing-defender — too deep, wrong-labeled, or trivial)
on non-forced real-game PVs. This milestone adds an "only-move" forcing-line gate modeled on
lichess-puzzler and persists MultiPV=2 evals so every threshold change is a seconds-fast
offline re-tag with no engine re-pass.

Phases 141–145 shipped the gate end-to-end and rolled it live across the corpus (release #229,
2026-06-30). Phases 146–147 were grouped in at close per the standalone-then-regroup pattern
(cf. v1.27/v1.28): the actual prod rollout — the upgraded remote-worker fleet that Phase 145's
drain was gated on — is the deliverable of Phase 146, and Phase 147 hardened the invariant so
only forcing-line-gated tags are ever persisted (go-forward suppression + a corpus data
migration + a server-authoritative atomic eval+blob worker pipeline).

**AGPL boundary respected** — lichess-puzzler heuristics/constants/names only, no source
copied (same boundary as Phase 124/125). The committed fixture precision gate cannot surface
this bug class (no non-forced tail), so validation was engine-free A/B over stored evals.

## Phases

### Phase 141: JSONB Schema + Gate Logic

**Goal**: The ORM model, DB migration, and forcing_line_gate module exist and are independently testable without any engine or DB
**Depends on**: Phase 140 (v1.29)
**Requirements**: STORE-01, STORE-02, GATE-01, GATE-02
**Plans**: 2 plans

- [x] 141-01: Alembic migration + ORM `allowed_pv_lines`/`missed_pv_lines` JSONB (`deferred=True` structural leak guard) + query-site audit (explicit column projections everywhere)
- [x] 141-02: pure engine-free `forcing_line_gate.py` — only-move win-prob margin (`ONLY_MOVE_WIN_PROB_MARGIN=0.35` via `eval_utils.LICHESS_K`), already-winning reject (`ALREADY_WINNING_CP_THRESHOLD=300`), still-winning floor (`STILL_WINNING_FLOOR_CP=200`), trailing-only-move strip, one-mover discard; all thresholds named constants; `PvNode` uses `eval_mate_to_expected_score` for perspective-safe mate checks

**Details:** No existing stats/read path regresses — every `select(GameFlaw)` query site audited and converted to explicit column projections so the JSONB blobs never leak into stats scans (`mistake-stats`, `flaw-comparison`, benchmark delta). The margin is a *tunable* parameter (defaults to lichess-puzzler's 0.35), final value chosen empirically in Phase 144.

### Phase 142: MultiPV=2 Engine Pass + Eval Drain + Remote Worker

**Depends on**: Phase 141
**Requirements**: MPV-01, MPV-02, MPV-03
**Plans**: 4 plans

- [x] 142-01: `_analyse_multipv2` + `evaluate_nodes_multipv2` EnginePool method (best + second eval cp/mate + second-best UCI per node; `list[InfoDict]` return forbids reusing `_analyse_with_pv`; `su` str sentinel `''` for single-legal-move positions)
- [x] 142-02: eval-drain multipv=2 switch + Option-B PV-walk blob assembly + JSONB write; `_fill_engine_game_flaw_second_best` mirrors SEED-056 for dedup-transplanted plies

**Details:** The MultiPV pass reuses the module-level `EnginePool` within the 4g container RSS budget. The pass is NOT gated on `lichess_evals_at` (second-best is new data, not a lichess freebie).

### Phase 143: Offline Re-tagger

**Goal**: A pure-offline scripts/retag_flaws.py re-derives tactic tags from stored JSONB in seconds with no engine pass, with mate combinations and defender branching covered by explicit unit tests
**Depends on**: Phase 142
**Requirements**: GATE-03, GATE-04, RETAG-01, RETAG-02
**Plans**: 3 plans

- [x] 143-01: parameterize the gate by margin (D-03) + GATE-03 mate-priority & GATE-04 multi-ply defender unit tests
- [x] 143-02: wire the gate into the live classify path (`_classify_tactic_gated` + thread in-memory blobs through the eval drain) — single classify path (SC4)
- [x] 143-03: offline re-tagger — extend + `git mv` to `scripts/retag_flaws.py` (blob loading, `--margin`, gated classify), per-motif delta report, idempotency test

**Details:** Mate-priority hierarchy runs *before* the sigmoid: only-best-is-mate → forced; both-mate → shorter distance forced; else fall through to sigmoid — so mate-in-1 and genuine mating combinations are never suppressed. Uniqueness is checked at solver nodes only; defender-node ambiguity does not kill a line (branch-then-reconverge treated as forced). Idempotent via the single classify path.

## Milestone Summary

**Requirements:** all 15 v1 requirements (STORE-01/02, GATE-01..04, MPV-01..03, RETAG-01/02, VALID-01/02, SHIP-01/02) complete — see `.planning/milestones/v1.30-REQUIREMENTS.md`.

**Key Decisions:**

- Inline JSONB on `game_flaws` (not a sidecar `game_flaw_pv_lines` table) — TOAST gives equivalent physical decoupling without a JOIN; `deferred=True` is the structural leak guard so blobs never load into stats scans.
- New `_analyse_multipv2` method required for the `list[InfoDict]` return type (can't reuse `_analyse_with_pv`).
- `blobs_pending` is an independent default-False signal, never derived from `flaw_pv_blobs` — changes behavior only at the `_apply_submit` write site (D-01, D-03 of Phase 147).
- Live `/submit` continuation eval offloaded to the remote fleet (Phase 146 D-03) — full-ply worker pass reduced to MultiPV-1; second-best only on the dedicated tier-4 blob rung.
- Versioned `/atomic-lease` + `/atomic-submit` contract lives alongside the old pair (Phase 147 D-02) so a mixed old+upgraded fleet runs simultaneously with instant rollback (flip workers back, no server redeploy).
- Two-stage Efraimidis-Spirakis tier-4 lottery (supersedes the top-50 recency window) so the whole analyzed corpus drains, fresh stays dominant.

**Issues Resolved:**

- Puzzle-trained tagger noise on non-forced PV tails (the milestone's reason for being) — FORK/allowed suppression, HANGING_PIECE, etc. now gated.
- Forcing-line gate Bug A (pre-flaw eval convention) + Bug B (whole-line only-move check rejecting flatly-winning conversion tails) — the gate is now depth-aware (quick 260630-jsr).
- SEED-072 tier-4 blob-routing starvation via the idle `/lease` path (release #231).
- SEED-073 fat-game `/flaw-blob-lease` 500 → over-cap 204 sentinel.

**Technical Debt / Known Gaps:**

- **Phase 147 strict-zero invariant is broken in prod** by the local in-process server-pool drain, which calls `_classify_and_fill_oracle` with default `blobs_pending=False` (`eval_drain.py`) and re-mints raw cp-based tactic tags. Impact is tiny (~9 of 3.36M rows) and self-heals slowly as tier-4 fills the blobs; **not rollback-class**. The atomic-submit worker path (rungs 1/3) is correctly gated; only the in-process drain leaks. Tracked in project memory (`project_local_drain_ungated_tactic_tags`).
- Phase 147-06 full atomic gated-write e2e is HUMAN-UAT-pending (dev DB has no queued eval_jobs); automated dry-run confirmed the wiring.
- v2 gate refinements deferred (GATEX-01..04): both-winning-captures exception, per-motif margin tuning, defender-node ambiguity rule, tablebase uniqueness.
