<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| worker submit → authoritative flaw write | Server re-derives flaws; worker-submitted blob/flaw claims are not trusted. |
| write_session transaction | Completion markers must not commit if the flaw write fails (fail-closed). |

Structure-only refactor with a data-integrity edge (silent blob wipe is a correctness regression, not a security threat, but is the phase's core risk).
