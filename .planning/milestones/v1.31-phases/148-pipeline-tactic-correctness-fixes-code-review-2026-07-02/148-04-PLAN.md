<objective>
Fix item 4 (D-06 seam: import robustness = item 4): one malformed platform game
aborts the whole import. `normalize_chesscom_game` / `normalize_lichess_game` are
called unguarded per game; a single `KeyError` (e.g. `game["white"]`, a direct
subscript with no `.get()`) propagates out of the async generator and fails the
entire import job.

Add a per-game try/except + skip + one aggregated Sentry capture, mirroring the
existing PGN-parse precedent (`import_service.py:922-932`, already CLAUDE.md-compliant:
one capture per failed game, variables via set_context, never in the message text).

Purpose: a single bad game no longer costs the user their whole import.
Output: two symmetric guards + two skip-and-continue tests.
</objective>
