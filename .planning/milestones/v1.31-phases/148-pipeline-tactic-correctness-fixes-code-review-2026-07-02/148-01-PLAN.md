Fix the two item-1 tactic production defects (D-06 seam: tactics = item 1):

`fen_map` stores `board_fen()`, dropping ep/castling state and corrupting PV replay.

geometry-dependent subtypes). Per D-02, store full `board.fen()` in the
detector-internal `fen_map` only — Zobrist/position-comparison call sites keep
`board_fen()`. Per D-03, re-run the tactic precision gate as a regression check.

Purpose: deep mates tag correctly and ep/castling flaws no longer silently fail
`parse_san` or replay a corrupt board.
Output: two source fixes, updated + new unit tests, green precision gate.
</objective>
