<objective>
Make the import-job concurrency guard durable (PRUNE-05 / R8): move the `import_jobs` row INSERT out of the fire-and-forget `_bootstrap_import_job` background task and into the `start_import` request handler before `asyncio.create_task`, backed by a partial unique index on `(user_id, platform) WHERE status IN ('pending','in_progress')`. The DB — not the in-memory `find_active_job` registry — becomes the source of truth for concurrency, closing the TOCTOU race where two concurrent requests both pass the empty in-memory check.

Purpose: prevent duplicate concurrent imports (resource exhaustion) at a level that survives process restarts and multi-worker deploys.
Output: one migration, a new repository query, the moved insert with IntegrityError-as-idempotency, and a race test.
</objective>
