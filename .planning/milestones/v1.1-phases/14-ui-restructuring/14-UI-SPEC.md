## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, move list item gaps |
| sm | 8px | Compact element spacing, button gaps within a group |
| md | 16px | Default element spacing, sidebar section padding, card padding |
| lg | 24px | Section padding, sidebar-to-content gap on mobile |
| xl | 32px | Layout gaps (not used — prefer the grid gap below) |
| 2xl | 48px | Empty state vertical padding (py-12 = 48px) |
| 3xl | 64px | Not used in this phase |

Grid gap: `gap-8` (32px) between sidebar and content on desktop — matches existing Dashboard pattern.
Page padding: `px-4 py-6 md:px-6` — matches existing layout.

Exceptions:
- Touch targets for icon-only buttons (BoardControls): minimum 44px height via `size="icon"` (shadcn default is 40px; wrap in `h-11` container if needed)
- Tab trigger height follows shadcn Tabs default (36px) — acceptable for this content-dense layout

Source: Existing Dashboard.tsx patterns (`mx-auto w-full max-w-7xl flex-1 px-4 py-6 md:px-6`, `md:grid-cols-[350px_1fr] md:gap-8`)

## Interaction Contract

### Auto-fetch behavior
All three sub-tabs auto-fetch when position or filters change. No Filter button and no Analyze button exist on OpeningsPage. Debounce: 300ms on filter state changes to prevent query spam on rapid toggle clicks.

### Tab state persistence
Filter state (unified `FilterState`) and chess position (from `useChessGame`) live in `OpeningsPage` parent — never inside sub-tab components. Tab switch does not reset these.

Games tab: pagination offset resets to 0 on tab switch (not preserved). Source: 14-CONTEXT.md decisions.

### Filter state unification
`Dashboard.FilterState` and `Openings.StatsFilters` are merged into a single `FilterState` type in `OpeningsPage`. The merged type must cover all fields needed by all three sub-tabs.

### Import progress — global toasts
`ImportProgress` is lifted to `App.tsx` level (or a global context). When the user navigates away from `/import` while a job is running, the toast still fires on completion. The inline progress display on `/import` is visible only when on that page.

### URL-based tab routing
shadcn `Tabs` `value` prop is controlled by the current route path segment. Tab clicks use React Router `navigate()`. Browser back/forward navigates between tabs.

### Games tab empty state
Games tab shows all games by default (no `positionFilterActive` gating). When no games are imported: show empty state with CTA linking to `/import`.
