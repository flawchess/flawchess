## Spacing Scale

Declared values (multiples of 4 only). Uses Tailwind v4 default scale via shadcn.

| Token | Value | Tailwind class | Usage in this phase |
|-------|-------|----------------|---------------------|
| xs | 4px | `gap-1`, `p-1` | Icon-to-text gap in transposition indicator |
| sm | 8px | `gap-2`, `p-2` | Table cell inner padding |
| md | 16px | `gap-4`, `p-4` | Explorer component padding, section spacing |
| lg | 24px | `gap-6`, `p-6` | Card-level padding |
| xl | 32px | `gap-8` | Major layout gaps |
| 2xl | 48px | `gap-12` | Not used in this phase |
| 3xl | 64px | `gap-16` | Not used in this phase |

**Exceptions:**
- Touch targets on mobile: minimum 44px height for move row click targets — use `min-h-[44px]` on `<tr>` or row wrapper on mobile breakpoints.
- Mini WDL bar height: 12px (`h-3`) — intentionally sub-scale as a data-dense inline element.

| Role | Light mode value | Dark mode value | Tailwind token | Usage |
|------|-----------------|-----------------|----------------|-------|
| Dominant (60%) | `oklch(1 0 0)` | `oklch(0.145 0 0)` | `bg-background` | Page background, explorer container background |

## Component Inventory

New components created this phase and their contracts:

### MoveExplorer (`frontend/src/components/move-explorer/MoveExplorer.tsx`)

**Purpose:** Displays next-moves table, computes board arrows, handles row click.

**Props:**
```typescript
interface MoveExplorerProps {
  fullHash: bigint;           // from useChessGame.hashes.fullHash
  position: string;           // board FEN — for SAN-to-squares resolution
  filters: FilterState;       // from Dashboard filter state
  onMoveClick: (from: string, to: string) => void;  // calls useChessGame.makeMove
}
```

**data-testid mapping:**
| Element | data-testid |
|---------|-------------|
| Component container | `move-explorer` |
| Section heading | `move-explorer-heading` |
| Table | `move-explorer-table` |
| Each move row | `move-explorer-row-{move_san}` (e.g. `move-explorer-row-e4`) |
| Transposition icon | `move-explorer-transpose-{move_san}` |
| Empty state | `move-explorer-empty` |
| Loading skeleton | `move-explorer-loading` |

**States:**
1. Loading — skeleton rows (3 rows at same height as data rows, bg-muted, animate-pulse)
2. Empty — empty state message (see Copywriting Contract)
3. Data — table with move rows
4. Error — error message (see Copywriting Contract)

### useNextMoves (`frontend/src/hooks/useNextMoves.ts`)

**Purpose:** TanStack `useQuery` hook. Auto-fetches `POST /analysis/next-moves` on hash or filter change. No manual trigger.

**Query key:** `['nextMoves', hashToString(fullHash), serializedFilters]`

**Enabled:** always `true` — explorer is always visible per CONTEXT.md decision.

**Filter fields forwarded (exact set — no additions):** `time_control`, `platform`, `rated`, `opponent_type`, `recency`, `color`. Omit `match_side`, pagination fields.

**Debounce:** none by default. Add 150ms debounce only if rapid-navigation thrashing is observed in manual testing.

### ChessBoard extension (`frontend/src/components/board/ChessBoard.tsx`)

**New prop added:**
```typescript
arrows?: Arrow[];   // from react-chessboard/dist/types; defaults to []
```

**Required option:** `clearArrowsOnPositionChange: false` — must be set whenever `arrows` prop is non-empty to prevent react-chessboard wiping arrows on position change.

## Explorer Table Layout Contract

**Structure:** 3 columns in an HTML `<table>` element (or shadcn `Table` component).

| Column | Header | Width | Content |
|--------|--------|-------|---------|
| Move | "Move" | `w-[3rem]` min-content | SAN string + optional transposition icon |
| Games | "Games" | `w-[4rem]` min-content | Integer game count |
| Results | "Results" | flex-1 / `w-full` | Mini WDL bar (12px height) |

**Row interaction:**
- Entire row is clickable (`cursor-pointer`)
- `hover:bg-accent` on hover
- `onClick`: resolve SAN to from/to squares, call `onMoveClick(from, to)`
- Row acts as a `<button>` role element for accessibility (use `role="button"` or wrap in semantic element)
- Mobile minimum height: `min-h-[44px]` to meet touch target standard

**Row ordering:** by `game_count` descending (backend already returns this order — no client-side sort needed).

## Transposition Indicator Contract

Appears in the Games column, inline after the game count number, when `transposition_count > game_count`.

| Property | Value |
|----------|-------|
| Icon | `ArrowLeftRight` from lucide-react (16px, `h-4 w-4`) |
| Color | `text-muted-foreground` |
| Position | Inline after game count, `inline-flex items-center gap-1` |
| Tooltip trigger | shadcn `Tooltip` + `TooltipTrigger` wrapping the icon |
| Tooltip content | "Position reached in {transposition_count} total games ({transposition_count - game_count} via other move orders)" |
| Tooltip component | shadcn `tooltip` — must be added: `npx shadcn add tooltip` |
| Tooltip provider | `TooltipProvider` must wrap `MoveExplorer` (or be present in parent) |

## Mobile Responsive Layout Contract

Source: CLAUDE.md requirement "UI must be mobile friendly."

| Breakpoint | Layout |
|------------|--------|
| Mobile (`< md` / `< 768px`) | Explorer stacks below board. Table columns compress: Move column min 2.5rem, Games column min 3rem, Results column takes remaining width. |
| Desktop (`>= md` / `>= 768px`) | Explorer in right column of Dashboard, above WDL bar and game cards. |

**Table column behavior on mobile:**
- SAN text truncates with `truncate` if overflow (rare — SAN strings are short)
- Game count uses `tabular-nums` for alignment stability
- Mini WDL bar scales to full available width
