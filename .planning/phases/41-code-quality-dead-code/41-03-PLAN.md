  <action>
    **Step 1: Run knip and capture the full report:**
    ```bash
    cd frontend && npm run knip 2>&1
    ```

    **Step 2: Categorize each reported issue:**

    For EACH item knip reports, determine if it is:

    a) **True dead export** — function/type/constant exported but never imported anywhere. Action: remove the `export` keyword (if still used internally) or delete the entire function/type/constant (if not used at all).

    b) **Test-only export** — exported function used only in `.test.ts`/`.test.tsx` files. Since `tsconfig.app.json` excludes test files, knip may not see the test imports. Action: do NOT remove. Instead:
       - First check if the Vitest plugin is detecting test files automatically (look for knip output mentioning vitest plugin).
       - If knip still reports test-only exports as dead, add `"ignoreExportsUsedInFile": true` to `knip.json` OR add the specific test patterns to the knip config.

    c) **False positive from dynamic import** — component loaded via React Router lazy import or dynamic `import()`. Action: add the file to `entry` in `knip.json`.

    d) **Unused dependency** — knip may report unused npm dependencies. Action: verify by grepping the codebase. If truly unused, remove from `package.json` with `npm uninstall {pkg}`. If used implicitly (e.g., Tailwind plugin, Vite plugin), add to `ignoreDependencies` in `knip.json`.

    e) **Unused file** — entire file with no imports. Action: verify by grepping. If truly unused, delete the file.

    **Step 3: Make the fixes:**

    For each true dead export/file:
    - Read the file to understand the export
    - Grep the entire frontend/src/ directory for the export name to confirm it's unused
    - Remove the dead export (or the entire file if everything in it is dead)

    For false positives, tune `knip.json`. Possible additions:
    ```json
    {
      "$schema": "https://unpkg.com/knip@6/schema.json",
      "entry": ["src/main.tsx", "src/prerender.tsx"],
      "project": ["src/**/*.{ts,tsx}"],
      "ignoreExportsUsedInFile": true
    }
    ```

    Or for specific ignore patterns:
    ```json
    {
      "ignoreDependencies": ["package-name-if-implicit"]
    }
    ```

    **Step 4: Verify clean report:**
    Run `cd frontend && npm run knip` again. It must exit with code 0. If not, repeat steps 2-3 for remaining items.

    **Step 5: Verify nothing broke:**
    Run `cd frontend && npm run build && npm test` to confirm no functional exports were accidentally removed.

    IMPORTANT per D-05: Do NOT install any additional tools (Oxlint, Biome, Madge, ts-prune). Knip is the only frontend dead code tool for this phase.
