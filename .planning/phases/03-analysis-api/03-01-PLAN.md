<task type="auto">
  <name>Task 1: Create Pydantic schemas and analysis repository</name>
  <files>app/schemas/analysis.py, app/repositories/analysis_repository.py</files>
  <action>
Create `app/schemas/analysis.py` with Pydantic v2 models:

- `AnalysisRequest(BaseModel)`: `target_hash: int`, `match_side: Literal["white", "black", "full"] = "full"`, optional filters: `time_control: list[Literal["bullet", "blitz", "rapid", "classical"]] | None = None`, `rated: bool | None = None`, `recency: Literal["week", "month", "3months", "6months", "year", "all"] | None = None`, `color: Literal["white", "black"] | None = None`, pagination: `offset: int = Field(default=0, ge=0)`, `limit: int = Field(default=50, ge=1, le=200)`.

- `WDLStats(BaseModel)`: `wins: int`, `draws: int`, `losses: int`, `total: int`, `win_pct: float`, `draw_pct: float`, `loss_pct: float`.

- `GameRecord(BaseModel)`: `game_id: int`, `opponent_username: str | None`, `user_result: Literal["win", "draw", "loss"]`, `played_at: datetime | None`, `time_control_bucket: str | None`, `platform: str`, `platform_url: str | None`.

- `AnalysisResponse(BaseModel)`: `stats: WDLStats`, `games: list[GameRecord]`, `matched_count: int`, `offset: int`, `limit: int`.

Create `app/repositories/analysis_repository.py` with two async functions:

1. `query_all_results(session, user_id, hash_column, target_hash, time_control, rated, recency_cutoff, color)` — returns list of tuples `(result, user_color)` for ALL matching games (lightweight query for stats computation). Uses `select(Game.result, Game.user_color).join(GamePosition).where(...).distinct(Game.id)` with dynamic filter chaining. This avoids loading full ORM objects for stats.

2. `query_matching_games(session, user_id, hash_column, target_hash, time_control, rated, recency_cutoff, color, offset, limit)` — returns `tuple[list[Game], int]` (paginated Game objects + total count). Uses the same base query pattern but selects full `Game` objects with `.offset().limit()` and a `func.count()` subquery for the total.

Both functions share a `_build_base_query` internal helper that constructs the filtered SELECT with DISTINCT to avoid code duplication. The helper takes `select_entity` as parameter (either `Game.result, Game.user_color` for stats or `Game` for full objects).

Dynamic filter application pattern (from research):
- `if time_control is not None: base = base.where(Game.time_control_bucket.in_(time_control))`
- `if rated is not None: base = base.where(Game.rated == rated)`
- `if recency_cutoff is not None: base = base.where(Game.played_at >= recency_cutoff)`
- `if color is not None: base = base.where(Game.user_color == color)`

Hash column mapping dict: `HASH_COLUMN_MAP = {"white": GamePosition.white_hash, "black": GamePosition.black_hash, "full": GamePosition.full_hash}`

Order paginated results by `Game.played_at.desc()`.

<task type="auto">
  <name>Task 2: Create analysis service and router with main.py wiring</name>
  <files>app/services/analysis_service.py, app/routers/analysis.py, app/main.py</files>
  <action>
Create `app/services/analysis_service.py`:

1. `derive_user_result(result: str, user_color: str) -> Literal["win", "draw", "loss"]` — pure function. If result is "1/2-1/2" return "draw". If ("1-0" and user_color=="white") or ("0-1" and user_color=="black") return "win". Else "loss".

2. `RECENCY_DELTAS` dict mapping recency strings to timedeltas: week=7d, month=30d, 3months=90d, 6months=180d, year=365d.

3. `recency_cutoff(recency: str | None) -> datetime | None` — returns None for None or "all", otherwise `datetime.now(tz=timezone.utc) - RECENCY_DELTAS[recency]`.

4. `analyze(session: AsyncSession, user_id: int, request: AnalysisRequest) -> AnalysisResponse` — the main orchestrator:
   - Map `request.match_side` to hash column via `HASH_COLUMN_MAP`
   - Compute `recency_cutoff` from `request.recency`
   - Call `query_all_results(...)` to get all `(result, user_color)` tuples for stats
   - Compute W/D/L: iterate tuples, call `derive_user_result`, count wins/draws/losses. Guard division by zero: percentages are 0.0 when total is 0. Use `round(x, 1)` for one decimal place.
   - Call `query_matching_games(...)` for paginated game list
   - Build `GameRecord` list from Game objects (calling `derive_user_result` for each)
   - Return `AnalysisResponse` with stats, games, matched_count, offset, limit
