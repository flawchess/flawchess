## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding |
| sm | 8px | Button internal padding, compact element spacing |
| md | 16px | Default element spacing, section padding |
| lg | 24px | Section grouping gaps |
| xl | 32px | Layout-level gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Board action buttons: reduced from `h-11 w-11` (44px) to `h-9 w-9` (36px) on mobile. 36px is the chosen compact size — large enough for reliable touch targets while freeing vertical space for the trigger row. (D-01 from CONTEXT.md — exact size left to Claude's discretion; 36px chosen.)
- Sidebar trigger buttons: `h-11` (44px) minimum touch target height, full-width row. Touch targets must never fall below 44px height on mobile.
- DrawerHeader padding: `p-4` (16px) — inherited from existing Drawer component.

## Component Inventory

### New: Sidebar Trigger Button Row

Location: below `BoardControls` (vertical) in the mobile sticky board container, outside and after BoardControls in the DOM.

```
[Filter icon + "Filters"]   [Bookmark icon + "Bookmarks"]
```

Structure:
- Wrapper: `flex gap-2 w-full` (horizontal row, full width)
- Each button: `flex-1 h-11` — flex-1 ensures even split; h-11 for 44px touch target
- Variant when inactive: `variant="ghost"` (matches board nav buttons)
- Variant when active (sidebar open): `bg-brand-brown text-white hover:bg-brand-brown-hover` — use `PRIMARY_BUTTON_CLASS` or equivalent inline for the active state
- Icons:
  - Filter trigger: `SlidersHorizontal` (from lucide-react) — 16px (`h-4 w-4`)
  - Bookmark trigger: `BookMarked` (from lucide-react) — 16px (`h-4 w-4`)
- Button labels: "Filters" (left), "Bookmarks" (right) — visible text alongside icon
- `data-testid`: `btn-open-filter-sidebar` and `btn-open-bookmark-sidebar`
- `aria-label`: "Open filters" and "Open bookmarks" (supplemental, as text is visible)

### Modified: BoardControls (mobile vertical variant)

- Reduce `h-11 w-11` to `h-9 w-9` on the base class (applies to all 4 buttons: reset, back, forward, flip)
- `sm:h-8 sm:w-8` override remains as-is
- Icon size stays `h-4 w-4` (unchanged)
- No structural change — size class change only

### Reused: Vaul Drawer (direction="right")

- Both sidebars use `<Drawer direction="right">` — existing `drawer.tsx`
- `DrawerContent` inherits `w-3/4 rounded-l-xl border-l inset-y-0 right-0` from existing styles
- `DrawerHeader` contains `DrawerTitle` + optional `DrawerClose` button (X icon, `aria-label="Close"`)
- Content area: `overflow-y-auto flex-1` — unconstrained vertical scroll
- Vaul swipe-to-dismiss: enabled by default (native Vaul behavior, do not disable)
- Transition: Vaul default (~300ms ease, slide from right) — do not override

Filter sidebar drawer:
- `DrawerTitle`: "Filters"
- Content: all filter sections — Played as, Piece filter, Recency, Time controls, Platforms, Rated, Opponent type
- Deferred apply: local filter state cloned on open; committed to real filters on close (D-10)
- No "Apply" button — closing IS the apply action (D-12)
- `data-testid="drawer-filter-sidebar"`

Bookmark sidebar drawer:
- `DrawerTitle`: "Position Bookmarks" (matches existing collapsible title)
- Content: InfoPopover + Save button + Suggest button + `PositionBookmarkList`
- Loading a bookmark closes the sidebar before/after applying position (D-13)
- `data-testid="drawer-bookmark-sidebar"`

### Removed: Mobile Collapsibles

These are deleted entirely (D-07, D-08):
- `<Collapsible>` wrapping "More filters" (lines 896-915 of `Openings.tsx`)
- `<Collapsible>` wrapping "Position bookmarks" (lines 918-981 of `Openings.tsx`)

State variables no longer needed: `mobileFiltersOpen`, `positionBookmarksOpen`.

## Interaction Contract

### Filter Deferred Apply Flow

1. User taps "Filters" trigger button.
2. `localFilters` is set to a shallow copy of current `filters`.
3. Filter sidebar opens (`filterSidebarOpen = true`). Trigger button shows active state (brand-brown).
4. User toggles filters inside sidebar — each change calls `setLocalFilters(...)`.
5. API calls are NOT triggered during sidebar interaction.
6. On any close event (overlay tap, swipe, X button): `handleFiltersChange(localFilters)` is called, then `filterSidebarOpen = false`.
7. Board updates with new filters. Trigger button returns to inactive ghost style.

### Bookmark Load Flow

1. User taps "Bookmarks" trigger button → sidebar opens.
2. User taps "Load" on a bookmark card.
3. Bookmark position is applied to board state.
4. Sidebar closes immediately (D-13).

### Mutual Exclusion

Only one sidebar open at a time. Opening one sidebar while the other is open: close the open one first (or let Vaul handle via separate state booleans — each drawer is independent; opening a second one while the first is open should not occur in the UI since there is only one trigger row with two distinct buttons).

### Touch Gestures

- Swipe left on the sidebar (right-direction drawer): closes and commits (filter) or closes (bookmark) — Vaul native behavior.
- Tapping overlay closes and commits (Vaul `onClose` event).

## Accessibility

- All icon-only buttons (if any) must carry `aria-label`.
- Sidebar trigger buttons have visible text labels — `aria-label` is supplemental only.
- DrawerTitle provides the accessible name for the dialog region (Vaul sets `role="dialog"` on DrawerContent).
- Focus must be trapped inside the open Drawer (Vaul handles this natively).
- Escape key closes the sidebar — Vaul native behavior, do not suppress.
- Swipe-to-dismiss must not interfere with vertical scrolling of sidebar content — Vaul handles this with its threshold logic.

---

## Responsive Boundary

All changes in this phase are wrapped in `md:hidden` (mobile) context:
- Trigger button row: rendered only inside the `md:hidden` mobile layout block
- Drawer components: rendered only inside the `md:hidden` mobile layout block
- Desktop sidebar (`md:grid` layout): completely untouched

If any component is shared between mobile and desktop, use conditional rendering (`{isMobile && <Drawer ...>}`) or render drawer markup conditionally inside the mobile layout branch.
