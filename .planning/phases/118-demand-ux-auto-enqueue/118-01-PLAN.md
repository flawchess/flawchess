## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated request → last_activity middleware | per-request user identity drives the throttled enqueue trigger |
| import worker → eval_jobs table | background task writes tier-2 rows on behalf of a user_id |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-118-01 | Elevation of Privilege | enqueue_tier2_window | mitigate | Guest guard via select(User.is_guest) returns 0 before any insert (QUEUE-08); mirrors existing enqueue_tier1_game guard. Verified by tier2_guest test. |
| T-118-02 | Denial of Service | last_activity trigger flooding tier-2 | mitigate | Existing ≤1-write/hr throttle in the middleware (the create_task sits behind the throttle gate); window capped at TIER2_AUTO_WINDOW_SIZE (200); ON CONFLICT DO NOTHING prevents duplicate rows. A user cannot enqueue more than ~200 rows/hr via activity. |
| T-118-03 | Tampering | bulk pg_insert / tier-3 ORDER BY SQL | mitigate | All values bound via SQLAlchemy params / sa.text with no f-string interpolation (existing security note in eval_queue_service.py header preserved). |
| T-118-04 | Information Disclosure | count_* functions leaking other users' counts | mitigate | Every count query is scoped `user_id == user_id`; no cross-user aggregate. Verified by the per-user count tests. |
| T-118-05 | Denial of Service | in-flight count full table scan at 3s polling | mitigate | ix_eval_jobs_user_active partial index makes the count O(log n); small at steady state (completed/failed excluded). |
| T-118-SC | Tampering | npm/pip/cargo installs | accept | Phase 118 installs zero external dependencies (RESEARCH.md Package Legitimacy Audit: N/A). No new package supply-chain surface. |
</threat_model>
