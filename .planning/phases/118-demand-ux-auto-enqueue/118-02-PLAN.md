| client → GET /imports/eval-coverage | per-user coverage read |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-118-06 | Elevation of Privilege (IDOR) | POST /eval/tier1/{game_id} | mitigate | game.user_id != user.id → 404 (ASVS V4, library.py pattern). Verified by test_tier1_idor. A user can only enqueue games they own. |
| T-118-07 | Elevation of Privilege | guest bypassing analysis gate | mitigate | Both endpoints check user.is_guest server-side and return skipped_guest; enqueue_tier1_game/enqueue_tier2_window also no-op for guests (defense in depth — guest exclusion is NOT merely hidden in the UI). |
| T-118-08 | Denial of Service | tier-2 spam (rapid re-clicks) | mitigate | count_tier2_in_flight gate returns in_flight before re-enqueue; ON CONFLICT DO NOTHING on uq_eval_jobs_game_active. A single user cannot pile multiple tier-2 windows into the shared pool. Verified by test_tier2_in_flight. |
| T-118-09 | Information Disclosure | eval-coverage leaking other users' counts | mitigate | All three counts scoped user.id; no cross-user query path. |
| T-118-10 | Input Validation | non-integer game_id | mitigate | game_id: int path param — FastAPI returns 422 for non-integers (ASVS V5). |
| T-118-SC | Tampering | npm/pip installs | accept | Zero new dependencies this phase. |
