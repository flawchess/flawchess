<objective>
Wire the shared compute path into the live full-eval completion seam.

This plan extends `app/services/eval_apply.py::_classify_and_fill_oracle` to call the Plan 02
`compute_game_accuracy_acpl` on the `positions` list it already loads, and to add the four
canonical accuracy/acpl columns to the existing atomic `UPDATE games` `.values()` (the same
statement that writes oracle counts and commits in the same transaction as the
`full_evals_completed_at` / `full_pv_completed_at` stamps). When the compute returns `None`
(interior hole or 0-move), the four columns are left NULL — the stamp-complete-does-not-imply-
hole-free gate (D-03). Because this is the single shared write body for both the server drain
and the remote atomic-submit path, one insertion covers every completion route; no second hook
is added at import.

Purpose: Every game that finishes full-eval analysis gets its uniform lichess-formula
accuracy/ACPL written atomically, going forward — the live half of D-06.

Output: The hook wiring in `eval_apply.py` + a drain-tick integration test.
</objective>
