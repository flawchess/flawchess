<objective>
Create the cold-lane drain coroutine and its helpers in a new `app/services/eval_drain.py` module — the structural fix for the OOM-kill failure mode.

Purpose: The hot lane (import) holds no Stockfish work, so a separate in-process loop must drain `evals_completed_at IS NULL` games over time. Crash-safe (idempotent on commit failure), pool-friendly (gathers outside any session scope), and bounded (batch=10, sleep=5s when empty).

Output: New module with `run_eval_drain()` + lifted eval-target helpers + pick/load/mark helpers. Tests cover the architectural invariants (gather-outside-session, LIFO, idempotency, engine-None contract).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Drain loop → engine pool | `asyncio.gather` over `engine_service.evaluate` shares the global EnginePool with API traffic; bounded by `_TIMEOUT_S = 2.0` per eval and `SCHED_IDLE` preexec |
| Drain loop → DB | Three short sessions per tick (pick, load, write); each opened from `async_session_maker()` |
| Drain loop ↔ lifespan | `asyncio.CancelledError` propagates on shutdown; no retry |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-91-05 | Denial of Service | Drain saturates Stockfish pool, starves API analysis | accept | RESEARCH R-01: `_TIMEOUT_S=2.0` + `SCHED_IDLE` preexec yields CPU to API requests; worst-case 0.4-1.6 s wait. Future mitigation (`_DRAIN_BATCH_SIZE` tuning) deferred to follow-up. |
| T-91-06 | Tampering / Repudiation | A pathological PGN causes infinite retry of the head of LIFO | mitigate | R-02: engine timeout → `(None, None)` → `_mark_evals_completed` marks the game done. Stuck game leaves NULL eval_cp permanently (operator can manually re-run `scripts/backfill_eval.py`). Bounded by `_DRAIN_BATCH_SIZE` per tick. |
| T-91-07 | Information Disclosure | Drain logs include user game IDs | accept | Game IDs are internal sequence integers, not PII. No usernames or PGN content logged. Sentry context carries `game_ids` list, not message body (`set_context`, not f-string). |
| T-91-08 | Tampering (session leak) | gather inadvertently runs inside open session in future edit | mitigate | Static test `test_gather_outside_session` parses AST/source on every CI run and fails if invariant broken. Documented inline comment immediately above the gather call. |
| T-91-09 | Denial of Service (DB-outage cascade) | Postgres restart mid-tick strands drain | mitigate | `_RETRIABLE_DB_OUTAGE_ERRORS` tuple catches the connection-class exceptions; loop sleeps `_DRAIN_IDLE_SLEEP_SECONDS` and retries. Idempotency invariant: no row stranded — UPDATE either commits all 10 game_id rows or none. |
</threat_model>
