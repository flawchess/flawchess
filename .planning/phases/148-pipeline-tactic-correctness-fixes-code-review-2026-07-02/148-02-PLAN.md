Fix the two eval-pipeline lease/submit correctness defects (D-06 seam: eval-lease =
items 2 + 5).

Item 2: the entry-ply drain stamps `evals_completed_at` even when a dead engine pool
returns `(None, None)` for every position. Mirror the full-ply WR-05 all-fail
breaker: on a non-empty batch with all-`(None,None)` results, do NOT stamp — leave
the lease to expire via its TTL, emit ONE aggregated Sentry event, sleep, continue.
Correct the misleading `EnginePool` docstring in the same pass.

Item 5 (D-05, minimum guard): `entry_submit_eval` stamps the full set leased under a
worker id with no expiry check — add `entry_eval_lease_expiry > now()` to the guard so
a stale/re-leased game is excluded. Do NOT build the fuller echoed-game_ids
intersection (deferred idea).

Purpose: a dead pool no longer permanently marks games complete-but-unevaluated, and
a late/stale submission cannot stamp a game whose lease already expired.
Output: two source fixes + one docstring fix + two new tests.
</objective>
