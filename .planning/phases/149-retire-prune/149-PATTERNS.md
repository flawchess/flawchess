**worker_heartbeats upsert target (already worked out in RESEARCH.md Pattern 1, verified this session against the analog above):**
```python
from sqlalchemy.dialects.postgresql import insert as pg_insert

stmt = pg_insert(WorkerHeartbeat).values(
    worker_id=worker_id, last_ip=last_ip, sf_version=sf_version,
    worker_schema_version=worker_schema_version, last_seen=sa.func.now(),
    submit_count=1, evals_submitted=n_evals,
)
stmt = stmt.on_conflict_do_update(
    index_elements=["worker_id"],
    set_={
        "last_ip": stmt.excluded.last_ip,
        "sf_version": stmt.excluded.sf_version,
        "worker_schema_version": sa.func.coalesce(
            stmt.excluded.worker_schema_version, WorkerHeartbeat.worker_schema_version
        ),
        "last_seen": stmt.excluded.last_seen,
        "submit_count": WorkerHeartbeat.submit_count + stmt.excluded.submit_count,
        "evals_submitted": WorkerHeartbeat.evals_submitted + stmt.excluded.evals_submitted,
    },
)
await write_session.execute(stmt)
```

### `app/routers/imports.py::start_import` (controller, request-response) — durable import guard

**Analog:** `app/services/guest_service.py` lines 162-180 (IntegrityError-as-idempotency)

**Exact existing precedent:**
```python
# Source: app/services/guest_service.py:162-180
# Wrap in try/except IntegrityError to handle double-submit race condition:
# if two promotion requests race, the second INSERT may violate an index;
# treat it as idempotent since the user row is already promoted.
session.add(oauth_account)
try:
    await session.commit()
except IntegrityError:
    await session.rollback()
    # Row already exists from a concurrent request — promotion is idempotent
```

**Current `start_import` shape to modify** (`app/routers/imports.py` lines 45-83) — the in-memory check (`find_active_job`) stays as a cheap pre-check, but the DB row creation must move here, before `create_task`:
```python
user_id = user.id
existing = import_service.find_active_job(user_id, request.platform)
if existing is not None:
    response.status_code = 200
    return ImportStartedResponse(job_id=existing.job_id, status=existing.status.value)

await user_repository.update_platform_username(session, user_id, request.platform, request.username)
await session.commit()

job_id = import_service.create_job(user_id, request.platform, request.username)
asyncio.create_task(import_service.run_import(job_id))
return ImportStartedResponse(job_id=job_id, status="pending")
```

**Target shape (per RESEARCH Pattern 2 / Pitfall 3 — insert now happens here, `_bootstrap_import_job`'s own `create_import_job` call is removed):**
```python
job_id = import_service.create_job(user_id, request.platform, request.username)  # in-memory registry
try:
    await import_job_repository.create_import_job(
        session, job_id=job_id, user_id=user_id, platform=request.platform, username=request.username
    )

except IntegrityError:

    return ImportStartedResponse(job_id=existing_row.id, status=existing_row.status)

asyncio.create_task(import_service.run_import(job_id))
return ImportStartedResponse(job_id=job_id, status="pending")
```

Note `create_import_job` (`app/repositories/import_job_repository.py` lines 21-49) already exists with the right signature/session-flush style — reuse verbatim. `get_active_job_for_user_platform` is new; copy the query style from `create_import_job`'s neighbors in the same file (uses `select(...)`, already imported at top of file).
