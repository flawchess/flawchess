<objective>
Replace `_normalize_chesscom_result`'s silent `return "1/2-1/2"  # safe fallback` (normalization.py:207) with an explicit unknown-result signal so a malformed/unrecognized chess.com outcome surfaces as a skipped game plus a Sentry capture, instead of silently polluting WDL stats with a fabricated draw (PRUNE-03 / R13).

Purpose: a fabricated draw corrupts the position-precise WDL that is the product's core value; skipping + alerting is strictly safer than a wrong stored result.
Output: an edited normalization path routing "unknown" through the existing `None`-skip channel, and a regression test.
</objective>
