<objective>
Add the server-side `worker_heartbeats` fleet-liveness registry (PRUNE-06) and land `worker_schema_version` telemetry (PRUNE-04) as a byproduct of the same upsert. Every live submit lane upserts one row by advisory worker identity, accumulating counts and recording the latest IP + versions. No consumer UI is built — this is raw telemetry that closes the fleet-visibility blind spot and makes the milestone's "fleet fully on atomic" claim a standing SQL query.

Purpose: give Phase 150's write-path consolidation a single already-present heartbeat insertion point, and end the reliance on hourly-rotating access logs for worker identity.
Output: new model, new repository upsert helper, one Alembic migration, wiring into all three live submit handlers, and a dedicated test file.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| remote worker → submit endpoint | operator-token-authenticated worker payload crosses here; X-Worker-Id and sf_version are self-reported |
| Caddy proxy → app | request.client.host resolves the true worker IP via trusted `--proxy-headers --forwarded-allow-ips='*'` |

## STRIDE Threat Register (ASVS L1, block on high)

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-149-01-SP | Spoofing | X-Worker-Id → worker_heartbeats.worker_id | low | accept | Advisory-only (T-123-03); last_ip is the cross-check. Never used for authz. Documented in model docstring. |
| T-149-01-TA | Tampering | Spoofed sf_version / worker_schema_version | low | accept | Telemetry only; server re-classifies authoritatively regardless of reported version (no 426 gate — PRUNE-04/D-10). |
| T-149-01-ID | Info Disclosure | last_ip stores worker IPs | low | accept | Operator-owned machines (local + Hetzner), not end-user IPs; column comment documents negligible GDPR surface (D-06). |
| T-149-01-DoS | Denial of Service | per-submit upsert write amplification | low | mitigate | Single-row upsert-by-PK inside the existing write session; ~7.5/s is trivial; no throttle needed (D-01). |
