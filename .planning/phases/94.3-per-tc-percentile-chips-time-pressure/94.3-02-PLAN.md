<objective>
Add three per-TC pooled-aggregate SQL builder families to `app/services/canonical_slice_sql.py` and widen the `per_user_cte_for` dispatcher with 12 new arms, so both `scripts/gen_global_percentile_cdf.py` (Plan C) and `app/services/user_benchmark_percentiles_service.py` (Plan D) can compute per-(metric × TC) percentile values without ever drifting. Add 12 new unit tests covering SQL shape + per-TC predicate + metric-specific floor and 12 new parity assertions confirming benchmark and single-user paths agree on the pooled body.

Purpose: Drift between CDF construction and per-user lookup is the central correctness risk (RESEARCH §Pitfall 2). Locating the per-TC restriction inside the shared builder — not in the script or service layer — makes drift structurally impossible. This plan is the first commit of the Wave-1 atomic-cutover trio (B → C → D); the byte-identical regression test in `tests/scripts/test_gen_global_percentile_cdf_unchanged.py` is EXPECTED RED between this plan and Plan C and stays red until Plan C regenerates the goldens.

Output: 3 new builder functions + 1 new helper CTE function + 12 new dispatcher arms + 4 new module-level constants + 24 new test cases. No consumer touches in this plan (Plans C and D do that).
</objective>
