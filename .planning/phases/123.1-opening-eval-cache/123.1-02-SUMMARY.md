## What Was Built

**`_fetch_dedup_evals` rewrite (Task 1)**: The function body was replaced with a single `select(OpeningPositionEval).where(full_hash.in_(full_hashes))` PK probe. Signature `(session, full_hashes) -> dict[int, tuple[int|None, int|None, str|None]]` and the `if not full_hashes: return {}` guard are byte-identical to before. The docstring was updated to document the SEED-053 / D-123.1-05 cache swap and that all read-side guards remain in the caller. The now-unused `sqlalchemy.orm.aliased` import was removed (ruff F401).

**`_upsert_opening_cache` helper (Task 2)**: New 55-line helper extracted before `_full_drain_tick` to keep the tick function within CLAUDE.md LOC limits. Builds `cache_rows` from `engine_targets` filtered to `ply <= _DEDUP_MAX_PLY and not is_terminal` with a real eval in `engine_result_map`. Emits one batched `INSERT INTO opening_position_eval ... ON CONFLICT (full_hash) DO NOTHING` using the module's CAST() / asyncpg-compatible VALUES convention. No-op when `cache_rows` is empty. Called inside the Step-4 write session after `_classify_and_fill_oracle`, so cache write + eval write commit atomically.

**Three new tests (Task 3)**: Added `TestOpeningEvalCacheRead` and `TestOpeningEvalCacheWrite` classes:

- `test_cache_backed_fetch_dedup_evals`: seeds `opening_position_eval` with known values, calls `_fetch_dedup_evals`, asserts exact (eval_cp, eval_mate, best_move) match + uncached hash absent.
- `test_empty_hash_list_returns_empty_dict`: verifies the early-return guard works.
- `test_write_population_includes_and_excludes`: constructs `_FullPlyEvalTarget` instances covering all exclusion cases (null eval, ply > DEDUP_MAX_PLY, is_terminal=True), runs `_upsert_opening_cache`, verifies correct inclusions and exclusions.
- `test_idempotency_first_write_wins`: pre-seeds hash A, runs upsert with a conflicting value, verifies original row unchanged (ON CONFLICT DO NOTHING semantics).

All 18 tests in `test_eval_drain.py` pass (15 pre-existing + 3 new).
