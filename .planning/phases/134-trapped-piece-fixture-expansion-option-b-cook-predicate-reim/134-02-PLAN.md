<objective>
Reimplement `detect_trapped_piece` in `app/services/tactic_detector.py` from cook's `util.is_trapped` predicate (PROSE ONLY — AGPL boundary, copy no source), anchoring the firing condition to the solution capture chain like cook's `trapped_piece` driver, and iterate it against the Plan-01 expanded fixture using `scripts/tactic_tagger_report.py --check-goals` / the `/loop` until post-dispatch precision is maximized.

Purpose: the current detector fires 0 TP / 153 FP (P 0.000) — the single biggest false-positive source. RESEARCH §"Current detect_trapped_piece — why it fires 0 TP / 153 FP" identifies the dominant cause: the detector scans EVERY opponent non-pawn piece on EVERY pov-result board and invents trapped judgments on pieces nowhere in the solution's capture chain. Cook's `trapped_piece` is far narrower — it only fires when the player actually captured a non-pawn piece in the line AND that piece was `is_trapped` on the board before the preceding opponent move. Re-anchoring to the capture chain plus mirroring cook's `is_trapped` is the precision lever.

Output:
- A rewritten `detect_trapped_piece` (same signature, same return tuple, same `TacticMotifInt.TRAPPED_PIECE`, same Tier 2 rank 6 dispatch slot) that reuses the existing faithful ports `_is_in_bad_spot` / `_is_defended` / `_piece_value` rather than hand-rolling new geometry.
- A `GOALS` entry for `trapped-piece` in `scripts/tactic_tagger_report.py` (precision ~0.80, recall None) so `--check-goals` can drive the loop.
- A recorded post-dispatch TRAIN+TEST precision/recall measurement on the expanded fixture (this number is the input to Plan 03's ship/suppress decision per D-EXP-03).

This plan does NOT change SUPPRESSED_MOTIFS, add a PRECISION_FLOOR, or touch the family map / frontend — all of that is Plan 03, gated on the precision this plan measures.
