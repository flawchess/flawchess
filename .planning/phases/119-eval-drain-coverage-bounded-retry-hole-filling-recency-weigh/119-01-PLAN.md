<objective>
Implement SEED-045 bounded-retry hole-filling and create the single phase-wide Alembic migration that all three plans share.

Today `_mark_full_evals_completed` (eval_drain.py:439) stamps `full_evals_completed_at` UNCONDITIONALLY (D-116-07), so a game with a transient mid-game engine hole (eval_cp NULL AND eval_mate NULL on a non-terminal ply) is permanently marked "fully analyzed" with a gap. With Phase 118's tier-1 "Analyze" button, users now trigger this path directly and see the gaps. Stop stamping while genuine holes remain; re-pick the game up to MAX_EVAL_ATTEMPTS; after the cap, stamp anyway plus ONE aggregated Sentry event.

This plan also owns the migration for the WHOLE phase (column add + the SEED-046 partial index + dropping the dead ix_eval_jobs_user_active) so the schema lands in one coherent revision with correct head ordering. Plans 02 and 03 depend on this migration but do not edit it.

Purpose: "Analyze" must mean every position is analyzed; keep D-116-07's infinite-re-pick protection (a deterministically-unevaluable ply cannot loop forever).
Output: full_eval_attempts column + migration, hole-aware completion gate, MAX_EVAL_ATTEMPTS constant, backfill sweep for already-stamped games, tests.
</objective>
