## Interaction Contract

### Collapsed Sidebar Strip

- Always visible on `md:` breakpoint and above. Hidden on mobile (`md:hidden` guards the strip).
- Width: 48px, full height of the content area (`h-full`), `flex flex-col items-center` with `py-2 gap-1`.
- Background: `bg-sidebar-bg` (`#171513`) with `charcoal-texture` class.
- Two icon buttons: `SlidersHorizontal` (filters) and `BookMarked` (bookmarks).
- Button variant: `ghost` size `icon` for inactive state; active icon (panel open) uses `brand-outline` variant.
- Tooltip on each icon button: "Filters" and "Bookmarks" â€” `side="right"` (using existing `Tooltip` component from `@/components/ui/tooltip`).
- `aria-label` on each button: "Open filters" / "Open bookmarks".
- `data-testid`: `sidebar-strip-btn-filters` / `sidebar-strip-btn-bookmarks`.
- Notification dot placement: top-right corner of the icon button (same pulsing dot pattern as current tab implementation).

### Open Panel

- Clicking an icon in the collapsed strip opens the respective panel directly (D-03). No intermediate "expanded but empty" state.
- Clicking the currently-active icon closes the panel (toggle behavior). Panel state: `null | 'filters' | 'bookmarks'`.
- Clicking the other icon while a panel is open switches panels without closing first (D-04).
- Clicking outside the open panel (on the board or content area) closes the panel. Implement via `useEffect` with `mousedown` listener or a transparent backdrop div.

### Overlay vs Push

- **Below 1280px (`xl`):** Open panel uses `position: absolute`, `z-index: 40`, `left: 48px` (directly right of the strip), overlaying the board. Board and content area width remain unchanged.
- **At 1280px and above:** Open panel is in-flow. Layout becomes 3-column: `48px strip | 280px panel | 1fr content`. Board area shrinks but does not overflow (sufficient space at xl+).
- Panel open/close animation: `transition-[width,opacity] duration-200 ease-in-out`. Collapsed width = 0 (with `overflow-hidden`), open width = 280px. Avoids layout shift below xl (absolute position) and provides smooth push above xl.

### Filter Live Updates

Filters in the sidebar panel apply live (D-05). No "Apply" button. Filter state flows through existing `useFilterStore` + `useDebounce(filters, 300)` unchanged. The panel simply renders `<FilterPanel>` directly.

### Board and Content Area

The board, `BoardControls`, opening name display, and `MoveList` all move from the `sidebar` variable into the main content area. The main content area layout on desktop becomes:

```
[ChessBoard + BoardControls + OpeningName + MoveList]
[Tabs: Moves / Games / Stats]
```

These two blocks stack vertically (single column) within the `1fr` right region, or use a sub-grid if space permits. The exact sub-layout within the content area is at executor discretion â€” the existing `moveExplorerContent`, `gamesContent`, and `statisticsContent` variables are reused unchanged.
