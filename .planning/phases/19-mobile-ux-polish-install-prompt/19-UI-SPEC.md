## Spacing Scale

Standard 8-point scale. All values are multiples of 4.

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1`, `gap-1` | Icon gaps, badge padding |
| sm | 8px | `p-2`, `gap-2` | Compact element spacing, filter button internal padding |
| md | 16px | `p-4`, `gap-4` | Default section/card padding |
| lg | 24px | `p-6`, `gap-6` | Section padding on desktop |
| xl | 32px | `p-8`, `gap-8` | Layout gaps |
| 2xl | 48px | `p-12` | Major section breaks |
| 3xl | 64px | `p-16` | Page-level spacing |

**Phase-specific exceptions:**

| Element | Value | Reason |
|---------|-------|--------|
| Touch target minimum height | 44px (`min-h-11`) | Apple HIG + WCAG 2.5.5 — mobile only; reset with `sm:min-h-0` on desktop |
| Sticky board vertical padding | 8px bottom (`pb-2`) | Visual breathing room below board before board controls; from RESEARCH.md Pattern 2 |
| InfoPopover trigger (info icon button) | 14px icon (`h-3.5 w-3.5`) | Established in info-popover.tsx; not a primary action — size acceptable |

## Typography

Font: Geist Variable. No custom font sizes defined — uses Tailwind default scale mapped to shadcn semantic classes. The project uses Tailwind CSS 4 with `@theme inline` (no tailwind.config file).

| Role | Size | Weight | Line Height | Tailwind / Class |
|------|------|--------|-------------|-----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` (default for most UI text) |
| Label / UI text | 12px | 400 (regular) | 1.4 | `text-xs` (filter labels, badges, table cells) |
| Subheading | 16px | 600 (semibold) | 1.4 | `text-base font-semibold` (collapsible section triggers, tab labels) |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold` (page section headings) |

**Weights in use: regular (400) and semibold (600) only.**

Phase 19 introduces no new typographic roles. The compact move list uses `text-xs` (12px) — this is intentional for density; MoveList move buttons are excluded from the 44px touch target requirement (CONTEXT.md: "keep as-is since they're not primary controls").

## Color

All colors use CSS custom properties from `index.css`. Values shown are light mode / dark mode.

| Role | CSS Variable | Light Mode | Dark Mode | Usage |
|------|-------------|------------|-----------|-------|
| Dominant (60%) | `--background` | `oklch(1 0 0)` (white) | `oklch(0.145 0 0)` (near-black) | Page background, body |
| Secondary (30%) | `--card` / `--secondary` | `oklch(1 0 0)` / `oklch(0.97 0 0)` | `oklch(0.205 0 0)` / `oklch(0.269 0 0)` | Cards, filter panels, collapsible sections, drawer backgrounds |
| Accent (10%) | `--primary` | `oklch(0.205 0 0)` (near-black) | `oklch(0.922 0 0)` (near-white) | Reserved list below |
| Destructive | `--destructive` | `oklch(0.577 0.245 27.325)` (red) | `oklch(0.704 0.191 22.216)` (light red) | Destructive actions only |

**Accent (`--primary`) reserved for:**
- Primary CTA buttons (filled `variant="default"`)
- Active state on ToggleGroupItem (selected filter)
- Active bottom nav tab indicator
- Sticky board wrapper background (`bg-background` — keeps board visually grounded at top)

**Phase 19 specific:**
- Sticky board wrapper: `bg-background` — matches page background so board appears flush at top, not as floating card
- Collapsible trigger (filters, bookmarks): `variant="ghost"` button — uses `--muted` on hover, no accent
- iOS install banner (PWA-05): `--card` background with `--border` border — subtle, not accent-colored; dismissal `×` uses `variant="ghost"`
- Android install prompt (PWA-04): shadcn `Dialog` or bottom `Drawer` — uses standard card/background tokens

**Chess-specific color (not a CSS variable):**
- Selected piece highlight: `rgba(255, 255, 0, 0.5)` — yellow overlay on selected square (existing logic, CONTEXT.md locked decision)
- Board arrows: green (win), red (loss), grey (draw/neutral) — existing pattern

## Interaction Contract

### Touch Targets (UX-01)

All interactive elements on mobile must meet 44x44px minimum. Pattern from RESEARCH.md:

```
min-h-11 sm:min-h-0        — height-only fix (for inline/wrapping elements)
min-h-11 sm:min-h-8 py-2 sm:py-0.5  — with vertical padding reset on desktop
h-11 w-11 sm:h-8 sm:w-8   — square icon buttons
```

Known violations to fix (RESEARCH.md Pattern 3):

| Element | Current | Fix Class |
|---------|---------|-----------|
| FilterPanel time/platform toggle buttons | `py-0.5` (~24px) | `min-h-11 sm:min-h-0 py-2 sm:py-0.5` |
| Board control icon buttons (`size="icon"`) | `size-8` (32px) | `h-11 w-11 sm:h-8 sm:w-8` on mobile breakpoint |
| `ToggleGroupItem size="sm"` | ~28px | `min-h-11 sm:min-h-0` |
| InfoPopover trigger | `h-3.5 w-3.5` (14px icon) | Not a primary control — acceptable; tap area may need `p-1` wrapper |

### Chessboard Touch (UX-03)

- **Drag-and-drop:** Test react-chessboard v5.10.0 touch drag on physical device first. If black screen occurs, set `allowDragging: false` in options.
- **Click-to-move:** Uses `onTouchEnd` in library. Two taps: first tap selects piece (yellow highlight `rgba(255, 255, 0, 0.5)`), second tap on destination moves it.
- **No legal move dot indicators** — CONTEXT.md locked decision.
- **Fallback sequence:** (1) Native touch drag via @dnd-kit TouchSensor, (2) `allowDragging: false` + click-to-move only.

### Openings Page Mobile Layout (UX-04)

Mobile single-column order (`md:hidden` section):

1. Sticky board wrapper: `sticky top-0 z-10 bg-background pb-2`
2. Board controls (flip, reset, etc.)
3. Compact move list (`text-xs`, fixed height `h-18` approximately)
4. `Collapsible` — "More filters" trigger + FilterPanel (default: closed, state: `mobileFiltersOpen` initialized `false`)
5. `Collapsible` — "Position bookmarks" + bookmark list + Save/Suggestions buttons (collapsed by default)
6. `Tabs` — Moves | Games | Statistics with scrollable tab content

MobileHeader hidden on `/openings/*` routes via `useLocation` in `ProtectedLayout` (CONTEXT.md + RESEARCH.md Pattern 1).

Desktop layout (`md:grid md:grid-cols-[350px_1fr]`) is unchanged.

### Horizontal Overflow (UX-02)

Baseline: 375px viewport width (iPhone SE). No horizontal scrollbar on any page.

| Page / Component | Fix |
|-----------------|-----|
| `body` / `html` | Add `overflow-x: hidden` to `body` in `index.css` as global safety net |
| GlobalStats `p-6` | Reduce to `px-4` on mobile: `px-4 sm:px-6` |
| MoveExplorer table | Wrap in `overflow-x-auto` if investigation shows overflow |
| ChessBoard | Already responsive via `ResizeObserver` — no change needed |

### Install Prompts

**Android (PWA-04):**
- Trigger: after user has imported games (meaningful engagement signal)
- UI: bottom `Drawer` (vaul) with "Install Chessalytics" heading, brief value prop, "Install" button (`variant="default"`)
- Dismiss: "Not now" link (`variant="ghost"`) — persisted to `localStorage` key `install-prompt-dismissed`
- Re-appearance: never after dismissal

**iOS (PWA-05):**
- Trigger: iOS Safari detected (`/iPad|iPhone|iPod/.test(navigator.userAgent)`) + not standalone (`display-mode: standalone`) + not dismissed
- UI: fixed bottom banner (above bottom nav bar), `bg-card border-t border-border` styling
- Copy: "Add to Home Screen: tap Share then 'Add to Home Screen'" with Share icon (Lucide `Share`)
- Dismiss: `×` icon button (`variant="ghost"`, `aria-label="Dismiss install banner"`) — persisted to `localStorage` key `ios-install-banner-dismissed`
