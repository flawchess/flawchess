# Phase 94.4 Plan 07: Frontend Chip Rewrite (Peer-Relative + Anchor Disclosure) Summary

## One-liner

Rewrites PercentileChip from the post-94.3 "Top X%" + per-flavor rating-correlation copy into the peer-relative `p23` chip face + 4-bullet anchor-disclosure tooltip, collapses the flavor enum 16 ‚Üí 8, rescues the Recovery chip per D-05a, and locks a no-flame regression guard against commit 6766898c.

## What was built

This is the **only frontend plan** in Phase 94.4 and the **terminal Wave 3 deliverable** of the milestone-equivalent atomic-cutover effort started in Plans 03 ‚Üí 04 ‚Üí 05 ‚Üí 06 (Wave 2). The frontend tolerates missing-percentile cells (chip gates on `!= null`), so it lands cleanly alongside or after Wave 2 in a single deploy.

### 1. PercentileChip.tsx ‚Äî full rewrite

**Chip face (CONTEXT D-06):** the bare `p${rounded}` form. `formatBarePercentile(pct)` clamps to `[MIN_PERCENT=1, MAX_PERCENT=99]`. NO direction word ‚Äî color band (red < p25 / blue p25-p75 / green > p75) carries direction. Single-branch `deriveBandColor` (all 8 flavors `higher_is_better` per CONTEXT D-07a). `min-w-[3rem]` keeps the shorter chip visually balanced.

**aria-label (CONTEXT D-06b):** preserves a direction word for screen readers:
```
`${metricLabel} percentile: p${rounded}${tc ? ` in ${tc}` : ''}, ${rounded < 50 ? 'bottom' : 'top'} of cohort`
```

**Popover body (CONTEXT D-07):** 4 bullets per chip via a new `PercentileChipPopoverBody({ tc, anchorRating, anchorSource, chesscomRawRating })`:

1. **Cohort framing.** Per-TC: `"Compared to other ~${anchor}-rated players in ${tc}."`. Aggregated: `"Compared to other ~${anchor}-rated players, aggregated across the time controls you play."` (CONTEXT D-07b).
2. **Recent-games basis.** Per-TC: `"Based on your most recent 1000 rated games in ${tc} over the last 36 months, vs opponents within +/-100 Elo."`. Aggregated: `"... most recent 1000 rated games per time control over the last 36 months, vs opponents within +/-100 Elo."`.
3. **Filter independence.** `COPY_FILTER_INDEPENDENCE = 'UI filters do not affect this percentile.'` kept verbatim from the post-94.3 module.
4. **Rating-anchor disclosure.** Lichess-anchored: `"Anchored on your Lichess ${tc} (${anchor})."`. chess.com-anchored WITH `chesscomRawRating`: `"Anchored on your chess.com ${tc} (${rawRating} -> ${anchor} Lichess-equivalent via ChessGoals snapshot 2026-05-26)."`. chess.com-anchored WITHOUT raw rating: simpler form `"Anchored on your chess.com ${tc} (${anchor})."`.

**Retired:** `RATING_NOTE_BY_FLAVOR`, all 12 `COPY_RATING_NOTE_*` constants, `DIRECTION_BY_FLAVOR`, `PercentileChipDirection`, `formatTopXPercent`, `formatBottomOrTopPhrase`, `PERCENTILE_METRIC_LABELS`, `recentGamesBasisFor`, `tcFromFlavor` (Phase 94.3 helpers). ### 5. pickDominantTcAnchor helper (new file)

`frontend/src/lib/percentileAnchor.ts`:

```ts
export type RatingAnchorsByTc = Partial<Record<TimeControlBucket, RatingAnchorOut>>;

export function pickDominantTcAnchor(
  anchors: RatingAnchorsByTc,
): RatingAnchorOut | undefined {
  const tcOrder: ReadonlyArray<TimeControlBucket> = ['bullet', 'blitz', 'rapid', 'classical'];
  let best: RatingAnchorOut | undefined = undefined;
  for (const tc of tcOrder) {
    const candidate = anchors[tc];
    if (candidate === undefined) continue;
    if (best === undefined || candidate.n_games > best.n_games) {
      best = candidate;
    }
  }
  return best;
}
```

Returns the highest-game-count anchor (deterministic tie-break by TC priority order). Returns `undefined` when no anchors are available ‚Äî caller suppresses the chip.
