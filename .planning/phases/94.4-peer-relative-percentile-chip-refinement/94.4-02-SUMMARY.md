# Phase 94.4 Plan 02: User Rating Anchors + Median-Anchor SQL Builder Summary

Landed the `user_rating_anchors` table, ORM model, repository, Alembic migration, and the `per_user_cte_median_anchor` SQL builder — the storage primitive plus the SQL primitive that Plan 04 (CDF generation) and Plan 05 (single-user Stage A wiring) both consume.

## What Shipped

- **Postgres table `user_rating_anchors`** with composite PK `(user_id, time_control_bucket)`, FK ON DELETE CASCADE to `users.id`, nullable `chesscom_raw_rating` column for D-07 bullet 4 tooltip provenance, and server-default NOW() `computed_at`.
- **Postgres ENUM `anchor_source`** with values `lichess` / `chesscom`, an implicit `varchar → anchor_source` CAST for asyncpg, and three-step Alembic upgrade/downgrade.
- **SQLAlchemy model** `UserRatingAnchor` in `app/models/user_rating_anchors.py` cross-importing the existing `time_control_bucket_enum` (Postgres type name: `timecontrolbucket`, not `time_control_bucket` — corrected after first migration attempt).
- **Repository helpers** `upsert_anchor` + `fetch_anchors_for_user` with V4-safe keyword-only `user_id` and a frozen `RatingAnchorRow` dataclass.
- **SQL builder** `per_user_cte_median_anchor(tc, *, source, snapshot_date, platform, min_games)` emitting `per_user_anchor(user_id, anchor_rating INT, n_games INT)` over the recent-1000-per-TC × 36-month pool, with `percentile_cont(0.5) WITHIN GROUP (ORDER BY <user-side rating>)::int` and the Daily-classical drop baked in.
- **Module-level constant** `MEDIAN_ANCHOR_MIN_GAMES: Final[int] = 30` (CLAUDE.md no-magic-numbers, default `min_games`).

## Generated SQL Sample (Daily-classical drop verified)

```sql
recent_capped_no_daily AS (
  SELECT rc.*
  FROM recent_capped rc
  JOIN games g ON g.id = rc.id
  WHERE NOT (g.platform = 'chess.com' AND g.time_control_str LIKE '1/%')
    AND g.platform = 'lichess'
),
per_user_anchor AS (
  SELECT
    rc.user_id,
    percentile_cont(0.5) WITHIN GROUP (ORDER BY (CASE WHEN g.user_color::text='white' THEN g.white_rating ELSE g.black_rating END))::int AS anchor_rating,
    count(*) AS n_games
  FROM recent_capped_no_daily rc
  JOIN games g ON g.id = rc.id
  GROUP BY rc.user_id
  HAVING count(*) >= 30
)
```

The `WHERE NOT (g.platform = 'chess.com' AND g.time_control_str LIKE '1/%')` substring is present in all 4 TC outputs (test `test_daily_classical_drop_present_for_all_tcs` parametrized over `bullet/blitz/rapid/classical`).
