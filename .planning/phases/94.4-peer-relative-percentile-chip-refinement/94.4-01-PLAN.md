<objective>
Ship `app/services/chesscom_to_lichess.py` as a NEW pure-Python lookup-table service exposing the ChessGoals rating-conversion snapshot (per D-02 SC-2 + D-14). The module ships all three tables verbatim from RESEARCH Pattern 8b: Table 1 (chess.com Blitz → Bullet/Rapid + USCF + FIDE), Table 2 (chess.com Blitz → Lichess per-TC), Table 3 (Lichess Blitz → Lichess Bullet/Rapid/Classical + USCF + FIDE). The consumed surface in Phase 94.4 is `convert_chesscom_to_lichess(rating, source_tc, target_tc) -> int | None` — called by Plan 05's Stage A anchor compute when a user's per-TC anchor source is `chesscom`. USCF + FIDE accessors ship as module-level data + minimal typed accessors per D-14; no Phase 94.4 caller consumes them.

Purpose: No principled closed-form chess.com↔Lichess conversion exists (chess.com is rating-deflated relative to Lichess by 100-300 Elo depending on TC). A hardcoded snapshot of the ChessGoals 2025 calibration table is the simplest honest pivot. Shipping USCF + FIDE columns alongside (D-14) avoids re-extracting the snapshot when future surfaces (federation cohorts, opponent normalisation, scout view) need it.

Output: NEW Python module + comprehensive pytest coverage. Independent of Plan 02 (anchor table) — both Wave 1 plans are parallel-safe. No call sites are added in this plan; Plan 05 (Wave 2) wires Stage A to call `convert_chesscom_to_lichess`.
</objective>
