The 4 Phase 94.2 ΔES ENUM values become per-TC. Every metric family stores up to 4 rows per user.

## Per-TC aggregation weight (for page-level ΔES chips)

| Option | Description | Selected |
|--------|-------------|----------|
| Weight by per-metric floor N | Each metric weights per-TC sub-percentiles by its own inclusion-floor denominator. Methodologically strict. | |
| Weight by raw game count uniformly | Every metric weights by raw game count regardless of denominator. Simpler; biases toward TCs the user plays most volume in. | ✓ |
| Display per-TC sub-percentiles directly | Skip aggregation; each ΔES row renders up to 4 mini-chips. ~32 chip surfaces total. Visually heavier. | |
| Empirical pass first | Researcher compares weighting schemes during planning. | |

**User's choice:** Weight by raw game count uniformly.

**Notes:** Simpler implementation, uniform across metrics, and the bias toward TCs the user plays most volume in matches realistic user expectation ("my main TC dominates my headline chip"). Aggregation lives in the backend API shaper (per-TC rows stored as-is; weighted mean applied at response shape time).

**Captured as:** D-08, D-08a, D-08b in CONTEXT.md.
