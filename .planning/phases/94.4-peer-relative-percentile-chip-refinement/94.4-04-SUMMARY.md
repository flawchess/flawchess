# Phase 94.4 Plan 04: Cohort CDF Artifact + Regen Script Summary

One-liner: Retire the Phase 94.3 flat `GLOBAL_PERCENTILE_CDF` (16-key, pooled across all anchors) for the cohort `COHORT_PERCENTILE_CDF` (8 metrics × per-(anchor, TC) sliding-window CDFs); ship the populated registry against the benchmark DB with K=200/±150-Elo strict defaults; Plan 05 finishes the cutover at every call site.

## What Was Built

### `app/services/global_percentile_cdf.py` — rewritten registry module

- **CdfMetricId** Literal collapses 16 → 8 (CONTEXT D-13). New value set: `score_gap`, `achievable_score_gap`, `section2_score_gap_conv`, `section2_score_gap_parity`, `recovery_score_gap`, `time_pressure_score_gap`, `clock_gap`, `net_flag_rate`. The 12 Phase 94.3 TC-suffixed entries (`time_pressure_score_gap_bullet`, `clock_gap_*`, `net_flag_rate_*`) retire entirely; TC is now an outer key on the registry, not a metric suffix.
- **COHORT_PERCENTILE_CDF** replaces `GLOBAL_PERCENTILE_CDF`. Shape: `Mapping[CdfMetricId, Mapping[tuple[int, TimeControlBucket], CdfTable]]`. The Task 3 regen wrote 98 non-suppressed cells across the 8 × 33 × 4 = 1,056 candidate grid.
- **interpolate_cohort_percentile(metric, value, anchor, tc) -> float | None** replaces `interpolate_percentile(metric, value)`. Anchor rounds to nearest `COHORT_ANCHOR_STEP_ELO` (50-Elo grid). Returns None when `(metric, rounded_anchor, tc)` is absent from the registry (suppressed cell → frontend chip suppresses).
- **interpolate_percentile** (2-arg) survives as a `DeprecationWarning`-emitting stub returning None unconditionally. Plan 05 deletes the stub.
- **TimeControlBucket** is duplicated locally to break a circular import with `canonical_slice_sql` (which imports `CdfMetricId` from this module).

### `scripts/gen_global_percentile_cdf.py` — rewritten regen script

- **5 module-level constants** driving the strict-default sliding-window cohort policy (CONTEXT D-11). One-line tunable; operator may relax K or window after reviewing the regen report.
- **`_build_per_user_with_anchor_query(metric, tc, *, snapshot_date)`** emits a JOIN of the Plan 03 per-TC `per_user_values` CTE × Plan 02 `per_user_cte_median_anchor` per_user_anchor CTE, returning `(user_id, metric_value, n_games, anchor_rating)`. A CTE-name-collision fix renames the anchor block's `recent_capped` to `recent_capped_anchor` (both builders inject the same name via `_recent_capped_per_tc_cte`).
- **`_build_cohort_cdf_for(rows, metric, tc)`** is the in-Python ranking loop: sort by `(|anchor_rating − anchor|, user_id)` (deterministic tiebreaker — REQUIRED for byte-identical regen), take K=200, suppress if `< K` or k-th distance `> 150` Elo, otherwise compute the 99-breakpoint CDF via the pure-Python `_percentile_linear` helper (numpy is not a project dependency).
- **`regenerate(target, snapshot_date, report_path)`** orchestrates the 3-level loop (8 metrics × 4 TCs × 33 anchors → 32 SQL queries + 1,056 Python rank sweeps), overwrites the `COHORT_PERCENTILE_CDF` block between `# --- BEGIN GENERATED REGISTRY ---` / `# --- END GENERATED REGISTRY ---` sentinels, and writes the regen report.
- **CLI** preserves the 94.1 Plan 09 safety guard (`--target benchmark` + `_assert_benchmark_db` requiring both `flawchess_benchmark` and `:5433` in the resolved URL). Adds `--snapshot-date YYYY-MM-DD` (default today UTC) and `--regen-report-path PATH` (default `reports/cohort-percentile-cdf-latest.md`).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Circular import] TimeControlBucket inlined in global_percentile_cdf.py**

- **Found during:** Task 1 (importing module via `python -c`)
- **Issue:** `canonical_slice_sql.py` imports `CdfMetricId` from `global_percentile_cdf`; trying to import `TimeControlBucket` from `canonical_slice_sql` inside `global_percentile_cdf` triggers `ImportError: cannot import name 'CdfMetricId' from partially initialized module`.
- **Fix:** Duplicate the 4-value Literal locally in `global_percentile_cdf.py` with a comment explaining the cycle break. The Literal value set is stable across the codebase.
- **Files modified:** `app/services/global_percentile_cdf.py`
- **Commit:** `f1b2f752`

**2. [Rule 1 - Bug] CTE name collision in _build_per_user_with_anchor_query**

- **Found during:** Task 3 (first regen run against benchmark DB)
- **Issue:** Both the per-TC metric builder (`per_user_cte_score_gap_tc` and siblings) and `per_user_cte_median_anchor` emit a top-level `recent_capped` CTE via their shared `_recent_capped_per_tc_cte` dependency. Joining them in a single WITH chain triggers `asyncpg.exceptions.DuplicateAliasError: WITH query name "recent_capped" specified more than once`.
- **Fix:** String-rename the anchor block's local CTEs to a unique prefix (`recent_capped_anchor` / `recent_capped_anchor_no_daily`) in the gen script's `_build_per_user_with_anchor_query`. The SQL builders remain pure and independently callable — the rename is gen-script-local, with an inline comment documenting why.
- **Files modified:** `scripts/gen_global_percentile_cdf.py`
- **Commit:** `ed0beb86`

**3. [Rule 3 - Missing dep] numpy not a project dependency**

- **Found during:** Task 2 (ty check)
- **Issue:** The plan's RESEARCH blueprint referenced `numpy.percentile`, but numpy is NOT in `pyproject.toml` (verified). Adding numpy as a dependency for a small CDF computation is heavy-handed.
- **Fix:** Write `_percentile_linear` in pure Python, matching numpy's default `'linear'` method by construction. Determinism is byte-identical given identical input. No new package added.
- **Files modified:** `scripts/gen_global_percentile_cdf.py`
- **Commit:** `63c9cd74`

**4. [Rule 1 - Report formatting bug] Per-metric coverage bullets collapsed onto one line**

- **Found during:** Task 4 (visual inspection of report)
- **Issue:** `_render_regen_report` joined the per-metric coverage bullets with `"".join(...)` instead of `"\n".join(...)`, producing 8 bullets on a single line.
- **Fix:** Swap to `"\n".join(...)` in the helper. Apply the same fix inline to the just-generated report (re-running the 19-minute regen for a markdown-only formatting fix would be wasteful).
- **Files modified:** `scripts/gen_global_percentile_cdf.py`, `reports/cohort-percentile-cdf-latest.md`
- **Commit:** `f7df17ad`
