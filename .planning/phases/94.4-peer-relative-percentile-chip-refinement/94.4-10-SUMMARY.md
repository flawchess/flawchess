## What Was Built

### SQL Builder (`canonical_slice_sql.py`)

Added `_chesscom_conversion_values_sql(target_tc: TimeControlBucket) -> str` helper that generates the raw `VALUES (anchor_int, lichess_equiv_int), ...` body from `CHESSCOM_BLITZ_TO_LICHESS`.

Rewrote `per_user_cte_median_anchor` to accept `blend: bool = False`. When `blend=True`:
- Emits `chesscom_conversion_lookup(chesscom_anchor, lichess_equiv) AS (VALUES ...)` CTE generated from the snapshot
- Emits `per_game_blended_rating` CTE via LATERAL nearest-anchor join for chess.com games (non-Daily) and passthrough for lichess games, combined via `UNION ALL`
- Emits 6-column `per_user_anchor(user_id, anchor_rating, n_chesscom_games, n_lichess_games, chesscom_median_native, lichess_median_native)`
- `blend=True` is mutually exclusive with `platform!=None` (raises `ValueError` at entry)

When `blend=False`, output is byte-for-byte unchanged (regression-guarded by Task 1.0 baseline constant in tests).

### Service Rewrite (`user_benchmark_percentiles_service.py`)

Deleted: `_user_has_lichess_games_in_tc`, `_compute_median_anchor_for_platform`, `_CHESSCOM_SOURCE_TC_BY_BUCKET`.

Dropped imports: `AnchorSource` (removed in Plan 09), `ChessComSourceTC`, `convert_chesscom_to_lichess`.

Rewrote `compute_anchors_for_user` to issue a single SQL call per TC with `blend=True`, parse the 5-column result, and persist via `upsert_anchor` with the blended schema kwargs.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Incorrect PostgreSQL VALUES CTE syntax**
- **Found during:** Task 1 real-DB test execution (syntax error)
- **Issue:** `_chesscom_conversion_values_sql` initially returned `(VALUES ...) AS chesscom_conversion_lookup(chesscom_anchor, lichess_equiv)` — aliased-expression form. PostgreSQL requires the column-declaration form `name(col1, col2) AS (VALUES ...)` inside a `WITH` clause. The aliased form is only valid in `FROM` clauses.
- **Fix:** Changed helper to return only the raw `VALUES (v1, v2), ...` body. The CTE head `chesscom_conversion_lookup(chesscom_anchor, lichess_equiv) AS (...)` is emitted by the caller `_per_user_cte_median_anchor_blended`.
- **Files modified:** app/services/canonical_slice_sql.py
- **Commit:** ec8a43a4

**2. [Rule 3 - Blocking] Wave 1 broken-state import (Plan 09 intentional left-break)**
- **Found during:** Task 2 (pytest collection failed before Task 3 ran)
- **Issue:** `user_benchmark_percentiles_service.py` imported `AnchorSource` which Plan 09 removed from the ORM. pytest conftest failed collection, blocking all tests.
- **Fix:** Task 3 (service rewrite) was executed before Task 2 tests, eliminating the import. This is the intended Plan 10 fix — documented as ordering adjustment, not unexpected deviation.
- **Files modified:** app/services/user_benchmark_percentiles_service.py
- **Commit:** 933103ac

**3. [Rule 1 - Bug] B3 test KeyError for anchor rating 1830**
- **Found during:** Task 4 test execution
- **Issue:** `_USER_RATING_BULLET` was set to 1830 but `CHESSCOM_BLITZ_TO_LICHESS` has no key 1830 (nearest keys are 1800 and 1850). The nearest-anchor LATERAL join would have returned an ambiguous row.
- **Fix:** Changed `_USER_RATING_BULLET` to 1800 (exact key in the lookup table), ensuring deterministic nearest-anchor selection.
- **Files modified:** tests/services/test_user_benchmark_percentiles_service.py
- **Commit:** 0c12037b
