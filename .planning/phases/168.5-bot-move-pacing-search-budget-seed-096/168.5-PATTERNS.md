### `scripts/lib/node-engine-providers.mjs` (utility, event-driven)

**Analog:** itself — `waitFor`/`stopAndSync` (lines 117-158), the primitive D-11 extends, not replaces

**Existing watchdog + recovery primitive (copy verbatim, do not reimplement):**
```javascript
waitFor(predicate, timeoutMs) {
  return new Promise((resolve, reject) => {
    const cleanup = () => { clearTimeout(timer); off(); this.pendingWaiters.delete(rejectWithCleanup); };
    const rejectWithCleanup = (err) => { cleanup(); reject(err); };
    const timer = setTimeout(() => {
      rejectWithCleanup(new Error(`Stockfish response timeout after ${timeoutMs}ms`));
    }, timeoutMs);
    const off = this.onLine((line) => { if (predicate(line)) { cleanup(); resolve(line); } });
    this.pendingWaiters.add(rejectWithCleanup);
  });
}

async stopAndSync() {
  this.send('stop');
  this.send('isready');
  await this.waitFor((line) => line === 'readyok', STOCKFISH_INIT_TIMEOUT_MS);
}
```
No code change expected in THIS file — only the timeout VALUES passed in by callers (`calibration-providers.mjs`) change, per D-11.
