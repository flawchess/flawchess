def _wdl_to_score(wdl: EndgameWDLSummary) -> float:
    """Convert WDL summary to score in range 0.0-1.0. Score = (Win% + Draw%/2) / 100."""
    if wdl.total == 0:

    return (wdl.win_pct + wdl.draw_pct / 2) / 100

def _compute_verdict(row_score: float, overall_score: float) -> Verdict:
    """Determine verdict relative to user's overall score.
    good: score >= overall, ok: within -0.05, bad: below -0.05."""
    if row_score >= overall_score:

Logic:
- Compute `endgame_score = _wdl_to_score(endgame_wdl)` and `non_endgame_score = _wdl_to_score(non_endgame_wdl)`
- Compute `score_difference = endgame_score - non_endgame_score`
- Compute `overall_score` from weighted combination: sum wins/draws/losses across both WDLs, then apply score formula
- Deduplicate entry_rows by game_id (keep first occurrence per game_id using a `seen_game_ids: set[int]` — row[0] is game_id)
- For each deduplicated row, read `user_material_imbalance` (row[4]). If None, skip. Assign to bucket: >= 100 -> "ahead", <= -100 -> "behind", else "equal"
- Derive user_result from row[2] (result) and row[3] (user_color) using `derive_user_result`
- Accumulate wins/draws/losses per bucket
- Build MaterialRow for each of the three buckets ("ahead", "equal", "behind") in that fixed order. If games=0, set all percentages to 0.0, score to 0.0, verdict to "bad"
- For non-zero buckets: compute win_pct, draw_pct, loss_pct as `round(count / total * 100, 1)`, score = `(win_pct + draw_pct / 2) / 100`, verdict = `_compute_verdict(score, overall_score)`
- Return `ScoreGapMaterialResponse` with all computed values
