Add two new async functions to app/repositories/analysis_repository.py:

**1. `query_next_moves`** — Main aggregation query

Parameters: `session: AsyncSession, user_id: int, target_hash: int, time_control: list[str] | None, platform: list[str] | None, rated: bool | None, opponent_type: str, recency_cutoff: datetime.datetime | None, color: str | None`

Returns: `list[tuple]` — rows of `(move_san, result_hash, game_count, wins, draws, losses)`

Implementation:
- Use `from sqlalchemy import case, func, select` and `from sqlalchemy.orm import aliased`
- Create aliased GamePosition: `gp1 = aliased(GamePosition, name="gp1")` (source position) and `gp2 = aliased(GamePosition, name="gp2")` (resulting position at ply+1)
- Build CASE expressions for W/D/L using Game.id as the DISTINCT target:
  ```python
  win_case = case(
      (
          ((Game.result == "1-0") & (Game.user_color == "white")) |
          ((Game.result == "0-1") & (Game.user_color == "black")),
          Game.id,
      ),
      else_=None,
  )
  draw_case = case((Game.result == "1/2-1/2", Game.id), else_=None)
  loss_case = case(
      (
          ((Game.result == "1-0") & (Game.user_color == "black")) |
          ((Game.result == "0-1") & (Game.user_color == "white")),
          Game.id,
      ),
      else_=None,
  )
  ```
- Build the SELECT:
  ```python
  stmt = (
      select(
          gp1.move_san,
          gp2.full_hash.label("result_hash"),
          func.count(Game.id.distinct()).label("game_count"),
          func.count(win_case.distinct()).label("wins"),
          func.count(draw_case.distinct()).label("draws"),
          func.count(loss_case.distinct()).label("losses"),
      )
      .join(Game, Game.id == gp1.game_id)
      .join(gp2, (gp2.game_id == gp1.game_id) & (gp2.ply == gp1.ply + 1))
      .where(
          gp1.user_id == user_id,
          gp1.full_hash == target_hash,
          gp1.move_san.isnot(None),
      )
      .group_by(gp1.move_san, gp2.full_hash)
  )
  ```
- Apply the same filter clauses as `_build_base_query` applies (but inline, since `_build_base_query` uses `DISTINCT ON Game.id` which conflicts with GROUP BY):
  - `if time_control is not None: stmt = stmt.where(Game.time_control_bucket.in_(time_control))`
  - `if platform is not None: stmt = stmt.where(Game.platform.in_(platform))`
  - `if rated is not None: stmt = stmt.where(Game.rated == rated)`
  - `if opponent_type == "human": stmt = stmt.where(Game.is_computer_game == False)`
  - `elif opponent_type == "bot": stmt = stmt.where(Game.is_computer_game == True)`
  - `if recency_cutoff is not None: stmt = stmt.where(Game.played_at >= recency_cutoff)`
  - `if color is not None: stmt = stmt.where(Game.user_color == color)`
- Execute and return: `rows = await session.execute(stmt); return list(rows.all())`

**IMPORTANT SQLAlchemy syntax note:** If `func.count(win_case.distinct())` does not compile correctly (test by running), use the fallback: `func.count(func.distinct(win_case))`. If neither works, fall back to fetching `(move_san, result_hash, game_id, result, user_color)` tuples with DISTINCT and aggregating W/D/L in Python — this is acceptable since there are typically <30 distinct moves and <1000 games per position.

**2. `query_transposition_counts`** — Batch transposition count

Parameters: `session: AsyncSession, user_id: int, result_hash_list: list[int], time_control: list[str] | None, platform: list[str] | None, rated: bool | None, opponent_type: str, recency_cutoff: datetime.datetime | None, color: str | None`

Returns: `dict[int, int]` — mapping `{result_hash: transposition_count}`

Implementation:
- ```python
  stmt = (
      select(
          GamePosition.full_hash.label("result_hash"),
          func.count(GamePosition.game_id.distinct()).label("transposition_count"),
      )
      .join(Game, Game.id == GamePosition.game_id)
      .where(
          GamePosition.user_id == user_id,
          GamePosition.full_hash.in_(result_hash_list),
      )
      .group_by(GamePosition.full_hash)
  )
  ```
- Apply same filter clauses (time_control, platform, rated, opponent_type, recency_cutoff, color) — identical to query_next_moves
- Execute and return dict: `rows = await session.execute(stmt); return {row.result_hash: row.transposition_count for row in rows}`
