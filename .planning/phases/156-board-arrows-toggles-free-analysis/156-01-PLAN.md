## Interface Context for Executors

- `BoardArrow` (ChessBoard.tsx L16): `{ startSquare, endSquare, color, width, label?, labelColor?, onTop?, ... }`. `width` is normalized 0–1. `onTop` arrows are drawn last (topmost).
- `uciToSquares(uci: string | null): { from: string; to: string } | null` (`@/lib/sanToSquares`) — already imported in Analysis.tsx (L63). Returns `null` for null/short input.
- `RankedLine.rootMove: string` (UCI) and `pvLines[0].moves[0]` (UCI) are the two arrow sources.
- `flawChessEnabled` / `engineEnabled` booleans live at Analysis.tsx ~L346–351.
- `BEST_MOVE_ARROW = 'rgba(37, 99, 235, 0.8)'` (theme.ts L346) — reuse for Stockfish.
- `NEXT_MOVE_ARROW_WIDTH = 0.18` and `QUALITY_HOVER_ARROW_WIDTH = 0.6` already exist in Analysis.tsx.
- **ChessBoard sorts arrows before drawing** (ChessBoard.tsx ~L142–152): order is `onTop` last, then by `arrowSortKey(color)`, then thicker-first (`b.width - a.width` → wider drawn first / at the bottom). `BEST_MOVE_ARROW` and the new amber token both fall in the default sortKey tier (3), so the concentric stack is produced by the width sort alone: FC 0.80 bottom, SF 0.50 middle, white 0.18 (`onTop`) top. Array order does NOT control draw order here — width + `onTop` do.
- **Mobile parity is automatic:** `boardRow` (Analysis.tsx L1182) is a single shared JSX block rendered by both the mobile (L1525) and desktop (L1668) trees, so `boardArrows` feeds both with no duplicate code.
