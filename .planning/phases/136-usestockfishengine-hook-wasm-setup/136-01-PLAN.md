<objective>
Vendor the single-thread WASM Stockfish 18 engine and harden the platform so the engine can ship without breaking the existing site. This plan delivers PLAT-01 (no COOP/COEP headers anywhere, CI-guarded) and PLAT-02 (iOS-safe PWA wasm handling, ~7 MB lite build vendored verbatim).

This plan ships NO user-visible UI. It installs/pins `stockfish@18.0.8`, copies the two engine binaries into `frontend/public/engine/` and commits them (D-03), wires `vite.config.ts` (`optimizeDeps.exclude` + Workbox `globIgnores`), and adds a CI `curl -I` guard that locks in the absence of COOP/COEP and the correct `application/wasm` MIME.

The `stockfish` npm package is flagged SUS by the legitimacy gate (v18.0.8 published ~2 weeks ago, a `too-new` date artifact). A blocking human-verify checkpoint precedes the install per SUS protocol.

Purpose: Foundation for Plan 02's hook + integration test. The integration test imports the `stockfish` package, and the production Worker loads the vendored `/engine/` binaries.
Output: Vendored binaries, pinned dependency, Vite wiring, CI guard, README provenance note.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| npm registry → build | Third-party `stockfish` package + vendored binaries enter the tree |
| browser → site headers | COOP/COEP headers (if present) change cross-origin isolation site-wide, affecting OAuth |
| static host → browser | Engine wasm served with a MIME type the browser must accept |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-136-SC | Tampering | `npm install stockfish` + vendored binaries | mitigate | Blocking `checkpoint:human-verify` (Task 1) before install; exact-version pin (18.0.8); binaries committed to git (D-03) so there is no runtime fetch and the bytes are SHA-fixed at commit time; postinstall is symlink-only (no network) |
| T-136-01 | Elevation of Privilege | Site-wide response headers / Vite bundler | mitigate | CI `curl -I` guard fails on any Cross-Origin-Opener-Policy / Cross-Origin-Embedder-Policy header; `optimizeDeps.exclude` prevents the bundler from pulling SharedArrayBuffer-requiring code paths; `window.crossOriginIsolated` stays false |
| T-136-02 | Denial of Service / Tampering | PWA SW precache + wasm MIME | mitigate | `globIgnores: ['**/*.wasm']` keeps the ~7 MB blob out of the iOS Cache-API quota; CI asserts `application/wasm` MIME so `instantiateStreaming` does not silently fail/slow |
</threat_model>
