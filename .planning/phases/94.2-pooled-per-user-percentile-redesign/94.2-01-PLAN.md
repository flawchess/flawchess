<objective>
Refactor `app/services/canonical_slice_sql.py` so its builders emit a single pooled per-user aggregate (not one row per `(user_id, elo_bucket, tc_bucket)` cell), apply the recent-1000-per-TC cap and 36-month recency filter, drop the `apply_floor` dual-mode, drop per-row `sparse_exclusion_sql` invocations, and add a deduped-by-`lichess_username` cohort CTE on the benchmark source per D-1. The three plans together form an atomic cutover per D-3a — Plan 01 alone leaves the byte-identical regression test red until Plans 02 and 03 land.

Output: A new pooled SQL builder API surface, updated unit tests, and updated parity tests. No commits to consumers in this plan.
