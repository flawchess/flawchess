    2. **Replace or add §1.1 "Pooled-per-user cohort construction"** describing the new recipe (cite D-5):
       - For each subject (cohort user during CDF generation; app user at lookup time):
         1. Take games per TC played, ordered by `played_at DESC`.
         2. Cap at the most recent 1000 games per TC (constant: `RECENT_GAMES_PER_TC_CAP = 1000`).
         3. Drop games older than 36 months from the snapshot date (constant: `RECENCY_WINDOW_MONTHS = 36`).
         4. Pool the per-TC subsets into one combined set.
         5. Apply universal filters: `rated = true`, non-computer, equal-footing ±100 ELO, standard variant only.
         6. Compute the metric once on the pool.
       - CDF: `percentile_cont` over the one-point-per-user values → 99 breakpoints. Globally pooled across all rating buckets and TCs.
       - Lookup: app user's pooled metric value is interpolated against the same global CDF.
       - Reference: `app/services/canonical_slice_sql.py` is the source of truth — every SQL builder there mirrors this section verbatim per D-10.

    3. **Add §1.2 "Cohort selection (benchmark side)"** describing D-1:
       - `benchmark_selected_users` is deduped by `lower(lichess_username)` (a user qualifying via multiple TC slots contributes one CDF point, not many).
       - Users whose ONLY selection slot is `(rating_bucket=2400, tc_bucket=classical)` are dropped (`bool_or(NOT (bsu.rating_bucket = 2400 AND bsu.tc_bucket = 'classical'))` in the cohort CTE). A user selected via that slot AND another slot stays.
       - Sparse-cell exclusion lives ONLY at cohort selection; the per-row CTE no longer filters by `(2400, classical)` (the pooled CDF has no cells to stabilise — pooling already mixes rating buckets together).

    4. **Add §1.3 "Snapshot date semantics"**:
       - CDF construction: snapshot date is operator-controlled via `scripts/gen_global_percentile_cdf.py --snapshot-date YYYY-MM-DD` (default derived from `BENCHMARK_DB_SNAPSHOT_MONTH`).
       - Per-user lookup: snapshot date is `NOW()` ("where I currently sit on this metric").
       - This asymmetry is intentional: the cohort is a snapshot in time; the user is asking "where am I now."

    5. **Annotate the prior "Per-cell stratification" subsections** (don't delete — flag as superseded). For each subsection describing per-cell stratification, prepend a callout block:

The pooled-per-user model in §1.1 replaces per-(elo_bucket, tc_bucket) stratification on both the CDF-construction side and the per-user lookup side.
