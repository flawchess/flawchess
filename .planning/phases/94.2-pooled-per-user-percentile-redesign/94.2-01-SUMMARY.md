# Phase 94.2 Plan 01: Refactor canonical_slice_sql for pooled-per-user shape — Summary

## One-liner

Reshaped `app/services/canonical_slice_sql.py` from per-cell `(elo_bucket, tc_bucket)` builders into pooled per-user CTEs with a recent-1000-per-TC cap, 36-month recency anchor (snapshot_date kwarg), single ≥30 inclusion floor, and deduped benchmark cohort — removing `apply_floor` from the public surface and `sparse_exclusion_sql` from every per-row invocation.

## What changed

### `app/services/canonical_slice_sql.py`

- **New module-level constants:** `RECENT_GAMES_PER_TC_CAP = 1000`, `RECENCY_WINDOW_MONTHS = 36`. Bumped `ACHIEVABLE_MIN_GAMES` and `SECTION2_MIN_SPANS_PER_BUCKET` from 20 → 30 (D-6: ≥30 on all four metrics).
- **New helper `_recency_window_sql(snapshot_date)`:** renders either `NOW() - INTERVAL '36 months'` (None, per-user lookup) or `DATE 'YYYY-MM-DD' - INTERVAL '36 months'` (explicit, CDF construction). f-string emission per Pattern 3 in RESEARCH (snapshot_date is a build-time `date | None`, not user input — `date.isoformat()` emits only digits).
- **New shared `_recent_capped_cte(snapshot_date)`:** the 1000-per-TC + 36-month + equal-footing prelude, identical on both sources per D-5. Builds the `ROW_NUMBER() OVER (PARTITION BY g.user_id, g.time_control_bucket ORDER BY g.played_at DESC)` window.
- **`selected_users_cte(source="benchmark")` rewrite (D-1):** now groups `benchmark_selected_users` by `lower(bsu.lichess_username)` and `HAVING bool_or(NOT (rating_bucket = 2400 AND tc_bucket = 'classical'))` to drop sparse-only members. Emits only `AS user_id` — downstream pooled CTEs no longer need `tc_bucket` / `selection_rating_bucket` / `median_elo` projections.
- **`selected_users_cte(source="single_user")`:** preserves `CAST(:user_id AS int) AS user_id` form verbatim (V5 / 94.1 Plan 09 fix). Dropped the now-unused `NULL::text AS tc_bucket` column.
- **`per_user_cte_score_gap` / `_achievable` / `_section2` rewrite:** all three now build on `_recent_capped_cte`, `GROUP BY user_id` only (no `elo_bucket` / `tc_bucket`), emit `per_user_values(metric_value, n_games)`. HAVING gates land on the pooled aggregate (≥30 endgame AND ≥30 non-endgame for score_gap; ≥30 d_i for achievable; ≥30 spans per bucket for section2). No per-TC predicate, no `sparse_exclusion_sql` invocation. `n_games` is the metric-relevant binding count per the D-9-amend.
- **`per_user_cte_for` signature:** drops `apply_floor`, adds `snapshot_date: date | None = None`. Dispatches threading `snapshot_date` through to each per-metric builder.
- **Helpers retained for external consumers:** `sparse_exclusion_sql`, `elo_bucket_expr`, `equal_footing_filter_sql` stay exported (`gen_global_percentile_cdf.py:_build_per_bucket_sanity_query` and the helper tests still call them). A module-level comment over `sparse_exclusion_sql` notes "D-1: NO LONGER invoked in per_user_cte_*; retained for the per-bucket sanity diagnostic and test reference."
- **Module docstring** rewritten to describe the pooled-per-user methodology with citations to D-1 / D-5 / D-6 / D-8 / D-10.

### `tests/services/test_canonical_slice_sql.py`

Rewritten top-to-bottom around five test classes:

- `TestPooledShape` — parametrised over 4 metrics × 2 sources: asserts `(metric_value, n_games)` projection, no `elo_bucket` leak, no `tc_bucket` leak (outside the `time_control_bucket` window-function), no `g.time_control_bucket::text = su.tc_bucket` predicate, no `tc_bucket = 'classical'` literal (sparse-exclusion fingerprint).
- `TestRecencyWindow` — `<= 1000` cap appears exactly once per builder; default emits `NOW() - INTERVAL '36 months'`; explicit `date(2026, 3, 31)` emits `DATE '2026-03-31' - INTERVAL '36 months'` (and no `NOW()`).
- `TestInclusionFloor` — per-metric ≥30 HAVING gates (score_gap: both filters; achievable: `WHERE d_i IS NOT NULL`; section2: `count(*) >= 30`); `apply_floor=False` raises `TypeError` for all four metrics.
- `TestCohortDedup` — D-1 `GROUP BY lower(bsu.lichess_username)` + `bool_or(NOT (rating_bucket = 2400 AND tc_bucket = 'classical'))` on benchmark; `CAST(:user_id AS int)` + SQLAlchemy bindparam round-trip on single_user; no benchmark-table joins on single_user.
- `TestExportedHelpers` — preserves `test_sparse_exclusion_sql_parametrises_columns` verbatim, plus `equal_footing_filter_sql` determinism and `elo_bucket_expr` alias substitution.

78 parametrised cases, all green.

### `tests/services/test_canonical_slice_per_user_vs_benchmark_parity.py`

Rewrote 94.1's DB-seeded numerical-parity gate (which aggregated over per-cell `per_user` CTE rows and asserted `abs(bm - su) < 1e-9` per `user_id`) into four structural assertions:

- Pooled CTE body is byte-identical between sources after whitespace normalisation (D-10).
- No `g.time_control_bucket::text = su.tc_bucket` on either source (D-5).
- Both sources include the `recent_capped` prelude with `<= 1000` and `INTERVAL '36 months'`.
- Both sources emit `per_user_values` with `metric_value` and `n_games`.

20 parametrised cases, all green. The numerical-parity claim is now structurally trivial: identical SQL emits identical values.

## Deviations from Plan

None — plan executed exactly as written.

The plan-acceptance "n_games projection per metric (D-9-amend)" criterion suggested `count(*) FILTER (WHERE has_endgame) AS n_games` etc. as one possible rendering, with the note "or equivalent grouped form". The implementation uses a cleaner shape: per-metric `per_user` already computes the binding count as a column (`eg_n` for score_gap, `di_n` for achievable, `span_n` for section2) and `per_user_values` aliases it to `n_games`. Substring `n_games` appears in the `per_user_values` projection for all four metrics — verified by the parametrised test `TestPooledShape::test_per_user_values_projection_emits_metric_value_and_n_games` (8 cases green).
