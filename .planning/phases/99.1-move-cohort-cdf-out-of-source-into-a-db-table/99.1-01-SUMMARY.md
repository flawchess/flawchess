## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed ENUM create_type handling in migration**

- **Found during:** Task 2 (alembic upgrade head failed)
- **Issue:** The autogenerated migration's `sa.Enum(...)` inline constructors emitted `CREATE TYPE benchmark_metric` even though the type already exists, causing `DuplicateObjectError`. Importing the SAEnum objects from model files also triggered this because Alembic's DDL rendering emits `CREATE TYPE` for any Enum column in `op.create_table`.
- **Fix:** Used `postgresql.ENUM(name="benchmark_metric", create_type=False)` and `postgresql.ENUM(name="timecontrolbucket", create_type=False)` as name-only references that never emit `CREATE TYPE`. Mirrors the exact pattern in `fc000b5d0134_add_user_rating_anchors.py`.
- **Files modified:** `alembic/versions/20260531_063037_d0a962b51c68_add_benchmark_cohort_cdf.py`
- **Commit:** 52742783

**2. [Rule 1 - Bug] Removed autogenerate noise from migration**

- **Found during:** Task 2 (autogenerate)
- **Issue:** `uv run alembic revision --autogenerate` also detected a pre-existing `ix_games_evals_pending` drop on `games` table (unrelated to this plan) and included it in the generated migration.
- **Fix:** Removed the `op.drop_index` / `op.create_index` pair for `ix_games_evals_pending` from both `upgrade()` and `downgrade()`. Pre-existing drift logged as deferred.
- **Files modified:** `alembic/versions/20260531_063037_d0a962b51c68_add_benchmark_cohort_cdf.py`
- **Commit:** 52742783
