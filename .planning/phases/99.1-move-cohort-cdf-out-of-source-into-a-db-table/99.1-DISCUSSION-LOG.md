## Table layout

| Option | Description | Selected |
|--------|-------------|----------|
| Long | One row per breakpoint (metric, anchor_elo, tc, percentile, value, n_users, snapshot_month). ~130k rows. PK (metric, anchor, tc, percentile). Best SQL-analysis surface; lookup rebuilds CdfTable from ~99 rows. | ✓ |
| Wide (float8[]) | One row per cell with percentiles/values float8[] columns. ~1.3k rows. Single-row fetch == one CdfTable; trivial upsert. SQL analysis needs unnest(). | |

**User's choice:** Long
**Notes:** Chosen for the stated SQL-analysis side-benefit. Matches the lean in the decision log.

---

## Seed file format

| Option | Description | Selected |
|--------|-------------|----------|
| TSV long rows | cohort_cdf.tsv, one row per breakpoint (~130k lines), 1:1 with table rows, mirrors openings.tsv + csv.DictReader. Drift test diffs this file. | ✓ |
| JSONL per cell | One JSON object per cell with nested arrays (~1.3k lines); seed script must explode each line into rows; diverges from openings pattern. | |

**User's choice:** TSV long rows
**Notes:** Pairs naturally with the long table; maximal consistency with the existing openings seed pattern.

---

## Lookup query shape

| Option | Description | Selected |
|--------|-------------|----------|
| Per-call SELECT | One indexed SELECT (~99 rows) per call; ~32 small queries per import. Simplest 1:1 refactor; clean helper signature. | |
| Batched prefetch | One load query up front into a dict cache; the 32 resolves hit memory. Fewer round-trips, adds a cache-shape layer + wider contract. | ✓ |

**User's choice:** Batched prefetch
**Notes:** Chosen against the recommendation (which flagged it as premature optimization for a once-per-import background path). Accepted — see follow-up on async boundary.

---

## Async boundary (follow-up triggered by batched-prefetch choice)

| Option | Description | Selected |
|--------|-------------|----------|
| Loader in repo/service, module stays pure | New async load_cohort_cells in a repository; global_percentile_cdf.py stays sync/pure; interpolate_cohort_percentile(table, value) takes a prefetched CdfTable. Supersedes notes #5 literal wording. | ✓ |
| Async loader inside the module | Module gains an async session-taking load fn; closer to notes #5 wording but breaks the module's stated purity invariant. | |

**User's choice:** Loader in repo/service, module stays pure
**Notes:** Surfaced because batched prefetch moves DB access out of `interpolate_cohort_percentile`, contradicting the literal wording of notes decision #5 / ROADMAP success criterion #3. Resolution preserves the module's purity invariant and sibling parity with `endgame_zones.py`; planner must reconcile criterion #3 to the repo-loader shape.
