## Decisions Made

- **D-04 sync-over-async:** `interpolate_cohort_percentile` takes a prefetched `CdfTable` and stays synchronous. The ROADMAP's earlier "becomes async" wording was superseded by the module-pure / repo-async split decided in CONTEXT.md. This avoids async leaking into the pure math layer and keeps the function easily unit-testable.
- **D-03 single prefetch:** `load_cohort_cells` is called once per import (before the `for tc, anchor` loop), not once per lookup cell. This bounds the DB round-trips to 1 per stage regardless of how many (metric, anchor, tc) combinations the user has.
- **Wave-3 ordering preserved:** This plan ran last. The dev DB was already seeded by Plan 03 before the registry was deleted, ensuring the DB path was live before the fallback was removed.

## Runbook Caveat

A fresh production DB (or a dev DB reset before Plan 03's seed runs) will suppress percentile chips until `scripts/seed_cohort_cdf.py` is executed. The seed script is idempotent (`ON CONFLICT DO UPDATE`) and is wired into `bin/run_local.sh` behind a row-count gate. This is documented behavior — not a regression.
