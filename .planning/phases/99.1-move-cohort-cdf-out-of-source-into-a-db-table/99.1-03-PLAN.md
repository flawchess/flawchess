## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cohort_cdf.tsv → seed script → DB | Trusted committed file read by an operator-run script; values bound as parameters |
| repository ← background service (Plan 04) | In-process call with a trusted AsyncSession; no external input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-99.1-05 | Tampering / Injection | seed script INSERT from TSV string values | mitigate | INSERT uses `text(...)` with bound `:params`, never f-string interpolation of TSV cell values; verified by grep gate. SQL injection surface closed even if the (trusted, committed) TSV were altered. |
| T-99.1-06 | Information disclosure | repository SELECT exposure | accept | The table holds aggregate cohort breakpoints (no PII, no per-user data); read in a trusted background task. Low value target. |
| T-99.1-07 | Denial of service | full-table load on a too-broad anchor/tc set | mitigate | Loader filters by the caller's relevant anchors+TCs (D-03 batched prefetch); empty-input guard prevents an unfiltered scan; ~32 cells per import is bounded. |
| T-99.1-SC | Tampering | npm/pip/cargo installs | mitigate | No new package installs in this plan; no audit checkpoint required. |
</threat_model>
