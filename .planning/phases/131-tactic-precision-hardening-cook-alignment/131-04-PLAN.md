<objective>
Add the Workstream B missed-vs-played false-alarm gate (D-03): on the missed pass, when the
flaw move's destination square equals the best line's first-move destination square, suppress
the missed tactic. This kills the wrong-recapture false alarm — the player captured the SAME
piece with the wrong piece type, so they plainly SAW it; it is not a "you missed it" tactic.

This bug is INVISIBLE to the CC0 puzzle fixture (a puzzle has no "move the player actually
played"), so it is validated SEPARATELY via hand-built `(flaw_move, best_line)` unit fixtures
(D-04), not the harness. Blast radius is large — the missed side is dominated by fork/pin/
discovered-attack/skewer (thousands of rows each), so a false suppression would be costly.

This plan is independent of the detector internals (different file, flaws_service.py) and runs
Wave 2 in parallel with the geometric ports (plan 02).

Purpose: stop false "you missed a tactic" chips on moves the player demonstrably saw.
Output: the call-site dest-square gate + unit fixtures proving suppress/non-suppress.
</objective>
