## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (components.json present) |

## Spacing Scale

4px grid throughout (Tailwind default scale). The following tokens are in use across sibling
components (`LibraryGameCard`, `TagChip`, `SeverityBadge`) and the new `FlawCard` must
match them identically.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, chip-internal padding |
| sm | 8px (`gap-2`, `p-2`) | Compact row spacing |
| md | 16px (`gap-4`, `p-4`) | Card body, grid column gap |
| lg | 24px (`gap-6`) | Section breaks |
| xl | 32px (`gap-8`) | Major layout gaps |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

**Card-internal specifics (mirror `LibraryGameCard`):**
- Card body padding: `p-3` (12px)
- Board-to-content gap: `gap-3` (12px)
- Content stack row gap: `gap-1.5` (6px) between most items; `gap-1` within metadata lines
- Grid card gap: `gap-4` (16px) between cards in the 2-up grid

Exceptions:
- `CardHeader` uses its own `size="compact"` preset (built into the component)
- 132px miniboard — fixed pixel size (DESKTOP_BOARD_SIZE constant from `LibraryGameCard`); not on the 4px grid, but already established by Phase 109

Source: `LibraryGameCard.tsx` layout lines + `SeverityBadge.tsx` padding classes.

## Typography

All text respects the CLAUDE.md `text-sm` floor. `text-xs` is reserved for hover/tap-activated
info tooltip bodies only (e.g. `TagChip` definition popovers, `TagLegend`).

| Role | Tailwind Class | Size | Weight | Line Height | Usage |
|------|---------------|------|--------|-------------|-------|
| Body / label | `text-sm` | 14px | 400 (normal) | 1.5 | Metadata items, move notation, eval swing, date/TC/termination |
| Label bold | `text-sm font-semibold` | 14px | 600 | 1.5 | Player names in CardHeader |
| Badge / count | `text-base font-bold` | 16px | 700 | 1.5 | Severity count number inside `SeverityBadge` |
| Badge label | `text-sm font-bold` | 14px | 700 | 1.5 | "Blunders" / "Mistakes" / "Inaccuracies" label inside `SeverityBadge` |

**CardHeader heading:** rendered as `h4`, `size="compact"` preset from `card.tsx` — do not
override the font size; it is set by the component.

**Move notation line:** `text-sm font-mono` (or default sans at `text-sm`) + eval swing
rendered inline separated by two spaces: `16...Nxd4  +4.7 → #-3`. The mono class is
discretionary — use whichever the existing `FlawRow` already uses; if none, use
`text-sm text-foreground`.

**Tooltip bodies** (`TagChip` definition popovers, `TagLegend` popover): `text-xs` is
permitted per CLAUDE.md exception for hover/tap-activated info tooltip surfaces.

Source: `SeverityBadge.tsx` classes, `LibraryGameCard.tsx` metadata div classes.

## Move Notation + Eval Swing Line Contract

**Format:** `{moveNotation}  {evalBefore} → {evalAfter}`

Examples:
- `16...Nxd4  +4.7 → #-3`
- `8.e4  +0.3 → -1.2`
- `22...Rxe4  +2.1 → —`  (when eval-before unavailable, e.g. ply=1)

**Move notation computation** (inline formula, not `formatCandidateMove` array):
```typescript
const moveNumber = Math.floor(ply / 2) + 1;
const notation = ply % 2 === 0
  ? `${moveNumber}.${move_san}`    // white move (even ply per kernel parity)
  : `${moveNumber}...${move_san}`; // black move (odd ply)
```
Parity rule: kernel stores `ply` where even = white mover, odd = black mover (verified from
`flaws_service._run_all_moves_pass`: `mover = "white" if n % 2 == 0`).

**Eval formatter** (extracted from `EvalChart.formatEval`, user-POV negated):
- For `user_color === 'black'`: negate both `eval_cp` and `eval_mate` before formatting
- `eval_mate !== null` → `#${evalMate}` (e.g. `#3` white mate, `#-3` black mate against user)
- `eval_cp !== null` → `formatSignedEvalPawns(evalCp / 100)` (1-decimal signed, e.g. `+4.7`, `-1.2`)
- `null` → `—` (em-dash, no eval available)
- Separator: ` → ` (space-arrow-space)
- No `"Eval: "` prefix — value string only

**POV rule:** stored `eval_cp`/`eval_mate` are white-POV. A flaw always drops the user's
own eval: for black, a blunder shows a worsening from the user's perspective only after negation.

**Placement:** `text-sm text-foreground` on the move notation, `text-sm text-muted-foreground`
on the eval swing portion, rendered on the same line. Use a `<span>` for the eval portion:
```tsx
<div className="text-sm">
  <span className="text-foreground font-medium">{moveNotation}</span>
  {'  '}
  <span className="text-muted-foreground">{evalSwing}</span>
</div>
```

Source: D-04, D-05, D-06, `112-RESEARCH.md` §Code Examples Pattern 2+3.

## Metadata Block Contract

Mirrors `LibraryGameCard` metadata. Rendered in `text-sm text-muted-foreground`.

**Desktop** (`flex flex-wrap items-center gap-x-3 gap-y-1`):
- Termination + result indicator (first)
- Date (`Calendar` icon, `h-3.5 w-3.5`)
- Time control bucket + raw string (`Clock` icon, `h-3.5 w-3.5`)

**Mobile** (`flex flex-col gap-1`):
- Date
- Time control
- Termination + result indicator

**Result indicator icons** (same as `LibraryGameCard`):
- Win: `Plus` icon, colored `WDL_BORDER_WIN`
- Loss: `Minus` icon, colored `WDL_BORDER_LOSS`
- Draw: `Equal` icon, colored `WDL_BORDER_DRAW`

**Termination copy:** capitalized (e.g. "Checkmate", "Timeout", "Resignation"). Omit if
`termination === 'unknown'` or null.

**Date format:** locale-aware short date (same `formatDate` helper as `LibraryGameCard`).
**TC format:** `{bucket} · {raw_string}` (e.g. "Rapid · 10+5"), same `formatTimeControl`
helper.

## "View game" Modal Contract

### Dialog sizing

| Viewport | Class |
|----------|-------|
| Desktop (≥ sm) | `sm:max-w-4xl` on `DialogContent` |
| Mobile (< sm) | Default near-fullscreen (`max-w-[calc(100%-2rem)]`) |
| Max height | `max-h-[90vh]` — prevents modal from exceeding viewport |
| Overflow | `overflow-y-auto` — internal scroll when `LibraryGameCard` content exceeds `max-h` |

**`DialogContent` className:** `"sm:max-w-4xl overflow-y-auto max-h-[90vh]"`

Note: `LibraryGameCard` uses `overflowVisible` for its `EvalChart` tooltip. Inside a
scrollable `DialogContent`, the Recharts tooltip may clip on mobile. This is acceptable on
mobile (chart renders at ~130px height). On desktop, no scrolling is needed so the tooltip
should not clip. Verify during UAT (CONTEXT.md Pitfall 2).

### Modal states

| State | Rendered Element | Notes |
|-------|-----------------|-------|
| Loading | `<div className="flex justify-center p-8"><Spinner /></div>` | Spinner centered with 32px vertical padding |
| Error | `<LoadError resource="game" variant="centered" />` | Mandatory `isError` pattern (CLAUDE.md) |
| Success | `<LibraryGameCard game={data} />` verbatim | No modifications to `LibraryGameCard` |

**Loading state:** Spinner (not skeleton). The single-game fetch is fast (one row + positions
for one game); a skeleton adds complexity for minimal UX gain.

### Modal a11y

- `data-testid="flaw-game-modal"` on `DialogContent`
- `aria-label="View full game"` on `DialogContent`
- Dialog title (visually hidden if desired): "Full Game View" — `DialogTitle` required by
  Radix for screen readers; may use `className="sr-only"` if title is not desired visually

### Modal trigger / fetch

- `selectedGameId: number | null` state in `FlawCard` (or in `FlawsTab` — either works;
  co-locate with `FlawCard` since each card manages its own modal)
- `useLibraryGame(gameId)` hook: `enabled` only when `gameId !== null`
- Fetch fires on first modal open per `gameId`; TanStack Query caches per `queryKey: ['library-game', gameId]`

Source: D-09 (CONTEXT.md), `112-RESEARCH.md` §Code Examples Pattern 5.

## Interaction States

### FlawCard

| State | Behavior |
|-------|----------|
| Default | Card renders with left accent edge (severity color), board, content stack |
| "View game" button hover | Standard `variant="default"` hover (brand-brown-hover) |
| Platform link hover | `Tooltip` shows "Open at this move on platform" |

| TagLegend click | "Explanation" popover opens listing all tag definitions |

### Miniboard

- Shows FEN of the board **before** the flawed move (`flaw.fen`)
- Arrow renders the flaw move (from-square → to-square derived from `move_san` + `Chess`)
- Board flipped when `flaw.user_color === 'black'`
- No click-to-move behavior (display only)

### Modal

| State | Behavior |
|-------|----------|
| Closed | `selectedGameId = null`, no fetch |
| Opening | `selectedGameId = flaw.game_id`, fetch starts if not cached |
| Open + loading | Spinner shown |
| Open + error | `LoadError` shown; user can close modal |
| Open + loaded | `LibraryGameCard` rendered; all eval chart interactions active |
| Closing | `onOpenChange(false)` resets `selectedGameId = null` |
