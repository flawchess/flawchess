<objective>
Slim the `game_flaws` schema (drop `es_before`/`es_after`/`move_san`, keep `fen`) and re-source those display values via a `game_positions` join in the flaw-list query, while adding `white_rating`/`black_rating` and before/after raw eval to `FlawListItem`. The column drop is gated behind a BLOCKING regression test that proves the new eval join reproduces the existing `es_before`/`es_after` (Pitfall 1 — eval-join off-by-one).

Purpose: SC-2 (ratings + schema slim) + SC-4 (eval swing source). Establishes the backend payload that the Wave 3 frontend `FlawCard` consumes.
Output: Updated `FlawListItem` schema, joined `query_flaws`, slimmed model + migration, classifier write-path change, regression + schema tests.
</objective>
