```python
def _render() -> str:
    """Build the full TypeScript source as a single string.

    The exact bytes emitted by this function are what gets committed. CI runs
    the script and uses `git diff --exit-code` to block any drift.
    """

        "// AUTO-GENERATED — do not edit by hand.\n"
        "// Source: <this script's data source>\n"
        "// Regenerate with: uv run python scripts/gen_bot_strength_curves.py\n"
        "\n"
        f"export const SOME_CONSTANT = {value};\n"
        ...
    )
```
For the nested `preset -> targetBlitzElo -> botElo` lookup table, follow
`gen_endgame_zones_ts.py`'s `_format_per_class_tc_gauge_zones()` (lines 155-176) pattern: a

**`--check` / drift-check `main()` pattern** (lines 79-99, identical in both analogs — copy
verbatim, just retarget `_OUTPUT` and the two error/regen messages):
```python
def main() -> None:
    check_mode = "--check" in sys.argv
    content = _render()
    if check_mode:
        if not _OUTPUT.exists():
            print(f"DRIFT: {_OUTPUT.relative_to(_REPO_ROOT)} does not exist.", file=sys.stderr)
            sys.exit(1)
        existing = _OUTPUT.read_text(encoding="utf-8")
        if existing != content:
            print(
                f"DRIFT: {_OUTPUT.relative_to(_REPO_ROOT)} is out of date. "
                "Run `uv run python scripts/gen_bot_strength_curves.py` to regenerate.",
                file=sys.stderr,
            )
            sys.exit(1)
        print(f"OK: {_OUTPUT.relative_to(_REPO_ROOT)} is up to date.")
    else:
        _OUTPUT.parent.mkdir(parents=True, exist_ok=True)
        _OUTPUT.write_text(content, encoding="utf-8")
        print(f"Wrote {_OUTPUT.relative_to(_REPO_ROOT)}")


if __name__ == "__main__":
    main()
```
Note: this phase emits TWO artifacts (JSON + TS), plus a second file
(`bot-strength-confirmation-predictions.json`) — either extend this `main()` to loop over
multiple `(content, output_path)` pairs, or (RESEARCH.md's suggestion) split into two small
scripts/modes. Either way, keep the same `--check` contract for both emitted files (the CI step
below diffs both paths in one `git diff --exit-code` call).

**Header comment convention** (every generated file, verbatim from `gen_flaw_thresholds_ts.py`

**Export shape:** simple scalar constants use `export const NAME = value;` (flat, one per line,
`flawThresholds.ts` style). Nested lookup structures (this phase's `preset -> targetBlitz ->
botElo` table) should follow `endgameZones.ts`'s `Record<>` + `as const` pattern:
```typescript
export const PER_CLASS_GAUGE_ZONES = {
  <entries>
} as const;

export type EndgameClassKey = keyof typeof PER_CLASS_GAUGE_ZONES;
```
Adapt for this phase: `export const BOT_STRENGTH_LOOKUP: Record<Preset, Record<number, number>> = { ... } as const;` plus a derived key type if presets are enumerated as a union (mirrors
`EndgameClassKey = keyof typeof ...`).

**Disclaimer string (D-06):** export as a plain `export const` string constant at module top
level, same treatment as any other named constant in `flawThresholds.ts` — no special pattern
needed beyond keeping it a single named export both JSON and TS reference identically.
