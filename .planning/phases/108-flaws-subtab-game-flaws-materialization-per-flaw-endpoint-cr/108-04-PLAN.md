<objective>
Migrate the Games-surface backend off the on-the-fly classifier re-call onto `game_flaws` (D-02): `get_library_games` card chips, per-game M+B severity counts, and the EXISTS filter source from `game_flaws`; `get_flaw_stats` severity counts, tag distribution, and trend come from a single `game_flaws JOIN games` scan. Inaccuracy counts stay the cheap aggregate / oracle columns (D-03). The `analyzed_n`/`total_n` denominator stays on the eval-coverage gate.

Purpose: D-02 makes `game_flaws` the single source of truth so there is no kernel re-call per filtered query and the cross-tab filter is unified in code. This is the HARD prerequisite that makes the Games tab read the materialized table (rollout via D-09 backfill).
Output: migrated `library_service.py` functions + parity regression tests; the per-game N+1 `_load_analyzed_flaws` loop retired.
</objective>
