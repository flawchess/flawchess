<objective>
Build the single materialization write path (D-10): a `game_flaws` repository (bulk insert, delete-for-game, and the `FlawRecord` → row mapping) and the import-pipeline hook in `eval_drain.py` that classifies just-evaluated games and persists their M+B flaws. This is the one classification path the import hook, `reclassify_positions.py`, and `backfill_flaws.py` all reuse.

Purpose: D-10 mandates ONE classify path so the materialized table never drifts from the live `classify_game_flaws` kernel. The import hook keeps `game_flaws` fresh on every new analyzed game; the shared mapping helper guarantees backfill and reclassify produce identical rows.
Output: `game_flaws_repository.py` (insert/delete/mapping), the `_classify_and_insert_flaws` hook in `eval_drain.py`, and a materialization round-trip test.
</objective>
