<objective>
Give the in-process server drain a tier-aware minimal path for tier-4b games so the freed
8-engine pool keeps contributing to gem-candidate backfill without re-evaluating and reclassifying
an already-analyzed game from scratch.

Purpose: `_full_drain_tick` captures `tier: int = claimed.tier` (eval_drain.py:691) and then discards
it (`_ = tier`, line 908) — every claimed game, including a TIER_BESTMOVE_BACKFILL pick, runs the
FULL Step-3 evaluate_nodes_multipv2 gather over every non-terminal ply and the full apply_full_eval
write. This bug is latent today (masked by fallback starvation) and will surface precisely as this
phase reduces fallback traffic. D-05 requires the drain to branch on tier and take the same minimal
candidate-only path the new /bestmove-submit handler uses server-side.

Output: the tier branch + a minimal drain tick reusing Plan 02's helpers, and drain-local Sentry
tagging.
</objective>
