<objective>
Add a new `§5 Flaw-Delta Zones` chapter to the benchmark generator that computes, for each
of the 15 flaw-delta metrics (13 tags + 2 combos `hasty+miss` / `low-clock+miss`, computed in the
same pipeline as two more metrics — D-05), every cohort user's **you−opponent delta**
on a **per-100-of-their-own-moves, paired-per-game** basis (D-01 unified estimator), then emits
per-(ELO×TC) Q1/Q3 quartiles + ELO/TC marginals + the established Cohen's-d collapse verdict per
axis, plus a per-metric viability diagnostic (D-06). Extend the `/benchmarks` SKILL.md to narrate
the new chapter into the committed report. Add a numeric-diff gate test.

Purpose: Produce the raw benchmark data (quartiles + marginals + verdicts + viability) that
Phase 115 hand-authors into the shipped flaw-delta zone constants (D-10). This phase is
benchmark-computation + narration ONLY — no endpoint, no UI, no committed zone constants, no DB
table, no migration (D-09).

Scope boundary (D-03): the `squandered` / `lucky` state-conditional residual (delta reads partly
as "how often the situation arose," not pure conversion skill) is NOT corrected with a separate
denominator here — it is disclosed via a Phase 115 tooltip line. A plan-time spot-check of that
exposure confound against the materialized cohort data is welcome but explicitly non-gating, so no
task blocks on it.
