## Spacing Scale

Standard 8-point scale inherited from project Tailwind defaults. All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tight label/icon pairs |
| sm | 8px | Compact element spacing, button internal gap |
| md | 16px | Default element spacing, card padding, section body padding |
| lg | 24px | Section padding, card group gaps |
| xl | 32px | Layout horizontal padding on desktop |
| 2xl | 48px | Major section breaks between homepage feature blocks |
| 3xl | 64px | Hero top/bottom padding, page-level vertical rhythm |

Exceptions:
- Touch targets (mobile filter buttons, CTA): minimum height 44px (`min-h-11`) — matches existing mobile pattern from Phase 19

## Typography

All values use the existing project type scale. No new fonts or sizes are introduced.

Scale uses exactly 2 weights: 400 (regular) for body-level roles, 700 (bold) for heading-level roles.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px | 400 (regular) | 1.5 | Feature descriptions, FAQ answers, privacy policy body text |
| Label | 14px | 400 (regular) | 1.4 | Callout pills ("Open source and free", "Mobile friendly"), nav links, small secondary text |
| Heading | 20px | 700 (bold) | 1.3 | Feature section headings (h2), FAQ question triggers |
| Display | 36px (mobile) / 48px (desktop) | 700 (bold) | 1.1 | Hero headline — "Engines are flawless, humans play FlawChess" in `.font-brand` (Fredoka) |

Notes:
- Display uses `.font-brand` (Fredoka), not Nunito Sans — consistent with the existing app header and Auth page
- Body, Label, and Heading use `font-sans` (Nunito Sans)
- Privacy policy body text uses the Body role — plain prose, no heading hierarchy beyond h2

## Color

The project uses shadcn CSS variables via `oklch()`. The values below are from `frontend/src/index.css` and `frontend/src/lib/theme.ts`. No new color values are introduced in this phase.

| Role | CSS Variable | Approximate Hex | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `--background` | #ffffff (light) / #191919 (dark) | Page background, hero background, privacy page background |
| Secondary (30%) | `--card` / `--muted` | #ffffff / #f7f7f7 (light) | Feature section cards, FAQ accordion background, callout pills |
| Accent (10%) | `#8B5E3C` (warm brown, `PRIMARY_BUTTON_CLASS`) | — | Primary CTA "Sign up free" button ONLY |
| Destructive | `--destructive` (`oklch(0.577 0.245 27.325)`) | ~#dc2626 | Import timeout error message ("Import timed out — re-sync to continue where it left off") |
| Semantic info | `--muted-foreground` | ~#737373 (light) | "X other users are importing" concurrent importer notice |

Accent reserved for: **the "Sign up free" CTA button only**. All other interactive elements (login link in public header, FAQ accordion triggers, internal links in privacy policy) use `ghost`, `outline`, or `link` variant — no warm brown accent.

The `PRIMARY_BUTTON_CLASS` constant from `frontend/src/lib/theme.ts` (`bg-[#8B5E3C] hover:bg-[#6B4226] text-white`) is the implementation mechanism. Use this constant on the CTA button.

## Page-Level Layout Contracts

### Homepage (`/`)

**Structure (top to bottom):**

1. **PublicHeader** — border-b border-border, sticky at top. Logo left (`h-10 w-10` icon + "FlawChess" in `.font-brand`). Right: "Log in" (`ghost` sm button) + "Sign up free" (`PRIMARY_BUTTON_CLASS` sm button). `data-testid="public-header"`.

2. **Hero section** — centered, `max-w-3xl mx-auto`, `px-4 py-16 lg:py-24`. Display headline in `.font-brand`. Tagline below in body text. Large "Sign up free" button (`PRIMARY_BUTTON_CLASS`, `size="lg"`, `min-h-11`). Below CTA: two small callout pills ("Open source and free", "Mobile friendly") using `bg-muted text-muted-foreground rounded-full px-3 py-1 text-sm`. `data-testid="hero-section"`.

3. **Screenshots section** — 1-2 screenshots provided by user as manual assets (placeholder state acceptable at build time). `max-w-5xl mx-auto`, `px-4 py-12`. Responsive: stacked on mobile, side-by-side on `sm:` if 2 images. `data-testid="screenshots-section"`.

4. **Feature sections** — 5 blocks, each using a `<section>` with `data-testid="feature-{slug}"`. Layout: alternating or grid depending on responsive width. On mobile: stacked (heading + description). On `lg:`: two-column grid (`grid grid-cols-2 gap-8`). Heading: Heading role (20px bold). Description: Body role (16px regular). No icons required — text-only feature blocks per the locked decision (D-03). Outer container: `max-w-5xl mx-auto px-4 py-12`.
   - `data-testid="feature-weaknesses"`
   - `data-testid="feature-scout"`
   - `data-testid="feature-move-explorer"`
   - `data-testid="feature-cross-platform"`
   - `data-testid="feature-filters"`

5. **FAQ section** — `max-w-2xl mx-auto px-4 py-12`. Heading: "Frequently asked questions" (Heading role). shadcn `Accordion type="single" collapsible`. 5 items per D-08. `data-testid="faq-accordion"`. Each item `data-testid="faq-item-{slug}"`.

6. **Footer CTA** — repeat "Sign up free" CTA with brief line: "Free to use. No credit card required." Center-aligned. `py-16`. `data-testid="footer-cta"`.

7. **Page footer** — minimal: copyright line + link to `/privacy`. `text-sm text-muted-foreground text-center py-6 border-t border-border`. `data-testid="page-footer"`.

**Responsive:** Full-width on mobile with `px-4` horizontal padding. `max-w-5xl` containers centered on desktop. Feature grid collapses to single column below `lg:`.

### Privacy Policy (`/privacy`)

**Structure:**
1. Same `PublicHeader` component as homepage.
2. Content area: `max-w-2xl mx-auto px-4 py-12`. `data-testid="privacy-page"`.
3. Page heading: "Privacy Policy" (Display size but left-aligned, not centered, weight 700). Subheading: "Last updated: March 2026" (Label role, muted-foreground).
4. Sections rendered as `<section>` with `<h2>` headings (Heading role): "What we collect", "Who we share it with", "Your rights", "Open source".
5. Body paragraphs in Body role. No accordion — prose layout only.
6. "Contact us" section: email link `support@flawchess.com` as `<a>` with `text-primary underline-offset-4 hover:underline` (link variant behavior). `data-testid="privacy-contact"`.

### Import Page — Concurrent Importer Notice (STAB-01)

This is an addition to the existing Import page, not a new page. No layout changes to the page shell.

**Notice component:** When `other_importers > 0`, show an info notice below the active import progress bar. Use `bg-muted border border-border rounded-md px-4 py-3 text-sm text-muted-foreground` with a lucide `Info` icon (16px, `inline-flex items-center gap-2`). `data-testid="import-concurrent-notice"`.

Copy: "X other users are also importing from chess.com — progress may be slower than usual." (one notice per platform, only shown for the platform currently being imported).

**Import timeout error:** Displayed in the existing `ImportProgressBar` error state, text color `text-destructive`. No new component needed — the existing `isError` branch already renders `data.error` content.

## Interaction Contracts

### PublicHeader
- Logo + brand name: `<Link to="/">` — navigates to `/` (homepage). No `data-testid` needed on the wrapping Link; the header itself has `data-testid="public-header"`.
- "Log in" button: `<Button variant="ghost" size="sm" asChild data-testid="nav-login">` wrapping `<Link to="/login">`.
- "Sign up free" button: `<Button size="sm" asChild className={PRIMARY_BUTTON_CLASS} data-testid="nav-signup">` wrapping `<Link to="/login?tab=register">`.

### Hero CTA
- Large "Sign up free": `<Button size="lg" asChild className={cn(PRIMARY_BUTTON_CLASS, 'min-h-11')} data-testid="hero-cta-signup">` wrapping `<Link to="/login?tab=register">`.

### FAQ Accordion
- Type: `single`, `collapsible` (only one item open at a time; can fully collapse)
- Each `AccordionTrigger` is keyboard-navigable via Tab and activatable via Enter/Space (shadcn Accordion provides this natively)
- `data-testid="faq-accordion"` on `<Accordion>`, `data-testid="faq-item-data"` / `"faq-item-free"` / `"faq-item-mobile"` / `"faq-item-requests"` / `"faq-item-who"` on each `<AccordionItem>`

### Homepage Auth Redirect
- `HomePage` checks `useAuth().token` on mount. If truthy: render `<Navigate to="/openings" replace />` immediately — no homepage content is rendered for authenticated users.
- No loading spinner or flash — the redirect is synchronous if token is in memory.

### Import Concurrent Notice
- Shown when `data.other_importers > 0` (per-platform, from `ImportStatusResponse`).
- Displayed inline below the import progress bar for the matching platform.
- No dismiss button — it disappears naturally when `other_importers` returns to 0 (polled every 2 seconds via existing `useImportPolling`).
