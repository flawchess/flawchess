   - CRITICAL: Restructure the loop. The existing code computes ply-0 hashes before the loop and iterates with `enumerate(moves, start=1)`. The NEW structure must:
     ```python
     results: list[tuple[int, int, int, int, str | None]] = []
     board = game.board()

     for ply, move in enumerate(moves):
         move_san: str = board.san(move)       # BEFORE push
         wh, bh, fh = compute_hashes(board)
         results.append((ply, wh, bh, fh, move_san))
         board.push(move)

     # Final position: no move played from it
     wh, bh, fh = compute_hashes(board)
     results.append((len(moves), wh, bh, fh, None))
     ```
   - Note: `board.san(move)` MUST be called BEFORE `board.push(move)` -- the board must be in the pre-move position
   - Note: `compute_hashes(board)` is called BEFORE push too, so hashes at ply N = the position BEFORE the move at ply N is played. This is a change from the existing code where ply-0 hashes are computed separately and ply 1+ hashes are computed AFTER push. The NEW code computes hashes BEFORE push for all plies, then appends the final position after the loop.
   - Update docstring to document the 5-tuple return and move_san semantics
