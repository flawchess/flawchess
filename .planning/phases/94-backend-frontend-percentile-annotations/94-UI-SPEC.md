## Spacing Scale

Standard 8-point scale; no new spacing tokens introduced in this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between flames inside chip (`gap-1`) |
| sm | 8px | Chip horizontal padding (`px-2`) |
| md | 16px | Default element spacing between rows |
| lg | 24px | Section padding on endgame cards |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Chip vertical padding: `py-0.5` (2px) ÔÇö tighter than the xs rung because the chip sits inline within a `text-sm` row and must not inflate row height.
- Chip-to-popover offset: `sideOffset={4}` (4px) ÔÇö matches the existing `MetricStatPopover` popover shell convention.
- Popover body padding: `px-3 py-1.5` (12px / 6px) ÔÇö matches existing `MetricStatPopover` and `InfoPopover` shells.

### Chip Color Bands (NEW ÔÇö all sourced from `theme.ts`)

The chip background fill is a solid colored pill with white text. Three discrete bands keyed by percentile:

| Band | Condition | Background constant | Value | Text color |
|------|-----------|---------------------|-------|------------|
| Red (low) | percentile < 25 | `ZONE_DANGER` (= `WDL_LOSS`) | `oklch(0.50 0.15 25)` | `oklch(0.98 0 0)` (near-white) |
| Blue (neutral) | percentile 25..75 | `GAUGE_NEUTRAL` | `oklch(0.55 0.18 260)` | `oklch(0.98 0 0)` (near-white) |
| Green (high) | percentile > 75 | `ZONE_SUCCESS` (= `WDL_WIN`) | `oklch(0.50 0.14 145)` | `oklch(0.98 0 0)` (near-white) |

Rationale: reuses the IQR zone-band palette already painted on raw % gauges and ╬öES zone bands on the same cards (D-02). No new theme constants ÔÇö import `ZONE_DANGER`, `GAUGE_NEUTRAL`, `ZONE_SUCCESS` from `@/lib/theme`.

The blue neutral uses `GAUGE_NEUTRAL` (not `ZONE_NEUTRAL`) because `GAUGE_NEUTRAL` is already the "typical skill-cohort range" blue used on the endgame gauges on the same page ÔÇö visual consistency is highest when the chip's "average tier" reads in the same hue family as the gauge's neutral band (RESEARCH.md Assumption A2).

Flame icon color: inherits the chip text color (`oklch(0.98 0 0)`) ÔÇö no separate constant needed.

Accent for chip: `ZONE_DANGER` / `GAUGE_NEUTRAL` / `ZONE_SUCCESS` are reserved for the chip fill only on this phase. They are already used on zone bands elsewhere; the chip joining that palette is additive and reinforcing, not a new usage context.

## Component Inventory

### New: `PercentileChip` (`frontend/src/components/charts/PercentileChip.tsx`)

The only new component this phase introduces. Single source for all 4 chip sites.

**Props:**

| Prop | Type | Required | Notes |
|------|------|----------|-------|
| `percentile` | `number` | yes | [0, 100] from backend; caller only renders chip when non-null |
| `flavor` | `'skill-isolating' \| 'improvement-focus'` | yes | Routes popover copy |
| `metricLabel` | `string` | yes | Used in `aria-label` and optionally popover heading |
| `testId` | `string` | yes | Provides `data-testid` on the chip trigger |

**Visual anatomy (single inline pill):**

```
[ ­ƒöÑ­ƒöÑ Top 3% ]   ÔåÉ green pill, 2 flames (p97, so top 3%)
[ Top 42% ]       ÔåÉ blue pill, no flames (p58, top 42%)
[ Top 88% ]       ÔåÉ red pill, no flames (p12, top 88%)
```

**Chip label formula:** `Top ${Math.max(1, Math.round(100 - percentile))}%`
- `Math.max(1, ÔÇª)` prevents "Top 0%" at p99.9.
- "Top 100%" at p0 is valid (honest literal render per D-07).
- Integer percent only ÔÇö no decimals (D-06).
- Capitalised "Top" (not "top") for readability as a standalone label element.

**Flame tier dispatch** (highest tier only; no stacking of lower tiers):

| Flames | Condition | Tier label |
|--------|-----------|------------|
| 0 | percentile < 90 | ÔÇö |
| 1 | percentile ÔëÑ 90 | Top 10% |
| 2 | percentile ÔëÑ 95 | Top 5% |
| 3 | percentile ÔëÑ 99 | Top 1% |

Flames rendered as `<Flame className="h-3 w-3" />` from `lucide-react`, stacked horizontally inline within the pill, `aria-hidden="true"`. Icon size `h-3 w-3` (12px) is consistent with existing inline icon convention in `EndgameMetricCard.tsx` lines 191/210.

**Pill shape:** `rounded-full px-2 py-0.5 text-sm font-medium` ÔÇö distinct from `Badge` (which uses `text-xs h-5`, too small per CLAUDE.md rule).

**Popover trigger:** The chip `<span>` itself is the trigger (`role="button" tabIndex={0}`). No adjacent HelpCircle (D-01). Uses Radix `PopoverPrimitive.Trigger asChild`.

**Hover/tap mechanic:** `HOVER_OPEN_DELAY_MS = 100` (matching `MetricStatPopover` convention). `onMouseEnter` starts a 100ms timeout before opening; `onMouseLeave` clears the timeout and closes immediately. Tap on mobile toggles open/closed via `onOpenChange`.

**Popover shell:** Radix `PopoverPrimitive.Root + Portal + Content` with `side="top" sideOffset={4}`. Animation classes:
```
data-[state=open]:animate-in data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95
data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95
data-[side=bottom]:slide-in-from-top-2 data-[side=top]:slide-in-from-bottom-2
```
Background: `bg-foreground`, text: `text-background`, body size: `text-xs`, padding: `px-3 py-1.5`, width cap: `max-w-xs`. All match the existing `MetricStatPopover` shell (RESEARCH.md ┬ºPattern 3).

**Focus:** `focus-visible:ring-2 focus-visible:ring-ring focus:outline-none` on the trigger span.

**Accessibility:**
- `aria-label={`${metricLabel} percentile: Top ${x}%`}` on the trigger
- `aria-hidden="true"` on the flame span
- `data-testid={testId}` on the trigger
- `data-testid={`${testId}-popover`}` on the popover Content

### Modified: `ScoreGapRow` (`EndgameOverallScoreGapRow.tsx`)

New optional prop `chipSlot?: ReactNode`. Renders as `{chipSlot && <span className="ml-auto">{chipSlot}</span>}` placed after the formatted value and before the existing `{tooltip}`. Default: `undefined` ÔÇö all existing consumers (including `EndgameTypeCard`) are unaffected.

Row layout after change (left-to-right flex):
```
[label]  [value]  [chip ÔÇö ml-auto, floats right]  [tooltip HelpCircle]
```

At narrow mobile widths (375px), if the row becomes too cramped, the chip is allowed to wrap to a second line via a natural flex-wrap ÔÇö label+value on line 1, chip+tooltip on line 2. This is acceptable per D-05.

### Modified: `EndgameOverallPerformanceSection.tsx`

Passes `chipSlot` to the 2 page-level `ScoreGapRow` calls:
- Achievable Score Gap row: `flavor="skill-isolating"` `testId="achievable-score-gap-percentile-chip"`
- Endgame Score Gap row: `flavor="skill-isolating"` `testId="endgame-score-gap-percentile-chip"`

Both chips render only when `percentile != null` (conditional expression at call site).

### Modified: `EndgameMetricCard.tsx`

New prop `scoreGapPercentile: number | null`. Passes `chipSlot` to its `ScoreGapRow` call:
- Parity bucket: `flavor="skill-isolating"` `testId={`${tileTestId}-percentile-chip`}`
- Conversion bucket: `flavor="improvement-focus"` `testId={`${tileTestId}-percentile-chip`}`
- Recovery bucket: `chipSlot={undefined}` ÔÇö explicitly no chip (D-12)

Parent `EndgameMetricsSection.tsx` routes `scoreGapPercentile={null}` to the recovery card.

## Copywriting Contract

### Chip labels

| State | Copy | Notes |
|-------|------|-------|
| Standard | "Top X%" | X = Math.max(1, Math.round(100 ÔêÆ percentile)); integer, no decimals |
| Edge: p = 0 | "Top 100%" | Honest literal; lowest possible ranking |
| Edge: p Ôëê 99.9 | "Top 1%" | Floor at 1 via Math.max |
| Near median: p = 50 | "Top 50%" | Literal render ÔÇö no "Ôëê average" suppression (D-07) |

Source: REQUIREMENTS.md PCTL-03, CONTEXT D-06/D-07.

### Popover body ÔÇö skill-isolating flavor

Used for: Endgame Score Gap, Achievable Score Gap, Parity ╬öES.

> "Where you rank vs all players. Mostly independent of rating ÔÇö reveals endgame ability separate from overall strength."

Source: CONTEXT D-08, locked. Within `feedback_popover_copy_minimalism.md` (WHAT + sign convention only; no jargon, no caveats, no "n=" callouts).

### Popover body ÔÇö improvement-focus flavor

Used for: Conversion ╬öES only.

> "Where you rank vs all players. Conversion tracks rating closely ÔÇö if you're in the lower tiers here, this is one of the biggest single improvements available to your ELO."

Source: CONTEXT D-08, locked. Both popover bodies must be read side-by-side during UAT to confirm they read as deliberate companions, not contradictions. The skill-isolating copy says "mostly independent of rating"; the improvement-focus copy says "tracks rating closely" ÔÇö these describe distinct properties of distinct metrics and are not contradictory when encountered in sequence on the same page.

### Empty state (chip absent)

No chip renders when `{metric}_percentile` is `null`. No visual placeholder, no greyed-out chip, no "N/A" label. The row renders exactly as before Phase 94 ÔÇö value, tooltip, nothing more. The chip is additive; its absence is invisible.

### Error state

No chip-specific error state. If the endgame API call fails, the existing row-level `isError` handling applies (per CLAUDE.md "Always handle `isError` in data-loading ternary chains" ÔÇö existing pattern in the parent component). No new error copy needed.

### Destructive actions

None in this phase. No chip removal, no data mutation, no destructive confirmation needed.

### Primary CTA

No new CTA in this phase. The chip is informational only; the primary CTA on the Endgame page remains "Generate Insights" (existing, unchanged).

## Mobile Parity (375px)

- Chip stays inline on the same row when space allows (label + value + chip + tooltip all fit).
- When the row is too narrow at 375px, flex wrapping moves the chip (and tooltip) to a second line ÔÇö acceptable per D-05.
- Chip font size is `text-sm` (16px in this project's cascade) ÔÇö readable at 375px.
- Flame icons are `h-3 w-3` (12px) ÔÇö small but legible; 3 flames at 12px = 36px total, still compact inline with the chip text.
- Tap opens the popover (no hover-only requirement); `onOpenChange` handles toggle on touch.

Source: CLAUDE.md "Always apply changes to mobile too", REQUIREMENTS.md PCTL-05, RESEARCH.md Open Question 1.
