## What Was Built

**Frontend cutover (Tasks 1-2):**

- `frontend/src/lib/chess.ts` — new shared file exporting `plysToFullMoves(plyCount: number): number { return Math.ceil(plyCount / 2); }`. Imported by all three cards to satisfy Knip (no dead export).
- `frontend/src/types/api.ts` L111: `move_count: number | null` → `ply_count: number | null`.
- `frontend/src/types/library.ts` L65, L208: both `move_count` fields renamed to `ply_count`.
- `GameCard.tsx`: `game.move_count !== null` → `game.ply_count !== null`; value `{game.move_count}` → `{plysToFullMoves(game.ply_count)}`. Desktop-only `<span>&nbsp;Moves</span>` preserved.
- `FlawCard.tsx`: same pattern with `flaw.ply_count`; desktop-only Moves span preserved.
- `LibraryGameCard.tsx`: same pattern; single-string markup `{plysToFullMoves(game.ply_count)} Moves` per Pitfall 6.
- Test fixtures: renamed `move_count → ply_count` and doubled numeric values (`40→80`, `42→84`, `30→60`, `41→82`, `58→116`); null fixture left as `ply_count: null`; rendered-text assertions unchanged.
- Verification: `grep -rn "move_count" frontend/src/` → 0 results; `npm run lint`, `npm test -- --run` (880 passed), `npm run knip` all green.

**Benchmark §5 fast path (Tasks 3-4):**

- `chapter5.py` `user_moves_per_game` CTE: replaced the 87M-row `game_positions` ply-parity scan with `FLOOR(g.ply_count / 2.0)` / `CEIL(g.ply_count / 2.0)` read from a `JOIN games g ON g.id = bg.game_id`. Result column name `user_moves` preserved so downstream aggregation is unaffected.
- Benchmark DB prerequisite: the benchmark DB was on Alembic head `f8a2d1c9b345` (still had `move_count`). Ran the migration chain through `07994baf3b15` via `DATABASE_URL=... alembic upgrade head`, which added `ply_count`, backfilled via `max(ply)` GROUP BY, and dropped `move_count`. The migration ran cleanly (no NULL window, sub-10s on the benchmark DB's ~87M game_positions rows).
- §5 fast-path gate: `uv run pytest tests/scripts/benchmarks/test_chapter5_diff.py -v` — **2 passed in 2:28** (the chapter5.compute() fast path). Both tests pass with expected values unchanged from before the basis change.
- §5 review outcome: all 15 metric sections, viability diagnostic, and headline table reviewed. **Numbers are byte-identical to the prior game_positions-based values** — FLOOR/CEIL(ply_count/2) reproduces the old ply-parity count exactly (proven by theory and confirmed by the green diff gate). Pooled n = 4,644, all verdicts unchanged. Updated the §5 D-01 callout in `benchmarks-latest.md` to note the `games.ply_count` basis and removal of the `game_positions` scan.
