**Part A: Create `frontend/src/components/move-explorer/MoveExplorer.tsx`**

This component receives move data and renders the explorer table. It does NOT call useNextMoves itself -- the hook is called in Dashboard so Dashboard can also compute arrows.

Props interface:
```typescript
import type { NextMoveEntry } from '@/types/api';

interface MoveExplorerProps {
  moves: NextMoveEntry[];
  isLoading: boolean;
  isError: boolean;
  position: string;           // current FEN, for SAN-to-squares resolution
  onMoveClick: (from: string, to: string) => void;
}
```

Implementation details:

1. **SAN to squares resolution:** Use `chess.js` to resolve SAN to from/to squares for click handling. On each render (when `position` or `moves` changes, via `useMemo`):
   ```typescript
   import { Chess } from 'chess.js';
   const moveMap = useMemo(() => {
     const chess = new Chess(position);
     const legalMoves = chess.moves({ verbose: true });
     return new Map(legalMoves.map(m => [m.san, { from: m.from, to: m.to }]));
   }, [position]);
   ```

2. **Row click handler:** When a row is clicked, look up `moveMap.get(entry.move_san)` to get `from`/`to`, then call `onMoveClick(from, to)`. If the SAN is not in moveMap (shouldn't happen with valid data), do nothing.

3. **Loading state:** Render 3 skeleton rows using `div` elements with `h-8 bg-muted animate-pulse rounded` classes. Wrap in a container with `data-testid="move-explorer-loading"`.

4. **Error state:** Show heading "Could not load moves" (`text-sm font-semibold text-foreground`) and body "There was a problem fetching next moves. Try adjusting filters or refreshing." (`text-xs text-muted-foreground`). Container has `data-testid="move-explorer-empty"`.

5. **Empty state (moves.length === 0, not loading, not error):** Show heading "No moves found" (`text-sm font-semibold text-foreground`) and body "No recorded games continue from this position with the current filters." (`text-xs text-muted-foreground`). Container has `data-testid="move-explorer-empty"`.

6. **Table layout:** Use an HTML `<table>` with `w-full text-sm` class. Table has `data-testid="move-explorer-table"`.
   - Header row: 3 columns with `text-xs text-muted-foreground font-normal` text:
     - "Move" -- `w-[3rem]` (left aligned)
     - "Games" -- `w-[4rem]` (right aligned)
     - "Results" -- remaining width
   - Data rows: Each `<tr>` has:
     - `data-testid="move-explorer-row-{move_san}"` (e.g., `move-explorer-row-e4`)
     - `className="cursor-pointer hover:bg-accent min-h-[44px]"` (44px touch target)
     - `role="button"` for accessibility
     - `tabIndex={0}` for keyboard navigation
     - `onClick` handler calling `onMoveClick(from, to)` via moveMap lookup
     - `onKeyDown` handler: Enter/Space triggers the same click action

7. **Move column (first cell):** Display `entry.move_san` in `text-sm text-foreground font-normal`.

8. **Games column (second cell, right aligned):**
   - Display `entry.game_count` with `tabular-nums` class for alignment
   - When `entry.transposition_count > entry.game_count`, show inline transposition indicator:
     ```tsx
     import { ArrowLeftRight } from 'lucide-react';
     import { Tooltip, TooltipTrigger, TooltipContent, TooltipProvider } from '@/components/ui/tooltip';

     {entry.transposition_count > entry.game_count && (
       <TooltipProvider>
         <Tooltip>
           <TooltipTrigger asChild>
             <span data-testid={`move-explorer-transpose-${entry.move_san}`}>
               <ArrowLeftRight className="inline h-4 w-4 text-muted-foreground ml-1" />
             </span>
           </TooltipTrigger>
           <TooltipContent>
             Position reached in {entry.transposition_count} total games ({entry.transposition_count - entry.game_count} via other move orders)
           </TooltipContent>
         </Tooltip>
       </TooltipProvider>
     )}
     ```

9. **Results column (third cell):** Render a mini WDL bar:
   ```tsx
   import { WDL_WIN, WDL_DRAW, WDL_LOSS } from '@/components/results/WDLBar';

   <div
     className="flex h-3 w-full overflow-hidden rounded"
     title={`W: ${entry.win_pct.toFixed(0)}% D: ${entry.draw_pct.toFixed(0)}% L: ${entry.loss_pct.toFixed(0)}%`}
   >
     {entry.win_pct > 0 && <div style={{ width: `${entry.win_pct}%`, backgroundColor: WDL_WIN }} />}
     {entry.draw_pct > 0 && <div style={{ width: `${entry.draw_pct}%`, backgroundColor: WDL_DRAW }} />}
     {entry.loss_pct > 0 && <div style={{ width: `${entry.loss_pct}%`, backgroundColor: WDL_LOSS }} />}
   </div>
   ```
   If all percentages are 0 (no games), render a full-width `bg-muted` bar.

10. **Component wrapper:** The outermost `div` has `data-testid="move-explorer"` and a heading "Move Explorer" with `text-base font-semibold` and `data-testid="move-explorer-heading"`. Add `p-4` padding.

11. **Mobile responsive:** The table compresses naturally. Move column uses `truncate` class. All styles use the semantic Tailwind tokens (`text-foreground`, `text-muted-foreground`, `bg-card`, `border-border`).

3. **Compute arrows** using `useMemo` based on `nextMoves.data` and `chess.position`:
   ```typescript
   const boardArrows = useMemo(() => {
     if (!nextMoves.data?.moves.length) return [];

     const chessInstance = new Chess(chess.position);
     const legalMoves = chessInstance.moves({ verbose: true });
     const moveMap = new Map(legalMoves.map(m => [m.san, { from: m.from, to: m.to }]));

     const moves = nextMoves.data.moves;
     const maxCount = Math.max(...moves.map(m => m.game_count), 1);
     const MIN_OPACITY = 0.15;

     return moves
       .map(entry => {
         const squares = moveMap.get(entry.move_san);
         if (!squares) return null;
         const opacity = MIN_OPACITY + (1 - MIN_OPACITY) * (entry.game_count / maxCount);
         const alpha = Math.round(opacity * 255).toString(16).padStart(2, '0');
         return {
           startSquare: squares.from,
           endSquare: squares.to,
           color: `#1d6ab1${alpha}`,
         };
       })
       .filter(Boolean);
   }, [nextMoves.data, chess.position]);
   ```

4. **Pass arrows to ChessBoard** -- add `arrows={boardArrows}` to the `<ChessBoard>` in the leftColumn:

   ```tsx
   <ChessBoard

     flipped={boardFlipped}

   ```
