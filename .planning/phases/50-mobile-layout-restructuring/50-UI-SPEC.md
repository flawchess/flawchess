## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), vertical column inter-button gap, notification dot inset |
| sm | 8px | Unified row horizontal gap (`gap-2`), sticky-wrapper inner padding |
| md | 16px | Default element spacing (content-area blocks) |
| lg | 24px | Section padding (not expected in this phase) |
| xl | 32px | Layout gaps (not expected in this phase) |
| 2xl | 48px | Major section breaks (not expected in this phase) |

**Exceptions (touch-target and layout-alignment values only):**
- **Unified control row height:** `h-11` = **44px** — meets the 44px touch-target minimum for the three icon buttons and matches Endgames after its alignment pass
- **Vertical board-action column button height:** `h-12` = **48px** — up from current `h-8` (32px); 5 buttons × 48px + 4 gaps × 4px = 256px, which stays within the current board footprint at 375px viewport
- **Vertical board-action column inter-button gap:** `gap-1` = **4px** — up from current `gap-0.5` (2px); preserves the segmented-column look while giving each button more breathing room
- **Collapse handle height:** `h-11` = **44px** — up from the current `py-0.5` sliver (~4px); meets touch-target minimum, matches the unified row height so the two form a visually even strip below the board
- **Sticky wrapper vertical gap between board region and unified row:** `mt-1` = **4px** — keeps the grid-rows collapse animation tight against the row
- **Notification dot size / position:** `h-2.5 w-2.5` at `top-0.5 right-0.5` — **unchanged** from current implementation (`Openings.tsx:980, 1001`). Dots move with their buttons; no visual adjustment needed in the new position.

Preserved unchanged on both filter and bookmark buttons in their new positions inside the unified row. The pulsing red dot is a semantic signal (not an accent color) and is documented as an exception to the brown-only accent rule.

The previous values were tuned against an opaque sticky wrapper; against backdrop-blur they read too bright. The chevron icon stays `text-muted-foreground`.

## Interaction Contract

### Unified Control Row (new — Openings mobile)

- **Container:** `<div className="flex items-center gap-2 h-11 px-1" data-testid="openings-mobile-control-row">` — placed inside the sticky top wrapper but **outside** the `grid transition-[grid-template-rows]` collapse region (D-03). Exactly one flex row.
- **Left-to-right order (D-02):**
  1. `TabsList variant="brand"` — `className="flex-1 h-full"` (flex-fills remaining space, height matches the row)
     - 3 `TabsTrigger` children: `explorer` (Moves) / `games` (Games) / `stats` (Stats) with `text-xs! flex-1` each — unchanged from current mobile tabs
  2. Color toggle button — `className="h-11 w-11 shrink-0 bg-toggle-active text-toggle-active-foreground hover:bg-toggle-active/80"`, `variant="ghost"`, `size="icon"`, existing handler (`filters.color` toggle + `setBoardFlipped` + navigate to explorer/games)
  3. Bookmark button — `className="h-11 w-11 shrink-0 bg-toggle-active text-toggle-active-foreground hover:bg-toggle-active/80 relative"`, `variant="ghost"`, `size="icon"`, `onClick={openBookmarkSidebar}`, existing notification dot logic
  4. Filter button — `className="h-11 w-11 shrink-0 bg-toggle-active text-toggle-active-foreground hover:bg-toggle-active/80 relative"`, `variant="ghost"`, `size="icon"`, `onClick={openFilterSidebar}`, existing notification dot logic
- **Scroll behavior:** plain flex, **no** `overflow-x-auto`, **no** swipe-to-paginate, **no** carousel (D-04). At 375px viewport: 44px + 44px + 44px = 132px reserved for icon buttons, `gap-2` × 3 = 24px of gaps, leaving ~219px (minus container padding) for the three-item `TabsList` — comfortable at `text-xs`. This is the exact target viewport for the success criterion "no horizontal scroll at 375px".
- **Persistence:** the row stays rendered and usable when `boardCollapsed === true`. This is the core Phase 50 ergonomic win.
- **Tooltips:** each icon button keeps its current `Tooltip` wrapper with `side="left"` — unchanged. Tooltip text: `Playing as {filters.color}` (color), `"Open bookmarks"`, `"Open filters"`.
- **aria-labels:** `"Playing as {filters.color}, tap to switch"`, `"Open bookmarks"`, `"Open filters"` — unchanged.

### Vertical Board-Action Column (shrunk — Openings mobile)

- **Container:** `<div className="flex flex-col gap-1">` (up from current `gap-0.5`).
- **Contents (5 items, D-06):**
  1. `BoardControls vertical` — renders Reset, Back, Forward, Flip, and the `infoSlot` (InfoPopover). **NOTE:** `BoardControls.tsx` currently hardcodes `h-8 w-8` per button; Phase 50 must bump these to `h-12 w-12` **only when** `vertical` is true (or introduce a size prop). This is the single component-level change outside `Openings.tsx`.
  2. Inter-button gap: `gap-1` (4px) between the 5 buttons inside `BoardControls`.
- **Width:** `w-12` (48px) — exact width of the square icon buttons. Column width does **not** expand beyond the button footprint (D-07). The board keeps its current horizontal footprint.
- **Removed from this column:** color toggle, filter button, bookmark button (moved to unified row per D-01).
- **Info popover:** **stays in the vertical column** inside `BoardControls.infoSlot`. Decision rationale: with 5 buttons at `h-12 + gap-1` = 256px the column is balanced against the board, and moving `info` into the unified row would crowd a 4th icon-button footprint into a row already at 375px ceiling. Info lives where it documents (the board), not where the tabs live.

### Sticky Top Wrapper (restructured)

- **Container:** `<div className="sticky top-0 z-20 bg-background/80 backdrop-blur-md border-b border-border">` — replaces the current `sticky top-0 z-20 bg-background shadow-[0_6px_20px_rgba(0,0,0,0.8)]`.
- **Children in order:**
  1. **Collapsible board region:** `<div className="grid transition-[grid-template-rows] duration-200 ease-in-out ${boardCollapsed ? 'grid-rows-[0fr]' : 'grid-rows-[1fr]'}">` containing `<div className="overflow-hidden">` → `<div className="flex items-stretch gap-1 pb-1">` → ChessBoard (flex-1) + vertical board-action column. Mechanism unchanged from current, just wrapping fewer children.
  2. **Unified control row** (see above). `mt-1` spacing from the collapsible region.
  3. **Collapse handle** (see below).
- **z-index:** remains `z-20`. Filter/bookmark drawers use Vaul's own stacking (z-50+) and still overlay correctly. No change required.
- **Background when board visible:** minimal visual change — the `react-chessboard` squares are opaque, so backdrop-blur only shows behind the padding, the unified row, and the collapse handle.
- **Background when board collapsed:** the remaining strip (unified row + collapse handle) becomes a glassy blurred surface. Scrolling content below is faintly visible through the blur — this is the intended "sibling of the desktop sidebar" look (D-10).

### Collapse Handle (enlarged)

- **Container:** `<button className="flex w-full items-center justify-center h-11 touch-none bg-white/5 border-t border-white/10 rounded-b-md">` — up from `py-0.5` (~4px) to `h-11` (44px).
- **Icon:** `ChevronDown className="h-5 w-5 text-muted-foreground transition-transform duration-200 ${boardCollapsed ? 'rotate-0' : 'rotate-180'}"` — bumped from `h-4 w-4` to `h-5 w-5` so it scales with the larger hit area.
- **Position:** below the unified row (D-11). This preserves current muscle memory while fixing the touch-target issue (D-12).
- **Touch handlers:** `onTouchStart={handleHandleTouchStart}` and `onTouchEnd={handleHandleTouchEnd}` — unchanged swipe logic.
- **Click handler:** `onClick={() => setBoardCollapsed((c) => !c)}` — unchanged.
- **aria-label:** `{boardCollapsed ? 'Expand board' : 'Collapse board'}` — unchanged.
- **data-testid:** `btn-board-collapse-handle` — unchanged.

### Endgames Mobile — Visual Alignment Only (D-13, D-14)

- **Container:** `<div className="sticky top-0 z-20 flex items-center gap-2 h-11 bg-background/80 backdrop-blur-md border-b border-border px-1" data-testid="endgames-mobile-control-row">` — replaces the current `sticky top-0 z-20 flex items-center gap-2 pb-2`.
- **TabsList:** `variant="brand" className="flex-1 h-full"` — change from current `flex-1 h-9!` to `flex-1 h-full` so the track grows into the unified 44px row height. No new tabs, no new triggers.
- **Filter button:** `className="h-11 w-11 shrink-0 bg-toggle-active text-toggle-active-foreground hover:bg-toggle-active/80"` — bumped from `h-9 w-9` to `h-11 w-11` to match Openings.
- **No other changes:** no bookmark button (endgames has no bookmarks), no color toggle (no board to flip), no board collapse handle (no board). Purely a background + height + testid alignment pass.
- **Result:** Endgames and Openings mobile sticky top rows become visual siblings at exactly 44px tall with the same backdrop-blur treatment.

### Drawers (Filter and Bookmarks) — Unchanged

- Vaul `Drawer direction="right"` with existing `DrawerContent className="!w-full sm:!w-3/4 !bottom-auto !rounded-bl-xl max-h-[85vh]"` — **no changes** (D-15, D-16). Only the trigger button positions move.
- Open handlers: `setFilterSidebarOpen(true)` / `setBookmarkSidebarOpen(true)` (Openings), `setMobileFiltersOpen(true)` (Endgames) — unchanged.

## Accessibility Contract

- **Every new interactive element reuses its current `aria-label`** — no re-authoring.
- **Touch targets:** all interactive elements in the sticky top region meet the **44×44px** minimum (three unified-row icon buttons, tab triggers via flex-filled 44px row, collapse handle). Vertical board-action column is an explicit exception at **48×48px** — larger than the minimum.
- **Semantic HTML:** `<button>` for the collapse handle (already correct), `<Button>` (→ `<button>`) for the icon buttons, Radix `Tabs` provides correct `role="tablist"` / `role="tab"` / `aria-selected` semantics automatically.
- **Tooltip on icon-only buttons:** already in place via the `Tooltip` wrapper — preserved as-is. These exist on both desktop (hover) and mobile (long-press).
- **Focus order:** tabs first, then color toggle, then bookmark, then filter — reading order matches visual left-to-right order.
- **`overflow-x-hidden` on body:** the layout contract is **no horizontal overflow at 375px viewport** — this is a hard success criterion. The `min-w-0` on the mobile `<Tabs>` root is preserved from the current code (`Openings.tsx:913`) to allow the flex children to shrink.

## Responsive & Layout Constraints

- **Breakpoint:** all changes live inside `md:hidden` branches on both `Openings.tsx` and `Endgames.tsx`. The `md:` breakpoint is `768px` — anything ≥768px sees the Phase 49 desktop `SidebarLayout`, untouched.
- **Target viewport:** 375px (iPhone SE / iPhone 13 mini baseline) — the hard success criterion from ROADMAP.md Phase 50 criterion 3.
- **Horizontal budget at 375px:**
  - Page body: 375px
  - Subtract app-level horizontal padding (if any — inspect main container during execution)
  - Sticky wrapper unified row, from right to left: 44 (filter) + 8 (gap) + 44 (bookmark) + 8 (gap) + 44 (color) + 8 (gap) = **156px** for the three icon buttons and their separators
  - `TabsList` flex-1 fills the remaining ~219px minus the row's own `px-1` container padding (~8px) = ~**211px** for 3 tab labels at `text-xs` (avg label ~42px per tab, well within budget)
- **Horizontal budget for the board region at 375px:**
  - Page body: 375px
  - Subtract `gap-1` (4px) between board and vertical column
  - Subtract vertical column width `w-12` (48px)
  - Board area: ~**323px** — unchanged from the current layout (current column is `w-9` ≈ 36px, so the board actually **shrinks by 12px**). This is acceptable because the board is square and react-chessboard will simply render 12px narrower; the success criterion is "remains fully usable", not "unchanged pixel width".
- **Vertical budget when board visible:** the 5-button vertical column at `h-12 + gap-1` sums to `48 × 5 + 4 × 4 = 256px`. On a 375px-wide board this is ~81% of the board's vertical footprint — the column now visually matches the board height more closely than the current 8-button `h-8 + gap-0.5` (258px). Near-identical total.
