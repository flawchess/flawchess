<objective>
Close REVIEW.md WR-01: the eval-drain post-commit Stage B fan-out at `app/services/eval_drain.py:556-564` iterates affected users and calls `count_pending_evals(read_session, user_id)` once per user inside a single read session. Worst case with `_DRAIN_BATCH_SIZE=10` is 10 sequential DB round-trips on the hot drain path before the next batch can begin. async def users_with_zero_pending(
    session: AsyncSession,
    user_ids: Sequence[int],
) -> list[int]:
    """Return the subset of `user_ids` whose pending-eval count is zero.

    Single aggregated query — replaces the per-user `count_pending_evals` loop in
    `eval_drain.py` post-commit Stage B fan-out (WR-01 fix). Hits the existing
    `ix_games_evals_pending` partial index.

    Args:
        session: AsyncSession (read-only is fine).
        user_ids: Sequence of internal user PKs to check.

    Returns:
        List of user_ids (subset of input) where COUNT(*) FILTER
        (WHERE evals_completed_at IS NULL) == 0. Returned in unspecified order.
        Empty list if no input users have zero pending evals, or if user_ids is empty.
    """

# FROM games
# WHERE user_id = ANY(:affected_ids)
# GROUP BY user_id
# HAVING count(*) FILTER (WHERE evals_completed_at IS NULL) = 0
