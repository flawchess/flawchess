## Decisions Made

- `import_jobs.status='completed'` used as the pre-skip filter instead of the plan-spec's `games.status='completed'` (games table has no status column — Rule 1 auto-fix)
- `CAST(:metric AS benchmark_metric)` in raw `text()` queries — asyncpg infers `VARCHAR` for all bindparams, causing `operator does not exist: benchmark_metric = character varying`; explicit CAST resolves this
- `engine.url.render_as_string(hide_password=False)` for the V4 guard URL comparison — `str(engine.url)` masks the password with `***`, so the comparison with the original URL always fails

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] `_iter_users` used non-existent `games.status` column**
- **Found during:** Task 1 (smoke test and test execution)
- **Issue:** Plan spec said `WHERE g.status = 'completed'` but the `games` table has no `status` column (that column lives on `import_jobs`)
- **Fix:** Changed query to `WHERE ij.status = 'completed'` using `import_jobs` table; added docstring note
- **Files modified:** `scripts/backfill_user_percentiles.py`
- **Committed in:** ebd65c81

**2. [Rule 1 - Bug] ENUM type mismatch in raw SQL `text()` queries**
- **Found during:** Task 1 (test execution)
- **Issue:** asyncpg infers all bindparams as `VARCHAR`; comparing against `benchmark_metric` ENUM column raises `UndefinedFunctionError: operator does not exist: benchmark_metric = character varying`
- **Fix:** Used `CAST(:metric AS benchmark_metric)` in `_row_exists` and the percentile probe query; same fix applied in test helpers
- **Files modified:** `scripts/backfill_user_percentiles.py`, `tests/scripts/test_backfill_user_percentiles.py`
- **Committed in:** ebd65c81

**3. [Rule 1 - Bug] V4 engine URL comparison used masked URL**
- **Found during:** Task 1 (first test run)
- **Issue:** `str(engine.url)` masks the DB password with `***`; comparing against the original URL (with real password) always fails the V4 guard
- **Fix:** Used `engine.url.render_as_string(hide_password=False)` for the comparison
- **Files modified:** `scripts/backfill_user_percentiles.py`
- **Committed in:** ebd65c81
