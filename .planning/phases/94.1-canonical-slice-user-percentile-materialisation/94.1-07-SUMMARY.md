# Phase 94.1 Plan 07: Chip Decoupling + Canonical-Slice Tooltip Summary

**One-liner:** Rewire all 4 endgame chip fields from per-request `interpolate_percentile()` to table-backed `fetch_for_user()` reads, and add a one-phrase canonical-slice clarifier to both PercentileChip popover flavors.

## What Was Built

**Task 1 (D-12 / PCTL-07): Chip read path rewired to materialised table**

Four `interpolate_percentile(metric_id, filter_applied_value)` call sites in `endgame_service.py` were removed and replaced with a single `await user_benchmark_percentiles_repository.fetch_for_user(session, user_id=user_id)` call at the top of both `get_endgame_overview` and `get_endgame_performance`. The returned `Mapping[CdfMetricId, PercentileRow]` is threaded into `_get_endgame_performance_from_rows` and `_compute_score_gap_material` as the new `percentile_rows` optional parameter.

The import of `interpolate_percentile` was removed from `endgame_service.py`. The helper remains in `global_percentile_cdf.py` as it is still consumed by the Plan 05 compute service.

Both compute helpers use `| None = None` defaults and a `_effective_rows` guard so existing unit tests that call them directly (without a DB session) continue to produce `None` chip fields.

**Task 2 (D-13): Canonical-slice clarifier in PercentileChip.tsx popovers**

Both `PercentileChipPopoverBody` flavors now end with: "Reflects your career under matched conditions, not the current filtered view." This satisfies the D-13 test regex and stays within the 280-char minimalism budget (skill-isolating: 194 chars, improvement-focus: 247 chars).

Three previously-skipped tests in `PercentileChip.test.tsx` were enabled (`it.skip` → `it`).
