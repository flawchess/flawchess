### Gaps Summary

The phase delivered the schema, the wiring, the chip rewire, and the backfill script structure — but its core behavior (compute-and-write per-user canonical-slice values) is broken at runtime by a SQL-parser interaction:

`selected_users_cte(source="single_user")` emits `"selected_users AS (SELECT :user_id::int AS user_id)"`. When this is passed to `sqlalchemy.text()`, the parser **does not recognize `:user_id` as a bindparam** because it is immediately followed by the Postgres-only `::int` cast operator. SQLAlchemy's tokeniser treats `:user_id::int` as a literal, leaving `text().compile().params` empty. The downstream `.bindparams(user_id=...)` call then raises `ArgumentError("This text() construct doesn't define a bound parameter named 'user_id'")`.

This is masked because:
1. `compute_stage_a` / `compute_stage_b` catch `Exception` and route it to Sentry (per Plan 05's D-04 contract). The trigger sites (`import_service`, `eval_drain`, backfill loop) see no exception.
2. The Plan 02 happy-path tests are `pytest.skip("implementation pending Plans 05/06")` — they were scaffolded but never enabled.
3. The Plan 02 SC-7 parity test is `pytest.skip("implementation pending Plan 03")` — never enabled.
4. The Plan 08 backfill tests seed users with zero games. The broken path returns "no row written", which matches the expected `no_canonical_games` outcome for zero-game users. Tests pass for the wrong reason.
5. The dev DB backfill smoke test reports `no_canonical_games=19` for all users, but **users 8 and 14 have 18,226 and 15,208 equal-footing rated games** respectively. The summary is reporting a phantom outcome.

REVIEW.md WR-03 flagged this as a possible runtime BLOCKER and asked for end-to-end verification against dev DB. That verification has now been performed and confirms the bug.

**Severity assessment:** This is a **goal-not-achieved** failure. The phase artifact exists structurally but does not produce its intended observable behavior (chip lighting up for any user). The phase cannot be considered complete; one tiny fix (change `:user_id::int` to `CAST(:user_id AS int)` or `SELECT CAST(:user_id AS INTEGER) AS user_id`) plus a real-data integration test should close all three gaps simultaneously.

**Suggested next-phase scope (compact):**
1. Fix `selected_users_cte(source="single_user")` to use a SQLAlchemy-parseable bindparam.
2. Add a Plan 05 integration test that seeds a user with canonical-slice-qualifying games and asserts a row IS written after `compute_stage_a` / `compute_stage_b`. Enable the SC-7 parity test.
3. Fix REVIEW.md IN-03 (`_compute_and_count` classifier — return False when row exists but percentile IS NULL).
4. Re-run backfill on dev to confirm real data flow; capture human-UAT for filter-toggle invariance.
5. Decide on REVIEW.md CR-01 (n_games column semantics) and WR-02 (cdf_snapshot column) — either rename columns or compute actual game count / CDF snapshot date.
