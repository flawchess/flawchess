# Phase 138 — UI Design Contract

> Visual and interaction contract for the standalone `/analysis` page shell + the "Analyze position"
> entry point on the Openings Explorer. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Scope note (read first).** This phase invents NO new visual primitives. It composes
> already-built, already-styled components from Phases 136/137 (`EvalBar`, `EngineLines`,
> `VariationTree`, `ChessBoard`, `BoardControls`) into a responsive page, and adds one secondary
> button to Openings. The contract below is therefore about **page-level composition** (responsive
> layout, the engine-loading state, the Suspense fallback, the entry button), not about colors,
> type, or spacing of the child components — those are locked by their own phases. All tokens come
> from the existing FlawChess design system; do not introduce new ones.

| Property | Value |

| Preset | not applicable |

(`oklch(0.145 0 0)`); cards are `--card` (`oklch(0.205 0 0)`). All values below assume dark mode.

## Spacing Scale

The project uses Tailwind v4's default 4px grid (CLAUDE.md: "4px spacing grid"). Phase 138 uses these
steps for the page-shell composition only.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px (`gap-1`) | Icon-to-label gap inside the "Analyze position" button |
| sm | 8px (`gap-2`, `p-2`) | Eval-bar↔board gutter on desktop; control cluster gaps |
| md | 16px (`gap-4`, `p-4`) | Default gap between layout regions (board column ↔ side panel) |
| lg | 24px (`gap-6`, `p-6`) | Page container padding (desktop); Suspense fallback padding |
| xl | 32px (`gap-8`) | Major desktop column gap (board area ↔ engine/tree panel) when space allows |

Exceptions: none. Touch targets rely on the existing `Button` sizing (`h-8`/`h-9`, `size-8`
icon buttons) which already meet the interaction floor; do not hand-roll smaller hit areas.

## Typography

Inherited from the design system — Phase 138 declares only what the page shell itself renders
(child components carry their own type). CLAUDE.md floor: **`text-sm` (14px) minimum** in all new
code (no `text-xs` except opt-in tooltips — none in this phase).

| Role | Size | Weight | Line Height | Where in this phase |
|------|------|--------|-------------|---------------------|
| Body / status | 14px (`text-sm`) | 400 (Nunito Sans regular) | 1.5 (`leading-normal`) | "Loading engine…" indicator, Suspense fallback copy |
| Label / button | 14px (`text-sm`) | 500 (`font-medium`, from `Button`) | 1.5 | "Analyze position" button label |
| Section heading (optional) | 16px (`text-base`) | 600 (`font-semibold`) | 1.2 (`leading-tight`) | Optional panel labels ("Engine", "Moves") if the layout adds them |

Two weights only: **400 regular** (body/status) and **500–600 medium/semibold** (labels, button,
any heading). Do not introduce additional weights. The page must not set a `<h1>` analysis title
(D-05: no nav chrome; `ROUTE_TITLES` drives the mobile header text instead).

## Color

Forced dark mode. The page shell itself paints almost nothing — it is a layout container around
pre-themed children. Semantic colors below come from CSS variables / `theme.ts`; **never hard-code
semantic hex/oklch in the page** (CLAUDE.md theme rule).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` `oklch(0.145 0 0)` (charcoal) | Page background, board-column backdrop, gutters |
| Secondary (30%) | `--card` `oklch(0.205 0 0)` | Side/below panel surface holding EngineLines + VariationTree (if wrapped in a `Card`/panel) |
| Accent (10%) | `--brand-brown` `#8B5E3C` (+ `brand-brown-light` `#A07850`) | "Analyze position" entry button (`brand-outline`), engine-enabled toggle active state, current-move highlight (already inside `VariationTree`) |
| Destructive | `--destructive` | Not used this phase — no destructive actions |

**Accent reserved for:** (1) the "Analyze position" entry button on Openings (`variant="brand-outline"`),
(2) the engine on/off toggle's active/enabled affordance, (3) the current-node highlight that
`VariationTree` already renders. Accent is NOT applied to the eval bar (grayscale `EVAL_BAR_WHITE`/
`EVAL_BAR_BLACK` from `theme.ts`), engine-line chips, board, or the "Loading engine…" text (that uses
`text-muted-foreground`). Do not paint generic interactive elements brown.

**Component-owned palettes (do not override from the page):** `EvalBar` → `EVAL_BAR_WHITE`/`EVAL_BAR_BLACK`;
board squares → `BOARD_DARK_SQUARE`/`BOARD_LIGHT_SQUARE`; best-move arrow (if shipped, optional) →
`BEST_MOVE_ARROW`. All already imported by the child components from `theme.ts`.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (entry, on Openings) | **"Analyze position"** — verb + noun; icon: lucide `Microscope` (or `Sparkles`); `variant="brand-outline"` (secondary action per CLAUDE.md). On a narrow mobile control row the label may collapse to icon-only with `aria-label="Analyze position"`. |
| Suspense fallback | **"Loading analysis board…"** — `text-sm text-muted-foreground`, `data-testid="analysis-loading"`, padded (`p-6`). Shown only while the lazy chunk downloads. |
| Engine-loading indicator (eval area) | **"Loading engine…"** — shown when `engineEnabled && !engine.isReady`; lives in the eval region only; board + stepper stay interactive. `text-sm text-muted-foreground`, with an optional lucide `Loader2` spinner (`animate-spin`). `data-testid="analysis-engine-loading"`. |
| Empty / blank free-play state | No empty-state copy. A no-param visit silently renders the standard start position (D-05 lean: silent, no onboarding chrome). The board is never "empty". |
| Engine-off state (eval area) | When the user toggles the engine off (ENGINE-04), the eval area shows a calm **"Engine off"** rest state (`text-sm text-muted-foreground`) rather than a spinner — distinguishes deliberate-off from loading. |
| Error state | The shell has no data fetch (no API in this phase). The one failure mode — a malformed `?fen=` param — is handled by **silently falling back to the standard start position** (security FEN-guard), not an error banner. No "Failed to load" copy needed. |
| Destructive confirmation | None — no destructive actions in this phase. ("Reset" clears the in-memory tree back to root via `loadMainLine([], rootFen)`; it is non-destructive of any persisted data and needs no confirmation.) |

Copy voice: terse, lowercase-after-first-word, no em-dashes (CLAUDE.md). Match the existing app's
status-line tone ("Loading…", "Analyzing…").

## Page Composition Contract (the load-bearing section for this phase)

> This replaces "new component visuals" — the deliverable is layout, not primitives. Tokens and
> component props are verified in `138-RESEARCH.md` § Exact Composition Contract.

### Layout — desktop (`sm:`/`lg:` and up)
Lichess/chess.com analysis convention, board-centric:

```
┌───────────────────────────────────────────────────────────┐
│  [EvalBar]  [   ChessBoard (square)   ]   [ Side panel ]   │
│   thin       wrapped in board.containerRef   ├ EngineLines │
│   vertical   (←/→ keys, drag + click moves)  ├ VariationTree (desktop list) │
│              [ BoardControls: ◀ ▶ ⟲ ⤢ + engine toggle ]    │
└───────────────────────────────────────────────────────────┘
```
- **EvalBar** sits flush-left of the board, height matched to the board via its `className`
  (white always at top — no `orientation` prop). Gutter `gap-2` (8px).
- **Board** is the visual anchor: square, max-width capped so it doesn't dominate ultrawide
  (e.g. `max-w-[min(70vh,640px)]`), centered in its column. Wrap in the `board.containerRef` div
  (`tabIndex={0}` safety net for ←/→ keys per Pitfall 5).
- **Side panel** (right column on desktop) holds `EngineLines` above `VariationTree`, on a `--card`
  surface, `gap-4`. `VariationTree` self-switches to its desktop vertical list (`hidden sm:block`).
- **BoardControls** sits directly under the board; the engine on/off toggle (ENGINE-04) goes in its
  `infoSlot`.

### Layout — mobile (below `sm:`)
Stacked, board-first, single column:

```
[ thin horizontal eval readout / EvalBar variant ]   (or above the board)
[            ChessBoard (full-width, square)        ]
[ BoardControls: ◀ ▶ ⟲ ⤢  + engine toggle           ]
[ EngineLines (top-2 PV)                             ]
[ VariationTree (horizontal scroll list, sm:hidden)  ]
```
- Do NOT keep the eval bar beside the board on mobile (Pitfall 6 — it squeezes the board). Either
  stack the vertical `EvalBar` above the board at reduced height, or render a thin horizontal
  variant via its `className`. Board takes full column width.
- `VariationTree` self-renders its mobile horizontal list (`sm:hidden`); pass a `heightClass` if the
  default is too tall on short viewports.
- Maintain `text-sm` floor throughout; reduce columns/whitespace, never type size.

### Engine-loading & on/off states (D-06, SC#3)
- Engine ON by default on mount. Board, stepper, `VariationTree`, `BoardControls` are interactive
  immediately — they never block on the engine.
- The **only** region that shows a loading state is the eval area: render **"Loading engine…"**
  (+ optional spinner) when `engineEnabled && !engine.isReady`. Once ready, `EngineLines` shows its
  own internal "Analyzing…" spinner, `EvalBar` shows the 0.50 midpoint until the first eval commits.
- Engine OFF (user toggled): eval area shows calm **"Engine off"** rest text, no spinner.

### Suspense boundary
- First `React.lazy` route in the app. Fallback: `<div className="p-6 text-sm text-muted-foreground"
  data-testid="analysis-loading">Loading analysis board…</div>`.

| Suspense fallback | `analysis-loading` | — |
| Engine on/off toggle | `btn-analysis-engine-toggle` | icon-only ⇒ `aria-label="Toggle engine"`; reflect state via `aria-pressed` |

  `tabIndex={0}`). All buttons reachable by Tab.

  browser-automation rule. Use semantic `<button>`/`<a>`, never `<div onClick>`.

## "Analyze position" Entry Button (Openings Explorer)

- **Placement:** near the Explorer board controls (the board-adjacent area on both
  `openings-board-container` desktop and the mobile settings/below-board column), so it reads as
  "take this position to the analysis board." Apply to **both** surfaces (parity rule).
- **Variant:** `Button variant="brand-outline"` (secondary action per CLAUDE.md — `brand-outline`,
  never hand-rolled `bg-*` colors). Icon: lucide `Microscope`. Label "Analyze position" (desktop);
  may collapse to icon-only with `aria-label` on a cramped mobile row.
- **Behavior:** `navigate('/analysis?fen=' + encodeURIComponent(chess.position))`. No backend, no
  write-back (D-01/D-02). `encodeURIComponent` is mandatory (FENs contain spaces and `/`).
