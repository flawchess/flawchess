<objective>
Wire the book into the bot's move path, latch the one-way exit, and warm the engine during the book window.

This is the plan where the phase becomes real, and where its two most dangerous properties live:

**The book's failure mode is invisible.** A book that matches nothing still produces perfectly legal moves — the bot just quietly searches every ply and nobody notices. Every assertion in this plan is therefore paired with a positive companion (the exit path really does re-engage the search) so that a test which "passes" by never engaging the book at all still fails. VALIDATION.md's non-inferable checks #1 and #3 both land here, each with its revert-check stated.

**The clock is honest and load-bearing.** Phase 169's D-15/D-16/D-20 (closed over two gap-closure rounds) mean the bot CAN lose on time by construction. A book move is CHEAP, not FREE: it flows through the exact same `chargeableElapsedMs()` → `flagIfOutOfTime()` → `commitMove()` pipeline as a searched move. Do not add a second commit path. Do not add a "book moves are untimed" bypass.

## Planner decision on RESEARCH.md Open Question 1 (post-commit draw-accept grade)

`runBotTurn` today fires a best-effort `pool.grade(fen, [uci])` after every commit to refresh `lastRootPracticalScoreRef` for the draw-accept heuristic (169 D-01, pre-dates this phase). It is a real Stockfish leaf eval and it would fire after book moves too if left alone.

**Decision: SUPPRESS it unconditionally on book plies — AND fix the latent defect that suppression exposes.**

First, the correction. An earlier draft of this plan justified the suppression partly on the claim that the draw-accept endgame gate could not be satisfied within the 16-ply book window. **That claim was false and is retracted.** `botDrawGate.ts:57` reads `queensAreOff(chess) || chess.moveNumber() >= DRAW_ACCEPT_MIN_FULLMOVE` — queens-off ALONE opens the gate; fullmove-40 is an OR-fallback, not a conjunct. And the shipped corpus this phase's book is built from contains cataloged lines that trade queens deep inside the cap — `openings.tsv:1065`, `B10  Caro-Kann Defense: Endgame Variation  1. e4 c6 2. Nf3 d5 3. d3 dxe4 4. dxe4 Qxd1+ 5. Kxd1`, has queens off the board by **ply 9**, comfortably inside `BOOK_PLY_CAP`.

Suppressing the grade for the whole book window therefore leaves `lastRootPracticalScoreRef` frozen at its `0.5` initial value. In a queens-off book position, `isNearEqual` (`|0.5 − 0.5| <= DRAW_ACCEPT_SCORE_BAND`) is trivially true and the endgame gate is open — so the bot would accept a draw in a position **it never evaluated**. That is a regression to a shipped 169 feature, and it is not hypothetical: the existing hook test at `useBotGame.test.ts:724` ("a queens-off position satisfying wouldBotAcceptDraw yields a draw-by-agreement outcome") passes *today* only because `mockGrade` resolves an empty `Map`, so the ref is never written and the accept comes off the untouched default. The test is itself a demonstration of the bug.

**The real defect is that `0.5` doubles as both "dead equal" and "never computed".** So fix that, not the symptom: `lastRootPracticalScoreRef` becomes `number | null`, initialized and reset to `null` = *no eval computed yet this game*, and `wouldBotAcceptDraw` returns `false` on `null` before it ever looks at the board.

Why the sentinel (option (c)) over gating the suppression on queens-off (option (a)) or seeding the ref from a cheap non-search signal (option (b)):
1. **It fixes the class, not the instance.** The conflation is a latent trap independent of this phase — any future code path that skips the grade (a resumed game, a fast-path move, an aborted turn) walks straight into it. (a) and (b) both leave `0.5`-means-two-things in place and buy only this phase's instance.
2. **It keeps SC1 literal.** "Zero Stockfish calls while in book" stays a hard, spy-testable invariant with no queens-off special case that re-admits a mid-book grade. (a) would make the invariant conditional on board state — exactly the "no search, except sometimes" shape that is untestable and rots.
3. **It is safe by construction, in the right direction.** The bot's worst failure mode becomes *declining* a draw it might have accepted — play simply continues, the user can re-offer after the D-04 cooldown, and the ref is real from the bot's first searched move onward. It can never *accept* off an uncomputed number. (b) inverts that: a synthetic seed is a number the bot would then be trusted on.
4. **It is cheap and narrowing, not widening.** Three lines in `botDrawGate.ts` (one param type, one guard, the doc comment) plus one ref type. `wouldBotAcceptDraw`'s contract gets *stricter*; `useBotGame` is its only production caller. This does not blow the phase's scope.

The other three original reasons for suppressing stand unchanged: the book window is exactly when the Stockfish workers should be free to *warm* (SC5), a per-book-ply movetime search competes with that prewarm, and the suppression needs no new gating logic in `runBotTurn` — it falls out of the branch structure.

Purpose: SC1, SC4, SC5 — without regressing 169 D-01.
Output: the book branch, the latch, the prewarm calls, the no-eval sentinel, and the hook tests that prove all four.
