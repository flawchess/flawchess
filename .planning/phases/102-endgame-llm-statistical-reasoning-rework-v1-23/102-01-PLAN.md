<objective>
Wire the statistical-rigor data the Endgame page already shows into the endgame-insights
LLM **payload** (no prompt teaching yet — that is Plan 02). Three concrete payload changes,
all verified against live code in 102-PATTERNS.md §D-07:

1. **Fix `net_timeout_rate`** — currently always emitted as an empty NaN finding and filtered
   out, so the LLM never sees Net Flag Rate. Compute a real n-weighted scalar from the
   time-pressure cards (LLM-04).
2. **Add the Score-Gap-by-Remaining-Time chart block** — the 5-quintile per-TC decomposition
   Phase 88.1 stripped from the payload. This is the only genuinely new payload structure (LLM-04).
3. **Plumb percentile annotations** — the page-level game-count-weighted percentile per in-scope
   metric (D-03) and the per-TC TPCTL for time-pressure metrics (D-06), threaded into the
   `[summary]` window lines as a `pctl=` field with cohort framing (D-05, LLM-01, LLM-05).

The zone field stays the sole gate (D-04 / LLM-03): percentile is metadata appended to an
already-emitted finding; no `if percentile … narrate` logic is added anywhere.

Purpose: give the LLM the same numbers the user is staring at, so the report can reference
percentile rank and time-pressure decomposition.
Output: a richer EndgameTabFindings payload; no schema break (all additions optional).
</objective>
