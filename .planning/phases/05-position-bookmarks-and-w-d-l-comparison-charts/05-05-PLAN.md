For each month, create an object with `{ month: "2025-01" }` plus one key per bookmark (`bkm_{id}`) set to that bookmark's win_rate for that month, or `undefined` if the bookmark has no data for that month (produces a gap in the line):
       ```typescript
       const allMonths = [...new Set(
         series.flatMap(s => s.data.map(p => p.month))
       )].sort();

       const data = allMonths.map(month => {
         const point: Record<string, string | number | undefined> = { month };
         for (const s of series) {
           const found = s.data.find(p => p.month === month);
           point[`bkm_${s.bookmark_id}`] = found?.win_rate; // undefined if no data
         }
         return point;
       });
       ```

    3. **Format month label**: X-axis `tickFormatter`:
       ```typescript
       const formatMonth = (m: string) => {
         const [year, month] = m.split('-');
         return new Date(Number(year), Number(month) - 1).toLocaleDateString('en-US', {
           month: 'short', year: '2-digit'
         });
       };
       ```

    4. **Empty state**: If `series.length === 0` or all series have empty data arrays, render a `<div className="text-center text-muted-foreground py-8">No game history available for saved positions yet.</div>` instead of the chart.

    5. **Chart JSX**:
       ```tsx
       <ChartContainer config={chartConfig} className="w-full h-64">
         <LineChart data={data}>
           <CartesianGrid vertical={false} />
           <XAxis dataKey="month" tickFormatter={formatMonth} />
           <YAxis domain={[0, 1]} tickFormatter={(v) => `${Math.round(v * 100)}%`} />
           <ChartTooltip content={<ChartTooltipContent />} />
           {bookmarks.map((b, i) => (
             <Line
               key={b.id}
               type="monotone"
               dataKey={`bkm_${b.id}`}
               stroke={`var(--color-bkm_${b.id})`}
               strokeWidth={2}
               dot={false}
               connectNulls={false}
             />
           ))}
         </LineChart>
       </ChartContainer>
       ```

    CRITICAL: `connectNulls={false}` — do NOT override this to true. Months with undefined win_rate must show as gaps, not connected lines. This implements the locked decision from CONTEXT.md.

    Derive WDL stats per bookmark from the time-series response (aggregate all months):
    ```typescript
    const wdlStatsMap = useMemo(() => {
      const map: Record<number, { wins: number; draws: number; losses: number; total: number }> = {};
      for (const s of tsData?.series ?? []) {
        let wins = 0, draws = 0, losses = 0;
        for (const p of s.data) {
          const total_month = p.game_count;
          const wins_month = Math.round(p.win_rate * total_month);
          // Note: win_rate = wins/total, so we can only approximately recover wins/draws/losses
          // Simpler: backend could return wins/draws/losses directly. Since it only returns win_rate,
          // use game_count * win_rate for wins, and distribute remainder.
          // HOWEVER: the backend TimeSeriesPoint only has win_rate and game_count.
          // For the WDL bar we need wins/draws/losses separately.
          // Solution: enhance the backend TimeSeriesPoint schema to include wins/draws/losses,
          // OR compute a simpler aggregate: show total W% across all time (games * win_rate summed).
          // Simplest correct approach: sum game_counts and use weighted average win_rate.
          wins += wins_month;
          losses += total_month - wins_month;  // approximation: no draws breakdown from backend
        }
        const total = s.data.reduce((acc, p) => acc + p.game_count, 0);
        map[s.bookmark_id] = { wins, draws: 0, losses: total - wins, total };
      }
      return map;
    }, [tsData]);
    ```

    IMPORTANT DESIGN NOTE: The TimeSeriesPoint schema from plan 02 only has `win_rate` and `game_count`. To properly show the WDL breakdown (wins/draws/losses separately), extend `TimeSeriesPoint` in the backend to include `wins: int`, `draws: int`, `losses: int`. Update `app/schemas/analysis.py` to add these fields to `TimeSeriesPoint`, and update `app/services/analysis_service.py` `get_time_series` to populate them. Then update the frontend `TimeSeriesPoint` type in `frontend/src/types/bookmarks.ts` to include `wins: number`, `draws: number`, `losses: number`. This allows accurate WDL bars.

    If extending the backend is complex, fall back to approximate stats (wins only, no draws breakdown) as shown above — mark it with a TODO comment.
