  <action>
    1. Create `pyproject.toml` with project metadata and dependencies per RESEARCH.md "Minimal pyproject.toml" example. Include:
       - Runtime: fastapi[standard], uvicorn[standard], sqlalchemy[asyncio], asyncpg, alembic, chess (python-chess), pydantic, pydantic-settings, python-dotenv
       - Dev: pytest, pytest-asyncio, ruff
       - [tool.pytest.ini_options] asyncio_mode = "auto"
       - [tool.ruff] line-length = 100

    2. Create `.python-version` with `3.12`.

    3. Create `.env.example` with:
       ```
       DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/chessalytics
       DB_ECHO=false
       ```
       Create `.env` with the same content (gitignored). Add `.env` to `.gitignore` if not already present.

    4. Run `uv sync` to install all dependencies.

    5. Create `app/core/config.py` — Pydantic `BaseSettings` with `DATABASE_URL` and `DB_ECHO` fields, loading from `.env`. Use `model_config = SettingsConfigDict(env_file=".env")` (Pydantic v2 style, NOT the legacy `class Config` inner class).

    6. Create `app/core/database.py` — async engine via `create_async_engine(settings.DATABASE_URL)`, `async_session_maker` with `expire_on_commit=False`, and `get_async_session` async generator.

    7. Create `app/models/base.py` — `Base(AsyncAttrs, DeclarativeBase)` with `type_annotation_map = {int: BIGINT, datetime.datetime: DateTime(timezone=True)}`.

    8. Create `app/models/game.py` — `Game` model per RESEARCH.md Pattern 2. All columns as specified:
       - id (BIGINT PK), user_id (BIGINT, indexed, not null)
       - platform (String(20)), platform_game_id (String(100)), platform_url (String(500) nullable)
       - pgn (Text, not null), variant (String(50), default "Standard")
       - result (String(10)), user_color (String(5))
       - time_control_str (String(50) nullable), time_control_bucket (String(20) nullable), time_control_seconds (nullable int)
       - rated (Boolean, not null, default True)
       - opponent_username (String(100) nullable), opponent_rating (nullable), user_rating (nullable)
       - opening_name (String(200) nullable), opening_eco (String(10) nullable)
       - played_at (nullable datetime), imported_at (not null datetime, server_default=func.now())
       - UniqueConstraint("platform", "platform_game_id", name="uq_games_platform_game_id")
       - relationship to positions with cascade="all, delete-orphan"

       IMPORTANT: Use `func.now()` from sqlalchemy for `imported_at` server default (not `datetime.datetime.utcnow` which is deprecated and evaluated at class definition time, not insert time). Use `server_default=func.now()`.

    9. Create `app/models/game_position.py` — `GamePosition` model per RESEARCH.md Pattern 3. Columns:
       - id (BIGINT PK), game_id (FK to games.id, CASCADE, indexed), user_id (BIGINT, not null), ply (int, not null)
       - full_hash, white_hash, black_hash (all BIGINT, not null)
       - Composite indexes: ix_gp_user_full_hash, ix_gp_user_white_hash, ix_gp_user_black_hash
       - relationship back to Game

    10. Create `app/models/__init__.py` that imports Game and GamePosition for convenient access.

    11. Create `app/main.py` — minimal FastAPI app with a `/health` endpoint returning `{"status": "ok"}`. No lifespan events needed yet.

    12. Create empty `__init__.py` files in: app/, app/routers/, app/services/, app/repositories/, app/schemas/.

    13. Verify by running `uv run python -c "from app.models.game import Game; from app.models.game_position import GamePosition; print('Models OK')"`.

  </files>
  <action>
    1. Run `uv run alembic init -t async alembic` to generate async Alembic scaffold.

    2. Edit `alembic.ini`: set `sqlalchemy.url` to empty string (will be overridden in env.py).

    3. Edit `alembic/env.py` to:
       - Import `settings` from `app.core.config` and override sqlalchemy.url: `config.set_main_option("sqlalchemy.url", settings.DATABASE_URL)`
       - Import `Base` from `app.models.base`
       - Import `Game` from `app.models.game` (noqa: F401) — side-effect import so Base.metadata sees the table
       - Import `GamePosition` from `app.models.game_position` (noqa: F401)
       - Set `target_metadata = Base.metadata`
       - Keep the async migration runner from the template (`run_async_migrations`)

    4. Run `uv run alembic revision --autogenerate -m "initial schema"` to generate the migration.

    5. Inspect the generated migration file to verify it contains:
       - CREATE TABLE games with all columns
       - CREATE TABLE game_positions with all columns
       - UniqueConstraint uq_games_platform_game_id
       - Indexes ix_gp_user_full_hash, ix_gp_user_white_hash, ix_gp_user_black_hash
       - All integer columns as BIGINT

    6. If the migration is empty or incomplete, check that all model modules are imported in env.py before target_metadata assignment.

    NOTE: This task requires a running PostgreSQL database at the URL specified in .env. If the database does not exist, create it: `createdb chessalytics` (or instruct the user). The migration generation (`--autogenerate`) needs a live database connection to diff against.
