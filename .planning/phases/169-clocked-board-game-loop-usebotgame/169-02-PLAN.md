<objective>
Build the three pure, React-free game-logic modules `useBotGame` composes:
- `botGameEnd.ts` — chess.js-backed end-condition detection (PLAY-06) + the
  outcome→result-copy mapping from the UI-SPEC copywriting table (PLAY-09).
- `botDrawGate.ts` — the two INDEPENDENT draw pieces (RESEARCH Pitfall 5): the
  D-04 offer-throttle counter and the D-01 wouldBotAcceptDraw eval+endgame gate
  (PLAY-07). The bot never offers/resigns (D-02/D-03) — encoded as absence.
- `botGamePgn.ts` — the `[%clk]`/`[Termination]`/`[Result]` PGN builder that
  produces a string the frozen Phase 167 backend validator accepts (PLAY-09).

Purpose: These are the non-clock pure decisions; extracting them makes the
draw-gate thresholds, end-condition mapping, and PGN encoding independently
testable without mounting React, and keeps the plan-04 hook thin.

Output: three modules + three test files. The PGN test includes a python-chess
round-trip check resolving RESEARCH Assumption A1 (exact `[%clk]` format).
</objective>
