# Phase 169 — UI Design Contract

> Visual and interaction contract for the clocked bot-play board + game loop
> (`useBotGame`). Ships on the real `/bots` route behind a D-14 hardcoded-settings
> stub (Phase 171 replaces the stub with the setup screen + nav entry).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (already initialized — `components.json` present) |
| Preset | `radix-nova`, baseColor `neutral`, Tailwind v4 CSS-variable tokens in `src/index.css` (no `tailwind.config.js`) |
| Component library | radix-ui (unified `radix-ui` package) via shadcn primitives in `src/components/ui/` |

plain composed components built from these primitives, per
`169-RESEARCH.md`'s recommended `components/bots/` structure — not new
registry installs.

## Spacing Scale

Declared values (must be multiples of 4) — this phase reuses the app-wide
scale already in effect (no new scale invented):

| Token | Value | Usage in this phase |
|-------|-------|----------------------|
| xs | 4px | Gaps inside the clock digit group, icon-to-label gaps in `GameControls` |
| sm | 8px | Row spacing in `MoveListPanel`, gap between clock and "thinking" dot |
| md | 16px | Default padding inside `ClockDisplay` cards, gap between board and side panel (desktop) |
| lg | 24px | Section padding around the board+clocks column, `GameResultDialog` body padding |
| xl | 32px | Gap between board column and move-list/controls column (desktop 2-column layout) |
| 2xl | 48px | Not used in this phase's UI |
| 3xl | 64px | Not used in this phase's UI |

Exceptions: **none.** Icon-only board control buttons (mute, resign, draw
offer) use the existing `size="icon"` (32px) / `size="icon-lg"` (36px) Button
variants already established app-wide (`ChessBoard.tsx` board controls) —
do not introduce a new 44px touch-target size for this phase; stay consistent
with the existing icon-button convention.

## Typography

Reuses the app's existing type scale (no new sizes/weights introduced):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / move list rows / control labels | 14px (`text-sm`) | 500 (`font-medium`) | 1.5 |
| Label / clock caption ("You" / "FlawChess Bot") | 14px (`text-sm`) | 500 (`font-medium`) | 1.5 |
| Heading / result dialog title, section captions | 18px (`text-lg`) | 700 (`font-bold`) | 1.2 |
| Display / clock digits | 20px (`text-xl`), tabular-nums | 700 (`font-bold`) | 1.2 |

Minimum font size is `text-sm` (14px) everywhere per CLAUDE.md — this
includes the move-list SAN entries and the low-time clock digits (do not
shrink the clock to fit; widen the clock card instead if space is tight).
Clock digits MUST use `tabular-nums` (or `font-variant-numeric: tabular-nums`)
so the countdown doesn't visually jitter as digit widths change.

## Color

This phase reuses `frontend/src/lib/theme.ts` tokens and the existing
shadcn CSS-variable palette — **no new color constants are invented.**

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` / `--card` (near-white light, near-black dark) | Page background, board card, move-list panel background |
| Secondary (30%) | `--secondary` / `--muted` | `GameResultStrip` background, move-list row hover, inactive clock card |
| Accent (10%) | `--brand-brown` (`#8B5E3C`) | Primary CTA buttons only: "New game", the active/live-position highlight in `MoveListPanel`, the bot-thinking pulsing dot (D-06) |
| Destructive | `--destructive` (`oklch(0.577 0.245 27.325)`) | Resign confirm button (destructive Button variant), low-time clock digits + urgent glow (D-07), the low-time warning `Alert` (if surfaced) |

Accent (`--brand-brown`, via `Button variant="default"`) reserved for:
**exactly** the "New game" button, the bot's turn-indicator pulsing dot, and
the active-ply highlight in the move list. Nothing else in this phase's UI
uses the brand accent — do not tint the board itself, the clocks (beyond the
existing `brand-outline` for "Analyze this game", see below), or move-quality
markers with brand-brown.

Additional existing (not new) tokens this phase draws on:

| Token (from `theme.ts`) | Value | Usage |
|--------------------------|-------|-------|
| `WDL_WIN` | `oklch(0.50 0.14 145)` (green) | Result dialog/strip when the user won |
| `WDL_LOSS` | `oklch(0.50 0.15 25)` (red) | Result dialog/strip when the user lost |
| `WDL_DRAW` | `oklch(0.60 0.02 260)` (grey) | Result dialog/strip on a draw |
| `--amber-500` family (used by the existing `Alert` `warning` variant) | amber | Draw-declined / low-time notification toast, if a toast is used instead of a modal for D-09's notification blip |

Secondary buttons in this phase ("Offer draw", "Analyze this game" before
game end context, mute toggle when rendered as a labeled button) use
`variant="brand-outline"` per CLAUDE.md's Primary/Secondary button rule —
never `variant="secondary"` (reserved for neutral gray chips/toggles) and
never a hand-rolled `bg-*` class.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (post-game) | **"New game"** — starts another game against the bot (D-11/D-14; reuses the same hardcoded-settings stub this phase ships behind) |
| Secondary CTA (post-game) | **"Analyze this game"** — deep-links to `/analysis?line=...` (D-12) |
| Empty state | **Not applicable this phase.** There is no "no game" empty state — the D-14 stub starts a hardcoded game immediately on route load; Phase 171's setup screen is the real "before a game exists" surface. |
| Error state (bot move selection fails) | **"The bot couldn't make a move. Try again or start a new game."** — shown inline near the board (not a full-page error) if `selectBotMove` rejects after its internal fallback also fails to resolve within a hard timeout; pairs with a "Retry" action and the "New game" CTA. This is a defensive/rare-path message — `selectBotMove` is designed to always resolve (BOT-04), so this copy should almost never surface. |
| Resign confirmation | **"Resign this game?"** — body: "You'll lose this game against the bot." — confirm button: **"Resign"** (`variant="destructive"`) — cancel button: "Cancel" (D-04, two-step: tap Resign → confirm dialog) |

| Draw offer (bot accepts) | Feeds directly into the result screen as a draw outcome — no separate confirmation copy needed |
| Result — win by checkmate | **"You won — checkmate"** |
| Result — loss by checkmate | **"You lost — checkmate"** |
| Result — win/loss by resignation | **"You won — the bot resigned"** *(never occurs per D-03, kept only if the copy table is reused elsewhere)* / **"You resigned"** |
| Result — win/loss on time | **"You won on time"** / **"You lost on time"** |
| Result — draw by agreement | **"Draw — by agreement"** |
| Result — draw by stalemate | **"Draw — stalemate"** |
| Result — draw by threefold repetition | **"Draw — repetition"** |
| Result — draw by 50-move rule | **"Draw — 50-move rule"** |
| Result — draw by insufficient material | **"Draw — insufficient material"** |

| Destructive confirmation | Resign: **"Resign this game? You'll lose this game against the bot."** — the only destructive action in this phase (draw offers are reversible/non-destructive, mute is not destructive) |

## Component Inventory (phase-specific)

**Visual hierarchy:** the chessboard is the primary focal point of the in-game
screen; clocks, move list, and game controls are secondary surfaces that must
never compete with it for attention (the brand-brown accent budget above is
what carries the eye to the bot-thinking dot and active ply, not size or
placement).

Concrete visual/interaction contract for the five new component groups
identified in `169-RESEARCH.md`'s recommended project structure. Executor
should treat this as the source of truth alongside the RESEARCH.md
architecture patterns (D-05 clock reconciliation math, Pattern 3 concurrency)
— this section only specifies the **visual/copy** layer, not the state logic.

### `ClockDisplay` (two instances: user + bot)
- Two clock cards, one per side, stacked vertically on mobile (bot's clock
  above the board, user's clock below — lichess convention) and stacked
  vertically in the side column on desktop next to the board.
- Each clock card: `Card` background (`--card`), `md` (16px) padding, rounded
  per the existing `--radius` token (no new radius).
- Contents: side label ("You" / "FlawChess Bot") at Label size (14px/500),
  clock digits at Display size (20px/700, tabular-nums) below or beside it.
- **Active clock** (the side to move) gets a subtle highlight: `--secondary`
  background tint (not brand-brown — accent is reserved per the Color
  section above) to read as "live" without competing with the 10% accent
  budget.
- **Bot-thinking indicator (D-06):** a small pulsing dot (brand-brown,
  `animate-pulse` Tailwind utility) rendered next to the bot's clock/name
  while `selectBotMove` is in flight. This is the ONE place brand-brown
  appears outside the "New game" button and the move-list active-ply
  highlight.
- **Low-time state (D-07):** below a named `LOW_TIME_THRESHOLD_MS` (~10s),
  that side's clock digits switch to `--destructive` red, display tenths
  (`0:09.4`), and gain an `animate-pulse` or `ring-2 ring-destructive/40`
  urgent treatment. Applies independently to whichever side is low — the bot
  visually never reaches true zero (168.5/D-05 clamp), so the bot's clock
  can show the low-time red state but should not flicker at exactly 0:00.0.
- Fischer increment: no special visual treatment on increment application —
  the digit simply updates; do not animate a "+2" pop-in (out of scope,
  keep this phase visually quiet).

### `GameResultDialog` (D-11)
- A `Dialog` (existing `components/ui/dialog.tsx`) shown once the game ends.
- Title: the outcome+reason string from the Copywriting Contract table above,
  colored by outcome using the existing `WDL_WIN`/`WDL_LOSS`/`WDL_DRAW`
  tokens (text color on the title, not a full-dialog background tint).
- Body: brief, no extra prose beyond the reason already in the title
  (per the project's established "Popover copy minimalism" convention —
  this dialog is even terser: title carries the full message).
- Actions row: **"New game"** (`variant="default"`, brand-brown, primary) and
  **"Analyze this game"** (`variant="brand-outline"`, secondary) side by side,
  primary on the right per the existing dialog action-row convention (mirror
  `Dialog` usages elsewhere in the app, e.g. confirmation dialogs).
- Dismissible via the existing `Dialog` close button (X) or clicking outside
  — dismissing reveals the final board position underneath (D-11).

### `GameResultStrip` (D-11, persistent post-dismiss)
- A thin inline bar (`--secondary` background, `sm`/8px vertical padding,
  `md`/16px horizontal padding) replacing the normal in-game controls area
  once the dialog is dismissed.
- Same outcome text (smaller — Body size 14px/500, not the dialog's Heading
  size) + the same two actions as compact buttons/links.
- Persists until "New game" is clicked (this phase's own scope ends here;
  Phase 170/171 own what happens across a page reload).

### `MoveListPanel` (D-13)
- A linear SAN move list in a scrollable panel beside (desktop) or below
  (mobile) the board — two-column ply layout (move number, White SAN, Black
  SAN), matching the existing app convention for move lists (mirror
  `VariationTree`'s visual density, not its branching-tree affordances —
  this list is linear only, no forking).
- Row text: Body size (14px/500). Active/live-position row (the actual
  current game position, not the scroll-back cursor) gets the brand-brown
  active-ply highlight (background tint at low alpha, e.g.
  `bg-brand-brown/10`, consistent with how `ACTIVE_FILTER_RING_CLASS`-style
  highlights are applied elsewhere).
- **View-only scroll-back (D-13):** clicking an earlier move or using arrow
  keys navigates the *displayed* board position without disabling the game
  — the board becomes read-only (no drag/click-to-move) while scrolled back,
  and a small inline affordance ("Return to live position" text link or an
  icon button) is shown to snap back. Do not silently disable the whole
  board without this affordance — the user must have an obvious way back.
- New moves played while scrolled back are impossible by construction (the
  live game only accepts moves at the live position) — no special error
  copy needed, this is a state, not a rejected action.

### `GameControls`
- A compact row of icon+label buttons: **Resign**, **Offer draw**, **Mute**
  toggle. Sits below the board on mobile, in the side column on desktop
  (same column as `MoveListPanel`, stacked above or below it).
- Resign: `variant="brand-outline"` at rest (not destructive until the
  confirm dialog appears — the confirm dialog's actual confirm button is
  `variant="destructive"`, per CLAUDE.md's primary/secondary button rule:
  don't pre-color the trigger button red, only the actual destructive
  confirm action).
- Offer draw: `variant="brand-outline"`, disabled (with tooltip, see
  Copywriting Contract) during the D-04 cooldown window.
- Mute: icon-only `size="icon"` Button (`variant="ghost"`), lucide `Volume2`
  / `VolumeX` icon swapped by mute state, `aria-label` per the Copywriting
  Contract table (CLAUDE.md ARIA rule for icon-only buttons).
- Every control gets a `data-testid` per CLAUDE.md's Browser Automation
  Rules naming convention: `board-btn-resign`, `board-btn-offer-draw`,
  `board-btn-mute`.

### Sound module UI surface (D-08/D-09/D-10)
- No dedicated "sound settings" screen — the single mute toggle in
  `GameControls` is the entire user-facing sound UI for this phase (D-10:
  boolean only, no volume slider).
- License correction from `169-RESEARCH.md` Pitfall 1: vendor lila's
  `sfx` (or `piano`/`futuristic`/`nes` — pick after a quick listen per
  RESEARCH Open Question 1) sound directory, NOT `standard` (non-free
  exception in lila's `COPYING.md`). Add a "## Sound Assets" section to
  `README.md` crediting the source + AGPLv3+ license, mirroring the existing
  "## Engine Binaries (GPLv3 License Note)" precedent — this is a licensing
  requirement, not a design one, but it gates what D-08 actually ships.
