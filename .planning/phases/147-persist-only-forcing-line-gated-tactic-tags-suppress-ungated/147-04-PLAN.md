`/eval/remote/atomic-lease` endpoint. Per D-02 this is a NEW endpoint pair with its own DoS caps and
rollback safety, running alongside the deprecated old pair during a mixed-fleet deploy.

- Q4 hint design: use the NARROWER `_run_all_moves_pass`-based worker hint → the lease payload stays
  FEN-per-ply (no PGN, no Game metadata). (RESEARCH A2 recommendation.)
- Q5 version tag: INCLUDE a `worker_schema_version: int` field for observability / rejection of egregiously
  stale workers — correctness is NOT gated on it (server re-classifies authoritatively + Part A's NULL net).

  flaw-ply pass (handled in 147-06). No lease-payload change follows from this.

Purpose: give Part B distinct, independently-capped schemas + a lease endpoint that selects games with the
existing queue logic and never 500s on fat games.
Output: schemas, the atomic-lease handler, and lease-level tests.
