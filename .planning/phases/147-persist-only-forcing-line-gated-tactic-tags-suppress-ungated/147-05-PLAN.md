<objective>
Part B, server side (2/2): the atomic `/eval/remote/atomic-submit` handler that eliminates the ungated
window at write time (D-01). The worker submits raw full-ply evals + MultiPV-2 continuation blobs together;
the server re-classifies AUTHORITATIVELY on its own game_positions with those blobs, applies the
forcing-line gate, and writes flaws + gated tactic tags + `allowed_pv_lines`/`missed_pv_lines` + both
completion markers in ONE transaction. Part A (147-01) is the graceful-degradation net for version skew.

Purpose: for games on the new path, tactic tags are gated the instant the game is stamped analyzed — no
raw window, no orphan, no tier-4 round-trip.
Output: the atomic submit handler + integration tests (atomic write, server-authoritative re-classify,
skew degradation, token tamper rejection).
