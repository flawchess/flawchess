<objective>
Part B, worker side: upgrade the fat `app.*` fleet worker (`scripts/remote_eval_worker.py`) with a new
atomic rung that computes full-ply evals + MultiPV-2 continuation blobs together and submits them to the
new `/atomic-submit` endpoint in one package. The worker classifies LOCALLY only as a hint (which plies are
flaws → which continuation lines to blob), using the narrower `_run_all_moves_pass` (Q4/RESEARCH A2 — no
PGN needed). The server remains the trust boundary.

Purpose: complete the atomic pipeline so remote games are gated at write time with no ungated window (D-01).
Output: the upgraded worker rung, worker unit tests, and a dev-first end-to-end verification gate.

## Artifacts this plan produces
- `scripts/remote_eval_worker.py`: a new atomic rung (`_handle_atomic_response` + `_eval_atomic_game` or
  equivalent) and its wiring into the worker's rung selection loop.
- New unit tests in `tests/test_remote_eval_worker.py`.
</objective>
