    Declare every threshold as a module-level named constant with a `#` comment citing its
    decision id (CLAUDE.md no-magic-numbers): `ONLY_MOVE_WIN_PROB_MARGIN: float = 0.35`
    (D-07 — cite the lichess-puzzler provenance in the comment: their +0.7 in -1..+1 space

    (D-08), `STILL_WINNING_FLOOR_CP: int = 200` (D-09).

    Implement the gate as a thin orchestrator `apply_forcing_line_filter()` that delegates each
    rule to its own small predicate helper (function-size discipline, nesting hard-cap 4):

    - Only-move margin (D-07, GATE-01): a solver node is forced when
      `eval_cp_to_expected_score(best_cp) - eval_cp_to_expected_score(second_cp) >
      ONLY_MOVE_WIN_PROB_MARGIN`, converting the stored white-perspective cp to side-to-move at
      read time. The motif is credited only when its FIRING node AND every solver node leading
      to it pass this test. This is checked at SOLVER nodes ONLY (D-10) — defender nodes play
      the single best reply with NO uniqueness check; a line that branches at a defender ply but
      re-converges to a single forcing continuation is fine.

    - Mate priority (D-01, pulled forward into THIS module; route through
      eval_mate_to_expected_score): if only the best move is a mate -> forced; if both best and
      second are mates -> the shorter distance to mate is forced (compare |bm| vs |sm|);
      mate-in-1 is NEVER suppressed; otherwise fall through to the win-prob margin above. A node
      with no legal second move (s/sm both None) also passes (only-move).

    - Already-winning reject (D-08, GATE-02): reject the whole motif if the pre-flaw position
      eval (white-perspective `game_positions.eval_cp` at the flaw ply, passed in as an arg) is
      already > ALREADY_WINNING_CP_THRESHOLD from the solver's perspective.

    - Still-winning floor (D-09, GATE-02): stop extending the line at the first solver node
      whose best-move eval drops below STILL_WINNING_FLOOR_CP (the tactic fizzled past there).

    - Trailing-only-move strip (D-10, GATE-02): strip trailing solver only-moves from the end
      of the line so the line ends on a real solver move with a genuine second-best.

    - One-mover discard (D-10, GATE-02): discard a line that reduces to a single solver move
      after stripping (one-movers are not credited).

    Add explicit return-type annotations on every function. No fenced reference to lichess
    source; constants/heuristics/names only.
