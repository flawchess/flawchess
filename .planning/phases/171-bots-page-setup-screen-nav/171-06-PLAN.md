<objective>
Wire the setup screen into `/bots` and make it the single entry point for every new game (D-09, D-11, D-13).

`BotsPage` currently has exactly one branch: mount `BotsGame` with a hardcoded `BOT_GAME_SETTINGS` stub.

exactly that branch and nothing else â€” the ResumeGate's snapshot-beats-setup precedence, the pending-store
drain, and `useBotGame` itself all stay as shipped.

Three paths must all converge on setup: a fresh visit with no snapshot, a Discard from the resume gate, and
"New game" from the result surfaces. That convergence is the point â€” there is then no second start path to
keep honest.

Purpose: PLAY-02's "start a new game from a setup screen", reachable by guests and logged-in users alike.
Output: a rewired `Bots.tsx` and the page's first test file.
