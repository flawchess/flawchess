<objective>
Close UAT gap 3 (major, test 5): on a phone the `/bots` setup screen's **Start** button is below
the fold AND occluded behind the fixed bottom nav.

Root cause (already diagnosed — do NOT re-investigate): **both** problems at once.
1. **Occlusion.** `SetupScreen`'s root is `mx-auto flex max-w-md flex-col gap-4 p-4` — it reserves
   only 16px below the CTA and is the **one screen in the app** that omits the established
   bottom-nav clearance (`Endgames.tsx:812`, `Analysis.tsx:2660`, `GamesTab.tsx:385`,
   `FlawsTab.tsx:362` all carry `pb-20`). `Bots.tsx:355` (the in-game view) has the same bare `p-4`.
2. **Density.** The setup stack costs ~632px on a phone, so the CTA is below the fold even before
   the nav eats into it. The 3 stacked TC bucket groups alone are ~228px (~40% of the screen).

Two independent fixes, one task each.

Purpose: the primary CTA of a brand-new page is unreachable on the device most users are on.
Output: a setup screen whose Start button is visible, tappable, and unobstructed on a real phone.
</objective>

<scope_fence priority="hard">
## Do NOT do these

- **Do NOT restructure the screen.** The control order (ELO → Play style → Play as → Time control →
  Start), the single-column `max-w-md` card, and the component split all stay exactly as the
  171-UI-SPEC specifies. This is a sizing/spacing pass, not a redesign.
- **Do NOT collapse the 3 TC groups into a single 9-preset grid.** The Blitz/Rapid/Classical
  grouping is a spec'd affordance (UI-SPEC "Time control picker (D-14)"). The sub-headers move from
  *above* each grid to *beside* it — they do not disappear.
- **Do NOT touch `frontend/src/App.tsx:567`** (`<main className='pb-16 sm:pb-0'>` is not
  safe-area-composed). That is a latent app-wide issue, out of scope here, and already masked
  everywhere else by each page's own `pb-20`. **Flag it as a seed at the end of this plan
  (`/gsd-capture --seed`), do not fix it.**
- **Do NOT edit `frontend/src/components/ui/slider.tsx`.** It is a shared primitive; its `min-h-11`
  is the documented app-wide 44px iOS-HIG/Material hit target. Scope any slider height change to
  the setup screen only (see Task 2).
- **Do NOT go below 40px on any touch target.** 44px is the WCAG 2.5.5 / Apple HIG minimum; 40px is
  already a deliberate, user-approved mild deviation and is the **hard floor**. No `h-9`, no `h-8`
  chips.
- **Do NOT use `text-xs`** anywhere here (CLAUDE.md floor; the only exception is info-popover bodies,
  which already exist and are unchanged).
</scope_fence>
