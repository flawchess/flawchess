<objective>
Wire the per-persona win-stars UI end-to-end on top of Plan 01's endpoint and Plan 02's transposed grid: (1) add `STAR_FILLED`/`STAR_EMPTY` theme constants; (2) add the `useBotPersonaWins` hook + `botsApi.getPersonaWins` + `PersonaWinsResponse` type; (3) send `persona_id` on the store-game request; (4) fetch win counts ONCE in `Bots.tsx` and prop-drill `winsByPersona` down through `PersonaGrid` → `PersonaCard`; (5) render a `PersonaStars` row of 3 stars (min(wins,3) gold + remainder grey-outline) at the bottom of each card.

This is the frontend expansion of the win-tracking tracer. It depends on Plan 01 (the `GET /bots/persona-wins` endpoint + `persona_id` schema field it round-trips to) and Plan 02 (it edits the transposed `PersonaGrid.tsx` and its props, building on Plan 02's layout — avoids a same-wave file conflict on `PersonaGrid.tsx`).

Purpose: surface a truthful, honest win signal per persona without breaking the components' prop-driven testability.
Output: theme constants, hook, API method, type, store-request field, prop-drill, and the stars row.
