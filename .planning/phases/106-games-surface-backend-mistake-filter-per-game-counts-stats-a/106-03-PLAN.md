<objective>
Build `GET /api/library/mistake-stats`: the stats-panel aggregate over the filtered analyzed-only
set — per-severity counts and rates normalized per game and per 100 moves, the full tag
distribution (tempo split, result-changing rate, phase histogram), a trend-over-time series, and
the explicit `% analyzed` (>=90%-per-ply-coverage) denominator with the analyzed-game N stated so
the panel never implies clean games where evals are merely absent.

Per D1 (locked, pragmatic-correctness path): the SQL handles the EXISTS filter (reused), the
per-severity flagged-row count aggregates, and the >=90% analyzed denominator. Tag distribution +
rates come from RE-CALLING the kernel over the analyzed filtered set. Per D3: trend bucketing via a
rolling-GAME window (get_time_series machinery; window size at executor discretion, default to the
existing ROLLING_WINDOW_SIZE / MIN_GAMES_FOR_TIMELINE constants) — NOT calendar buckets. Reconcile
the two aggregations RESEARCH flags: the B/M/I count path uses all three tiers
(count_game_severities / SQL aggregate) while the tag/window path is mistakes+blunders only — keep
them distinct; do NOT derive the inaccuracy count from the M+B set.

Purpose: backend the Games-subtab stats panel consumes (LIBG-09 / backend of LIBG-03).
Output: the stats schemas, the coverage-denominator + analyzed-set repository queries, the service
orchestration (split by stage to stay shallow), the thin router handler, and tests.
</objective>
