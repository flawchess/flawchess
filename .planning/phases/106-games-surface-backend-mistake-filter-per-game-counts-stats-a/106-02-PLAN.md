<objective>
Build `GET /api/library/games`: a paginated game archive (mirroring the endgame archive shape)
filterable by a boolean mistake-severity EXISTS, where each returned game carries its per-game
B/M/I counts and a curated/deduped set of card tag-chips — by re-calling the Phase 105 kernel
per game on the returned page only. chess.com / unanalyzed-lichess games surface the explicit
"no engine analysis" card state, never a false zero-flaw game.

Per D1 (locked): counts + chips come from RE-CALLING the kernel (`count_game_severities` for
B/M/I incl. inaccuracy; `classify_game_mistakes` for the chip tag set). Per D2: new `/library`
router/service/repository/schema files. Per SEED-036 chip curation: aggregate to game level,
exclude `phase-*` AND inaccuracy-level tags, one chip per remaining tag type.

Purpose: this is the backend the Games subtab card list consumes (LIBG-08 / backend of LIBG-01).
Output: the schemas, the archive repository query, the service orchestration with chip curation,
the thin router, the /api mount, and DB + service tests.
</objective>
