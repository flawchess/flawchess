# Phase 41.1 Plan 01: Unified process_game_pgn Function Summary

**One-liner:** Single-pass PGN processing function returning hashes, classification, eval, clock, move_san, endgame_class, move_count, and result_fen from one mainline walk via `PlyData`/`GameProcessingResult` TypedDicts.

## What Was Built

Added `process_game_pgn()` to `app/services/zobrist.py` ‚Äî a unified function that walks the PGN mainline exactly once and returns all per-ply data needed for `game_positions` rows. This eliminates the triple-PGN-parse bottleneck (D-01, D-02) identified in the phase research.

Converted `hashes_for_game` to a thin wrapper delegating to `process_game_pgn` for full backward compatibility with all existing callers and tests.

### New exports in `app/services/zobrist.py`

- `PlyData` ‚Äî TypedDict with 16 fields: `ply`, `white_hash`, `black_hash`, `full_hash`, `move_san`, `clock_seconds`, `eval_cp`, `eval_mate`, `material_count`, `material_signature`, `material_imbalance`, `has_opposite_color_bishops`, `piece_count`, `backrank_sparse`, `mixedness`, `endgame_class`
- `GameProcessingResult` ‚Äî TypedDict with 3 fields: `plies`, `result_fen`, `move_count`
- `process_game_pgn(pgn_text: str) -> GameProcessingResult | None` ‚Äî the unified processing function

### New imports in `app/services/zobrist.py`

- `classify_position` from `app.services.position_classifier`
- `classify_endgame_class, _CLASS_TO_INT` from `app.services.endgame_service`
- `ENDGAME_PIECE_COUNT_THRESHOLD` from `app.repositories.endgame_repository`
