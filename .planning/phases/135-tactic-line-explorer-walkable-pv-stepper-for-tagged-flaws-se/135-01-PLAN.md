<objective>
Build the backend `tactic-lines` contract: a new `GET /library/flaws/{game_id}/{ply}/tactic-lines` endpoint that surfaces the stored Stockfish PVs (`game_positions.pv`) as display-ready SAN, plus depth/motif/FEN metadata, for the Tactic Line Explorer.

Purpose: The explorer needs both the missed line (engine PV the player should have played) and the allowed line (opponent punishment), converted server-side via python-chess so the frontend renders SAN directly. This is the data source for the entire phase (D-01..D-05 surfaces all read from this one endpoint).
Output: `TacticLinesResponse` schema, `fetch_tactic_lines()` repository function with server-side UCI→SAN conversion and payoff truncation, the IDOR-guarded route, and Wave-0 backend tests covering anchoring/offset/short-PV/IDOR.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-135-01 | Information Disclosure | get_tactic_lines route | mitigate | IDOR guard: `user.id` strictly from `current_active_user` JWT dependency; repository filters `GameFlaw.user_id == user_id`; repo returns None → route raises 404 (not 403), so existence of another user's game/ply is never confirmed (mirrors get_library_game / T-112-01). Covered by test_404_wrong_user. |
| T-135-02 | Tampering | game_id / ply path params | mitigate | Both typed `int`; FastAPI auto-rejects non-integers with 422 before handler runs (T-112-05 pattern). No manual parsing. |
| T-135-03 | Information Disclosure | TacticLinesResponse payload | mitigate | Response model exposes only FEN (board placement + side-to-move), SAN move lists, raw depths, motif strings, flaw_ply, best_move_uci — never the internal Zobrist hashes (white_hash/black_hash/full_hash) stored on game_positions (CLAUDE.md). Covered by test_no_hash_leak. |
| T-135-04 | Denial of Service | _pv_to_san_list on malformed PV | accept | `_parse_pv` raises ValueError on illegal UCI; helper catches it and returns None gracefully. PV strings are server-originated (~12 plies, written by our own eval drain), not user-supplied, so unbounded/abusive input is not a realistic vector. No capture_exception (expected condition). |
</threat_model>
