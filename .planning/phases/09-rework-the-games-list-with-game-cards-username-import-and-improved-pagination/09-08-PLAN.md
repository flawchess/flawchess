  <action>
Add a session-scoped autouse fixture to tests/conftest.py that:

1. Overrides `settings.ENVIRONMENT = "production"` so the bypass condition is false.
2. Reassigns `app.users.current_active_user = app.users._jwt_current_active_user` to restore JWT auth for all test-session requests.
3. Uses yield so the fixture spans the entire test session.
4. After yield, restores the original values (good practice but not strictly needed since tests run in a subprocess).

The fixture must be `autouse=True` and `scope="session"` so it applies to every test without explicit request. Place it after the existing fixtures in conftest.py.

Important: The assignment at module level in app/users.py (`current_active_user = _dev_bypass_user`) happens at import time. The conftest fixture runs after import, so it must patch the module-level attribute directly: `import app.users; app.users.current_active_user = app.users._jwt_current_active_user`. All routers that do `from app.users import current_active_user` import the Depends()-wrapped callable at their own import time — but since FastAPI resolves Depends lazily, patching the module attribute works IF the routers use `Depends(current_active_user)` where `current_active_user` is looked up at call time. Check: if routers do `from app.users import current_active_user` at top level, the local name is already bound. In that case, also patch each router's reference OR use a different approach: patch `app.dependency_overrides` on the FastAPI app.

Safest approach: Use `app.dependency_overrides[app.users._dev_bypass_user] = app.users._jwt_current_active_user` on the FastAPI app instance. This tells FastAPI "whenever _dev_bypass_user is the dependency, use _jwt_current_active_user instead." This works regardless of how routers imported the callable.

Actually, the cleanest approach: Since `current_active_user` is assigned at module level and routers do `Depends(current_active_user)`, and `current_active_user` IS `_dev_bypass_user` in dev mode, we need `app.dependency_overrides[_dev_bypass_user] = _jwt_current_active_user`. This is the standard FastAPI testing pattern.

```python
@pytest.fixture(autouse=True, scope="session")
def disable_dev_auth_bypass():
    """Force JWT auth in tests even when ENVIRONMENT=development."""
    from app.main import app as fastapi_app
    from app.users import _dev_bypass_user, _jwt_current_active_user

    fastapi_app.dependency_overrides[_dev_bypass_user] = _jwt_current_active_user
    yield
    fastapi_app.dependency_overrides.pop(_dev_bypass_user, None)
```

This is the correct FastAPI testing pattern — dependency_overrides intercepts at resolution time.
