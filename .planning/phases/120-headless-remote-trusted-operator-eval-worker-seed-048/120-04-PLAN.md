<objective>
Per D-7 (SEED-048, added 2026-06-14): replace the deterministic within-user game pick in `_claim_tier3_derived` (Step 2) with the same Efraimidis–Spirakis weighted-random key already used for the user pick (Step 1), so that when the server pool and the two remote workers land on the same recency-hot user they usually evaluate *different* games instead of the identical top game.

Purpose: With server-pool + 2 remote workers and no tier-3 lease / no `in_flight` (Phase 119 removed it), two workers on the same user currently collide on the exact same deterministic top game for the whole ~10s eval window — a guaranteed wasted eval cycle (D-4 accepts the residual, but D-7 cuts the avoidable bulk). This is a stateless, schema-less refinement of D-4; the principled zero-collision endgame (ephemeral TTL lease) stays deferred.

Output: named game-weight constants (TC-bucket multipliers + recency decay), a rewritten Step-2 game pick using `ORDER BY -ln(random()) / game_weight LIMIT 1` with all values bound as `:params`, the same spread applied to the residual PV-backfill fallback for symmetry, and tests proving the spread is real while weighting still biases toward classical/recent.
</objective>
