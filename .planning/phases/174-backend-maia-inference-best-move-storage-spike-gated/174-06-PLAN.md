<objective>
Close 174-VERIFICATION Truth 2 by retiring the lichess-eval special-case lane so lichess-eval
games (`lichess_evals_at IS NOT NULL`, ~6% of prod games / 18% of lichess games) get the SAME
uniform full Stockfish MultiPV-2 pass as chess.com / non-eval-lichess games. That single change
makes `engine_result_map` + `second_best_map` cover every ply, which lets the existing 174-05
candidate builder nominate every out-of-book played==best ply — not just the holes + flaw-adjacent
subset it sees today.

**Locked design (SEED-109, Option C — do not re-litigate):** stop special-casing lichess imports
that arrive with `%eval`; run them through the full analysis pass. This plan implements the
"going-forward / analysis-pass" half; 174-07 implements the 43k backlog backfill.

**Provenance decision (SEED-109 item 4 — reconciled here, PRESERVE):** keep the stored lichess
`%evals` and `games.lichess_evals_at` exactly as-is. Rationale: (1) gem/great detection reads the
FRESH `engine_result_map` / `second_best_map` from the MultiPV-2 pass, never `game_positions.eval_cp`,
so preserving the stored evals does NOT compromise uniform coverage; (2) `.planning/notes/eval-completion-columns.md`
establishes `lichess_evals_at` as durable provenance and gates opening-cache donation on
`lichess_evals_at IS NULL` (it needs our own best_move) — preserving keeps those invariants intact;
(3) overwriting would rewrite ~43k games' eval rows with dev-vs-prod-nondeterministic Stockfish values
for zero coverage benefit. The existing `is_lichess_eval_game=True` write path already does exactly
this (never shift/overwrite lichess evals, write best_move only), so preservation is the minimal,
rules-consistent reading of "first-class". We therefore overlay best_move / PV / second-best across
ALL plies and leave the provenance columns untouched.

Purpose: re-establish "every newly analyzed game stores a candidate row for each out-of-book
best-move ply that clears the inaccuracy gate" for ALL lanes.
Output: a single unified analysis lane; lichess-eval games covered end-to-end; the book-depth
correctness bug (CR-01) fixed; guarding invariants preserved via adapted tests.
</objective>
