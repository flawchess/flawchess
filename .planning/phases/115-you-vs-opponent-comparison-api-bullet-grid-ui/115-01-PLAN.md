<objective>
Build the backend for the you-vs-opponent flaw comparison: a new sibling endpoint
`GET /api/library/flaw-comparison` that computes, for all 15 flaw-delta metrics, the
mean paired per-game delta (per-100-of-your-own-moves, per 114 D-01) with a normal/t
(Wald-z) 95% CI per bullet, plus the hand-authored §5 benchmark "typical" zone bounds and
per-metric axis domains embedded in the response (D-04/D-05/D-07).

Purpose: This is the data layer the Phase 115 bullet-grid UI (Plan 02) renders. The
unified estimator (FLAWCMP-01, amended), full 15-bullet inventory (FLAWCMP-03), combo
viability (FLAWCMP-04, validated in RESEARCH), and the 20-analyzed-game section gate +
zero-event handling (FLAWCMP-05) all live here.

Output: `flaw_delta_zones.py` registry, `FlawBullet`/`FlawComparisonResponse` schemas,
`fetch_flaw_comparison` repository fn, `get_flaw_comparison` service fn + `_compute_mean_ci`
helper + `FLAW_COMPARISON_GATE` constant, the `/flaw-comparison` route, and a backend test
suite.
</objective>
