## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|

| sm | 8px | Compact element spacing (gap-2, px-2, py-1) |

## Typography

| Role | Size (desktop) | Size (mobile) | Weight | Line Height |
|------|---------------|---------------|--------|-------------|
| Body | 14px (text-sm) | 16px (mobile bump) | 400 (normal) | 1.5 (1.25rem) |
| Label / nav item | 14px (text-sm) | 16px (mobile bump) | 500 (medium) | 1.5 |
| Tab trigger | 14px (text-sm) | 16px (mobile bump) | 500 (medium) | — (inline) |
| Heading / page title | 18px (text-lg) | 20px (mobile bump) | 600 (semibold) | 1.2 |

Notes:
- Mobile typography bump is global: `@media (max-width: 639.98px)` lifts text-sm → 16px, text-base → 18px. Applies automatically to all text in the new Library page and its subtabs.
- Minimum font size is `text-sm` (14px desktop / 16px mobile). Never use `text-xs` in new code except hover/tap-activated info tooltips.
- Brand logotype ("FlawChess") uses `font-brand` (Fredoka) — not used in Library page body content.

## Layout Patterns

### Library Page Shell (`LibraryPage.tsx`)

Pattern: mirrors `Endgames.tsx` exactly — no sidebar panels (Import and Overview each manage their own sidebars internally).

**Focal point:** the primary visual anchor of the shell is the charcoal-textured `TabsList variant="brand"` bar at the top; the active subtab trigger (brand-brown background) is the strongest single element. Below it the focal point belongs entirely to the inherited subtab content (Import sync form / Overview dashboard), unchanged by this phase.

```
<div data-testid="library-page" className="flex min-h-0 flex-1 flex-col bg-background">
  <main className="mx-auto w-full max-w-7xl flex-1 px-4 py-2 md:py-6 md:px-6">

    {/* Desktop: Tabs above content */}
    <div className="hidden md:block">
      <Tabs value={activeTab} onValueChange={(val) => navigate(`/library/${val}`)}>
        <TabsList variant="brand" className="w-full" data-testid="library-tabs">
          <TabsTrigger value="import" data-testid="tab-import" className="flex-1">
            <DownloadIcon /> Import
          </TabsTrigger>
          <TabsTrigger value="overview" data-testid="tab-overview" className="flex-1">
            <LayoutDashboard /> Overview
          </TabsTrigger>
        </TabsList>
        <TabsContent value="import" className="mt-4"><ImportTab .../></TabsContent>
        <TabsContent value="overview" className="mt-4"><OverviewTab /></TabsContent>
      </Tabs>
    </div>

    {/* Mobile: sticky subnav row + content */}
    <div className="md:hidden flex flex-col min-w-0">
      <Tabs value={activeTab} onValueChange={(val) => { navigate(`/library/${val}`); window.scrollTo({ top: 0 }); }}>
        <div className="sticky top-0 z-20 flex items-center gap-2 h-[52px] bg-white/20 backdrop-blur-md rounded-md px-1 py-1"
             data-testid="library-mobile-control-row">
          <TabsList variant="brand" className="flex-1 !h-full !p-0" data-testid="library-tabs-mobile">
            <TabsTrigger value="import" className="flex-1" data-testid="tab-import-mobile">
              <DownloadIcon /> Import
            </TabsTrigger>
            <TabsTrigger value="overview" className="flex-1" data-testid="tab-overview-mobile">
              <LayoutDashboard /> Overview
            </TabsTrigger>
          </TabsList>
        </div>
        <TabsContent value="import" className="mt-4"><ImportTab .../></TabsContent>
        <TabsContent value="overview" className="mt-4"><OverviewTab /></TabsContent>
      </Tabs>
    </div>

  </main>
</div>
```

Note: The mobile `md:hidden` / desktop `hidden md:block` split mirrors Endgames.tsx. No filter button in the Library mobile control row at this phase (Import and Overview manage their own filter controls internally).

## Interaction Contracts

### Tab Switching
- Trigger: `<TabsTrigger>` click (desktop) or tap (mobile)
- Effect: `navigate('/library/<tab>')`, active tab derived from `location.pathname`
- Mobile only: `window.scrollTo({ top: 0 })` on switch (matches Endgames pattern)
- Deep-linkable: `/library/import` and `/library/overview` are valid direct URLs

### Nav-Lock Exemption (UPDATED)
- Current: `to !== '/import' && to !== '/admin'`
- New: `to !== '/library' && to !== '/admin'`
- Library is always unlocked (hosts Import, cannot be import-gated)
- Applies in: `NavHeader`, `MobileBottomBar`, `MobileMoreDrawer`
- Source: CONTEXT.md D-08, D-09

### isActive Helper (UPDATED)
Must detect `/library` prefix for subtab routes:
```tsx
if (to === '/library') return pathname.startsWith('/library');
```
Source: App.tsx lines 89-93 (existing isActive pattern to extend)

### Notification Dot (MOVED)
- Condition: `noGames` (profile loaded + `totalGames === 0`)
- Appears on: Library nav item in all three surfaces
- Removed from: Import nav item (no longer exists as standalone item)
- test-ids: `library-notification-dot` (desktop), `library-notification-dot-mobile` (bottom bar), `library-notification-dot-drawer` (More drawer)
- Markup: identical pulsing red-500 dot pattern from existing import-notification-dot
- Source: CONTEXT.md D-10

Implementation note (Claude's Discretion): `ImportTab.tsx` and `OverviewTab.tsx` may either re-export / thin-wrap the existing page components or absorb them. Prefer thin-wrap over move to minimize diff and test disruption.
