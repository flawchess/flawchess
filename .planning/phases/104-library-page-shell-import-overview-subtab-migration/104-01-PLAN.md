<objective>
Create the new `frontend/src/pages/library/` component trio that turns the existing Import and Overview pages into deep-linkable, URL-routed subtabs of a single Library page shell — mirroring the established `Endgames.tsx` two-subtab `<Tabs variant="brand">` pattern exactly. No App.tsx wiring happens in this plan; these are net-new files that compile and test in isolation.

Purpose: Establishes the Library page structure (LIB-02) and the Import/Overview subtab containers (LIB-03, LIB-04) so Plan 02 can repoint the router and nav at them. Identical runtime behavior for the migrated surfaces — they move location, they do not change.

Output: `LibraryPage.tsx` (shell + state-dependent default-subtab `<Navigate>`, desktop + mobile tabs, prop threading), `ImportTab.tsx` (thin wrapper forwarding the three App-level job props), `OverviewTab.tsx` (zero-prop wrapper of `GlobalStatsPage`), and a one-word `export` addition to `Import.tsx`'s props interface.
</objective>
