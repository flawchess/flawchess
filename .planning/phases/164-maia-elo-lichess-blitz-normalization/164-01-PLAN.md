<objective>
Add the model-facing rating-conversion math that normalizes any player rating to its
Lichess-blitz equivalent (Maia-3's training scale), with full unit coverage.

Purpose: chess.com ratings run ~150-250 below the Lichess-blitz equivalent and Lichess-non-blitz
ratings run above it, so seating Maia's ELO from a raw rating mis-conditions its move prediction.
This plan is the pure-Python primitive; Plans 02 (serialization) and 03 (frontend) consume it.

Output: two new functions in the existing `chesscom_to_lichess.py` module (`_invert_table2_column`,
`normalize_to_lichess_blitz`) and a new block of unit tests in the existing
`tests/services/test_chesscom_to_lichess.py`, covering every source path and the two research-found
table edges (classical tie, classical None-gap) plus every None/fallback edge.
</objective>
