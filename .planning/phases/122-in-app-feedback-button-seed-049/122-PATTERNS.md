### `app/models/feedback.py` (model, CRUD)

**Analog:** `app/models/position_bookmark.py` (whole file, 31 lines)

```python
        nullable=False, server_default=func.now(),
```

**Replicate:** `user_id` FK with `ondelete="CASCADE"` + `index=True` (D-05 / CLAUDE.md DB rules);

`sentiment: Mapped[str | None] = mapped_column(String(16), nullable=True)` (nullable = optional, wide enough for `"negative"` per UI-SPEC);
bare `Mapped[datetime.datetime]` is auto-mapped to `DateTime(timezone=True)` by `Base` (no explicit type needed). No `UniqueConstraint` (no natural key beyond `id`).

### `app/services/feedback_service.py` (service, transform: Sentry + ELO derivation)

**Analog A (Sentry signal):** `app/routers/eval_remote.py:263-275`

**Analog B (ELO bucket anchors):** `app/services/canonical_slice_sql.py:231-248` (`elo_bucket_expr`)
— a SQL `CASE`; mirror its anchors in a Python helper:
```
< 800 → None ;  <1200 → 800 ;  <1600 → 1200 ;  <2000 → 1600 ;  <2400 → 2000 ;  else → 2400
```

**Analog C (rating source):** `app/repositories/user_rating_anchors_repository.py:129`
`fetch_anchors_for_user(session, *, user_id=user.id) -> dict[TimeControlBucket, RatingAnchorRow]`
(V4: pass the authenticated user's id, never a client param).

**Analog D (username/platform):** `app/models/user.py:19-20` — `user.chess_com_username` /
`user.lichess_username` are already on the injected `User` object (no extra query).

**Replicate (D-05):**
```python
_ELO_ANCHORS = ((2400, 2400), (2000, 2000), (1600, 1600), (1200, 1200), (800, 800))

def elo_bucket(rating: int) -> int | None:
    if rating < 800:
        return None
    for threshold, bucket in _ELO_ANCHORS:   # descending; first match wins
        if rating >= threshold:
            return bucket
    return None

async def push_sentry_signal(session, user, data) -> None:
    anchors = await fetch_anchors_for_user(session, user_id=user.id)
    rating = max((a.anchor_rating for a in anchors.values()), default=None)  # [ASSUMED A1] highest anchor
    bucket = elo_bucket(rating) if rating is not None else None
    platform = "chess.com" if user.chess_com_username else "lichess" if user.lichess_username else "none"
    sentry_sdk.set_tag("source", "feedback")
    sentry_sdk.set_tag("platform", platform)
    sentry_sdk.set_tag("elo_bucket", str(bucket) if bucket is not None else "unknown")
    sentry_sdk.set_context("feedback", {"user_id": user.id, "page_url": data.page_url, "sentiment": data.sentiment})
    sentry_sdk.capture_message("feedback submitted", level="info")
```
**Critical:** static message string only — NEVER `f"feedback from {username}"` (fragments Sentry
grouping — Pitfall 2 / CLAUDE.md). Variable data goes in tags/context. Use `capture_message`
(non-exception, D-05), not `capture_exception`. The service must be imported by the router or knip's
backend equivalent / unused-export sense doesn't apply (Python), but keep it referenced (A2 — if it
stays trivial it could fold into the router; a thin service is the cleaner home for this logic).

### `frontend/src/hooks/useScrollDirection.ts` — **NO ANALOG (genuinely new)**

No scroll-direction hook exists in the codebase. Build: debounced `window` `scroll` listener
comparing `window.scrollY` to the previous value; return `'up' | 'down'` (show on `up` or at-top,
hide on `down`). Unit-testable in jsdom (D-02 test). This + the overlay signal are the phase's only
real implementation risk.

### Overlay-yield signal — **NO ANALOG (genuinely new); planner's choice (D-02 / A3)**

There is **no global "overlay open" signal** — drawer state is per-page local state
(`frontend/src/pages/openings/useSidebarState.ts` holds `sidebarOpen` / `filterSidebarOpen` /
`bookmarkSidebarOpen` in component `useState`, not a store). Two acceptable mechanisms (UI-SPEC
leaves the mechanism to the planner; the yield behavior is the binding requirement):
- **(a) radix DOM-presence check** (lower wiring, more brittle):
  `document.querySelector('[data-slot="dialog-content"], [data-slot="drawer-content"], [role="dialog"]')`
  — radix sets these `data-slot`s (verified in `dialog.tsx:48`, drawer analog). Hide the button when present.
- **(b) global zustand `useOverlayStore`** (explicit; requires wiring each drawer to increment/decrement on
  open/close). zustand is already used in the project (`useFilterStore`, `useFlawFilterStore`).
Verify against the ACTUAL components: `MobileBottomBar`/`MobileMoreDrawer` (`App.tsx`), and the
filter/bookmark drawers driven by `useSidebarState` on `/openings` (Pitfall 4 — the seed's stated
design risk).

### Auth / access control (V2 / V4)


### Session auto-commit (never commit in repo)
**Source:** `app/core/database.py` `get_async_session` (auto-commits on success);
`position_bookmark_repository.py:69`
**Apply to:** `feedback_repository.create_feedback`.

### Sentry non-exception signal (tags vs context)
**Source:** `app/routers/eval_remote.py:263-275`
**Apply to:** `feedback_service.push_sentry_signal`.
Static `capture_message(..., level="info")`; variable data via `set_tag` (filterable:
`source`/`platform`/`elo_bucket`) + `set_context`. Never embed variables in the message string.
No double-capture on the frontend (`MutationCache.onError` covers mutations).

### Router prefix convention
**Source:** `position_bookmarks.py:33`, `main.py:143`
**Apply to:** the feedback router. `APIRouter(prefix="/feedback", tags=["feedback"])` + relative
decorator paths (`@router.post("")`), registered `prefix="/api"`.

### Button variants (never hand-roll colors)
**Source:** `frontend/src/components/ui/button.tsx:11-27`
**Apply to:** both buttons. Submit = `variant="default"` (brand brown, line 14-15); trigger =
`variant="secondary"`/`outline`. Never `className="bg-*"` on a button.

### Browser-automation a11y
**Source:** CLAUDE.md Browser Automation Rules; UI-SPEC Component Inventory.
**Apply to:** every interactive element. `data-testid` (kebab, component-prefixed) on
button/modal/textarea/each sentiment segment/submit/cancel; `aria-label` on the icon-only trigger;
semantic `<button>`/`<form>`.
