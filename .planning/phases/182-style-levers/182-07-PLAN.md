<objective>
Wire the style layer into the game loop (STYLE-01/02): thread an optional `BotGameSettings.style?` into the book-weighting call, the draw-accept contempt, and a new resign-hysteresis path — so a styled bot follows its curated book, accepts draws per its contempt, and can resign (D-07: resignation surfaces now). Every seam is gated on `settings.style`, so all current callers (unstyled) are byte-identical (D-03). The persona UI that actually SELECTS a style is Phase 183; this plan makes the engine wiring ready.

Purpose: The book wiring, contempt, and resign hysteresis are the three CONTEXT.md canonical-ref seams in useBotGame. Hysteresis is inherently stateful across turns and MUST live as a per-game ref (Pitfall 3), mirroring the four existing latches; the pure predicate from Plan 02 stays call-and-forget. Book plies must keep the null-score sentinel so a styled bot never resigns/draws off an unevaluated position.
Output: `BotGameSettings.style?` + style-aware `resolveBookMove` + contempt-threaded draw accept + `consecutiveLowScoreTurnsRef` + resign check in `useBotGame.ts`, plus a `useBotGame`-level behavioral test proving the styled-resign wiring (counter increment/reset, hysteresis-floor firing, unstyled unreachability).
</objective>
