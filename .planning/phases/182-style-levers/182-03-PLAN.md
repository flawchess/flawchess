<objective>
Curate the per-style opening-line prefix lists (STYLE-01, D-05) as plain exported data: for each of the 4 styles (Attacker / Trickster / Grinder / Wall), a white-side and black-side `ReadonlySet<string>` of SAN move-sequence prefixes drawn from the real ECO corpus (`frontend/public/openings.tsv`). This is pure data with no code dependencies, so it lands independently in Wave 1; Plan 04's `styleBookWeighting` and Plan 05's bundles consume it.

Purpose: D-05 requires FRESH SAN-prefix curation (trollOpenings.ts stores position keys, not SAN lines — its lines live in comments). The prefix strings must be joined full sequences (Pitfall 2: `styleBookWeighting`'s membership test needs the joined history+san key, not a bare per-ply SAN), split by color (Open Question #3), and every one must actually exist as a prefix in openings.tsv or it can never be booked (D-04/D-06).
Output: `frontend/src/lib/engine/styleOpeningLines.ts` (data + `styleLinesFor` accessor) + a validation test that checks membership against openings.tsv.
</objective>
