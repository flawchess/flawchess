    has_queen = "Q" in sig
    has_rook = "R" in sig
    has_minor = "B" in sig or "N" in sig

Endgame transition point (from RESEARCH.md):
- First position per game where material_count < ENDGAME_MATERIAL_THRESHOLD (2600)
- material_imbalance at that ply determines conversion/recovery classification
- user_material_imbalance = imbalance if user_color == "white" else -imbalance

            (3, "1-0", "white", "KP_KP", 0),

        ]

        """Percentages computed correctly."""

        assert rook.losses == 1

            (3, "1-0", "white", "KR_KR", -100),   # down, won → not conversion

from pydantic import BaseModel
from app.schemas.analysis import GameRecord

class ConversionRecoveryStats(BaseModel):
    """Inline conversion/recovery stats for one endgame category (D-06, D-08, D-09)."""
    conversion_pct: float        # win rate when up material (0-100), per D-08
    conversion_games: int        # games where user entered this endgame type up
    conversion_wins: int         # wins among those games
    recovery_pct: float          # draw+win rate when down material (0-100), per D-09
    recovery_games: int          # games where user entered this endgame type down
    recovery_saves: int          # draws+wins among those games

class EndgameCategoryStats(BaseModel):
    """W/D/L + inline conversion/recovery for one endgame category (D-06, D-10)."""
    endgame_class: str           # rook|minor_piece|pawn|queen|mixed|pawnless
    label: str                   # display label: "Rook", "Minor Piece", etc.
    wins: int
    draws: int
    losses: int
    total: int
    win_pct: float
    draw_pct: float
    loss_pct: float
    conversion: ConversionRecoveryStats  # inline, not separate section (D-10)

class EndgameStatsResponse(BaseModel):
    """Response for GET /endgames/stats. Categories sorted by total desc (D-05)."""

class EndgameGamesResponse(BaseModel):

    offset: int
    limit: int

**2. Create `app/repositories/endgame_repository.py`** with two query functions:

a) `query_endgame_entry_rows(session, user_id, time_control, platform, rated, opponent_type, recency_cutoff)`:
   - Returns one row per game: (game_id, result, user_color, material_signature, user_material_imbalance)
   - Uses subquery with `func.min(GamePosition.ply)` where `material_count < ENDGAME_MATERIAL_THRESHOLD` to find the endgame transition point per game
   - Joins back to get material_signature and material_imbalance at that ply
   - Computes user_material_imbalance = imbalance if white, else -imbalance (add comment explaining sign flip for black)
   - Applies all filters: time_control (list), platform (list), rated (bool), opponent_type ("human"/"bot"/"all"), recency_cutoff (datetime)
   - NO color filter (per D-02)
   - Uses `ENDGAME_MATERIAL_THRESHOLD = 2600` named constant

b) `query_endgame_games(session, user_id, endgame_class, time_control, platform, rated, opponent_type, recency_cutoff, offset, limit)`:
   - Returns paginated GameRecord-shaped rows for games whose endgame entry point classifies as the requested endgame_class

   - Returns (games_list, matched_count) tuple for pagination

**3. Create `app/services/endgame_service.py`**:

a) `classify_endgame_class(material_signature: str) -> str`:
   - Pure function, follows RESEARCH.md Pattern 1
   - Parse material_signature, classify into rook/minor_piece/pawn/queen/mixed/pawnless
   - Classification priority: mixed (Q+any, R+minor, minor+pawn) > queen > rook > minor_piece > pawn > pawnless

b) `_aggregate_endgame_stats(rows) -> list[EndgameCategoryStats]`:
   - Takes raw (game_id, result, user_color, material_signature, user_material_imbalance) tuples
   - Groups by classify_endgame_class(material_signature)
   - For each category: count W/D/L using derive_user_result, compute percentages
   - For each category: compute conversion (wins / games_up when user_material_imbalance > 0) and recovery (wins+draws / games_down when user_material_imbalance < 0) — per D-08, D-09
   - Sort by total descending (D-05) — NOT a fixed order
   - Return list[EndgameCategoryStats]

c) `get_endgame_stats(session, user_id, time_control, platform, rated, opponent_type, recency) -> EndgameStatsResponse`:
   - Convert recency to cutoff using `recency_cutoff()` from analysis_service
   - Call query_endgame_entry_rows
   - Call _aggregate_endgame_stats
   - Return EndgameStatsResponse

d) `get_endgame_games(session, user_id, endgame_class, time_control, platform, rated, opponent_type, recency, offset, limit) -> EndgameGamesResponse`:
   - Call query_endgame_games or query_endgame_entry_rows + filter by classify_endgame_class
   - Build GameRecord objects matching the existing schema
   - Return EndgameGamesResponse with pagination
