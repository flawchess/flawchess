## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, gap between filter chips |
| md | 16px | Default element spacing, section padding |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Touch targets (filter toggle buttons, tab triggers) must be at minimum 44px height on mobile. Use `min-h-11` (44px) on mobile, `sm:min-h-0` to revert on sm+. This matches the existing FilterPanel pattern.
- Category row height in the endgame bar chart: 64px per row (matching WDLBarChart `data.length * 64 + 60` formula).

## Component Inventory

All components already exist in the project. No new shadcn components need to be installed.

### Reused without modification
- `Tabs`, `TabsList`, `TabsTrigger`, `TabsContent` — sub-tab structure
- `Collapsible`, `CollapsibleTrigger`, `CollapsibleContent` — "More filters" collapsible on sidebar
- `Button` (variant="ghost", variant="outline") — collapsible triggers, nav buttons
- `FilterPanel` — reused as-is; Endgames page does not pass `color` or `matchSide` to it (those fields are hidden/unused since no color filter for endgames per D-02)
- `GameCardList` — Games sub-tab

### New component to create
- `EndgameWDLChart` — new chart component at `frontend/src/components/charts/EndgameWDLChart.tsx`. Follows the exact same visual pattern as `WDLBarChart` (stacked horizontal bar + game count outline bar) but:
  - Y-axis labels are the six endgame categories (Rook, Minor Piece, Pawn, Queen, Mixed, Pawnless), not bookmark labels
  - Below each category bar row, render inline conversion and recovery metrics as a `text-xs text-muted-foreground` line: `Conversion: X% (n/m)  ·  Recovery: Y% (n/m)`
  - Props: `EndgameWDLChartProps` — takes API response data typed to the new endgame stats response schema
  - `selectedCategory: string | null` prop — highlights the currently selected row (used to drive Games tab filter)
  - `onCategoryClick: (category: string) => void` — clicking a category row sets selection

### Filter state for Endgames page
The Endgames page uses a subset of `FilterState`:

```ts
// Only these fields are used — no color, no matchSide
type EndgameFilterState = Pick<FilterState, 'timeControls' | 'platforms' | 'recency' | 'rated' | 'opponentType'>;
```

The main filter row (Time Control, Platform, Recency) appears directly in the sidebar. Rated and Opponent Type are under "More filters" collapsible, identical to the Openings page pattern.

## Page Layout

### Desktop (md+): two-column grid
```
[ Filter sidebar 280-320px ]  |  [ Tabs content — flex-1 ]
                               |  [ Statistics | Games ]
```

Column widths: `md:grid-cols-[280px_1fr]` (slightly narrower than Openings since no chessboard in sidebar).

### Mobile (< md): single column

2. "More filters" collapsible

4. Tab content

### Tab structure
- Tab 1: "Statistics" — default, URL `/endgames/statistics`
- Tab 2: "Games" — URL `/endgames/games`
- Redirect: `/endgames` and `/endgames/` redirect to `/endgames/statistics`

Tab trigger icons (lucide-react):
- Statistics tab: `BarChart2Icon` (h-4 w-4 mr-1.5)
- Games tab: `Gamepad2Icon` (h-4 w-4 mr-1.5)

### Statistics tab content
1. Section heading: "Results by Endgame Type" (`text-lg font-medium mb-3`) with InfoPopover
2. `EndgameWDLChart` — stacked horizontal bar chart, categories sorted by game count descending
3. Below each category bar: inline conversion / recovery line

### Games tab content
1. If no category selected: prompt text `text-sm text-muted-foreground` — "Select an endgame category in the Statistics tab to view matching games."
2. If category selected: show `GameCardList` with selected category label as subtitle above the list

## Interaction Contract

### Category selection
- User clicks any bar row in `EndgameWDLChart`
- Selected row gets a visible highlight: `bg-muted/50` background on the row container, `ring-1 ring-primary/40` border
- Selection is stored in page-level React state (`selectedCategory: string | null`)
- Switching to Games tab shows games for the selected category
- Clicking the same row again deselects it (sets to null)

### Filter changes
- Any filter change resets `gamesOffset` to 0 (pagination reset)
- Filters apply to both Statistics and Games sub-tabs simultaneously (D-03)
- Filter state persists when switching between Statistics and Games tabs

### Empty / loading states
See Copywriting Contract section below.

### Mobile filter collapsible
- Default state: collapsed
- Trigger label: "Filters" with ChevronDown / ChevronUp icon
- `data-testid="section-filters-mobile"`
