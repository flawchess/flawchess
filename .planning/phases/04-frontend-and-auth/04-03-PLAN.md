  <action>
    1. Create `frontend/src/hooks/useChessGame.ts`:
       - Manages a `Chess` instance via `useRef` (not useState — avoid re-render on internal mutations).
       - State: `position` (FEN string for react-chessboard), `moveHistory` (string[] of SAN moves), `currentPly` (number — which move we're viewing), `hashes` (ZobristHashes from zobrist.ts).
       - `makeMove(sourceSquare, targetSquare)`: calls `chess.move({from, to, promotion: 'q'})` wrapped in try/catch. If the move succeeds AND currentPly === moveHistory.length (we're at the end), update position, push to moveHistory, increment currentPly, recompute hashes. If currentPly < moveHistory.length (user navigated back then makes a new move), truncate moveHistory at currentPly, then apply the move — this replaces the future line.
       - `goToMove(ply)`: Create new Chess(), replay moveHistory[0..ply-1], set position from the new chess FEN, update currentPly. Recompute hashes.
       - `goForward()`: goToMove(currentPly + 1) if currentPly < moveHistory.length.
       - `goBack()`: goToMove(currentPly - 1) if currentPly > 0.
       - `reset()`: New Chess(), clear moveHistory, set currentPly to 0, set position to 'start', recompute hashes (starting position hash).
       - `getHashForAnalysis(matchSide)`: Returns the appropriate hash as a string based on matchSide ('white' -> whiteHash, 'black' -> blackHash, 'full' -> fullHash). Uses `hashToString()` from zobrist.ts.
       - Recompute hashes on every position change by calling `computeHashes(chess.current)`.

    2. Create `frontend/src/components/board/ChessBoard.tsx`:
       - Receives `position` (FEN) and `onPieceDrop` callback from useChessGame.
       - Renders `<Chessboard position={position} onPieceDrop={onPieceDrop} />` from react-chessboard.
       - Set `boardWidth` responsively — use a ref to measure container width. On mobile (<768px) use full container width; on desktop use a fixed width (e.g., 400px or container-constrained).
       - Dark theme board colors: use `customDarkSquareStyle` and `customLightSquareStyle` with muted colors matching the dark UI.

    3. Create `frontend/src/components/board/MoveList.tsx`:
       - Receives `moveHistory: string[]` and `currentPly: number` and `onMoveClick: (ply: number) => void`.
       - Renders moves in pairs: "1. e4 e5  2. Nf3 Nc6 ..." format.
       - Each move is clickable — calls `onMoveClick(ply)` where ply is the 1-indexed position.
       - The currently viewed move (matching currentPly) is highlighted with a distinct background.
       - Scrollable container if many moves. Auto-scroll to keep current move visible.

    4. Create `frontend/src/components/board/BoardControls.tsx`:
       - Receives `onBack`, `onForward`, `onReset`, `canGoBack: boolean`, `canGoForward: boolean`.
       - Row of icon buttons (using lucide-react icons): SkipBack (reset), ChevronLeft (back), ChevronRight (forward).
       - Buttons disabled when at boundaries (can't go back at ply 0, can't go forward at last ply).
  </action>
