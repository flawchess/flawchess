<feature>
  <name>Position Classifier</name>
  <files>app/services/position_classifier.py, tests/test_position_classifier.py</files>
  <behavior>
    ## Game Phase (PMETA-01, per D-01)
    - Starting position (phase_score=62) -> "opening"
    - Early queen trade, both queens off (phase_score=44) -> "middlegame" (NOT endgame)
    - Phase score exactly 25 -> "middlegame" (boundary: >= 25 is middlegame)
    - Phase score 24 -> "endgame"
    - All major pieces off, only pawns + kings (phase_score=0) -> "endgame"
    - endgame_class is None when game_phase != "endgame"

    ## Material Signature (PMETA-02, per D-02)
    - KRP vs KR: asymmetric, stronger side first -> "KRP_KR"
    - K vs K: bare kings -> "K_K"
    - KQR vs KQR: symmetric, lexicographic (equal) -> "KQR_KQR"
    - Starting position: "KQRRBBNNPPPPPPPP_KQRRBBNNPPPPPPPP" (33 chars, needs String(40))
    - Symmetric position produces same signature when viewed from white or black side

    ## Material Imbalance (PMETA-03, per D-03)
    - Equal material -> 0
    - White up a pawn -> +100
    - Black up a queen -> -900
    - Starting position -> 0

    ## Endgame Class (PMETA-04, per D-04)
    - K+P vs K -> "pawn" (priority 1: kings and pawns only)
    - K+R vs K -> "pawnless" (priority 2: no pawns either side)
    - K+R+P vs K+R -> "rook" (priority 3: rooks + pawns, no other pieces)
    - K+B+P vs K+N -> "minor_piece" (priority 4: minor pieces + pawns)
    - K+Q+P vs K -> "queen" (priority 5: queens + pawns)
    - K+R+B+P vs K+Q -> "mixed" (priority 6: catch-all)
    - K+B+N vs K (no pawns) -> "pawnless" not "minor_piece" (priority 2 before 4)

    ## Tactical Indicators (per D-05)
    - White has 2 bishops -> has_bishop_pair_white=True
    - White has 1 bishop -> has_bishop_pair_white=False
    - Each side exactly 1 bishop on different colors -> has_opposite_color_bishops=True
    - Both bishops on same color -> has_opposite_color_bishops=False
    - One side has 0 or 2+ bishops -> has_opposite_color_bishops=False
  </behavior>
  <implementation>
    Create `app/services/position_classifier.py` with:

    1. A `PositionClassification` dataclass with 7 fields:
       - `game_phase: str` ('opening' | 'middlegame' | 'endgame')
       - `material_signature: str` (canonical form)
       - `material_imbalance: int` (signed centipawns)
       - `endgame_class: Optional[str]` (None for non-endgame)
       - `has_bishop_pair_white: bool`
       - `has_bishop_pair_black: bool`
       - `has_opposite_color_bishops: bool`

    2. Named constants (no magic numbers per CLAUDE.md):
       - `_PHASE_WEIGHT = {chess.QUEEN: 9, chess.ROOK: 5, chess.BISHOP: 3, chess.KNIGHT: 3}`
       - `_OPENING_THRESHOLD = 50`
       - `_ENDGAME_THRESHOLD = 25`
       - `_MATERIAL_VALUE_CP = {chess.PAWN: 100, chess.KNIGHT: 300, chess.BISHOP: 300, chess.ROOK: 500, chess.QUEEN: 900}`
       - `_SIGNATURE_ORDER = [chess.QUEEN, chess.ROOK, chess.BISHOP, chess.KNIGHT, chess.PAWN]`
       - `_SIGNATURE_LETTER = {chess.QUEEN: 'Q', chess.ROOK: 'R', chess.BISHOP: 'B', chess.KNIGHT: 'N', chess.PAWN: 'P'}`

    3. Internal helper functions:
       - `_compute_phase_score(board) -> int`: sum of non-pawn, non-king material weights for both sides
       - `_compute_game_phase(board) -> str`: apply thresholds (>= 50 opening, >= 25 middlegame, < 25 endgame)
       - `_side_string(board, color) -> str`: 'K' + piece letters in _SIGNATURE_ORDER, repeated per count
       - `_side_material(board, color) -> int`: sum of centipawn values for one side (excludes king)
       - `_compute_material_signature(board) -> str`: canonical form, stronger side first, lex tie-break
       - `_compute_material_imbalance(board) -> int`: white_material - black_material in centipawns
       - `_compute_endgame_class(board) -> Optional[str]`: priority chain pawn->pawnless->rook->minor_piece->queen->mixed
       - `_compute_opposite_color_bishops(board) -> bool`: each side exactly 1 bishop on different square colors, using `chess.BB_DARK_SQUARES & chess.BB_SQUARES[sq]`

    4. Public function:
       - `classify_position(board: chess.Board) -> PositionClassification`: calls all helpers, sets endgame_class to None if game_phase != 'endgame'

    Mirror the style of `app/services/zobrist.py`: module docstring, private helpers prefixed with `_`, public function documented with Args/Returns docstring.
  </implementation>
</feature>
