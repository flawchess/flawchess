<objective>
Land the three load-bearing correctness fixes the rest of Phase 145 sits on, BEFORE any
sentinel blob is ever written to the corpus:

1. **D-06 sentinel tolerance** in `_classify_tactic_gated` — treat the empty list `[]` as
   "no gate-eligible line → no suppression" (today `[]` enters the gate and the one-mover
   discard suppresses the motif). This is the PREREQUISITE: writing sentinels before this
   fix would incorrectly strip tags (RESEARCH Pitfall 1).
2. **`_apply_submit` gate gap** (RESEARCH Priority 6, SHIP-02) — the live remote-worker submit
   handler writes blobs but calls `_classify_and_fill_oracle` WITHOUT `blob_map`, so new
   remote-worker games get UNFILTERED tags. Thread `blob_map` through so the live drain writes
   gated tags for all new games.
3. **Partial index** for the tier-4 backfill predicate `allowed_pv_lines IS NULL` (RESEARCH
   Open Question 1) so the lottery in Plan 02 does not seq-scan game_flaws.

Purpose: De-risk the rollout — these are small, independently-testable fixes to the EXISTING
live path plus a query-efficiency index. Nothing here depends on the new endpoints.
Output: Corrected `_classify_tactic_gated`, corrected `_apply_submit`, a new Alembic partial index.
</objective>
