<objective>
Build the **submit half** of the flaw-blob backfill (D-04): the
`POST /eval/remote/flaw-blob-submit` endpoint that ingests an upgraded worker's MultiPV-2 results,
reassembles per-node `PvNode` blobs, writes `allowed_pv_lines`/`missed_pv_lines` (plus the `[]`
sentinel for un-fillable lines, D-06), and then — INSIDE the same handler — runs the single gated
classify path (`_classify_tactic_gated`) for that game's flaws and bulk-updates only the 8 tactic
columns (D-07). This makes the rollout **rolling**: each backfilled game lands fully done (blobs +
gated tags) with no end-of-backfill barrier.

The handler is PURE (Pitfall 2): it reads worker evals from the request body, assembles blobs, and
writes — no engine calls, no asyncio.gather. It reuses `_batch_update_flaw_pv_lines` for the JSONB
write and `bulk_update_tactic_tags` for the tag write. It depends on Plan 01's D-06 sentinel fix so
the `[]` sentinel skips the gate (no suppression) at classify time, and on Plan 03's schemas + lease.

Purpose: Close the loop — worker results become gated tags atomically per game (transitional path, D-07).
Output: The `/flaw-blob-submit` endpoint + blob-assembly helper, end-to-end tested.
</objective>
