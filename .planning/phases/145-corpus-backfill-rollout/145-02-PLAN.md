<objective>
Add the lowest-priority **tier-4 blob-backfill lottery** that feeds the new flaw-blob lease
endpoint (Plan 03). It is a thin clone of the existing tier-3 table-less lottery
(`_claim_tier3_derived`), but with a simpler single-level random pick (no ES user weighting —
tier-4 is spare-capacity only) over the predicate `game_flaws.allowed_pv_lines IS NULL` (D-03).

Per D-02 it fires only when no tier-1/2/3 work remains and only under `EVAL_AUTO_DRAIN_ENABLED`,
so live analysis and the not-yet-analyzed backlog always win and the fleet self-paces on spare
capacity. Per D-05 old/un-upgraded workers never reach this work (the new endpoint in Plan 03 is
the capability signal). Per D-03 the lottery is idempotent-by-construction: once a game's flaw
blobs (or D-06 sentinels) are written it stops matching — no `eval_jobs` row, `job_id=None`,
acceptable double-claim (idempotent write).

Purpose: Distribute the bulk MultiPV-2 compute across the remote fleet without a server engine pass.
Output: `_claim_tier4_blob`, a `TIER_BLOB_BACKFILL` constant, and the tier-4 dispatch branch.
</objective>
