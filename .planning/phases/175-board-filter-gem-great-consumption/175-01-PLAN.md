<objective>
Deliver FILT-01's backend: a "has gem" / "has great" Library game filter built on
the existing flaw/tactic EXISTS machinery, plus the SQLAlchemy expression twin of
`classify_best_move` that the filter needs (the same classifier the board read path
reuses in Python). Board and filter agree by construction because both derive from
`classify_best_move`'s constants and math.

Purpose: FILT-01 forces backend classification in SQL (the EXISTS must know the
tier), so this plan builds the one authoritative SQL twin and wires two booleans
through the single `apply_game_filters()` implementation.
Output: `best_move_tier_sql`/`_es_sql`, `best_move_exists_from_table`, `has_gem`/
`has_great` params on `apply_game_filters` and `GET /library/games`.
</objective>
