<amendment>
**2026-06-05 (post-plan, user-approved) — emit mistakes + blunders only; inaccuracies are count-only.**

The serving design is settled as **Option 2** (see 105-CONTEXT.md "On-the-fly" amendment + SEED-036 2026-06-05 decision log): the cross-game filter + stats panel run as a **SQL window-scan** that returns only flagged mistake/blunder rows; **Python tags that reduced set** (this module's tag functions); **inaccuracy/mistake/blunder counts come from SQL aggregates / the `games.white_/black_*` oracle columns**, never from this function. **No materialization, no backfill; thresholds are bound parameters.** Game filters are boolean `EXISTS` ("≥1 blunder"), not counts.

Concrete change to THIS plan: `classify_game_mistakes` **emits tagged FlawRecords for severity in {mistake, blunder} only** — inaccuracies are **not emitted** (count-only). `_classify_severity` still returns all three bands and `INACCURACY_DROP` stays defined and tested (SQL uses it for counts; the oracle test in plan 02 tallies inaccuracies from the internal all-moves pass); only *emission* is filtered to mistakes/blunders. A game whose only flaws are inaccuracies therefore returns an **empty list** (still "analyzed" — distinct from `GameNotAnalyzed`).
</amendment>
