<objective>
Complete the mistake-detection service by adding all eight attribution tags to the flaw objects produced by `classify_game_mistakes`, create the repository read helper that loads a game's positions ordered by ply, and add the DB-backed repository test plus the oracle-closeness sanity test.

Purpose: Tags (LIBG-06) are the per-flaw signal the Games/Flaws/Analysis surfaces and SEED-037 Train consume; the repository (LIBG-02/07) is the on-the-fly data path that keeps SQL out of the service; the oracle test proves the derived severities track Lichess's own classification closely without claiming identity.

Output: tag-populated `FlawRecord`s, `app/repositories/mistakes_repository.py`, `tests/test_mistakes_repository.py`, and the tag + oracle test classes in `tests/services/test_mistakes_service.py`.
</objective>

<amendment>
**2026-06-05 (post-plan, user-approved) — tags apply to mistakes + blunders only.**

Per the Option-2 serving design (SQL window-scan returns flagged mistake/blunder rows; Python tags + stats over them; inaccuracies are count-only via SQL aggregates / `games.white_/black_*`; boolean `EXISTS` filters; no materialization — see 105-CONTEXT.md and the SEED-036 2026-06-05 decision log), `classify_game_mistakes` emits only mistake/blunder records (changed in 105-01). So the tag pass here runs over mistakes/blunders only — the tempo-exclusivity invariant holds for every emitted flaw by construction. The oracle-closeness test tallies mistake/blunder from the emitted records and the **inaccuracy** count from the plan-01 internal all-moves severity pass (inaccuracies aren't emitted).
</amendment>

| caller -> repository | The caller supplies `game_id` + `user_id`; the repository filters on both so one user cannot read another user's positions. No raw SQL string interpolation (parameterized SQLAlchemy). |
