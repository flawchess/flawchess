## Spacing Scale

Declared values (8-point scale with 4px base, matching the existing token set):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip inner padding, inline badge gaps |
| sm | 8px | Gap between chips in a chip row, gap between severity badges |
| md | 16px | Card horizontal padding, panel section padding |
| lg | 24px | Section gap between panel zones (band → trend → tags) |
| xl | 32px | Vertical gap between Flaw-Stats panel and card list |
| 2xl | 48px | Reserved — page-level breaks |
| 3xl | 64px | Reserved — not used at this surface |

Exceptions:
- Card header bottom-border padding: 9px (matches sketch 001-A `pb-[9px]` for board alignment).
- Flaw-column left-border padding: 14px (dashed divider to flaw content, sketch 001-A).
- `text-chip` (13px) font used on severity badges, tag chips, and panel KPI labels — this is a
  tooltip/aside exception per CLAUDE.md (display-only inline chips, not primary body text).

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm, 0.875rem) | 400 | 1.5 | Card opening line, metadata items, filter labels |
| Label | 13px (text-chip, 0.8125rem) | 700 | 1.3 | Severity badge labels, tag chip text, KPI sub-labels, denominator pill |
| Subheading | 14px (text-sm) | 800 | 1.4 | Card player names at desktop (bold emphasis), panel stat values in cells |
| Heading | 18px (text-lg, 1.125rem) | 600 (Fredoka) | 1.2 | "Flaw-Stats" panel title (brand font) |
| KPI number | 24px (text-2xl, 1.5rem) | 800 | 1.0 | Severity-rate cells in the stat band (blunders/game number) |

Notes:
- Body floor is `text-sm` (14px) per CLAUDE.md. The `text-chip` (13px) exception applies only to
  severity badges, tag chips, KPI labels, and the denominator pill — all short, display-only,
  non-primary surfaces explicitly covered by the CLAUDE.md tooltip/aside exception.
- Player names in the card header: `text-sm` (14px) / weight 700 on desktop (single-line);
  `text-sm` / weight 400 on mobile (stacked, both lines).
- Panel trend chart axis labels use `text-chip` (11–13px) inside an SVG — acceptable inside
  Recharts/SVG rendering where normal Tailwind font classes are unavailable.

Source: sketch `themes/default.css` type scale; CLAUDE.md typography rules; GameCard.tsx `text-sm`
baseline.

## Analyzed Game Card (Sketch 001-A) — Detailed Contract

### Structure

```
┌──────────────────────────────────────────────────────────────┐
│ CARD HEADER (full width, bottom-border)                      │
│  ■ WhitePlayer (1842)  vs  □ BlackPlayer (1871)  [Platform ↗]│
├─────────────┬────────────────────┬───────────────────────────┤
│ MiniBoard   │ Game info          │ Flaw column               │
│ (100px desk)│  Opening name      │  (dashed left border)     │
│ (105px mob) │  Metadata row      │  [2 Blunders] [1 Mistake] │
│             │                    │  [4 Inacc.]               │
│             │                    │  ┄ chip row ┄             │
│             │                    │  ⑤result-changing         │
│             │                    │  ⏱time-pressure           │
│             │                    │  ◎miss                    │
└─────────────┴────────────────────┴───────────────────────────┘
```

Desktop: 3 columns (board / info / flaws). The flaw column uses `flex: 0 0 auto` —
its width is dictated by the severity badge row (which must not wrap).

Mobile: full-width header (names stacked, no "vs"), then board + info row, then
full-width flaw block below both columns.

### Header

- Desktop: single-line `■ White (rating) vs □ Black (rating)` — `text-sm` weight 700.
- Mobile: two stacked lines (`■ White (rating)` / `□ Black (rating)`) — `text-sm` weight 400.
- Platform icon + external link pinned `ml-auto` in both cases.
- Bottom border: `1px solid var(--color-border)` with `pb-[9px] mb-3`.
- The card still carries the WDL left-border (4px, color from `BORDER_COLORS[result]`).

### Severity count row (analyzed games only)

Three pills in a nowrap row: `[N Blunders] [N Mistake] [N Inacc.]`. Full labels (not just
"B/M/I") — the count number is `font-weight: 800 text-base` and the label is `font-weight: 700
text-chip`. Each pill has a colored border and background at the severity alpha values above.
Pills must NOT wrap — the flaw column is sized to the full-label nowrap row.

### Tag chips (analyzed games — display-only)

Rendered below the severity row. `flex-wrap` is allowed — chips may wrap to a second line.
Color = family (tempo violet / opportunity cyan / impact magenta). Each chip carries:
- A small glyph/icon (SVG or lucide icon, distinguishes members within the family).
- Short label: `result-changing`, `time-pressure`, `miss`, `hasty`, `unpunished`, `from-winning`,
  `knowledge-gap`.
- `cursor: pointer` and `hover: filter brightness(1.15) translateY(-1px)` — visual affordance for
  the future deep-link. In Phase 107 chips are non-navigable; clicking shows nothing (or a future
  placeholder toast — planner's choice on whether to wire a toast or leave it as cursor-pointer only).
- `data-testid="chip-{tagName}-{gameId}"` on each chip.

Excluded from card chips (per Phase 106 curate rules): inaccuracy-level tags, `phase-*` tags.
Cards show only mistake- and blunder-level tags, game-level deduped (one chip per tag type present).

### "No engine analysis" state (chess.com and unanalyzed lichess)

Replaces the entire severity row + chips section. A single dashed pill:

```
○  No engine analysis
```

- `border: 1px dashed var(--color-border-strong)`
- `border-radius: var(--radius-full)`
- `color: var(--color-text-muted)`
- `background: oklch(1 0 0 / 5%)`
- `font-size: text-chip` / `font-weight: 700`
- `padding: 3px 10px`

This state must NEVER show "0 Blunders 0 Mistakes 0 Inacc." — the schema enforces this via
`severity_counts: null` when `analysis_state === "no_engine_analysis"`.

### Desktop metadata row

Order (matching existing GameCard): termination (with result chip) · date · time control · move count.
Wrapping flex row, `text-sm text-muted-foreground`.

### Mobile metadata: vertical stack

Order: date · time control · termination (with result chip). Move count is omitted on mobile
(matches existing GameCard mobile behavior).

### `data-testid` conventions for the card

| Element | testid |
|---------|--------|
| Card root | `library-game-card-{gameId}` |
| Severity badge row | `severity-row-{gameId}` |
| Blunder badge | `severity-blunder-{gameId}` |
| Mistake badge | `severity-mistake-{gameId}` |
| Inaccuracy badge | `severity-inaccuracy-{gameId}` |
| Tag chip | `chip-{tagName}-{gameId}` |
| No-analysis state | `no-analysis-{gameId}` |
| Platform link | `game-card-link-{gameId}` (existing convention, preserve) |

## Flaw-Stats Panel (Sketch 002-A) — Detailed Contract

### Panel shell

- `background: var(--color-surface)` — `oklch(0.205 0 0)`
- `border: 1px solid var(--color-border)`
- `border-radius: var(--radius-lg)` — 10px
- `padding: 18px 20px`
- `data-testid="flaw-stats-panel"`

### Panel header row

Left-to-right: `"Flaw-Stats"` title (Fredoka, `text-lg`, weight 600) · segmented toggle (per game /
per 100 moves) · denominator pill (pinned `ml-auto`).

**Segmented toggle** (`data-testid="flaw-stats-norm-toggle"`):
- Two segments: "per game" / "per 100 moves"
- `border: 1px solid var(--color-border)` outer pill, `border-radius: var(--radius-full)`
- Active segment: `background: var(--brand-brown)` / `color: white`
- Inactive segment: `background: transparent` / `color: var(--color-text-muted)`
- `font-size: text-chip` / `font-weight: 700`
- `data-testid="flaw-stats-toggle-game"` / `data-testid="flaw-stats-toggle-100"`

**Denominator pill** (`data-testid="flaw-stats-denominator"`):
- Copy: `📊 {analyzed_pct}% analyzed · N = {analyzed_n}`
- `{analyzed_pct}` value in `color: var(--brand-brown-highlight)` / `font-weight: 700`
- `{analyzed_n}` value in `color: var(--brand-brown-highlight)` / `font-weight: 700`
- Background: `oklch(1 0 0 / 4%)` / border: `1px solid var(--color-border)` / pill shape
- When `analyzed_n === 0`: render "No analyzed games in the current filter"

### Zone 1: Severity-rate band

A flex row of 4 cells (`flex: 1 min-width: 120px flex-wrap`). Each cell:
- `background: var(--color-charcoal)` — `#161412`
- `border: 1px solid var(--color-border)` / `border-radius: var(--radius-md)`
- `padding: 12px 14px`

| Cell | Number color | Label copy | delta copy |
|------|-------------|------------|------------|
| Blunders | `SEV_BLUNDER` | "Blunders / game" (or "/ 100 moves") | optional delta vs prior window |
| Mistakes | `SEV_MISTAKE` | "Mistakes / game" (or "/ 100 moves") | optional delta |
| Inaccuracies | `SEV_INACCURACY` | "Inaccuracies / game" (or "/ 100 moves") | optional delta |
| Result-changing | `FAM_IMPACT` | "Result-changing" | "of blunders" |

Number size: `text-2xl` (24px) `font-weight: 800`. Label: `text-chip` uppercase muted, weight 700.

Toggle behavior: switching the normalization toggle updates only the number displayed (per_game or
per_100_moves value from the API); the label suffix updates accordingly ("/ game" vs "/ 100 moves").
No re-fetch — both values are in the same `FlawStatsResponse`.

The result-changing cell shows `rates.result_changing_rate` formatted as a percentage (e.g. "31%").
It is not affected by the per-game / per-100 toggle.

`data-testid="flaw-stats-band"`, individual cells: `data-testid="stat-cell-blunders"` etc.

Mobile: cells wrap (the 120px min-width and flex-wrap produce a 2-up layout at 360–420px). No
collapse needed — let the flex layout handle it naturally.

### Zone 2: Trend chart

Full-width. Header row: "Blunders / game" label in `SEV_BLUNDER` color + period annotation
("last N games · rolling window") pinned right.

**Chart**: Recharts `AreaChart` or `LineChart` (use the existing chart pattern from
`EndgameScoreOverTimeChart.tsx` or `EndgameClockDiffOverTimeChart.tsx`):
- X-axis: `date` field from `FlawTrendPoint` (date label of the last game in each window).
- Y-axis: `rate` field.
- Area fill: `SEV_BLUNDER` at 32% opacity, fading to 0 (same gradient pattern as the sketch SVG).
- Line stroke: `SEV_BLUNDER` / stroke-width 2.5.
- Dots: small circles at each point (`r=3`, fill `SEV_BLUNDER`).
- X-axis tick color: `var(--color-text-muted)`.
- No grid lines (matches existing Recharts pattern on charcoal).
- Container: `background: var(--color-charcoal)` / `border: 1px solid var(--color-border)` /
  `border-radius: var(--radius-md)` / `padding: 14px 16px` / `margin-top: 14px`.
- `data-testid="flaw-trend-chart"`.

When `trend` array is empty or has fewer than 2 points: show "Not enough games to show a trend"
in `text-sm text-muted-foreground text-center py-4` inside the chart container.

### Zone 3: Tag distribution block

Container: `background: var(--color-charcoal)` / border / `border-radius: var(--radius-md)` /
`padding: 14px 16px` / `margin-top: 14px`. Section heading "Tag distribution" in `text-sm
font-weight: 800`.

**Tempo split stacked bar** (`data-testid="tempo-stacked-bar"`):
- Full-width horizontal bar, height 14px, `border-radius: var(--radius-full)`.
- Three segments (proportional width):
  - Time-pressure: `FAM_TEMPO_TIME_PRESSURE` / `oklch(0.74 0.16 290)`.
  - Hasty: `FAM_TEMPO_HASTY` / `oklch(0.62 0.15 300)`.
  - Knowledge-gap: `FAM_TEMPO_KNOWLEDGE_GAP` / `oklch(0.50 0.13 305)`.
- Legend below: dot + label + percent for each segment.
- Source: `tag_distribution.tempo` dict from API (keys: `time-pressure`, `hasty`, `knowledge-gap`).

**Three sub-columns** (desktop: `grid grid-cols-3 gap-2 mt-3`; mobile: `flex flex-col gap-3 mt-3`):

1. By phase histogram (`data-testid="phase-histogram"`):
   - Horizontal bar rows: Opening / Middlegame / Endgame.
   - Fills: `PHASE_OPENING` / `PHASE_MIDDLEGAME` / `PHASE_ENDGAME`.
   - Source: `tag_distribution.phase_histogram`.

2. Opportunity rates (`data-testid="opportunity-rates"`):
   - Miss rate + Unpunished rate as horizontal bar rows.
   - Fill: `FAM_OPPORTUNITY`.
   - Source: `tag_distribution.opportunity` key from the API. Note: the FlawStatsResponse
     `TagDistribution` carries `tempo` dict and `phase_histogram` dict; miss/unpunished rates
     must be derived from the full `tag_distribution` data. The planner must verify the exact
     field name on the backend `TagDistribution` schema — see `app/schemas/library.py`. If the
     schema does not directly expose miss/unpunished rates, derive them as
     `miss_count / total_mb_flaws` and `unpunished_count / total_mb_flaws` client-side.

3. Impact rates (`data-testid="impact-rates"`):
   - From-winning rate + Result-changing rate as horizontal bar rows.
   - Fill: `FAM_IMPACT`.
   - `result_changing_rate` comes from `tag_distribution.result_changing_rate`.

Each bar row: 3-column grid (`label | bar track | value`). Bar track height 8px,
`background: oklch(1 0 0 / 7%)`, fill `border-radius: var(--radius-full)`.
Label: `text-sm text-muted-foreground font-weight: 700`.
Value: `text-sm font-weight: 800`.

`data-testid="tag-distribution-block"` on the container.

### Empty analyzed-set state

When `analyzed_n === 0`:
- Denominator pill shows "0% analyzed · N = 0".
- Band cells show "—" for all numbers.
- Trend chart shows "Not enough games to show a trend".
- Tag distribution shows "No analyzed games in the current filter" in `text-sm text-muted-foreground`.

## Filter Panel Extension — Library Games

`LibraryFilterPanel` composes the existing `FilterPanel` (with specific `visibleFilters`) and adds
the mistake-severity filter above the standard filters.

### Filters shown on this surface

| Filter | Control type | Source |
|--------|-------------|--------|
| Mistake severity | Toggle buttons (Blunders / Mistakes, multi-select) | NEW — wires to `severity[]` query param |
| Time control | Toggle buttons (Bullet / Blitz / Rapid / Classical) | Existing FilterPanel |
| Platform | Toggle buttons (Chess.com / Lichess) | Existing FilterPanel |

| Rated | Toggle (All / Rated / Casual), under "More" | Existing FilterPanel |
| Opponent Strength | Slider/presets | Existing FilterPanel |

**Omitted on this surface:** color (Played-as) filter — the Games subtab shows all colors; there
is no opening/position filter and no chessboard.

### Mistake-severity filter (new)

Label: "Show games with:" (text-sm muted above the toggles).
Two `<button>` toggles side by side: **Blunders** and **Mistakes**.
- Both deselected = no severity filter (all games shown).
- Either or both selected = only games containing ≥1 of those severity levels.
- API param: `severity=blunder&severity=mistake` (multi-value query param).
- Active style: `border-toggle-active bg-toggle-active text-toggle-active-foreground` (matches
  existing FilterPanel button style).
- Inactive style: `border-border bg-inactive-bg text-muted-foreground`.
- `data-testid="filter-severity-blunder"` / `data-testid="filter-severity-mistake"`.
- Height: `h-11 sm:h-7` (matches existing FilterPanel touch targets).

The mistake-severity filter state is NOT part of the shared `FilterState` interface (which is
used by Openings and Endgames). It lives as a separate `severityFilter: ('blunder' | 'mistake')[]`
local state in `GamesTab` and is passed directly to the API query.

### Reset Filters button

Resets all filters including severityFilter. Uses existing `variant="brand-outline"` button.
`data-testid="btn-reset-filters"` (same testid, same behavior as other pages).

## Page Layout — Mobile

Mobile uses the Openings drawer pattern exactly:

1. Sticky top row with "Filters" button (with modified-indicator dot if any filter is active).

## Interaction Contract

### Per-game / per-100 toggle

- State is local to the Flaw-Stats panel (no URL param, no re-fetch).
- Switching updates the displayed number for Blunders/Mistakes/Inaccuracies cells only.
- Label suffix updates: "Blunders / game" ↔ "Blunders / 100 moves".
- Result-changing % is unaffected by the toggle (it is always a ratio, not a rate).
- Default: "per game".

### Filter interaction

- Desktop: filters apply live (same as Openings desktop sidebar).
- Mobile: filters apply on drawer close (same as Openings mobile drawer).
- Both the `GET /api/library/games` and `GET /api/library/flaw-stats` queries share the same
  filter params and are both refetched on filter change.
- The severity filter (`blunder`, `mistake`) is passed as multi-value `severity[]` to BOTH
  endpoints so the stats panel reflects the same game set as the card list.

### Pagination

- Page size: 20 cards (same as Openings, matching the API's default `limit=20`).
- Scrolls to `[data-testid="library-game-card-list"]` on page change.
- Pagination controls: prev / numbered pages / next — same component logic as `GameCardList`.

### Tag chip interaction

- Chips are display-only in Phase 107. `cursor: pointer` is set (visual affordance for future
  deep-link) but no navigation occurs. No toast required in Phase 107.
- `aria-label="Tag: {tagName} (deep-link to Flaws view not yet available)"` — honest ARIA.

### Returning-user default subtab

- The `LibraryPage.tsx` default redirect changes from `overview` to `games` for users with
  games: `Navigate to={noGames ? '/library/import' : '/library/games'}`.
- The Games tab is the third `TabsTrigger` in the existing Library `<Tabs>`.
- URL: `/library/games`.

## API Data Mapping

### `GET /api/library/games` → `LibraryGamesResponse`

| Response field | UI target |
|----------------|-----------|
| `games[].user_result` | WDL left-border color, result chip |
| `games[].white_username`, `black_username`, ratings | Card header names |
| `games[].opening_name` | Opening line |
| `games[].played_at` | Date metadata item |
| `games[].time_control_bucket`, `time_control_str` | TC metadata item |
| `games[].termination` | Termination metadata item |
| `games[].move_count` | Move count (desktop only) |
| `games[].result_fen` | LazyMiniBoard `fen` prop |
| `games[].user_color` | LazyMiniBoard `flipped` prop |
| `games[].platform`, `platform_url` | Platform icon + external link |
| `games[].analysis_state` | `"analyzed"` → severity row + chips; `"no_engine_analysis"` → NoAnalysisState |
| `games[].severity_counts` | `{ blunder, mistake, inaccuracy }` → SeverityBadge × 3 |
| `games[].chips` | `FlawTag[]` → TagChip × N (color by family) |
| `matched_count`, `offset`, `limit` | Pagination |

### `GET /api/library/flaw-stats` → `FlawStatsResponse`

| Response field | UI target |
|----------------|-----------|
| `rates.per_game[severity]` | Band cell numbers (per-game mode) |
| `rates.per_100_moves[severity]` | Band cell numbers (per-100 mode) |
| `tag_distribution.result_changing_rate` | Result-changing band cell (%) |
| `tag_distribution.tempo` | Tempo stacked bar (time-pressure / hasty / knowledge-gap counts) |
| `tag_distribution.phase_histogram` | Phase histogram bars (opening / middlegame / endgame) |
| `trend[].date`, `trend[].rate`, `trend[].window_size` | Trend chart X/Y + period label |
| `analyzed_pct` | Denominator pill percentage |
| `analyzed_n` | Denominator pill N |
| `total_n` | Context for empty state decisions |

**Note on opportunity/impact rates:** The `TagDistribution` schema exposes only `tempo`,
`result_changing_rate`, and `phase_histogram`. The miss / unpunished / from-winning rates are NOT
directly in the schema as separate rate fields — they must be derived by the frontend from the raw
chip data on individual game cards, or the backend schema must be extended. The planner must verify
this gap and either: (a) derive rates client-side by counting chip types across the fetched page
(fragile — only covers the current page), or (b) request a backend schema extension to add
`opportunity_rates` and `impact_rates` dicts to `TagDistribution`. Recommendation: option (b) —
flag as a planner decision before implementation.

## State Management

All data fetching via TanStack Query (`useQuery`). No global state mutations.

| Query key | Endpoint | Refetch trigger |
|-----------|----------|----------------|
| `['library-games', filters, severityFilter, offset]` | `GET /api/library/games` | Filter change, page change |
| `['library-flaw-stats', filters, severityFilter]` | `GET /api/library/flaw-stats` | Filter change (no offset) |

Both queries share the same filter params object (derived from `LibraryFilterPanel` state +
`severityFilter`). When filters change, both queries refetch and the card list resets to page 1
(offset 0).

### `isError` branches (mandatory per CLAUDE.md)

Every `useQuery` chain must include an `isError` branch showing the error copy above.
Never let an error fall through to the empty-state message.
