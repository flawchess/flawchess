## Typography

Two weights only, matching the real codebase:

| Role | Size | Weight | Line Height | Tailwind class | Usage |
|------|------|--------|-------------|----------------|-------|
| Body | 14px | 400 | 1.5 | `text-sm` | Card opening line, metadata items, filter labels, tag distribution bar labels and values |
| Label | 13px | 700 | 1.3 | `text-sm font-bold` (chip surfaces) | Severity badge labels, tag chip text, KPI sub-labels, denominator pill, toggle segment text |
| Heading | 18px | 700 | 1.2 | `text-lg font-bold font-brand` | "Flaw-Stats" panel title (Fredoka brand font) |
| KPI number | 24px | 700 | 1.0 | `text-2xl font-bold` | Severity-rate cells in the stat band (blunders/game number) |

Notes:
- Weight pair is 400 (regular) and 700 (bold/`font-bold`). The app uses `font-semibold` (600) in some panel headers (e.g. `EndgameOverallEntryCard.tsx`); for this phase use `font-bold` (700) throughout to stay within a strict 2-weight contract. Either 600 or 700 is visually acceptable; 700 is the primary emphasis weight in GameCard and the broader codebase.
- KPI numbers (24px) gain visual contrast over label text (13px) through **size alone** ŌĆö the 24px vs 13px ratio is sufficient; no additional weight beyond 700 is needed or declared.
- Body floor is `text-sm` (14px) per CLAUDE.md. The 13px Label row applies only to severity badges, tag chips, KPI sub-labels, and the denominator pill ŌĆö all short, display-only, non-primary surfaces explicitly covered by the CLAUDE.md tooltip/aside exception.
- Player names in the card header: `text-sm font-bold` on desktop (single-line, emphasis); `text-sm` (weight 400) on mobile (stacked, both lines ŌĆö matches existing GameCard mobile behavior).
- Panel trend chart axis labels render inside Recharts SVG and may use smaller sizes (11ŌĆō13px) ŌĆö acceptable inside SVG rendering context where Tailwind font classes are not applicable.

Source: `GameCard.tsx` (no explicit weight class = 400 body); `EndgameOverallEntryCard.tsx` (`font-semibold` panel heading pattern); CLAUDE.md typography rules.

### Zone 1: Severity-rate band

A flex row of 4 cells (`flex: 1 min-width: 120px flex-wrap`). Each cell:
- `background: var(--color-charcoal)` ŌĆö `#161412`
- `border: 1px solid var(--color-border)` / `border-radius: var(--radius-md)`
- `padding: 12px 16px` (12px top/bottom, 16px left/right ŌĆö 4px grid)

| Cell | Number color | Label copy | delta copy |
|------|-------------|------------|------------|
| Blunders | `SEV_BLUNDER` | "Blunders / game" (or "/ 100 moves") | optional delta vs prior window |
| Mistakes | `SEV_MISTAKE` | "Mistakes / game" (or "/ 100 moves") | optional delta |
| Inaccuracies | `SEV_INACCURACY` | "Inaccuracies / game" (or "/ 100 moves") | optional delta |
| Result-changing | `FAM_IMPACT` | "Result-changing" | "of blunders" |

Number size: `text-2xl font-bold`. Label: `text-sm uppercase text-muted-foreground font-bold`.

Toggle behavior: switching the normalization toggle updates only the number displayed (per_game or
per_100_moves value from the API); the label suffix updates accordingly ("/ game" vs "/ 100 moves").
No re-fetch ŌĆö both values are in the same `FlawStatsResponse`.

The result-changing cell shows `rates.result_changing_rate` formatted as a percentage (e.g. "31%").
It is not affected by the per-game / per-100 toggle.

`data-testid="flaw-stats-band"`, individual cells: `data-testid="stat-cell-blunders"` etc.

Mobile: cells wrap (the 120px min-width and flex-wrap produce a 2-up layout at 360ŌĆō420px). No
collapse needed ŌĆö let the flex layout handle it naturally.

### Zone 2: Trend chart

Full-width. Header row: "Blunders / game" label in `SEV_BLUNDER` color + period annotation

**Chart**: Recharts `AreaChart` or `LineChart` (use the existing chart pattern from

- Y-axis: `rate` field.
- Area fill: `SEV_BLUNDER` at 32% opacity, fading to 0 (same gradient pattern as the sketch SVG).
- Line stroke: `SEV_BLUNDER` / stroke-width 2.5.
- Dots: small circles at each point (`r=3`, fill `SEV_BLUNDER`).
- X-axis tick color: `var(--color-text-muted)`.
- No grid lines (matches existing Recharts pattern on charcoal).
- Container: `background: var(--color-charcoal)` / `border: 1px solid var(--color-border)` /

- `data-testid="flaw-trend-chart"`.

When `trend` array is empty or has fewer than 2 points: show "Not enough games to show a trend"

## Interaction Contract

### Per-game / per-100 toggle

- State is local to the Flaw-Stats panel (no URL param, no re-fetch).
- Switching updates the displayed number for Blunders/Mistakes/Inaccuracies cells only.
- Label suffix updates: "Blunders / game" Ōåö "Blunders / 100 moves".
- Result-changing % is unaffected by the toggle (it is always a ratio, not a rate).
- Default: "per game".

### Filter interaction

- Desktop: filters apply live (same as Openings desktop sidebar).
- Mobile: filters apply on drawer close (same as Openings mobile drawer).
- Both the `GET /api/library/games` and `GET /api/library/flaw-stats` queries share the same
  filter params and are both refetched on filter change.
- The severity filter (`blunder`, `mistake`) is passed as multi-value `severity[]` to BOTH
  endpoints so the stats panel reflects the same game set as the card list.

### Pagination

- Page size: 20 cards (same as Openings, matching the API's default `limit=20`).
- Scrolls to `[data-testid="library-game-card-list"]` on page change.
- Pagination controls: prev / numbered pages / next ŌĆö same component logic as `GameCardList`.

### Tag chip interaction

- Chips are display-only in Phase 107. `cursor: pointer` is set (visual affordance for future
  deep-link) but no navigation occurs. No toast required in Phase 107.
- `aria-label="Tag: {tagName} (deep-link to Flaws view not yet available)"` ŌĆö honest ARIA.

### Returning-user default subtab

- The `LibraryPage.tsx` default redirect changes from `overview` to `games` for users with
  games: `Navigate to={noGames ? '/library/import' : '/library/games'}`.
- The Games tab is the third `TabsTrigger` in the existing Library `<Tabs>`.
- URL: `/library/games`.

| Response field | UI target |
|----------------|-----------|

(fragile ŌĆö only covers the current page), or (b) request a backend schema extension to add
`opportunity_rates` and `impact_rates` dicts to `TagDistribution`. Recommendation: option (b) ŌĆö
flag as a planner decision before implementation.

## State Management

All data fetching via TanStack Query (`useQuery`). No global state mutations.

| Query key | Endpoint | Refetch trigger |
|-----------|----------|----------------|
| `['library-games', filters, severityFilter, offset]` | `GET /api/library/games` | Filter change, page change |
| `['library-flaw-stats', filters, severityFilter]` | `GET /api/library/flaw-stats` | Filter change (no offset) |

Both queries share the same filter params object (derived from `LibraryFilterPanel` state +
`severityFilter`). When filters change, both queries refetch and the card list resets to page 1
(offset 0).

### `isError` branches (mandatory per CLAUDE.md)

Every `useQuery` chain must include an `isError` branch showing the error copy above.
Never let an error fall through to the empty-state message.
