<objective>
The D-01 backend slice: extend `TagDistribution` (returned by `GET /api/library/flaw-stats`)
with three flat float fields — `miss_rate`, `lucky_escape_rate`, `while_ahead_rate` — computed
in `_compute_tag_distribution`, each as `count / total M+B flaws` (`0.0` when no M+B flaws),
mirroring the existing `result_changing_rate` precedent exactly. These feed the UI-SPEC Zone 3
Opportunity (miss + lucky-escape) and Impact (while-ahead + result-changing) sub-columns (D-03).

Purpose: The UI-SPEC's Flaw-Stats panel needs these rates as true `count / total_mb_flaws`
metrics. D-02 explicitly rejected deriving them client-side from page-limited card chips. This is
the one deliberate, pre-approved backend expansion in an otherwise frontend-only phase.
Output: Three new computed (not stored) schema fields + service counters + test coverage. No
migration (the fields are computed on-the-fly). The endpoint is `flaw-stats`, never `mistake-stats`.
</objective>
