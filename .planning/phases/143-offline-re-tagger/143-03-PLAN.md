<objective>
Deliver the offline re-tagger by extending the optimized `scripts/backfill_tactic_tags.py` with
blob loading + `--margin` + gate application, then `git mv` it to `scripts/retag_flaws.py` (D-01),
and emit a committed per-motif tag-delta report (D-04).

Purpose: RETAG-01/RETAG-02 — a seconds-fast, engine-free, idempotent re-derivation of tactic tags
from the stored MultiPV-2 JSONB, tunable by `--margin` for the Phase 144 A/B sweep. It reuses the
existing keyset paging, spawn-worker pool, and change-only batched UPDATE verbatim; the only delta
is three new worker inputs (allowed/missed blob + flaw-ply eval_cp), the `--margin` flag, and a
report writer. It classifies through `_classify_tactic_gated` (Plan 02), guaranteeing zero drift
from the live drain (SC4).

Output: `scripts/retag_flaws.py`, the `reports/retag/` report, the two docstring fixes in
game_flaws_repository.py, and a pytest that proves the dry-run delta report + idempotency against
the per-run test DB.
</objective>
