<objective>
Implement the missed-opportunity detector pass. Today `_detect_tactic_for_flaw` runs one pass — `board_after_flaw` + the `flaw_ply+1` (refutation) PV — which becomes the `allowed_*` tag. Add a second pass: `board_before` (the pre-flaw decision position, turn = the mover) + the `flaw_ply` PV (the SEED-054 "instead-of" line) + `pov = board_before.turn`, populating `missed_*`. `_build_flaw_record` calls the detector twice and fills both 4-tuples on `FlawRecord`.

Purpose: D-03 (orientation-agnostic dispatcher, called with the mover board/PV/pov for the missed pass), D-04 (reuse the 127 gate/floor/query-suppression unchanged — no new harness; precision is a property of the detector, not the PV source), D-06 (parametrize/split `_detect_tactic_for_flaw`, two calls in `_build_flaw_record`).

Output: both `allowed_*` and `missed_*` 4-tuples populated on every FlawRecord from the live drain and backfill paths; no new PV plumbing (pv_by_ply already carries both flaw_ply and flaw_ply+1 PVs).
</objective>
