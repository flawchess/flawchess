## Why

Phase 94.4 ships the D-12 Reversal Amendment — a game-weighted blended anchor that replaces the original Lichess-precedence anchor. The schema migration (`reshape_user_rating_anchors_for_blended_anchor`, revision `c70f5d94b243`) TRUNCATEs both `user_rating_anchors` and `user_benchmark_percentiles`. Until prod runs the migration + backfill, the amendment's user-visible behavior change does not land for real users — prod chips remain blank for the population whose anchor row was destroyed by the migration.

Plan 12 Tasks 1+2 (dev backfill) shipped during `/gsd-execute-phase 94.4` on 2026-05-27. Tasks 3+4 (prod backfill + mixed-platform UAT) were deferred at the HUMAN gate per operator decision so the phase could close on schedule. This TODO captures the obligation.

## What

When the v1.19 milestone-cut deploy ships Phase 94.4 to production (per CLAUDE.md milestone-boundary release discipline):

### 1. Prod backfill execution

After `bin/deploy.sh` completes and Alembic head includes `c70f5d94b243`:

```bash
# 1. Pre-flight — confirm migration applied
ssh flawchess "cd /opt/flawchess && docker compose logs --tail=80 backend | grep -i 'alembic\|reshape_user_rating_anchors'"

# 2. Confirm both tables are empty (truncated by Plan 09 migration)
bin/prod_db_tunnel.sh
psql 'postgresql://flawchess_ro:<password>@localhost:15432/flawchess' \
  -c "SELECT count(*) FROM user_rating_anchors; SELECT count(*) FROM user_benchmark_percentiles;"
# Both expected = 0

# 3. Confirm schema reshape — n_chesscom_games + n_lichess_games present, source_platform absent
psql 'postgresql://flawchess_ro:<password>@localhost:15432/flawchess' -c "\d user_rating_anchors"

# 4. Run prod backfill (5-30 min depending on user count)
uv run python scripts/backfill_user_percentiles.py --target prod --snapshot-date <YYYY-MM-DD>
# Monitor Sentry at https://flawchess.sentry.io during the run

# 5. Verify summary tables print at end (Anchor Composition Summary + Percentile Summary)
# Verify both tables now have rows
psql 'postgresql://flawchess_ro:<password>@localhost:15432/flawchess' \
  -c "SELECT count(*) FROM user_rating_anchors; SELECT count(*) FROM user_benchmark_percentiles;"

# 6. Close tunnel
bin/prod_db_tunnel.sh stop
```

**Recovery contract:** the backfill is UPSERT-idempotent. If it fails mid-run, re-running end-to-end is safe. DO NOT manually edit prod data.
