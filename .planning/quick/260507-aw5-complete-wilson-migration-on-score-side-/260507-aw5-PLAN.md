# Plan: Complete Wilson migration (score side only)

## Goal

Replace the trinomial Wald p-value in `score_confidence.compute_confidence_bucket` with a Wilson score-test p-value (uses null variance instead of estimated variance). The CI bounds (`wilson_bounds`) are already Wilson — this closes the inconsistency.

Eval side (`eval_confidence.py`) is **out of scope** — Wilson doesn't apply to continuous z-tests. Keep eval as-is.

## Background

Discussion thread above. Key points:
- Wilson uses null SE = `sqrt(π₀(1-π₀)/n) = 0.5/sqrt(n)` at H0=0.5; doesn't depend on observed W/D/L.
- No more SE=0 degeneracy: Wilson is well-defined for all p̂ ∈ [0,1] including all-wins / all-losses.
- p_value still = `erfc(|z|/√2)`; bucket thresholds (p<0.01 high, p<0.05 medium) unchanged.
- Empirical (trinomial) SE retained in 3-tuple return as informational; existing callers ignore SE.
