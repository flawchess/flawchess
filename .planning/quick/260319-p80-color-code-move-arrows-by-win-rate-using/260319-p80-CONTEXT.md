# Quick Task 260319-p80: Color-code move arrows by win rate using WDL chart green-grey-red gradient - Context

**Gathered:** 2026-03-19
**Status:** Ready for planning

<domain>
## Task Boundary

Replace blue move arrows on the chessboard with a green-grey-red color scale based on win rate, using the same colors as the WDL chart. Win rate >= 66% = green, <= 33% = red, 33%-66% = gradient with 50% = grey.

</domain>

<decisions>
## Implementation Decisions

### Frequency encoding
- Keep opacity for frequency (game count) AND use color for win rate — dual encoding
- Arrow opacity continues to reflect how often a move was played (game_count / maxCount)
- Arrow color reflects win rate on a green-grey-red gradient

### Hovered arrow style
- Hovered arrows keep their win-rate color but rendered brighter (higher lightness)
- No separate highlight color — just a brighter version of the win-rate color

### Low sample sizes
- No special handling needed — low sample size moves already have very low opacity due to frequency encoding, which naturally de-emphasizes unreliable win rates

</decisions>

<specifics>
## Specific Ideas

- WDL colors: green = oklch(0.45 0.16 145), grey = oklch(0.55 0.01 260), red = oklch(0.45 0.17 25)
- Win rate thresholds: >= 66% full green, <= 33% full red, 33%-66% gradient
- 50% win rate = grey color
- Arrow creation logic exists in both Openings.tsx and Dashboard.tsx — both need updating

</specifics>
