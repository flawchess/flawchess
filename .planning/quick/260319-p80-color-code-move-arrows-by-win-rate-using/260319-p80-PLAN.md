Create `getArrowColor(winPct: number, opacity: number, isHovered: boolean): string` that returns an oklch CSS color string.

Color interpolation logic using oklch components:
- **Anchor colors** (from WDL chart): green = `oklch(0.45 0.16 145)`, grey = `oklch(0.55 0.01 260)`, red = `oklch(0.45 0.17 25)`
- **Win rate mapping**: winPct is 0-100
  - winPct >= 66 -> full green (L=0.45, C=0.16, H=145)
  - winPct <= 33 -> full red (L=0.45, C=0.17, H=25)
  - winPct 33-66 -> interpolate between red(33%) -> grey(50%) -> green(66%)
    - 33-50: lerp red->grey (t = (winPct - 33) / 17)
    - 50-66: lerp grey->green (t = (winPct - 50) / 16)
  - Interpolate L, C, H independently (hue interpolation: red=25 -> grey=260 crosses 0, so for red->grey go 25->260 via the short arc; simplest: just lerp the three components directly since 25->260 is fine as increasing)

- **Opacity**: Apply the passed-in opacity value (from frequency encoding). Use oklch alpha: `oklch(L C H / alpha)` where alpha is 0-1.

- **Hovered state**: When isHovered=true, increase lightness by 0.15 (e.g., 0.45 -> 0.60) and set opacity to 1.0 (full opacity). This makes hovered arrows brighter without changing hue.

Export the function as named export.
