<objective>
Persist import job visibility across page refresh and login cycles, and live-update game counts during import.

Currently, activeJobIds are stored in React state only -- lost on refresh/logout. The backend already tracks active jobs in-memory and persists completed jobs to DB, but there is no endpoint for the frontend to ask "what jobs are active for me?" Also, the profile game count only refreshes when a job completes, not during import.

Purpose: Users who start a large import (e.g., 5000 chess.com games taking minutes) should not lose visibility of that import if they refresh the page or get logged out by token expiry and re-login.

Output: Backend endpoint + frontend restoration hook + live game count updates during import.
</objective>
