<objective>
Fix the tactic filter so depth and orientation are independently meaningful (not silent no-ops when no family is selected), and make a tactic chip/badge show IFF that specific slot satisfies the active predicate (family ∩ orientation ∩ depth-range), evaluated per-slot. Apply identical per-slot suppression at BOTH serialization sites (/library/flaws and /library/games), nulling non-matching slots. Flip the default depth preset to High (full range) so the out-of-box state shows everything.

Purpose: The current implementation traps depth+orientation predicates inside `if tactic_families:`, so they do nothing alone; and both response paths emit both tactic slots gated only by confidence>=70, never by the active filter — violating the type contract that claims slots are null when their chip is hidden.

Output: Refactored backend filter + a single shared per-slot display predicate applied at both sites, threaded tactic params on the Games path, frontend defaults flipped to High, Games-card empty-column hiding, and backend tests proving the five scenarios.
</objective>
