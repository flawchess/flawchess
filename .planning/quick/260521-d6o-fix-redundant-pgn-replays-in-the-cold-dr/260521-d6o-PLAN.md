<objective>
Eliminate redundant PGN replays in the cold-drain eval lane. Today, a single game with 1 midgame entry + M endgame span entries triggers 1+M independent calls to `_board_at_ply`, each of which parses the PGN from scratch and walks the mainline from move 1. Replace this with a single PGN parse + single mainline walk per game that snapshots `board.copy()` at each pre-determined target ply.

Purpose: ~3-7% cold-drain wall-time savings on large imports; correctness is the must-have, perf is incidental. The cold drain is the lane that runs after hot import; eliminating O(N) parses for N target plies makes the lane scale cleanly as endgame-class richness grows.

Output: A refactored `app/services/eval_drain.py` where target collection is single-walk-per-game, plus a regression test that asserts the single-parse invariant.

Scope guard: Cold-drain only. `app/services/zobrist.py:process_game_pgn` (the fetch/hot lane) is OUT OF SCOPE per task spec. Public signatures of `_collect_midgame_eval_targets` and `_collect_endgame_span_eval_targets` must be preserved because `app/services/import_service.py:_collect_covered_game_ids` imports them.
</objective>
