# Quick Task 260417-o2l: Score % Difference Timeline Chart

## Goal

Add a weekly rolling-window timeline chart showing the Score % Difference
(endgame Score % minus non-endgame Score %) directly below the
"Games with vs without Endgame" table in the Endgame Overall Performance
section. Mirrors the layout of the "Average Clock Difference over Time" chart
in `EndgameClockPressureSection`.

- 100-game trailing window per side (endgame games and non-endgame games each
  carry their own trailing 100-game window — analogous to how the existing
  Win Rate by Endgame Type per-type series tracks each type independently).
- Weekly cadence (one point per ISO week, on the Monday).
- Y axis fixed at ±20 percentage points (zone backgrounds and dot colors use
  the existing ±5 pp parity neutral band already used by the table's
  `MiniBulletChart`).
- Single line; dots colored by zone (green/blue/red) using the existing
  `SCORE_DIFF_NEUTRAL_MIN/MAX = ±0.05` thresholds.

## Tasks

### Task 1 — Backend: schema + computation

**Files**
- `app/schemas/endgames.py`
- `app/services/endgame_service.py`
- `tests/test_endgame_service.py`

**Action**

1. `schemas/endgames.py`:
   - Add `ScoreGapTimelinePoint(BaseModel)` with
     `date: str` (Monday of ISO week, YYYY-MM-DD),
     `score_difference: float` (endgame_score - non_endgame_score on 0.0-1.0 scale,
     i.e. signed; 0.05 = +5%),
     `endgame_game_count: int` (games in the trailing endgame window, ≤ window),
     `non_endgame_game_count: int` (games in the trailing non-endgame window, ≤ window).
   - Extend `ScoreGapMaterialResponse` with
     `timeline: list[ScoreGapTimelinePoint]` and `timeline_window: int`.

2. `services/endgame_service.py`:
   - Constant `SCORE_GAP_TIMELINE_WINDOW = 100`.
   - New `_compute_score_gap_timeline(endgame_rows, non_endgame_rows, window)`:
     - Tag each row with side ("endgame"/"non_endgame") and outcome score
       (1.0 win / 0.5 draw / 0.0 loss) via `derive_user_result`.
     - Merge into one chronological event stream sorted by played_at.
     - Maintain two trailing windows (`endgame_scores`, `non_endgame_scores`)
       capped at `window` items each via list slicing (`[-window:]`).
     - For each event: append to its side's window, then record the latest
       state per ISO week using the same Monday-of-ISO-week logic as
       `_compute_clock_pressure_timeline` / `_compute_weekly_rolling_series`.
     - Per-week record: `{date, score_difference, endgame_game_count,
       non_endgame_game_count}`.
     - Filter output: emit only weeks where both window counts are
       ≥ `MIN_GAMES_FOR_TIMELINE` (=10) — matches the cold-start handling
       used elsewhere and avoids early-data noise from one-sided windows.
   - Update `_compute_score_gap_material` signature to accept
     `endgame_rows: list[Row[Any]]` and `non_endgame_rows: list[Row[Any]]`
     and populate the new `timeline` + `timeline_window` fields on the
     response. Existing bucket math is unchanged.
   - In `get_endgame_overview`, pass the existing `endgame_rows` and
     `non_endgame_rows` from `query_endgame_performance_rows` into
     `_compute_score_gap_material` (no new DB query needed).

3. Tests in `tests/test_endgame_service.py`:
   - Add `TestComputeScoreGapTimeline` class:
     - `test_empty_inputs_returns_empty`
     - `test_drops_weeks_with_either_side_below_min_games` (e.g. 10 endgame
       games but only 5 non-endgame games -> no points)
     - `test_emits_one_point_per_iso_week_using_rolling_diff` (10 endgame
       wins + 10 non-endgame losses in week 1; 10 endgame draws +
       10 non-endgame wins in week 2; check the diff and counts evolve
       per the rolling window).
     - `test_rolling_window_caps_at_window_size` (window=10; week 2 reflects
       only week-2 games per side).

**Verify**
- `uv run ruff check app/ tests/`
- `uv run ty check app/ tests/`
- `uv run pytest tests/test_endgame_service.py -x`

**Done**
- `GET /api/endgames/overview` returns `score_gap_material.timeline`
  populated with weekly rolling-100 score-difference points and
  `timeline_window=100`.
