<design_decisions>
LOCKED — do not revisit:

1. Two-line rows for the AGGREGATED chip's per-TC list (bullet 2). Each
   renderable TC `<li>` renders two lines:
     line 1: `{tc} — anchored at ~{anchor} Lichess Elo`
     line 2: `{value} over {n_games} games -> {percentile} percentile`
   Use the EXACT label wording `anchored at ~{anchor} Lichess Elo` (anchor is the
   stored int, "~" prefix). The em-dash between `{tc}` and `anchored` is part of
   the locked copy — this is data/label text, not prose, so the project's
   em-dash-sparing style does NOT apply here.

2. REMOVE bullet 4 entirely (the `bullet4` IIFE and its
   `{bullet4 !== '' && <p>{bullet4}</p>}` render line). No replacement generic
   note. The rating-matching method is conveyed ONLY by the per-row "anchored
   at ~X Lichess Elo" text.

3. AGGREGATED bullet 1 (tc === undefined): drop the `~{anchorRating}-rated`
   number. New text:
   `Your <em>recent</em> {metricLabel} is {phrasing} of similarly-rated players, aggregated across the time controls you play.`
   `{phrasing}` keeps the existing logic: clampedPct > PERCENTILE_MEDIAN →
   `better than {pct}%` else `in the bottom {pct}%`. Keep `<em>recent</em>`.

4. PER-TC bullet 1 (tc !== undefined): UNCHANGED — still
   `…of ~{anchorRating}-rated players in {tc}`.

5. Bullet 3 (COPY_FILTER_INDEPENDENCE): UNCHANGED.

6. Edge cases (two-line adaptation of current behavior):
   - Below-floor TC (value == null && n_games > 0): SINGLE line
     `{tc}: insufficient games` — no stats line, no anchor line (unchanged).
   - value != null && percentile == null (CDF out-of-range): DROP the entry
     (existing `renderable` filter already does this — do not change it).
   - n_games <= 0: omit (unchanged, existing `renderable` filter).
   - Defensive: renderable entry (value+percentile present) but anchor
     null/undefined → render the stats line WITHOUT the anchor label line. Never
     crash (noUncheckedIndexedAccess / strict-null rules apply).

7. PROP CLEANUP: `anchorRating` becomes OPTIONAL (`anchorRating?: number`). Per-TC
   chips still pass it (bullet 1 uses it); aggregated chips OMIT it (their bullet 1
   no longer shows a number). Remove `nChesscomGames`, `nLichessGames`,
   `chesscomMedianNative`, `lichessMedianNative` from `PercentileChipProps`,
   `PopoverBodyProps`, the body destructure, the trigger destructure, the
   `<PercentileChipPopoverBody .../>` pass-through, AND all 4 call sites.

8. Aggregated chip gating: the aggregated call sites currently gate chipSlot on
   `percentile != null && dominantAnchor !== undefined`. After this change the
   aggregated chip no longer displays a single anchor, so DROP the
   `dominantAnchor !== undefined` condition — gate on `percentile != null` only.
   This makes `dominantAnchor` / `ratingAnchors` / `pickDominantTcAnchor` dead in
   the two aggregated call sites. Remove `dominantAnchor`, the `ratingAnchors`
   prop (and its `RatingAnchorsByTc` import + interface field), and DELETE
   `pickDominantTcAnchor` from `frontend/src/lib/percentileAnchor.ts`
   (knip will fail otherwise). The per-TC `EndgameTimePressureCard` does NOT use
   `pickDominantTcAnchor` (it passes its own TC's anchor directly), so deletion
   is safe — VERIFY with the knip run.

NOT IN SCOPE: the 260529-js1 anchor recompute follow-up. Displayed anchors may be
stale until that separate human step runs — do NOT gate completion on it.
</design_decisions>
