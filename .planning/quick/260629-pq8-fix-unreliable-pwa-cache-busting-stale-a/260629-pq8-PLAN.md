# Quick Task 260629-pq8: Fix unreliable PWA cache-busting

## Problem

Installed Android PWA sometimes launches a many-deploys-old layout (e.g. missing the
"Library" nav). Root cause: the service worker **precaches `index.html` and serves it
cache-first**. The PWA launches at `start_url` `/`, which matches the Workbox precache
route (default `directoryIndex: 'index.html'`), so the SW returns the old precached
shell + old precached hashed JS — a self-consistent stale snapshot. The existing
`controllerchange` auto-reload + hourly `setInterval` update check only help *after* a
new SW activates, but Android freezes backgrounded PWAs: a resumed app fires no fresh
`load` and the interval is unreliable while suspended, so no update check runs on resume.
A manual reload forces a network fetch + SW update, which is why it fixes things. The
HTTP layer (Caddy `no-cache` on HTML/SW/manifest, `immutable` on `/assets/*`) is already
correct — this is purely a service-worker-layer problem.

Full plan: `~/.claude/plans/gleaming-painting-lantern.md` (approved).

## Tasks

### Task 1 — Stop precaching the HTML shell; serve navigations NetworkFirst
- **File:** `frontend/vite.config.ts` (workbox block, ~lines 89-103)
- **Action:** Add `'**/*.html'` to `globIgnores` so `index.html` is dropped from the
  precache (removes the cache-first precache route for `/`). Keep `navigateFallback: null`
  (OAuth fix, commit b953abad). Keep the `/^\/api\//` `NetworkOnly` route FIRST. Add a
  second runtime route: navigations (`request.mode === 'navigate'` guarded with
  `!url.pathname.startsWith('/api/')`) handled `NetworkFirst` into an `html-shell` cache,
  with `cacheableResponse: { statuses: [200] }`. No `networkTimeoutSeconds`.
- **Why both changes:** Workbox registers the precache route before runtime routes and
  matches in registration order, so a NetworkFirst nav route alone never gets to handle
  `/` while `index.html` is still precached. Removing HTML from the precache is what lets
  the NetworkFirst route take over. A fresh `index.html` references current hashed
  `/assets/*` (not in the old precache → old SW network-fetches them) → fully fresh app.
- **Verify:** `npm run build`; inspect `dist/sw.js` — `precacheAndRoute([...])` contains
  NO `*.html` entries; hashed `assets/*` remain; `NetworkFirst` nav route + `/api/`
  `NetworkOnly` route present.
- **Done:** Build succeeds, sw.js shows the expected routes.

### Task 2 — Resume-aware, debounced SW update checks
- **File:** `frontend/src/main.tsx` (interval block, ~lines 25-31)
- **Action:** Keep the `controllerchange` reload block and `registerType: 'autoUpdate'`.
  Replace the bare `setInterval` with a debounced `checkForSwUpdate()` (named constants
  `SW_UPDATE_INTERVAL_MS`, `SW_UPDATE_DEBOUNCE_MS`) wired to `setInterval`,
  `document visibilitychange` (→ `visible`), and `window focus` — the events that fire
  when Android resumes a frozen PWA.
- **Verify:** `npm run lint && npm test -- --run && npm run build` all pass.
- **Done:** Gate green, no magic numbers.
