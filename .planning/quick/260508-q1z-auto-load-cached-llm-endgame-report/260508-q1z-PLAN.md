# Quick Task 260508-q1z: Auto-load cached LLM endgame report on Endgames page mount

## Description

When users open or reload the Endgames page (or change filters that match a previously-cached report), automatically render the cached LLM endgame insights report without requiring a click on Generate. Implemented as a cache-only GET endpoint that never invokes the LLM and never accounts against rate-limit budget.

## Tasks

### Task 1 — Backend: GET /insights/endgame/cached

**Files:**
- `app/routers/insights.py` (add new route)
- `tests/test_insights_router.py` (add cache-only test class)

**Action:**
1. Add `GET /insights/endgame/cached` to `app/routers/insights.py`.
2. Mirror query params from `POST /insights/endgame` (time_control, platform, recency, rated, opponent_gap_min/max, color).
3. Build the `FilterContext` exactly as the POST handler does, but skip `_validate_full_history_filters` (a non-default filter just naturally produces a 404 because no cache row exists for that opponent_strength preset).
4. Implementation mirrors `app/services/insights_llm.py:1830-1857`:
   - Call `get_latest_successful_log_for_user(session, user_id, prompt_version=_PROMPT_VERSION, model=settings.PYDANTIC_AI_MODEL_INSIGHTS, opponent_strength=preset, max_age=INSIGHTS_CACHE_MAX_AGE)`.
   - On hit, call `get_latest_completed_import_with_games_at` for staleness check.
   - If valid, build `EndgameInsightsResponse(report=_maybe_strip_overview(report), status="cache_hit")`.
5. Returns 404 (no body content needed beyond simple JSON) on miss.
6. Never invokes `compute_findings`, never the LLM, no rate-limit accounting.
7. Custom (non-preset) opponent gap range → return 404 (filter that does not match any cache row).

**Verify:**
- `uv run ruff check app/ tests/`
- `uv run ty check app/ tests/`
- `uv run pytest tests/test_insights_router.py -x`

**Done:** Endpoint returns 200 with `status="cache_hit"` when a fresh cache row exists, 404 otherwise. No LLM invocation. Auth required.

### Task 2 — Frontend: useCachedEndgameInsights hook + Endgames.tsx wiring

**Files:**
- `frontend/src/hooks/useEndgameInsights.ts` (add new hook export)
- `frontend/src/pages/Endgames.tsx` (wire query, upsert into insightsCache)
- `frontend/src/hooks/__tests__/useEndgameInsights.test.tsx` (add 200 vs 404 tests)

**Action:**
1. Export `useCachedEndgameInsights(filters: FilterState)` using TanStack `useQuery`.
2. Use the same `buildFilterParams` serialization as `useEndgameInsights`, including `color`.
3. Query key: `['endgame-insights', 'cached', serializedParams]` — re-fires when filters change.
4. Returns parsed `EndgameInsightsResponse` on 200, `null` on 404 (no error). Other axios failures propagate as errors.
5. In `Endgames.tsx`: call the hook with `appliedFilters`. On `data` arrival (truthy), upsert into `insightsCache` if not already present (mirror existing `setInsightsCache` upsert logic).
6. Existing POST/Generate path is untouched.

**Verify:**
- `cd frontend && npm run lint`
- `cd frontend && npm test -- useEndgameInsights`

**Done:** Page mount with cached report renders insights without clicking Generate. Filter changes that match an old cache hit auto-load. Cache miss is silent (no error toast).
