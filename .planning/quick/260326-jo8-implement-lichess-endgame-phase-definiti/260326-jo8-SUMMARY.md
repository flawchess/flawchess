## What Was Built

### piece_count classifier field
- New `_compute_piece_count(board)` function counts Q+R+B+N pieces for both sides combined
- Added `piece_count: int` field to `PositionClassification` frozen dataclass
- Updated `classify_position()` docstring to mention 5 fields (was 4)

### Database column + migration
- Added `piece_count: Mapped[Optional[int]] = mapped_column(SmallInteger, nullable=True)` to `GamePosition` model
- Alembic migration adds the column and backfills existing rows from `material_signature` using character counting SQL (`LENGTH(sig) - LENGTH(REPLACE(sig, 'Q', ''))` etc.)
- Backfill runs in 50,000-row batches to avoid long table locks on production
- Spot check confirmed 0 rows with NULL piece_count and non-NULL material_signature

### Import pipeline
- Added `"piece_count": classification.piece_count` to the row.update block in import_service.py
- Updated chunk_size from 2300 to 2100 (now 15 columns per row: 32767 / 15 = 2184, using 2100 for safety)

### Endgame threshold switch
- Replaced `ENDGAME_MATERIAL_THRESHOLD = 1500` with `ENDGAME_PIECE_COUNT_THRESHOLD = 6`
- Updated `query_endgame_entry_rows` WHERE clause from `material_count < 1500` to `piece_count <= 6`
- Updated endgame repository tests: `_seed_game_position` now takes `piece_count` parameter (default=2)

### Frontend tooltip
- Updated EndgameWDLChart.tsx tooltip to explain the Lichess piece-count definition (threshold=6, excludes kings/pawns)
