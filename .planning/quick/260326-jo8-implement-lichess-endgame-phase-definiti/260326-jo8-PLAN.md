    After the `add_column`, add a raw SQL UPDATE that computes piece_count from material_signature. The formula: count characters in material_signature that are in {Q, R, B, N} (case-sensitive). In PostgreSQL:
    ```sql
    UPDATE game_positions
    SET piece_count = (
        LENGTH(REPLACE(REPLACE(REPLACE(REPLACE(REPLACE(REPLACE(
            material_signature, 'K', ''), 'P', ''), '_', ''), 'Q', 'Q'), 'R', 'R'), 'B', 'B'))
    )
    WHERE material_signature IS NOT NULL AND piece_count IS NULL;
    ```

    Actually, simpler approach — count Q+R+B+N characters:
    ```sql
    UPDATE game_positions
    SET piece_count = (
        LENGTH(material_signature)
        - LENGTH(REPLACE(material_signature, 'Q', ''))
        + LENGTH(material_signature)
        - LENGTH(REPLACE(material_signature, 'R', ''))
        + LENGTH(material_signature)
        - LENGTH(REPLACE(material_signature, 'B', ''))
        + LENGTH(material_signature)
        - LENGTH(REPLACE(material_signature, 'N', ''))
    )
    WHERE material_signature IS NOT NULL;
    ```
    This counts occurrences of Q, R, B, N in the signature string. Kings and pawns are excluded. The underscore separator contains none of these letters.

    Run in batches if the table is large — use a DO block with batched updates of 50,000 rows at a time to avoid long locks on production.
