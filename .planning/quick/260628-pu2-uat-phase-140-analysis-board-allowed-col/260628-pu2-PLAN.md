# Quick Task 260628-pu2: Analysis board UAT (phase 140)

UAT feedback for Phase 140's full-game analysis board. Three interrelated arrow /
depth-overlay tweaks, executed inline (well-scoped frontend changes).

## Tasks

1. **Allowed-color flaw arrow (item 1)** — On a flaw position the board drew the
   engine's following-best move in plain blue (`BEST_MOVE_ARROW`). When the flaw
   allowed a tactic, that arrow IS the opponent's punishing response, so it should
   read in the allowed-tactic crimson (`TAC_ALLOWED`), mirroring the game-card
   miniboard (`LibraryGameCard.boardArrows`).
   - `frontend/src/hooks/useGameOverlay.ts`: color the following-best arrow
     `TAC_ALLOWED` when `depths.allowed != null`, else `BEST_MOVE_ARROW`.

2. **Depth 0 = payoff (item 2)** — The countdown arrow showed a "0" label in the
   orientation color on the move after the tactic resolves. Depth 0 should show no
   number and drop the orientation color (the tactic is over by then).
   - `frontend/src/pages/Analysis.tsx`: `isPayoff = stepIntoPv >= rootDisplayDepth`
     (was `>`), so display-depth 0 becomes the neutral payoff arrow.
   - `frontend/src/lib/tacticArrows.ts`: drop the now-dead `isFlawLeadIn` branch /
     param.

3. **Allowed line starts at the flaw position (item 3)** — Allowed PVs forked one
   ply BEFORE the flaw (decision board) and replayed the flaw move as the first
   sideline move (the backend prepends it to `allowed_moves`). Start the line AT
   the flaw position instead: fork at `mainLine[ply]` and drop the prepended flaw
   move (`allowed_moves.slice(1)`). This also fixes the depth counter, which was
   counting one level too low (now counts down to 1 at the punchline like missed).
   - `frontend/src/pages/Analysis.tsx`: new `forkPlyForOrientation()` helper applied
     to the insertPvLine effect, `pvSidelineArrows`, and `handlePvChipClick`; shift
     the sideline-coloring `resolveIdx` by -1 for allowed (lead-in dropped). Missed
     lines unchanged (still fork at the decision board).
