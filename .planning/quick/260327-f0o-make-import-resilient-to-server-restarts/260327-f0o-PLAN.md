<objective>
Make the game import pipeline resilient to server restarts by fixing three issues:
1. DB job progress counters (games_fetched/games_imported) only update at completion -- if server crashes mid-import, they stay at 0 despite games being committed to DB
2. Platform username only saves to user profile after successful completion -- if import fails, the username is lost and the sync button appears empty after restart
3. Incremental counter updates also provide accurate progress data for the orphaned job cleanup

Purpose: Users who experience a mid-import server restart can immediately see accurate progress and re-sync without confusion.
Output: Updated import_service.py with incremental DB updates, imports router that saves username at start.
</objective>

**1. Move username save to import start (app/routers/imports.py):**

In the `start_import` endpoint, after `create_job` and before `asyncio.create_task`, save the platform username to the user profile immediately. This requires a DB session. Add `session: Annotated[AsyncSession, Depends(get_async_session)]` parameter and call:
```python
await user_repository.update_platform_username(session, user_id, request.platform, request.username)
await session.commit()
```
This ensures the username persists even if the import fails mid-way, so after restart the Import page pre-populates the username field and the Sync button is enabled.

**2. Update DB job counters after each batch (app/services/import_service.py):**

In `run_import`, after each `_flush_batch` call (both the loop batch and the trailing batch), update the DB row with current counters. After the line `job.games_imported += imported`, add:
```python
await import_job_repository.update_import_job(
    session,
    job_id=job_id,
    status="in_progress",
    games_fetched=job.games_fetched,
    games_imported=job.games_imported,
)
await session.commit()
```
Note: the commit for counter updates piggybacks on the same session, which is already committed in `_flush_batch`. We need a separate commit here for the import_job update. Actually, `_flush_batch` already commits at the end (`await session.commit()`), so the counter update needs its own flush+commit cycle. Call update_import_job and then commit.

Also update games_fetched periodically. Currently `_on_game_fetched` only increments in-memory. Add a DB update after each batch flush that writes the current `job.games_fetched` value too (already included in the update_import_job call above).

**3. Remove the duplicate username save at completion (app/services/import_service.py):**

Remove the best-effort `update_platform_username` block at lines 246-252 (the try/except after completion), since username is now saved at import start in the router.

**4. Add an `update_job_progress` helper to import_job_repository.py (optional optimization):**

The existing `update_import_job` uses generic **kwargs with setattr. This is fine, no new function needed -- just use the existing function.
