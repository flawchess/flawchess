# Quick Task 260619-uq6: Implement SEED-055

Drop the 7 columns of `game_positions` that are computed and written at import
but never read by any serving query: `material_count`, `material_imbalance`,
`has_opposite_color_bishops`, `material_signature`, `piece_count`,
`backrank_sparse`, `mixedness`.

**Decision (2026-06-19):** Drop all 7, including `material_signature` (the seed
flagged it for a conscious decision — confirmed dropped). Reversible: re-add +
PGN-replay backfill if a future `endgame_class` reclassification ever needs it.

## Key principle: derived values stay

`endgame_class` and `phase` are derived from these columns **in-memory at import
time** (`classify_endgame_class(material_signature)` and
`assign_game_phases(piece_count, backrank_sparse, mixedness)`), then persisted.
The in-memory computation in `position_classifier.py` for the 4 "intermediate"
columns MUST stay; only the persisted copy is dropped. The 3 "fully dead"
columns (`material_count`, `material_imbalance`, `has_opposite_color_bishops`)
stop being computed entirely.

## Tasks

1. **Source** — remove the 7 columns from the persistence path:
   - `app/models/game_position.py` — drop 7 `mapped_column`s.
   - `app/services/position_classifier.py` — drop 3 fully-dead fields from
     `PositionClassification` + `classify_position`; remove `_compute_material_count`,
     `_compute_material_imbalance`, `_compute_opposite_color_bishops` and the
     centipawn-value constant they alone use. Keep signature/piece_count/
     backrank_sparse/mixedness computation (needed for endgame_class + phase).
   - `app/services/zobrist.py` — drop the 3 fully-dead fields from `PlyData`; keep
     the 4 intermediate ones (used locally for endgame_class + phase), then they
     are no longer copied into rows downstream.
   - `app/services/import_service.py` — remove the 7 keys from the position-row dict.
   - `app/repositories/game_repository.py` — remove the 7 from `POSITION_COPY_COLUMNS`.
   - `app/services/eval_drain.py` — remove the 7 placeholder-dict keys.
   - `scripts/backfill_eval.py` — remove the now-obsolete PHASE-FILL-01 phase
     backfill block (it computes `phase` FROM the dropped columns; phase is fully
     populated by the import path + prior backfill). Drop the now-unused threshold
     imports / `PHASE_BACKFILL_CHUNK_SIZE`.

2. **Migration** — Alembic `DROP COLUMN` x7 (downgrade re-adds nullable). Document
   that `DROP COLUMN` does not reclaim disk on existing rows — a `pg_repack` /
   `VACUUM FULL` prod window is a separate operational step (out of scope here).

3. **Tests** — remove the 7 columns from every fixture / assertion that constructs
   `GamePosition` rows or `PlyData`/row dicts, and the column-coverage test's
   `optional_cols` set. Remove the `_compute_material_imbalance` /
   `material_count` / `has_opposite_color_bishops` unit tests; keep
   signature/piece_count/backrank_sparse/mixedness tests.
