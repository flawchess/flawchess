# Quick 260529-015: Gate Endgame Tier-2 Readiness on Stage-B Summary

**In-memory Stage-B compute registry gates `GET /imports/readiness` tier2, closing the race where the endgame page unlocked the instant pending evals hit zero — before `compute_stage_b` had written the 7 eval-dependent percentile families.**

## Accomplishments
- New `percentile_compute_registry` module: `mark` / `clear` / `is_computing` on a module-level `set[int]`, no lock (GIL-atomic), no Sentry (no I/O), with a docstring documenting the race it closes and restart semantics.
- Both Stage-B trigger sites (`eval_drain.py`, `import_service.py`) now call `mark(uid)` synchronously immediately before `asyncio.create_task(compute_stage_b(uid))`, closing the window where the 3s readiness poll could observe `pending==0` before the task started.
- `compute_stage_b` clears the mark in a `finally` covering all exit paths: success, the early no-anchors return, the top-level exception path, and cancellation (the finally runs before the `CancelledError` re-raise). This prevents permanent lockout for below-floor / no-anchor users where zero rows are written.
- `get_readiness` tier2 now additionally requires `not is_computing(user.id)` (a sync set lookup, no extra query, so the "at most 3 queries" bound is unchanged).
