<objective>
Close the Stage-B readiness race: the endgame page Tier-2 gate currently flips
`tier2=True` the moment pending evals reach zero, BEFORE `compute_stage_b` has
written the 7 eval-dependent percentile metric families. On first import the
eval-dependent badges are missing on unlock; on re-import stale prior-import
badges are shown until recompute lands.

The fix is an in-memory "Stage-B-in-progress" registry mirroring the existing
in-memory active-job registry in `import_service`. A user is marked
synchronously immediately before `asyncio.create_task(compute_stage_b(uid))` at
both trigger sites, and cleared in a `finally` inside `compute_stage_b` that
covers every exit path (success, early no-anchors no-op, exception). The
readiness endpoint adds the membership check to its `tier2` condition.

Purpose: badges are present and current the instant the page unlocks; no
permanent lockout for below-floor users (the `finally` clears even when zero
rows are written).
Output: one new registry module, four wired call sites, two test files.
</objective>

    - Double mark(uid) then a single clear(uid) leaves is_computing(uid) False (idempotent set add/discard — no refcount).

    (set.discard — idempotent, never raises), `is_computing(user_id: int) -> bool`
    (membership test). Add a module docstring explaining: (a) this gates Tier-2
