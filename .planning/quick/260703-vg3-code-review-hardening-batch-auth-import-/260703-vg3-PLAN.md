# Quick Task 260703-vg3 — Code-review security/ops hardening batch

Trivial, low-blast-radius security + ops fixes from the 2026-07-02 fable review.
Each item is an independent atomic commit.

## Investigation findings (pre-execution)

- **#1**: `GET /api/imports/{job_id}` (`imports.py:333`) is the only import route with no
  `current_active_user` dependency and no user scoping. Raw `str(exc)` is written to
  `job.error` / `error_message` at `import_service.py:619` + `:632` (the generic-exception
  handler) and echoed to clients by BOTH `/{job_id}` and `/active`. Sanitizing at the
  source fixes both endpoints; the timeout handler already sets a safe message.
- **#3**: `ix_game_flaws_blob_backfill` is NOT in `_AUTOGEN_INDEX_IGNORELIST` (env.py) and
  is not ORM-declared → next `--autogenerate` emits `op.drop_index` on prod's 348M-scan
  index. The 4 "dev-missing" partials (`ix_games_evals_pending`, `ix_games_full_evals_pending`,
  `ix_games_full_pv_pending`, `ix_games_needs_engine_full_evals`) are created unconditionally
  by their migrations and never dropped in `upgrade()` — a fresh `alembic upgrade head`
  (i.e. the per-run test template) builds them. Their absence in the local dev DB is
  historical drift, NOT a migration bug. Fix = ignorelist line + a drift-detection test
  (runs against the freshly-migrated test DB) + restore the 4 in the local dev DB (idempotent).
- **#1.2**: default `ENVIRONMENT="production"` + default `SECRET_KEY="change-me-in-production"`.
  conftest sets a real 32-byte SECRET_KEY before app import (and leaves ENVIRONMENT default),
  so a guard keyed on `SECRET_KEY == default` will NOT trip tests. Mirror D-22: a testable
  validator called from the lifespan (not an import-time model_validator, which would trip
  alembic/tooling that imports `settings`).
- **#11**: `eval_cp`/`eval_mate` are `SmallInteger` (±32767), `best_move` is `String(5)`,
  `pv` is `Text`; `ply` max ~600. An out-of-range worker value → DBAPIError → 500 → an
  unresolvable retry-loop. Add Field bounds to `SubmitEval`, `AtomicSubmitEval`,
  `EntrySubmitEval`.
- **#5 ANALYZE**: `opening_position_eval` planner stats stale. Prod-only op (prod MCP is
  read-only; requires the SSH tunnel or an `ssh flawchess` docker exec). Flagged as a manual
  follow-up — NOT bundled into a committed code change (autovacuum tuning deferred per triage).
