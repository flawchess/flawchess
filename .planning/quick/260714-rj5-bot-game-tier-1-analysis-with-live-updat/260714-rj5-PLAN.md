<objective>
After a bot game, "Analyze this game" enqueues a tier-1 eval job and navigates straight to the analysis board in GAME mode (`/analysis?game_id=X`). The board renders the move list immediately, shows the Pending…/Analyzing… pill where the eval chart would be, and when analysis lands the eval chart, move-list flaw icons and missed/allowed tactic badges appear IN PLACE — no remount, no navigation — so the move cursor and any user-built variation tree survive automatically.

Purpose: today the button drops the freshly stored `game_id` on the floor and opens a free-play `?line=` board with no analysis path at all; guests are told "use 'Analyze this game' above" by a button that enqueues nothing. Same code path also fixes the pre-existing dead end where an unanalyzed IMPORTED game opens an empty board.

Output: a backend payload change (moves for unanalyzed single-game cards), a live-polling analysis board with a shared pending pill, and a one-click Analyze on both bot result surfaces.
</objective>
