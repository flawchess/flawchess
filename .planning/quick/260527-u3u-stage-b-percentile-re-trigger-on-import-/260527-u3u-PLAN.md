<objective>
Fix stale eval-dependent percentile rows after Stage-5c-covered incremental imports by adding a second Stage B trigger inside `_complete_import_job`, gated on the existing `users_with_zero_pending` check.

Purpose: When an incremental import contains only lichess games whose entry plies already carry `%eval` annotations, every new game is marked `evals_completed_at = NOW()` at import time (Stage 5c). The cold-drain coroutine in `eval_drain.py` never ticks for that user, so the existing Stage B trigger at `eval_drain.py:566-568` never fires. Stage A (`score_gap`) refreshes; the 7 eval-dependent metric families (`achievable_score_gap`, `score_gap_conv`, `score_gap_parity`, `recovery_score_gap`, `clock_gap`, `net_flag_rate`, `time_pressure_score_gap`) go stale — verified on prod for user 28 (Stage A 18:35:34 vs Stage B 16:24:59 after three Stage-5c-covered imports).

Output: Stage B re-fires from import completion whenever the user's pending-eval count is zero. The cold-drain trigger is preserved; both sites are idempotent (write-disjoint rows from Stage A, repeated runs upsert the same values).
</objective>
