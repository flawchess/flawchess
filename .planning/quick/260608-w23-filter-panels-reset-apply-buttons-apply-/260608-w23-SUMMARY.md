## What Was Built

**FilterActions component** (`frontend/src/components/filters/FilterActions.tsx`): reusable 50/50 footer with Reset (`variant="brand-outline"`) on left and Apply (`className="btn-brand"`) on right. Default testids `btn-filter-reset`/`btn-filter-apply`; Tags panel uses `btn-tags-reset`/`btn-tags-apply`.

**Staged Apply-only model** across all 5 pages:
- Filter edits update `pendingFilters` / `pendingFlawFilter` draft state only
- Apply commits draft to shared store AND closes the panel
- Opening a panel snapshots committed state into the draft
- Closing without Apply discards the draft (re-snapshot on next open)
- Reset clears draft to defaults (with color/playedAs preserved; Tags Reset → DEFAULT_FLAW_FILTER)

**Library "Tags" panel** renamed from "Flaw filters" in both desktop SidebarLayout and mobile drawer.

**FlawFilterControl cleanup**: removed `onClear` prop, "Show flaws with:" section header, and "Clear flaw filter" conditional button.

**Openings pulse preservation**: `justCommittedFromDrawerRef.current = true` set before `setFilters(localFilters)` in Apply handlers so existing useEffect detects the commit and fires the board arrow pulse.
