<objective>
Add a Reset+Apply button row to every filter panel and switch all filter panels to a unified
staged-apply model: edits mutate pending/draft state only; nothing commits to the shared store
until Apply is clicked; Apply also closes the panel; closing any other way discards the draft.
Rename the Library "Flaw filters" panel to "Tags" and bring it under the same treatment.

Purpose: Consistent, predictable filter UX across the app — no surprise live-apply, no ambiguous
"applies on close" hint, one button layout everywhere.
Output: New FilterActions component + reworked staged-state wiring across 5 pages and shared panels.
</objective>
