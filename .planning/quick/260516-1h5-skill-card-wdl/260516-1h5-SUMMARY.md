# Quick Task 260516-1h5: Skill Card WDL Summary

One-liner: Replaced the plain `Games: N` text in the Endgame Skill card with the same games-count row + `MiniWDLBar` layout used by the Conv/Parity/Recov cards, driven by `perfData.endgame_wdl` (Games with Endgame).

## What Changed

### `EndgameSkillCard.tsx`
- New required prop `endgameWdl: EndgameWDLSummary` (replaces the now-redundant `totalGames` prop).
- Imports `Swords` (lucide-react), `MiniWDLBar`, and the `EndgameWDLSummary` type.
- Inside the existing `hasSkill` branch, the simple `Games: {totalGames}` span is replaced with a flex column wrapping:
  1. A header row with `Win/Draw/Loss` label on the left and a right-aligned `Games: {endgameWdl.total}` + `Swords` icon span (`data-testid="tile-endgame-skill-games-count"`).
  2. A `MiniWDLBar` driven by `endgameWdl.win_pct/draw_pct/loss_pct`.
- The new WDL block is guarded by `endgameWdl.total > 0` so it does not render when there are no games (e.g. filters wipe the cohort).
- Gauge, Score Gap bullet, InfoPopover copy, the `Cpu` icon on the eval-derived row, and the connector-arrows overlay are untouched.
- Header docstring updated to reflect the new layout and call out the quick task.

### `EndgameMetricsSection.tsx`
- Added `endgameWdl: EndgameWDLSummary` to the section's props and forwarded it to `<EndgameSkillCard endgameWdl={endgameWdl} ... />`.
- Imported `EndgameWDLSummary` from `@/types/endgames`.

### `Endgames.tsx`
- Passes `endgameWdl={perfData.endgame_wdl}` to `<EndgameMetricsSection ... />`.
- Tightened the conditional from `{scoreGapData && (...)}` to `{scoreGapData && perfData && (...)}` so TypeScript can narrow `perfData` to non-null inside the block.

### `__tests__/EndgameSkillCard.test.tsx`
- Replaced `totalGames={300}` / `totalGames={0}` props with `endgameWdl={DEFAULT_ENDGAME_WDL}` / `endgameWdl={EMPTY_ENDGAME_WDL}` fixtures.
- Flipped the obsolete `does NOT render MiniWDLBar` test to its inverse: now asserts the bar renders and the games-count row contains `Games: 100` (using the default fixture).
- Added a new test verifying the games-count row and MiniWDLBar are hidden when `endgameWdl.total === 0`.
- All existing assertions (ScoreGapRow, popover, sign convention, CI whiskers, empty state) preserved.

### `__tests__/EndgameMetricsSection.test.tsx`
- Added a `DEFAULT_ENDGAME_WDL` fixture and threaded the new `endgameWdl` prop through every `<EndgameMetricsSection ... />` render call so the section-level integration tests keep compiling and passing.

## Deviations from Plan

None for the core work.

One Rule 2 (auto-add) adjustment: the plan only listed `EndgameSkillCard.test.tsx` under files to update, but `EndgameMetricsSection.test.tsx` also breaks under the new required prop. The constraint says "If a test breaks because of the new prop signature, update the test in the same task — don't leave a broken test", so the section test was updated in the same commit.

One Rule 3 (blocker fix): the wrapping `{scoreGapData && (...)}` block in `Endgames.tsx` did not narrow `perfData` to non-null for TypeScript, so passing `perfData.endgame_wdl` raised a TS error. Added `&& perfData` to the guard. Functionally identical at runtime because the parent section is already gated by `showPerfSection = !!(perfData && perfData.endgame_wdl.total > 0)`.
