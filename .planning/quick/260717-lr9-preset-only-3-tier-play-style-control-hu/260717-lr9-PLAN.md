# Quick Task 260717-lr9 — Preset-only 3-tier play-style control

## Goal

Replace the Bots setup-screen play-style **slider + preset-chips hybrid** with a
**preset-only three-button segmented control**. Three discrete opponents:

| Preset | blend | Behavior copy |
|--------|-------|---------------|
| Human | 0 | Plays on instinct, no calculation |
| FlawChess Light (**default**) | 0.05 | Calculates a little |
| FlawChess Deep | 0.5 | Calculates hard |

## Rationale (from /gsd-explore session)

- The ELO selector × blend cannot be disentangled or predicted into a combined
  strength number, so we ship the "three distinct opponents to experiment with"
  framing (calibrated-strength sparring is the north star — see SEED-112).
- Behavior-relative names ("Light/Deep" = calculation depth) make **no rating
  promise** — the engine is not ELO-calibrated yet.
- Dropping the slider retires the entire D-01 "slider cannot reach 0 / cannot
  advertise a smooth continuum" honesty problem. With three named buttons there
  is no continuum to over-promise.

## Design decisions

- **Keep a validation ceiling constant at 1** (rename `ENGINE_BLEND` →
  `BLEND_MAX`). blend=1 is no longer a preset, but the valid-range must stay
  `[0, 1]` so legacy stored blobs (old Engine preset = 1.0) still *validate*
  rather than being treated as corruption — the WR-01 bug comment in
  `botSetupSettings.ts` warns that out-of-range blends silently discard finished
  games. Downstream `selectBotMove` still treats blend≥1 as argmax.
- **`formatPlayStyleSummary` becomes behavior prose** keyed off the active
  preset (no blend number, no "% search", no ELO). A non-preset legacy blend
  (e.g. a stored 1.0) falls back to a neutral line and shows no active preset
  until the user clicks one — acceptable for a pre-release feature.
- **Default flips to Light (0.05)** via `PLAY_STYLE_DEFAULT_BLEND`.
