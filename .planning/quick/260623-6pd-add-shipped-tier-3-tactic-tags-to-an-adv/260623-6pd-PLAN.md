# Quick Task 260623-6pd

Surface the shipped Phase-132 tier-3 tactic motifs everywhere the existing tactic
families appear. The tactic taxonomy is registry-driven, so this is primarily an
additive change to the shared `FAMILY_TO_MOTIF_INTS` (backend) + `tacticComparisonMeta.ts`
(frontend) registries plus a collapsible "Advanced" group in the filter panel.

## Decisions (locked with user)

- **Comparison grid:** full parity — the 5 new families flow into the `FlawStatsPanel`
  you-vs-opponent grid (10 → 15 families). Update the hardcoded "10 families" tests.
- **Motif set / grouping:** the new "Advanced" group contains **6** families — the 5
  newly-shipped tier-3 (`deflection`, `intermezzo`, `interference`, `clearance`,
  `capturing_defender`) **plus `x_ray` relocated** from "Piece Attacks". `attraction`
  and `sacrifice` stay suppressed (0 TP, no data) and are NOT surfaced.
- **Colors:** keep the intentional single-blue convention (new `TAC_*` alias `TAC_BLUE`).
- **Advanced group is collapsed by default** with a "Advanced · N" selected-count badge,
  mirroring the existing Context collapsible.

## Mechanism (why this is sufficient)

Cook-aligned tier-3 detectors fire at `TACTIC_CONFIDENCE_HIGH` (100), so shipped tier-3
motifs already pass the `_TACTIC_CHIP_CONFIDENCE_MIN=70` lever and are returned by the
backend chip-read path. They are invisible today only because (a) the frontend drops
motifs with no `TACTIC_FAMILY_FOR_MOTIF` entry, and (b) the family filter / comparison
grid iterate `FAMILY_TO_MOTIF_INTS`, which lacks them. Adding the families to both
registries makes chips, eval-chart cycling, the eval-chart tooltip, the tags-info legend,
and the family filter all work automatically (every consumer derives from the registry).
