# Summary: Aggregate Time Controls in Time Pressure vs Performance Chart

## What Changed

`frontend/src/components/charts/EndgameTimePressureSection.tsx` rewritten:
- Dropped the time-control tabs. Chart now renders a single aggregated view.
- Added `aggregateSeries()` helper — weighted-average score per bucket across all rows (weight = game_count), with pooled game counts.
- X-axis tick labels now show full `"0-10%"`..."90-100%"` bucket labels (removed `tickFormatter` that stripped to left edge).
- X-axis label: "% of base time remaining at endgame entry".
- Y-axis: `domain={[0.2, 0.8]}`, ticks `[0.2..0.8]`, label "Avg Score".
- Info-popover copy updated to mention aggregation across all time controls + filter-panel drill-down.
- Removed unused `Tabs*` imports.

Backend, schemas, tests, and API shape untouched — aggregation is purely client-side.

## Why

Prod data (266k games, 40 users) showed bullet/blitz/rapid curves have near-identical *shape* (steep 0→40%, plateau 40→100%, inflection ~30%). Rapid sits ~3–5pt below as a vertical offset, not a shape change. Classical has n≈177 and is unusable as a separate line. Pooling keeps the "danger zone <30% time" story while giving more games per datapoint; users can still drill down via the side-panel time-control filter.
