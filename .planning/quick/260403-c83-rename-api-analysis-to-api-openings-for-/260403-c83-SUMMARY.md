# Phase quick Plan 260403-c83: Rename /api/analysis to /api/openings Summary

**One-liner:** Renamed all three openings endpoints from `/api/analysis/*` to `/api/openings/*` to align with the `/api/endgames` URL convention.

## What Was Built

Renamed the HTTP-facing URL prefix for the analysis router from `/analysis` to `/openings`. All three endpoints now live under `/api/openings/`:

- `POST /api/openings/positions` (was `/api/analysis/positions`)
- `POST /api/openings/time-series` (was `/api/analysis/time-series`)
- `POST /api/openings/next-moves` (was `/api/analysis/next-moves`)

Internal Python module names (`analysis_service`, `analysis_repository`, `schemas/analysis`) were intentionally left unchanged — they describe the operation, not the URL.
