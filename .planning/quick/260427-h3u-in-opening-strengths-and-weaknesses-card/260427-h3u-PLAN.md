# Quick Task 260427-h3u: OpeningFindingCard link refactor

## Goal

In each Opening Strengths/Weaknesses finding card:

- Remove the whole-card deeplink (no more `<a>` wrapping the card body).
- Remove the `(n=<x>)` indicator from the prose line.
- Remove the `ExternalLink` icon in the top right of the card header.
- Add a separate row inside the card with two links:
  - `Moves` — uses the `ArrowRightLeft` icon (same icon as the Moves tab trigger). Loads the finding's entry move sequence onto the board and navigates to `/openings/explorer`.
  - `<n> Games` — uses the `FolderOpen` icon. Loads the finding's entry move sequence onto the board and navigates to `/openings/games`.
- Both links have hover state (`hover:text-foreground`) and `Tooltip`s, matching the Games button in `MostPlayedOpeningsTable`.

## Files

- `frontend/src/components/insights/OpeningFindingCard.tsx` — primary refactor.
- `frontend/src/components/insights/OpeningInsightsBlock.tsx` — add `onOpenGames` prop.
- `frontend/src/pages/Openings.tsx` — add `handleOpenFindingGames` callback that mirrors `handleOpenFinding` but routes to `/openings/games`.
- `frontend/src/components/insights/OpeningFindingCard.test.tsx` — drop tests asserting `(n=…)`, ExternalLink, whole-card click; add tests for Moves and Games links.
- `frontend/src/components/insights/OpeningInsightsBlock.test.tsx` — pass `onOpenGames` prop in tests.

## Plan

1. Edit `OpeningFindingCard.tsx`:
   - Add a new `onOpenGames: (finding) => void` prop to `OpeningFindingCardProps`.
   - Replace the wrapping `<a>` with a non-interactive `<div>` (keep the same `data-testid`, border styling, no `href` / no card-level click).
   - In `headerLine`: remove the `<ExternalLink>` block.
   - In `proseLine`: remove the `(n={n_games})` span.
   - Add a new `linksRow` with two `<button>` elements wrapped in `Tooltip` (component from `@/components/ui/tooltip`):
     - Moves button: `ArrowRightLeft` icon, label "Moves", tooltip `"Open ${display_name} in the Move Explorer"`, calls `onFindingClick(finding)`.
     - Games button: `FolderOpen` icon, label `${n_games} Games`, tooltip `"View ${n_games} games for ${opening_name}"`, calls `onOpenGames(finding)`.
     - Reuse the styling pattern from `MostPlayedOpeningsTable` Games button: `flex items-center gap-1 text-sm text-muted-foreground hover:text-foreground transition-colors`, `tabular-nums` on the count.
   - Place `linksRow` on its own row in both the mobile and desktop layouts (after the prose).

2. Edit `OpeningInsightsBlock.tsx`:
   - Add `onOpenGames: (finding: OpeningInsightFinding) => void` prop.
   - Plumb it through `SectionsContent` → `FindingsSection` → `OpeningFindingCard`.

3. Edit `Openings.tsx`:
   - Add `handleOpenFindingGames` (mirrors `handleOpenFinding` but `navigate('/openings/games')`).
   - Pass it to `OpeningInsightsBlock` as `onOpenGames`.

4. Update tests:
   - `OpeningFindingCard.test.tsx`: replace `(n=18)` / `(n=25)` assertions with assertions about the `<n> Games` link text. Drop the `aria-label` assertion (the card no longer has one). Replace the whole-card click test with separate tests for the Moves button and Games button.
   - `OpeningInsightsBlock.test.tsx`: pass an `onOpenGames` prop to every `<OpeningInsightsBlock />` render. The existing card-click test currently clicks the card div by `data-testid`; switch it to click the Moves button (or remove if redundant).

5. Run `npm run lint` and `npm test -- OpeningFinding OpeningInsights` to verify.
