# Quick Task 260619-fk1 — Phase 126 UAT: tactic tags

## Problem (UAT feedback, Games card)

1. Tactic tags don't appear in the eval-chart flaw-datapoint tooltip.
2. The tactic tag has its own hover/tap definition popover that duplicates the
   shared Tags legend.
3. Hovering a tactic tag should highlight matching eval-chart datapoints and dim
   the rest (same as flaw-tag chips / severity badges).
4. Clicking/tapping a tactic tag should cycle the eval chart through its
   datapoints (same as the other tags).

## Root cause

- (1) `EvalChart` already renders the tactic motif in its tooltip, gated on the
  `betaEnabled` prop. The **mobile** `EvalChart` call site in `LibraryGameCard`
  passes `betaEnabled`; the **desktop** call site does not, so the motif line is
  silently off on desktop.
- (2) `TacticMotifChip` carries its own Radix popover. `FlawCard` and
  `LibraryGameCard` already surface motif definitions via the shared `TagLegend`
  (and their `TagChip`s use `definition={false}`), so the per-chip popover is
  redundant and inconsistent.
- (3/4) `TacticMotifChip` exposes no `onHover`/`onActivate`, and the card's
  highlight/cycle machinery (`FlawRef`) only knows `tag` and `severity` kinds.

## Tasks

### Task 1 — wire tactic motifs into the highlight/cycle machinery

**Files:** `frontend/src/components/results/LibraryGameCard.tsx`

- Add `betaEnabled={userProfile?.beta_enabled ?? false}` to the **desktop**
  `EvalChart` call site (fixes item 1).
- Extend `FlawRef` with `{ kind: 'motif'; motif: string }`; update `sameFlawRef`.
- Add a `motifPlies` memo (user markers grouped by `tactic_motif`, sorted),
  mirroring `tagPlies`.
- Update the `highlightedPlies` derivation to match `fm.tactic_motif === ref.motif`
  for motif refs.
- Update `pliesForRef` to resolve motif plies.
- Pass `onHover` / `onActivate` to each `<TacticMotifChip>` (mirroring `<TagChip>`).

**Verify:** `npx tsc -b`; hovering/clicking a tactic chip drives the chart like a
flaw-tag chip; tooltip shows the motif on desktop.

### Task 2 — make TacticMotifChip a highlight/cycle chip (drop popover)

**Files:** `frontend/src/components/library/TacticMotifChip.tsx`

- Remove the Radix popover (trigger/content/portal), the `open` state, the
  open/close timers, and `useIsMobile` (only used for popover side).
- Add optional `onHover?(active)` and `onActivate?()` props.
- `interactive = Boolean(onHover || onActivate)` (mirror `TagChip`): when
  interactive, render `role="button"`, `tabIndex=0`, `cursor-pointer`,
  hover-brighten, and wire mouseenter/leave → highlight + `onHover`, click +
  Enter/Space → `onActivate`. When not interactive (FlawCard), render a plain
  decorative span (matches `TagChip definition={false}` there).
- Keep the chip `data-testid`, `aria-label`, family color, and `text-xs`.

### Task 3 — update tests

**Files:** `frontend/src/components/library/__tests__/TacticMotifChip.test.tsx`

- Drop popover-wiring assertions.
- Interactivity assertions (`role=button`, `tabIndex`, `cursor-pointer`) now
  render with callbacks; add a decorative-default test (no callbacks → no role).
- Add `onHover` (true on enter / false on leave) and `onActivate`
  (click + Enter/Space) tests.
