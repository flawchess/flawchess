# Quick Task 260709-k9r

## Problem

On a forced-checkmate line (e.g. `/analysis?game_id=687512&ply=63`) the FlawChess
Engine card mis-rendered two things:

1. The objective eval next to the gold practical badge showed the `…` placeholder
   instead of `#-4`.
2. The agreement verdict showed the muted "Turn on Stockfish to compare picks."
   prompt even though standalone Stockfish was on.

## Root cause

Both symptoms trace to a single dropped value. `MoveGrade` carries both `evalCp`
and `evalMate`, but `RankedLine`/the search-tree node only kept `objectiveEvalCp`.
At a forced-mate leaf the grade is `{evalCp: null, evalMate: -4}`, so
`objectiveEvalCp` became `null` and the mate distance was lost:

- `FlawChessEngineLines` formatted `formatScore(objectiveEvalCp, null)` → `…`.
- `computeFlawChessVerdict` bailed at `if (objectiveEvalCp == null) return null`,
  so the verdict fell into its Stockfish-off slot.

The primary display path is `Analysis.tsx`'s `reconciledRankedLines`, which
overrides `objectiveEvalCp` from the on-page eval lookup but did NOT carry the
lookup's `evalMate`.

## Fix (additive plumbing — no search-logic change)

Thread the already-computed `evalMate` through to display:

- `lib/engine/types.ts` — add `objectiveEvalMate: number | null` to `RankedLine`
  and `ModalPlyStat`.
- `lib/engine/treeCommon.ts` — carry it on the node, into `buildModalPath` +
  `buildRankedLines`.
- `lib/engine/mctsSearch.ts` + `fallbackExpectimax.ts` — pass `grade?.evalMate ??
  null` (both runners, symmetric).
- `components/analysis/FlawChessEngineLines.tsx` — format
  `(objectiveEvalCp, objectiveEvalMate)` for the badge + chip preview.
- `lib/flawChessVerdict.ts` — accept a mate-only FC eval (bail only when BOTH cp
  and mate are null), set `flawChessMove.evalMate`, feed mate into
  `evalToExpectedScore`, and null the cp gap when the FC side is a mate.
- `pages/Analysis.tsx` — `reconciledRankedLines` now pulls both `evalCp` and
  `evalMate` from the same resolved grade.

## Tasks

1. Thread `objectiveEvalMate` through the engine data contract + both search
   runners.
2. Reconcile mate in `Analysis.tsx` and render it in the card + verdict.
3. Add regression tests (verdict mate classification, card `#-4` render) and
   update fixtures.
