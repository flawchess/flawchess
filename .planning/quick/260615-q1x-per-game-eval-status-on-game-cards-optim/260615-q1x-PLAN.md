<objective>
Surface per-game eval status on Library game cards so a user clicking "Analyze"
gets immediate feedback. The pill goes optimistic-Pending on click (no server
round-trip wait), transitions to "Analyzing…" once the eval job is leased, and
clears when the game becomes analyzed.

The transition rides the existing `library-games` poll: GamesTab already polls
that endpoint every 3s while any analyze is in flight, so enriching its payload
with `active_eval_status` delivers the pending→leased transition with zero new
requests. No new endpoint, no progress bar — keep the existing pulsing pill.

Purpose: Tighter feedback loop on the on-demand analyze action; distinguishes
"queued" from "actively running" without polling a new endpoint.

Output: A new `active_eval_status` field on the library-games payload, a batch
repository query that LEFT JOINs the single active eval_jobs row per game, and an
updated three-state pill (Analyze button → Pending… → Analyzing…) wired into both
the desktop and mobile card layouts with optimistic click + error rollback.
</objective>
