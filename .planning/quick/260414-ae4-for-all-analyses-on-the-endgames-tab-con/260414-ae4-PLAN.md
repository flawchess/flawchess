<objective>
Apply the 6-ply minimum endgame duration rule uniformly to ALL analyses on the endgames tab.

Currently, `ENDGAME_PLY_THRESHOLD = 6` already exists in `app/repositories/endgame_repository.py`
and is applied to per-class queries (WDL by class, per-type timeline series, clock stats,
paginated game list). However, the **binary** "reached an endgame phase" path uses a different
rule (`endgame_class IS NOT NULL`, with no ply threshold). This drives:

- `count_endgame_games` → used for "X of Y games reached an endgame" summary line
- `query_endgame_performance_rows` → Games with vs without Endgame WDL bars + score gap
- `query_endgame_bucket_rows` → conversion/even/recovery material table
- `query_endgame_timeline_rows` Query B (non-endgame rows) → overall timeline split

The result: a game that touched an endgame class for 1–5 plies (a tactical transition) was
counted as "reached an endgame" for the binary split but NOT counted in any per-class stat.
That's an inconsistency the quick task fixes by unifying on the 6-ply rule everywhere.

Purpose: One semantic definition of "has endgame" across the entire tab; eliminates the
Phase 59 bucket-sum carve-out (query_endgame_bucket_rows previously justified dropping the
threshold to preserve `sum(material_rows.games) == endgame_wdl.total` — that invariant is
preserved automatically now because both the binary split AND the bucket query use the
same 6-ply threshold).

Output: Updated backend queries, updated popover text, updated tests.
</objective>
