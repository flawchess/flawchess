# Quick Task: Phase 140 analysis-board UAT round 6

Six (5 items) frontend UAT tweaks for the full-game analysis board. Frontend-only.
The alignment item (item 2) is the 4th failed attempt — root cause found empirically
in the browser (DOM geometry), not guessed.

## Items

### 1. Miniboard hover tooltip: no border, rounded corners, green played-move overlay
The engine-line hover miniboard (`EngineLines.tsx` → `<Tooltip>` → `<MiniBoard>`).
The "border" is the shared tooltip's `px-3 py-1.5 bg-foreground` padding frame; the
board's square corners poke out of the `rounded-md`.
- `ui/tooltip.tsx`: add optional `contentClassName?: string`, appended LAST in the
  `cn()` (tailwind-merge lets it override the base padding).
- `EngineLines.tsx`: pass `contentClassName="overflow-hidden p-0"` (kills the frame +
  clips the board to rounded corners) and `lastMove={{from,to}} lastMoveColor={MOVE_HIGHLIGHT_GOOD}`
  to MiniBoard (green overlay on the played PV move; MiniBoard already supports it).

### 2. Eval-chart slider ↔ board-controls bottom alignment (4th try)
**Root cause (measured live):** the outer row is `lg:items-stretch`; the LEFT column
(board 480 + gap + eval-chart 147 = 635px) is SHORTER than the RIGHT column. The move
list's `max-h-[455px] mb-[25px]` makes the right column 677px tall, so items-stretch
stretches the left column to 677 and leaves 42px dead space *below* the eval-chart
slider while the controls pin to the true bottom (742) → 49px misalignment. Prior
`mb`/`max-h` tweaks can't fix it because they don't stop the move list from inflating
the row's intrinsic height.

**Fix (verified in browser → controls bottom 742→700 = slider container bottom, no
magic numbers):** make the move-list scroller `absolute inset-0` inside a `relative`
flex parent so its content can't inflate the row; the board column then drives the
height and the controls bottom-align with the slider.
- `VariationTree.tsx:681` wrapper → add `relative`.
- `VariationTree.tsx:624` scroller → `absolute inset-0 overflow-y-auto` (drop
  `min-h-0 max-h-[455px] mb-[25px] flex-1`).
