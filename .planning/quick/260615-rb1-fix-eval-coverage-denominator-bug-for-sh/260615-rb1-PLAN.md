<objective>
Fix a confirmed production bug: `_compute_eval_coverage` divides eval-carrying positions by the total position count, but the terminal position (after the last move) structurally never carries an eval. For an N-ply game there are N+1 positions and at most N can have an eval, capping max coverage at N/(N+1). Short games (7-8 plies) fall below `EVAL_COVERAGE_MIN = 0.90` even when fully analyzed, so `classify_game_flaws` returns GameNotAnalyzed, the oracle columns are never written, and the frontend "Analyze" pill never resolves. 445 prod games are stuck.

Fix: exclude the single structurally-unevaluable terminal position from the denominator in BOTH mirrored gates (Python kernel + SQL subquery), keeping them in lockstep.

Purpose: short fully-analyzed games correctly count as analyzed and get oracle columns populated.
Output: corrected denominator in two places + tests proving the fix and the over-correction guard + CHANGELOG entry.
</objective>
