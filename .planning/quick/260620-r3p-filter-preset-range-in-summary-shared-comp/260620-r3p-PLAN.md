# Quick Task 260620-r3p: Filter preset polish + shared component

## Goal

Polish the **Tactic Difficulty** and **Opponent Strength** filters and de-duplicate
their near-identical wrapper components.

User-facing changes:
- Move the range into the **top-right active-preset label** (the summary), not the
  preset chips. Format: `Intermediate: 0-5` (preset name + bare range, no "Depth" word).
  Custom ranges render bare (`2-4`, or `3` when min === max).
- **Remove the scale labels below the slider** (the `0` / `11` and `≤−200 / 0 / ≥+200`
  tick rows) in both filters.
- Same change applied analogously to **Opponent Strength** (summary shows
  `Stronger: ≥+50`, `Similar: ±100`, `Weaker: ≤−50`, `Any`).

Refactor:
- Extract the duplicated `TacticDepthFilter` / `OpponentStrengthFilter` markup into a
  shared presentational `PresetRangeFilter` component (label row + InfoPopover +
  summary + preset chip grid + dual-thumb slider). Each wrapper keeps only its domain
  logic and passes results in as props.

## Notes / decisions

- Iterated with the user: an earlier pass put the range inside the preset chips
  (`Beginner: Depth 0-1`); the user moved it to the summary label and dropped the
  "Depth" word, so chips are back to plain names (`Beginner`).
- Magic numbers in the Opponent Strength info-popover prose replaced with the existing
  threshold constants (`STRONG_WEAK_THRESHOLD`, `PRESET_THRESHOLD`, `SLIDER_STEP`).
