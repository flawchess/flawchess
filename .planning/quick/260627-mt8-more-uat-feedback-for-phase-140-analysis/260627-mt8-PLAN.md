# Quick Task 260627-mt8 — Phase 140 analysis-board UAT polish (round 3)

UAT feedback for the `/analysis` game-mode board. Seven frontend tweaks, all in
`frontend/src`. No backend, no schema, no new deps.

## Tasks

1. **Engine info line.** Add a header row above the two engine PV lines:
   `<engine toggle> SF 18, Depth <d>`. The toggle (Cpu icon) enables/disables the
   engine; depth comes from the live engine. Always visible (so the toggle is
   reachable when the engine is off/loading). `SF 18` matches the bundled
   Stockfish 18 WASM.

2. **Eval badges + single-line PV rows.** In `EngineLines`, render each PV line as
   one row: a colored eval badge followed by the move chips. Best line = blue badge
   (`BEST_MOVE_ARROW`), second line = grey badge (`ARROW_NEUTRAL`) — matching the
   board arrow colors. Remove the separate score-header row.

3. **Remove relocated controls.** Drop the engine on/off toggle from `BoardControls`
   (now in the info line) and the `d{depth}` badge from the best PV line (now in the
   info line). Remove the now-unused `depth` prop from `EngineLines`.

4. **Fixed-height loading container.** Wrap the engine-lines region in a fixed
   `min-h` container and replace the spinning "Analyzing…" indicator with a
   fixed-height loading placeholder, so the panel no longer jumps as the engine
   transitions loading → analyzing → 2 lines.

5. **Move-list sizing + bottom alignment.** Narrow the right column (move list)
   and let `VariationTree` fill the column (flex-1, scroll) with the row set to
   `lg:items-stretch`, so the bottom of `BoardControls` vertically aligns with the
   bottom of the eval-chart slider.

6. **Eval-chart sync on navigation.** Add a `syncPly` prop to `EvalChart` that
   parks the slider at a parent-driven ply (no tooltip/focus). Analysis drives it
   from the board's current main-line ply, so clicking a move in the move list moves
   the eval-chart slider.

7. **Sideline → fork point.** When the board is off the main line, `syncPly`
   resolves to the nearest main-line ancestor (the position the sideline branches
   from), so clicking a sideline move parks the eval chart at the fork point.

## Files

- `frontend/src/pages/Analysis.tsx` — info line, fixed-height engine container, layout sizing, syncPly wiring, remove BoardControls infoSlot
