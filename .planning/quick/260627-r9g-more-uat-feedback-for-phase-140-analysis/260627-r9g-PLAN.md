# Quick Task 260627-r9g — Phase 140 analysis-board UAT polish (round 4)

Six frontend tweaks to the `/analysis` game board. All in `frontend/src`. No
backend, no schema, no new deps.

## Tasks

1. **Move list max-height −25px.** `VariationTree` desktop list `max-h-[480px]` →
   `max-h-[455px]` (update the row-count comment). Mobile is a horizontal chip
   list (no rows) — not applicable.

2. **Zebra rows + sideline inherits fork-row bg.** Desktop move list: subtle
   alternating row background (local `ZEBRA_ROW_BG` const, no semantic color).
   The injected `variation-pv-section` (and nested sub-PV) gets the fork row's
   zebra background so the whole sideline reads as one band. Variation rows
   themselves stay transparent (inherit the section band).

3. **Engine info → Card.** Wrap the engine area in `<Card>` (charcoal surface).
   The info line (engine toggle + `SF 18, Depth d`) goes in `<CardHeader>`.
   `<CardBody>` is fixed-height so the panel never jumps. Replace the
   "Analyzing…" text and the "Loading engine…" spinner with a shared
   `EngineLinesSkeleton` (animate-pulse bars). "Engine off" stays as text.

4. **Severity arrows → corner icons.** Drop the red/orange/yellow flaw arrow from
   `useGameOverlay`; instead return `squareMarkers` (square + severity + optional
   allowed-depth label). `ChessBoard` renders the Blunder/Mistake/Inaccuracy glyph
   in the target square's **top-right** corner. Add a new `InaccuracyIcon` (`!?`
   on a yellow `SEV_INACCURACY` dot) to `SeverityGlyphIcon.tsx`.

5. **Severity-colored last-move overlay.** The played-move square overlay
   (currently yellow `MOVE_HIGHLIGHT_SQUARE`) becomes severity-colored on the
   main line: blunder→red, mistake→orange, inaccuracy→yellow (unchanged),
   otherwise→green. `useGameOverlay` resolves `lastMoveHighlightColor`;
   `ChessBoard` gains an optional `lastMoveColor` prop (defaults to the current
   yellow, so Openings/TrainSketch are unaffected). Off the main line falls back
   to the default yellow (no precomputed severity).

6. **Depth label → top-left, smaller.** `ChessBoard` arrow depth labels move from
   the top-right to the **top-left** corner and shrink via a max-px cap so they
   read smaller on the large desktop board while keeping their size on the
   smaller mobile board. Only labeled arrows (analysis tactic/best-move arrows)
   are affected; Openings/TrainSketch arrows carry no labels.
