# Quick Task 260712-r4s

## Goal

Make `scripts/calibration-harness.mjs` resumable. Today every run opens a NEW
timestamped TSV and sweeps the full `(bot_elo, bot_blend, anchor)` grid from
`gameIndex = 0`, so a killed 12h run re-invoked with the same command replays
the entire grid into a fresh file. Add `--resume <prior.tsv>`: read the prior
TSV, skip every cell already present, fast-forward the global `gameIndex`
through the skipped cells, and append the remaining cells to the same file so
the finished map is byte-identical to an uninterrupted run.

Implements SEED-097 (Phase 168 follow-up).

## Landmines (from SEED-097)

1. **Fast-forward `gameIndex` through skipped cells — never reset it.** Each
   game's opening/color/seed derives from the global running `gameIndex`, so a
   skipped cell must still `gameIndex += gamesPerCell` or the resumed map is a
   different experiment (D-09 reproducibility).
2. **Reload skipped cells into `cellRows`** so the advisory ELO summary spans
   the whole grid, not just the newly-played cells.
3. **Cell identity = `(bot_elo, bot_blend, anchor)`**; `--games-per-cell` must
   match the prior TSV's `games` column — throw (WR-02) if it disagrees.
4. **Guard against a changed grid/seed/budget on resume** — refuse (throw) if
   `--seed`, the D-11 budget, or the grid axes differ from the prior TSV.

## Plan

- Add `--resume <path>` to `parseArgs` (reuses `requireFlagValue`, WR-02).
- Teach `openMainTsvWriter` an `{ append }` option (append mode, no re-written
  header) for the resume path.
- Add a resume section: `readPriorTsvLines` (refuses a truncated/schema-
  mismatched file), `parsePriorRow`, and `loadPriorSweep` (validates
  games-per-cell / seed / budget / grid membership, returns completed cell keys
  + reconstructed rows).
- In `main()`: build the current grid's cell-key set, load the prior sweep when
  `--resume` is set, point the main TSV at the prior file (append), skip
  completed cells (advance `gameIndex`, push the reloaded row to `cellRows`),
  derive the summary path from the main path, and guard the throughput report
  against an all-cells-complete resume (zero games played).

## Scope

Single file: `scripts/calibration-harness.mjs`. No engine/app changes. Reuses
the existing durable TSV writer and CLI validation.

## Verification

Run a small `blend=0` grid, truncate the TSV to simulate a mid-sweep kill,
`--resume` it, and confirm the finished main TSV **and** the advisory summary
are byte-identical to an uninterrupted run of the same command (directly
exercises landmine #1). Confirm each validation guard throws.
