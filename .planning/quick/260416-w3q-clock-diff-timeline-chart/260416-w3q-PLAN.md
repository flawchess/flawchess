# Quick Task 260416-w3q: Clock Diff Timeline Chart

## Goal

Add a weekly rolling-window timeline chart showing the average clock-diff % at
endgame entry directly below the "Time Pressure at Endgame Entry" table. Uses
the same pattern as the "Win Rate by Endgame Type" chart: 100-game rolling
window, one point per ISO week. Collapsed across all time controls (single
line). Y-axis centered at 0%, range ±30%. Dots colored by zone (green/blue/red)
using the existing NEUTRAL_PCT_THRESHOLD=10.

Reference: Win Rate by Endgame Type (`EndgameTimelineChart`) for the
rolling/weekly shape; Time Pressure vs Performance (`EndgameTimePressureSection`)
for axis labeling.

## Tasks

### Task 1 — Backend: extend clock-stats rows with played_at + add timeline

**Files**
- `app/repositories/endgame_repository.py`
- `app/schemas/endgames.py`
- `app/services/endgame_service.py`

**Action**

1. `query_clock_stats_rows`: append `Game.played_at` to the select (index 8).
   Keep existing indices 0-7 unchanged so `_compute_clock_pressure` and
   `_compute_time_pressure_chart` stay untouched. Update docstring.

2. `schemas/endgames.py`:
   - New `ClockPressureTimelinePoint(BaseModel)` with
     `date: str` (Monday of ISO week),
     `avg_clock_diff_pct: float` (mean user_clock% - opp_clock% over window),
     `game_count: int`.
   - Extend `ClockPressureResponse` with
     `timeline: list[ClockPressureTimelinePoint]` and `timeline_window: int`.

3. `services/endgame_service.py`:
   - Constant `CLOCK_PRESSURE_TIMELINE_WINDOW = 100`.
   - New `_compute_clock_pressure_timeline(clock_rows, window)`:
     - Iterate rows; skip rows missing `time_control_bucket`,
       `base_time_seconds`, `played_at`, or either entry clock.
     - Apply the same 2x clamp as `_compute_clock_pressure`.
     - Record `(played_at, diff_pct)` where
       `diff_pct = (user_clock - opp_clock) / base_time_seconds * 100`.
     - Sort by played_at; walk with trailing `window` list; for each ISO week
       emit the latest state using `statistics.mean(window_slice)`, mirroring
       `_compute_weekly_rolling_series`.
     - Return list filtered to entries with `game_count >= MIN_GAMES_FOR_TIMELINE`.
   - `_compute_clock_pressure`: include `timeline` +
     `timeline_window=CLOCK_PRESSURE_TIMELINE_WINDOW` in the returned
     `ClockPressureResponse`.

**Verify**
- `uv run ruff check app/`
- `uv run ty check app/`
- Start backend, hit `/api/endgames/overview` for a user with enough games,
  confirm the new `clock_pressure.timeline` array and `timeline_window=100`.

**Done**
- `GET /api/endgames/overview` returns `clock_pressure.timeline` with weekly
  points, each carrying a mean of the trailing 100 games' clock-diff %.
