1. In the `data` mapping (around line 31), add `color: b.color` to each data point so the color value is available in the chart data.

2. Create a custom YAxis tick component that renders an SVG foreignObject containing the color circle and label text. The custom tick replaces the default `tick` prop on the YAxis:

```tsx
const CustomYTick = ({ x, y, payload }: { x: number; y: number; payload: { value: string; index: number } }) => {
  const item = data[payload.index]; // access from closure
  return (
    <g transform={`translate(${x},${y})`}>
      <foreignObject x={-120} y={-10} width={120} height={20}>
        <div xmlns="http://www.w3.org/1999/xhtml" className="flex items-center justify-end gap-1 h-full pr-1">
          {item?.color && (
            <span
              className={`inline-block h-2.5 w-2.5 rounded-full border border-muted-foreground shrink-0 ${item.color === 'white' ? 'bg-white' : 'bg-zinc-900'}`}
            />
          )}
          <span className="text-xs truncate">{payload.value}</span>
        </div>
      </foreignObject>
    </g>
  );
};
```

3. Replace the YAxis `tick` prop: change `tick={{ fontSize: 12 }}` to `tick={<CustomYTick x={0} y={0} payload={{ value: '', index: 0 }} />}` (Recharts will override these props with actual values).

Note: The foreignObject approach is needed because SVG text elements cannot render inline HTML circles. The existing circle CSS classes from PositionBookmarkCard (`inline-block h-3 w-3 rounded-full border border-muted-foreground bg-white/bg-zinc-900`) should be reused but slightly smaller (h-2.5 w-2.5) to fit the chart tick area.

If foreignObject causes rendering issues in Recharts, fall back to pure SVG: render a `<circle>` element (r=4, fill white or #18181b for black, stroke #a1a1aa) before a `<text>` element.
