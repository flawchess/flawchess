# Quick Task 260621-mq4 — Tactic Depth relabel + 1-12 numbering + depth on miniboards

## Goal

1. Relabel the "Tactic Difficulty" filter to "Tactic Depth".
2. Show depth as 1..12 in the frontend (instead of the internal 0..11).
3. On the library miniboards (game card + flaw card), show the tactic depth number:
   - **Missed** tactics → at the square the **blue** Stockfish (best-move) arrow points to.
   - **Allowed** tactics → at the end of the **orange/red** flaw-move arrow.

## Design decisions

- **Display offset, not domain shift.** Internal / DB / API depth stays 0-based
  (matches `game_flaws.*_tactic_depth`). A single `DEPTH_DISPLAY_OFFSET = 1`
  converts to the user-facing 1..12 at every display site (filter summary, info
  copy, miniboard labels). Query params and the slider domain are untouched.
- **Depth must be plumbed through the API.** The depth columns are SELECTed for
  filtering but never returned. Expose `missed_tactic_depth` / `allowed_tactic_depth`
  on both `FlawMarker` (game-card eval markers) and `FlawListItem` (flaws tab),
  gated by the SAME `_TACTIC_CHIP_CONFIDENCE_MIN` rule as the motif — i.e. depth
  is non-null exactly when its motif chip is shown.
- **Label pairing:** blue best-move arrow ↔ `missed_tactic_depth`; flaw-move
  arrow ↔ `allowed_tactic_depth`. The label is rendered only when that depth is
  non-null (motif present).
- **Rendering:** extend `MiniBoardArrow` with an optional `label?: string`; the
  arrow overlay draws the label as a small badge centered on the arrow's `to`
  square.
