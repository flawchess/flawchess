<objective>
Fix the IllegalMoveError 500 on POST /api/insights/openings for users with games imported from non-standard starting positions (PGNs with `[SetUp "1"][FEN "..."]` headers).

Purpose: Phase 71's frontend OpeningInsightsBlock cannot render because compute_insights raises chess.IllegalMoveError when it tries to replay a SAN sequence that originated in a game started from a custom FEN (e.g. user 7 / hikaru has 4 such chess.com puzzle / themed-event games starting from `rnbqkbnr/ppp1pppp/8/3p4/8/1P6/P1PPPPPP/RNBQKBNR w KQkq - 0 1` where ply 0 already has `b3` played, so move_san at ply 0 is `Bb2`). When `func.min(entry_san_sequence)` over a transposition group picks the chain originating in such a game, the resulting array starts mid-game and `chess.Board().push_san('Bb2')` blows up.

Output:
1. Repository excludes those games from the CTE entirely (correctness fix — they shouldn't seed transition rows because their SAN sequences are unreachable from the starting board).
2. Service wraps `_compute_prefix_hashes` and `_replay_san_sequence` calls so a single bad row logs to Sentry and gets skipped instead of 500-ing the whole endpoint (defensive guard — handles future corruption / new edge cases).
3. Regression test covers both the SQL exclusion and the defensive guard.

Conventional commit subject: `fix(70): exclude non-standard-FEN games from opening-insights transitions (#71 hotfix)` — the underlying defect was introduced by Phase 70's SQL CTE (commit 6045a7e brought the entry_san_sequence column online), so the `fix(70)` prefix matches the prior #71 hotfix style on this branch.
</objective>

Reproducer (confirmed against local dev DB, user_id=7):

```
chess.IllegalMoveError: illegal san: 'Bb2' in rnbqkbnr/pppppppp/8/8/8/8/PPPPPPPP/RNBQKBNR w KQkq - 0 1
```

Root cause: 4 chess.com games (ids 210464–210467) imported with `[SetUp "1"][FEN "rnbqkbnr/ppp1pppp/8/3p4/8/1P6/P1PPPPPP/RNBQKBNR w KQkq - 0 1"]` headers. In these games, ply 0's stored move_san is `Bb2` (which IS legal from the custom FEN, but illegal from the standard starting position).

The CTE in `query_opening_transitions` aggregates `entry_san_sequence = array_agg(move_san) ... ROWS UNBOUNDED PRECEDING TO 1 PRECEDING`. For these games, the prefix is `[Bb2, Nf6, Nf3, ...]` — which is unreachable from `chess.Board()`. After GROUP BY (entry_hash, move_san), `func.min(entry_san_sequence)` lexicographically picks `[Bb2, ...]` over `[Nf3, ..., Bb2, ...]` because 'B' < 'N'. That min() value is what `_compute_prefix_hashes` then tries to replay → 500.

The starting position's full_hash for python-chess Polyglot zobrist is well-defined; we'll compute it once at module-import time:

```python
import ctypes
import chess
import chess.polyglot
STARTING_POSITION_HASH: int = ctypes.c_int64(
    chess.polyglot.zobrist_hash(chess.Board())
).value
```

(Same signed-int64 conversion already used elsewhere in opening_insights_service.py — see _compute_prefix_hashes line 129.)

Filtering the CTE by `WHERE NOT EXISTS (SELECT 1 FROM game_positions gp0 WHERE gp0.game_id = gp.game_id AND gp0.user_id = gp.user_id AND gp0.ply = 0 AND gp0.full_hash <> STARTING_POSITION_HASH)` cleanly excludes them.

       ```python
       def _safe_replay(
           san_sequence: list[str],
           *,
           entry_hash: int,
           candidate_san: str,
       ) -> tuple[str, list[int]] | None:
           """Replay san_sequence and return (entry_fen, prefix_hashes) or None on failure.

           Phase 71 hotfix: a corrupt or non-standard-start row would otherwise
           500 the whole endpoint. Captures to Sentry once per bad row using
           set_context (no variable data in the exception message — preserves
           Sentry grouping per CLAUDE.md) and returns None so the caller skips.
           """
           try:
               entry_fen = _replay_san_sequence(san_sequence)
               prefix_hashes = _compute_prefix_hashes(san_sequence)
           except (chess.IllegalMoveError, chess.InvalidMoveError, ValueError) as exc:
               sentry_sdk.set_tag("source", "opening_insights")
               sentry_sdk.set_context(
                   "opening_insights_replay",
                   {
                       "entry_hash": entry_hash,
                       "candidate_san": candidate_san,
                       "san_sequence": san_sequence,
                   },
               )
               sentry_sdk.capture_exception(exc)
               return None
           return entry_fen, prefix_hashes
       ```
