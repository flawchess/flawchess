# Quick Task 260619-uk7: PR_SET_PDEATHSIG orphan guard for Stockfish children

## Goal

Close the bare-metal + adversarial-exit orphan leak for every `EnginePool`
consumer in one place: when the parent process dies by any means (mashed
ctrl-c that aborts the graceful `EnginePool.stop()` UCI quit loop, `kill -9`,
or a hard interpreter crash), the kernel should SIGKILL each child `sf` engine
so no orphaned ~180-320 MB RSS processes leak and accumulate.

See `.planning/seeds/SEED-059-engine-pool-pdeathsig-orphan-guard.md`.

## Approach (single task)

Add a `prctl(PR_SET_PDEATHSIG, SIGKILL)` request to the existing
`_sched_idle_preexec` in `app/services/engine.py:149`. Because the preexec
already runs only inside the Linux branch of `_engine_popen_kwargs`
(`engine.py:180`), this is automatically Linux-only and covers the module-level
pool (uvicorn lifespan) and all script consumers (`remote_eval_worker.py`,
`backfill_best_move_pv.py`, `backfill_eval.py`) at once — the decisive reason to
fix it in `engine.py` rather than per-script.

- `files`: `app/services/engine.py`
- `action`: append a swallow-on-error `ctypes` `prctl(1, SIGKILL)` block after
  the existing `SCHED_IDLE` call; document the spawning-thread-death invariant.
- `verify`: smoke-test `prctl` on the box; start an `EnginePool`, `kill -9` the
  parent, assert no `sf` children survive; run engine test suite; ruff/ty clean.
- `done`: orphan test shows 0 survivors; 26 engine tests pass; lint+type green.

## Out of scope

- Optional secondary hardening (tighten `backfill_best_move_pv.py` single-ctrl-c
  `KeyboardInterrupt` handling) — skipped per seed; PDEATHSIG is the real fix and
  the single-ctrl-c path already cleans up.
- macOS/Windows keep today's behavior (no preexec) — by design.

## must_haves

- truths:
  - The Linux preexec requests PR_SET_PDEATHSIG=SIGKILL so the kernel kills each
    `sf` child when its spawning (main) thread dies.
  - The call is error-swallowed (seccomp hosts that block `prctl` must not crash
    worker spawn), matching the SCHED_IDLE defensive posture.
- artifacts:
  - `app/services/engine.py` `_sched_idle_preexec` contains the prctl call + the
    thread-death invariant comment.
- key_links:
