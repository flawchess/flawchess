2. Instead of immediately showing the error toast and resetting `googleLoading`, implement a single automatic retry:
   - Use a local `retryCount` ref or variable (simplest: pass a `retry` param defaulting to 0).
   - On first failure (retry === 0), wait 500ms then call `handleGoogleSignIn` again with retry=1.
   - On second failure (retry >= 1), show the existing error toast and set `googleLoading(false)`.

Implementation approach — convert to an inner async function or add an optional parameter:

```typescript
const handleGoogleSignIn = async (attempt = 0) => {
  setGoogleLoading(true);
  try {
    const response = await apiClient.get<{ authorization_url: string }>(
      '/auth/google/authorize',
    );
    window.location.href = response.data.authorization_url;
  } catch (error) {
    console.error('Google sign-in failed:', error);
    if (attempt < 1) {
      await new Promise((r) => setTimeout(r, 500));
      return handleGoogleSignIn(attempt + 1);
    }
    toast.error('Could not start Google sign-in. Please try again.');
    setGoogleLoading(false);
  }
};
```

The onClick handler already passes no args so `attempt` defaults to 0.
