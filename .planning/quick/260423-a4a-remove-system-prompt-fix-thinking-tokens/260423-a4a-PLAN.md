## Task 2 — Investigate why `thinking_tokens` is always NULL for Gemini

**Outcome: not a code bug. Config choice.**

Verified with live Gemini-3 API calls:
- Current config (`app/core/config.py:52`): `GEMINI_THINKING_LEVEL: Literal["low", "high"] = "low"`.
- With `thinking_level='low'`, Gemini-3 returns `usage_metadata.thoughts_token_count=0` for all prompt shapes tested (short, long, structured, unstructured).
- pydantic-ai's Google adapter (`.venv/.../models/google.py:1453`) only adds `details['thoughts_tokens']` when the value is **non-zero** — `if thoughts_token_count := (metadata.thoughts_token_count or 0):`.
- Therefore `usage.details.get("thoughts_tokens")` returns `None`, which our code correctly persists as NULL.
- Switching to `thinking_level='high'` in the same test produced `details={'thoughts_tokens': 463, ...}`.

Extraction logic at `app/services/insights_llm.py:693` is correct.

**Recommendation for the user (not implemented here — needs explicit decision):** if real thinking is desired, set `GEMINI_THINKING_LEVEL=high` in `.env`. This will increase cost and latency materially (my test: ~3× output tokens at high vs low). No code change required.
