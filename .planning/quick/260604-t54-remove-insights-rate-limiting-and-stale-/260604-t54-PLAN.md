<objective>
Remove the per-user insights rate-limiting protection and the tier-2 stale-serve
fallback (`status="stale_rate_limited"`), now that LLM report caching makes them
redundant. The natural cache-miss ceiling (~4 fresh calls per user per era, keyed
on the 4 opponent_strength presets) is already below the old
`INSIGHTS_MISSES_PER_HOUR = 3` limit, so the limit blocks legitimate use (viewing
all 4 variants trips it on the 4th) while doing nothing against guest/bot fan-out.

Purpose: stop blocking legitimate users; delete dead code paths.
Output: a slimmer insights contract — `generate_insights()` goes straight from a
tier-1 cache miss to the fresh LLM call. Tier-1 caching and the 502
provider_error / validation_failure paths stay fully intact.
</objective>
