# Quick Task 260413-qg0: Apply Openings Stats responsive layout to endgame chart sections

## Goal

Adopt the same desktop/mobile responsive pattern used in the Openings Stats tab for:
1. `EndgamePerformanceSection` — "Endgame vs. Non-Endgame Games"
2. `EndgameWDLChart` — "Results by Endgame Type"

**Pattern (from Openings Stats / `MostPlayedOpeningsTable`):**
- Desktop (`lg+`): label + games count + constrained WDL bar on the same row using a grid
- Mobile (`<lg`): label + games count on top, full-width WDL bar below (the current stacked `WDLChartRow` layout)

**Why:** Current behavior on desktop produces very wide WDL bars that hurt readability; mobile can overflow horizontally due to long inline headers.

## Files

- `frontend/src/components/charts/EndgamePerformanceSection.tsx` — split WDL comparison rows into desktop (grid row) + mobile (stacked `WDLChartRow`)
- `frontend/src/components/charts/EndgameWDLChart.tsx` — split category rows into desktop (grid row) + mobile (stacked `WDLChartRow`)

Both should reuse `MiniWDLBar` on desktop (matches Openings Stats) and keep `WDLChartRow` for mobile (unchanged).

## Task 1 — Update `EndgamePerformanceSection`

**Action:**
- For the WDL comparison block (endgame vs non-endgame), render two variants:
  - `<div className="hidden lg:block">` with a grid: label | games count | `MiniWDLBar` (`minmax(120px,200px)`)
  - `<div className="lg:hidden">` keeping the existing `WDLChartRow` rendering (stacked)
- Preserve existing `data-testid` attributes (`perf-wdl-endgame`, `perf-wdl-non-endgame`) — apply them on the outermost row wrapper so tests keep finding them
- Keep `InfoPopover` accessible in the header (unchanged)

**Verify:** Section renders with constrained bars on lg+ and stacked full-width bars on <lg.

**Done:** Component renders both layouts correctly; testids remain.

## Task 2 — Update `EndgameWDLChart`

**Action:**
- Replace the single `EndgameCategoryRow` rendering with a responsive pair:
  - Desktop (`hidden lg:block`): grid-based row with label (+ `InfoPopover`) | games link (`FolderOpen`) | `MiniWDLBar` (`minmax(120px,200px)`) — matches `MostPlayedOpeningsTable` column pattern
  - Mobile (`lg:hidden`): existing `EndgameCategoryRow` using `WDLChartRow` with `gamesLink` (stacked, full-width bar)
- Preserve `data-testid` attributes: `endgame-category-${slug}`, `endgame-category-${slug}-row`, `endgame-games-link-${slug}`, `endgame-type-info-${slug}`
- Keep `maxTotal` proportional frequency bar behavior visible in both layouts (desktop under bar via small wrapper, mobile already handled by `WDLChartRow`)

**Done:** Both layouts render; testids preserved; game link and popovers functional.

## must_haves

- Desktop layout (`lg+`) of both sections puts label + games + WDL bar on a single grid row, with the bar constrained (not full viewport width).
- Mobile layout (`<lg`) keeps the current stacked full-width `WDLChartRow` appearance.
- `data-testid` attributes on rows, games links, and info popovers are preserved.
- No horizontal scroll at mobile widths (≤ 640px) on `/endgames`.
- TypeScript type-check and linter pass: `npm run lint` and `npx tsc --noEmit` in `frontend/`.

## Out of scope

- Any change to `WDLChartRow`, `MiniWDLBar`, or `MostPlayedOpeningsTable` itself.
- Changes to other endgame sub-sections (gauges, material breakdown, timeline).
- Backend or data changes.
